IBM OpenAdmin Tool (OAT) para Informix

O IBM® OpenAdmin Tool (OAT) for Informix® é um aplicativo da web para administrar e analisar o desempenho de servidores de banco de dados IBM Informix. É possível administrar várias instâncias do servidor de banco de dados a partir de uma única instalação do OAT em um servidor da web. É possível acessar o servidor da Web através de qualquer navegador para administrar todos os servidores de banco de dados.

É possível usar o OAT para monitorar e analisar o desempenho de um servidor de banco de dados:

  • Monitore o estado de um servidor de banco de dados com relatórios de status detalhados, informações de alerta e logs de mensagens.
  • Analise o estado de um servidor de banco de dados e os resultados e recomendações de email usando o IBM Informix Health Advisor Plug-in for OpenAdmin Tool (OAT).
  • Monitore as estatísticas de desempenho ao longo do tempo com gráficos detalhados do histórico de desempenho.
  • Monitore os dados reunidos pelo Planejador, que executa tarefas administrativas e coleta informações usando instruções SQL.
  • Execute verificações de validação do sistema nos bancos de dados e tabelas em um servidor de banco de dados.
  • Execute relatórios do sistema e comandos onstat.
  • Faça uma pesquisa detalhada nas instruções SQL para revisar os detalhes de desempenho e otimizar as instruções SQL.

É possível usar o OAT para executar estas tarefas administrativas:

  • Gerenciar o servidor de banco de dados incluindo a configuração, os privilégios do usuário e os processadores virtuais.
  • Criar e monitorar tarefas para o Planejador.
  • Gerenciar o sistema de manutenção Auto Update Statistics (AUS), que atualiza automaticamente estatísticas para otimizar o desempenho de consulta.
  • Gerenciar objetos de banco de dados e de tabela usando o IBM Informix Schema Manager Plug-in for OpenAdmin Tool (OAT).
  • Gerenciar o espaço de armazenamento e os logs de recuperação para o servidor de banco de dados.
  • Otimizar espaço compactando e defragmentando tabelas.
  • Faça backup dos espaços de armazenamento para um servidor de banco de dados.
  • Executar instruções SQL e executar operações SQL padrão em tabelas.
  • Monitorar e administrar clusters de alta disponibilidade.
  • Monitorar e administrar o Enterprise Replication usando o IBM Informix Replication Plug-in for OpenAdmin Tool (OAT).
  • Monitorar e administrar objetos de banco de dados relacionados a uma série temporal, incluindo tabelas, índices e tabelas virtuais, além de subtipos, contêineres e calendários do TimeSeries, usando o IBM Informix TimeSeries Plug-in for OpenAdmin Tool (OAT).

É possível instalar o IBM OpenAdmin Tool (OAT) for Informix ao instalar o IBM Informix Client Software Development Kit (Client SDK) ou ao instalar o pacote configurável do software IBM Informix e selecionar o Client SDK ou o IBM Informix Connect. Esse programa de instalação inclui o OAT, o plug-in Health Advisor, o plug-in Replication, o plug-in Schema Manager e o plug-in TimeSeries e todo o software necessário para executar o OAT, incluindo o Apache PHP e PDO_informix pre-configurados.

Alternativamente, o OpenAdmin Tool Community Edition está disponível a partir da página de Download do Produto Grátis do IBM Informix: https://www14.software.ibm.com/webapp/iwm/web/preLogin.do?lang=en_US&source=swg-informixfpd (abre em uma nova janela). O OpenAdmin Tool Community Edition inclui o OAT e os plug-ins opcionais, mas não inclui o software de apoio. Esta edição deve ser instalada manualmente.

Como Me Conecto a um Servidor de Banco de Dados?

Efetuando login em um servidor de banco de dados

Efetue login em um servidor de banco de dados usando o IBM OpenAdmin Tool (OAT) for Informix.

Para efetuar login em um servidor de banco de dados:

  1. Acesse a URL raiz do OAT: http://server name/openadmin.
  2. Na página Login para OAT, preencha os campos Detalhes do Servidor.
  3. Clique em Login.

Você está conectado ao servidor de banco de dados por meio do OAT.

Dica: Alternativamente, se o servidor de banco de dados for um membro de um grupo do OAT, será possível conectar-se ao servidor de banco de dados efetuando login no grupo do OAT. Um grupo do OAT contém informações de conexão para servidores de banco de dados. Quando você efetua login em um grupo do OAT, é possível alternar para outros servidores de banco de dados no grupo sem efetuar login novamente.

Efetuando Login em um Servidor de Banco de Dados em um Grupo do OAT

Conecte-se a um servidor de banco de dados efetuando login em um grupo de conexões do servidor de banco de dados no IBM OpenAdmin Tool (OAT) for Informix.

Um grupo do OAT contém informações de conexão para servidores de banco de dados. Quando você efetua login em um grupo do OAT, é possível alternar para outros servidores de banco de dados no grupo sem efetuar login novamente.

Para efetuar login em um servidor de banco de dados em um grupo do OAT:

  1. Acesse a URL raiz do OAT: http://server name/openadmin.
  2. Na página Login do OAT, na lista Grupo do OAT, selecione o grupo do OAT que contém o servidor ao qual você deseja se conectar.
  3. Insira a senha para o grupo do OAT.
  4. Clique em Obter Servidores.
  5. Selecione o servidor na lista de servidores. Os campos Detalhes do Servidor são preenchidos com as informações do servidor.
  6. No campo Senha na área Detalhes do Servidor, aceite a senha se ela for fornecida ou insira uma senha válida e clique em Efetuar Login.

Alternando para outro Servidor em um Grupo do OAT

Conecte-se a outro servidor de banco de dados em um grupo de conexões do servidor de banco de dados no IBM OpenAdmin Tool (OAT) for Informix.

Um grupo do OAT contém informações de conexão para servidores de banco de dados. Quando você efetua login em um grupo do OAT, é possível alternar para outros servidores de banco de dados no grupo sem efetuar login novamente.

Para alternar para outro servidor de banco de dados em um grupo do OAT, selecione o servidor de banco de dados na lista Servidor na parte superior da página do OAT.

Como monitoro a atividade do servidor de banco de dados?

Monitorando o Status do Servidor de Banco de Dados

Monitore o status de um servidor de banco de dados específico em tempo real ou visualize o status de todos os servidores em um grupo do OAT ao mesmo tempo.

É possível monitorar o status de um servidor de banco de dados específico usando um painel. Esse painel mostra o uso de memória e espaço, o número de bloqueios, transações e consultas e as tabelas com a maioria das leituras e gravações.

É possível visualizar o status de todos os servidores de banco de dados em um grupo do OAT em um painel multisservidor. A partir desse painel, é possível acessar outras páginas no IBM OpenAdmin Tool (OAT) for Informix para obter informações adicionais.

Para monitorar o status do servidor de banco de dados:

  1. No menu do OAT, clique em Health Center > Painel.
  2. Para atualizar o painel, clique no ícone de atualização ou altere a taxa de atualização.

Customizando o Painel

Altere as informações de status que o painel exibe para um servidor de banco de dados específico.

Pré-requisito: Se o menu Administração no IBM OpenAdmin Tool (OAT) for Informix estiver protegido por senha, a senha para executar esta tarefa será necessária.

O painel OAT exibe informações sobre um servidor de banco de dados; por exemplo, o uso de memória e o número de transações. É possível especificar as informações a serem exibidas sobre um servidor de banco de dados customizando o painel.

Para customizar o painel:

  1. No menu Administração no OAT, clique em Gerenciador de Painel.
  2. Para abrir o editor de layout para um painel, clique duas vezes no painel.

Modificar os Limites de Status para o Painel

Modifique os limites para os indicadores de status que são exibidos no painel multisservidor.

Pré-requisito: O IBM Informix Health Advisor Plug-in for OpenAdmin Tool (OAT) deve estar instalado.

O painel multisservidor no IBM OpenAdmin Tool (OAT) for Informix exibe indicadores de status para os servidores de banco de dados em um grupo do OAT. Por exemplo, se o espaço livre para um dbspace em um servidor de banco de dados ficar abaixo do limite especificado, um indicador de aviso será exibido para o servidor de banco de dados.

É possível modificar os limites de status para a memória, o espaço e a porcentagem de chunk do total de E/S. Esses limites são as configurações padrão a partir do plug-in Health Advisor.

Para modificar os limites de status para o painel:

  1. No menu do OAT, clique em Health Center > Health Advisor.
  2. Na página Perfil, verifique se o perfil atual é o perfil padrão.
  3. Clique na guia Alarmes.
  4. Modifique os limites para qualquer um destes alarmes:
    Limite de alarme do plug-in Health Advisor Limite de status do painel
    Memória Livre do S.O. Memória
    Espaço Livre do Dbspace Espaço
    Operações de E/S de Chunk E/S: Porcentagem de chunk do total de E/S

Monitore os alarmes de eventos gerados pelo servidor de banco de dados e os alertas gerados pelo Planejador de Tarefas ou por tarefas e sensores integrados.

Para monitorar alertas e alarmes de eventos:

No menu do OpenAdmin Tool (OAT) for Informix, clique em Health Center > Alertas.

Excluindo o log de mensagens on-line

Exclua ou trunque o arquivo de log de mensagens on-line para o servidor de banco de dados.

Pré-requisito: Para esta ação, o IBM OpenAdmin Tool (OAT) for Informix requer o IBM Informix versão 11.70.xC3.

Para excluir o log de mensagens on-line:

  1. No menu do OAT, clique em Logs > Log Online.
  2. Proceda de uma das seguintes formas:
    • Para excluir o arquivo de log, clique em Excluir.
    • Para excluir as mensagens no arquivo de log, mas reter o arquivo de log, clique em Truncar.

Girando o Log de Mensagens On-line

Ativar, desativar ou modificar a rotação automática do arquivo de log de mensagens on-line para o servidor de banco de dados.

Pré-requisito: Para esta ação, o IBM OpenAdmin Tool (OAT) for Informix requer o IBM Informix versão 11.70.xC3.

É possível controlar o tamanho do arquivo de log de mensagens on-line configurando a rotação de log automático. Quando o log de mensagens on-line é girado, o servidor de banco de dados alterna para um novo arquivo de log de mensagens on-line e incrementa em um os números de ID para os arquivos de log anteriores. Quando o número máximo de arquivos de log é atingido, o arquivo de log com o ID mais alto é excluído.

Para girar o log de mensagens on-line:

  1. No menu do OAT, clique em Logs > Log Online.
  2. Clique na guia Rotação do Log de Mensagens On-line. É possível ativar ou desativar a tarefa, modificar o planejamento ou editar o parâmetro que determina o número máximo de arquivos de log.

Como Gerencio o Espaço do Sistema?

Monitorando o Uso do Espaço

Revise as informações sobre o uso de espaço e os espaços de armazenamento para um servidor de banco de dados. Para um espaço de armazenamento específico, revise informações sobre o uso e a otimização do espaço, e as partes, tabelas e índices do espaço.

Para monitorar o uso do espaço:

  1. No menu do IBM OpenAdmin Tool (OAT) for Informix, clique em Administração de Espaço > Armazenamento. A página exibe painéis de informações sobre o uso do espaço para o servidor de banco de dados.
  2. Para exibir informações adicionais, expanda os painéis clicando no botão de expansão em um painel.
  3. Para exibir informações sobre um espaço de armazenamento específico, selecione o espaço na lista. A página exibe painéis de informações sobre o uso do espaço no espaço.
  4. Para exibir informações adicionais sobre o espaço selecionado, expanda os painéis.

Gerenciando o Uso do Espaço

Gerencie o espaço de armazenamento de um servidor de banco de dados.

Criando um Espaço

Crie um dbspace, dbspace temporário, blobspace ou sbspace para um servidor de banco de dados.

Quando você cria um espaço usando o IBM OpenAdmin Tool (OAT) for Informix, também é possível tornar o espaço expansível. O servidor de banco de dados expande automaticamente o tamanho do espaço de armazenamento quando necessário, usando ou estendendo um chunk. O fornecimento de armazenamento automático requer o IBM Informix versão 11.70.

Para criar um espaço:

  1. No menu do OAT, clique em Administração de Espaço > Armazenamento.
  2. Clique em Ações > Criar Espaço.
  3. Preencha os campos na página Criar um Espaço.

  2. Expanda o painel Espaços.
  3. Na página Espaços, selecione o espaço.
  4. Clique em Modificar.
  5. Preencha os campos na página.

Configurando a Conectividade para Clusters de Alta Disponibilidade

Configure o daemon idsd em um servidor IBM Informix para ativar o início de um servidor em um cluster de alta disponibilidade ou incluir um servidor shared disk secondary (SDS).

Pré-requisito: São necessários privilégios de administrador no servidor Informix para este procedimento.

Para usar o IBM OpenAdmin Tool (OAT) for Informix para tarefas de administração remota, incluindo iniciar um servidor em um cluster de alta disponibilidade, ou incluir um servidor secundário, instale e configure os daemons a seguir no servidor Informix:

  • Um daemon de serviço da Internet, como os daemons de Internet xinetd ou inetd para UNIX e Linux.
  • idsd, um processo daemon que permite que o OAT inicie um servidor Informix remoto. O daemon reside no mesmo host que o servidor Informix e deve estar localizado em um diretório acessível ao usuário informix. O daemon idsd é invocado através dos daemons xinetd ou inetd.
Importante: Não é possível usar os recursos de administração remota no Windows ou MAC OS.

Para configurar um daemon da Internet e o daemon idsd:

  1. Assegure-se de que o daemon idsd esteja instalado com o Informix, no diretório $INFORMIXDIR/bin. Assegure-se de que o proprietário e o grupo sejam informix e que as permissões sejam 0754:

    Por exemplo:

    [informix@myhost]/usr/informix/bin% ls -la idsd 
    -rwxr-xr--  1 informix informix 23717 Oct 15 16:34 idsd
  2. No servidor, configure a criptografia para a conexão. Para Informix 11.70, use a ferramenta GSKCmd que está disponível como parte do IBM JRE 1.6 ou mais recente. Para versões anteriores do Informix, use a ferramenta GSKCmd que está incluída no IBM Global Security Kit.
    1. Crie um banco de dados de chaves usando os seguintes parâmetros da linha de comandos GSKCmd para sua plataforma: 
      command -keydb -create -db keydb_name.kdb -pw password -type cms -stash
      Por exemplo: 
      gsk8cmd -keydb -create -db mydb.kdb -pw mypassw0rd -type cms -stash
    2. Crie um certificado autoassinado usando os seguintes parâmetros da linha de comandos GSKCmd: 
      command -cert -create -db keydb_name.kdb -pw password -label label_text
    -dn distinguished name 
    -size <1024 | 512 | 2048> -default_cert yes
      Em que: distinguished name é um nome distinto de X.500. O distinguished name é uma sequência de caracteres entre aspas com o seguinte formato: CN=common name (necessário), O=organization, OU=organization unit, L=location, ST=state or province, C=country.

      Por exemplo: 

      gsk8cmd -cert -create -db mydb.kdb -pw mypassw0rd -label mycertificatelabel
    -dn "CN=lenexa.ibm.com,O=ibm,OU=IBM HTTP Server,ST=KS,C=US" 
    -size 1024 -default_cert yes
  3. Abra o arquivo de configuração xinetd ou inetd para edição e inclua uma entrada de serviço para o daemon idsd.
    Importante: Se o parâmetro de configuração GSKIT_VERSION for configurado para uma versão não padrão do GSKit, inclua o argumento de programa do servidor -g NUMBER na entrada de serviço xinetd ou inetd para especificar a versão do GSKit.
    • Para o daemon do servidor xinetd: /etc/xinetd.conf. 
      service idsd
             {
             disable = no
             socket_type = stream 
             protocol = tcp
             user = root
             wait = no 	    
             server = /usr/informix/bin/idsd
             server_args = -l /usr/informix/idsd.log 
                 -k /usr/informix/idsd -n idsd_label
             }
    • Para o daemon do servidor inetd no UNIX: /etc/inetd.conf: 
      idsd stream tcp nowait root /usr/informix/bin/idsd idsd -l
/usr/informix/idsd.log  -k /usr/informix/idsd -n idsd_label
  4. Abra o arquivo /etc/services para edição e inclua um serviço denominado idsd. Por exemplo: 
    idsd 10000/tcp
    Neste exemplo, o número da porta 10000 é usado, mas é possível usar qualquer outra porta maior que 1024 que ainda não esteja em uso. Entretanto, o nome deve ser idsd e o protocolo deve ser tcp.
  5. Reinicie os daemons de serviço xinetd ou inetd. Na maioria das implementações, é possível reiniciar enviando o sinal SIGHUP para o processo inetd ou xinetd em execução. Entretanto, em algumas plataformas Linux, é possível executar o comando a seguir como raiz: 
    service xinetd restart

Gerenciando Estatísticas de Atualização Automática

Revise e ajuste o sistema de manutenção de Estatísticas de Atualização Automática (AUS) para estatísticas da tabela.

O sistema de manutenção de AUS identifica tabelas e índices que exigem novas estatísticas do otimizador e executa as instruções UPDATE STATISTICS apropriadas para otimizar o desempenho da consulta. Dependendo do seu sistema, talvez seja necessário ajustar as políticas de expiração ou o planejamento de AUS.

É possível controlar e monitorar as políticas de AUS na página Administração do Servidor > Auto Update Statistics > Configuração.

É possível revisar estatísticas de tabela usando o IBM Informix Schema Manager Plug-in for OpenAdmin Tool (OAT) na página Caixa de Ferramentas SQL > Schema Manager > Estatísticas.

Configurando e Ativando Estatísticas de Atualização Automática

  1. No menu para o IBM OpenAdmin Tool (OAT) for Informix, clique em Administração do Servidor > Auto Update Statistics > Configuração.
  2. Com base no desempenho e requisitos do sistema, determine se os valores padrão são válidos para seu sistema.
    1. Para especificar o tempo máximo entre as reconstruções de estatísticas, insira o número de dias para o parâmetro AUS_AGE, como 30, 60 ou 90.
    2. Para especificar qual percentual de alteração em uma tabela inicia uma reconstrução de estatísticas, insira um número de porcentagem, tal como 20 ou 50 para o parâmetro AUS_CHANGE.
    3. Para especificar se o AUS usa as regras de diretriz recomendada, selecione Ativado ou Desativado na lista para o parâmetro AUS_AUTO_RULES. O padrão é Ativado.
    4. Para especificar o número de linhas abaixo do qual as estatísticas são sempre reconstruídas, insira um número para o parâmetro AUS_SMALL_TABLES. O padrão é 100.
    5. Para especificar a prioridade de PDQ, insira um número de -1 a 100 para o parâmetro AUS_PDQ. O padrão é 10. A prioridade Parallel Database Queries (PDQ) é um dos fatores que determina como o servidor de banco de dados usa recursos para processamento paralelo.
    6. Para especificar o número de encadeamentos a ser usado para executar as tarefas de estatísticas, insira um número de encadeamentos. O OAT envia esse número de tarefas do AUS para o Planejador de Tarefas. Esta opção requer o IBM Informix versão 11.70.
  3. Clique em Salvar para manter suas alterações.
  4. Para determinar se as tarefas do AUS estão ativadas, clique na guia Informações e, em seguida, clique nestas tarefas:
    • Avaliação de Estatísticas de Atualização Automática
    • Atualização das Estatísticas de Atualização Automática
      Nota: Se não houver trabalho a ser executado pela tarefa Atualização de Auto Update Statistics, a tarefa desativará a si mesmo. Entretanto, na próxima vez que a tarefa de Avaliação de Estatísticas de Atualização Automática for executada e criar trabalho para a tarefa de atualização, a tarefa de avaliação ativará automaticamente a tarefa de atualização.
    A página Detalhes da Tarefa é exibida.
  5. Role para baixo até o campo Ativar Tarefa e assegure que ele esteja selecionado.
  6. Especifique a frequência com que as tarefas são executadas com os campos que configuram a janela de execução. Como o AUS é uma operação cheia de recursos, uma janela de execução específica pode ser definida. A definição da janela de execução inclui um horário de início e um horário de encerramento, e os dias da semana para executar. Por padrão, o espaço de tempo de execução é 1h – 5h diariamente.
  7. Clique em Salvar para salvar qualquer alteração.

Como Gerencio Tarefas do Planejador?

Incluindo Tarefas do Planejador

É possível criar tarefas do Planejador que executam tarefas ou sensores em tempos e frequências especificados.

  • As tarefas fornecem os meios para executar uma tarefa específica em um intervalo ou tempo específico.
  • Os sensores coletam e salvam informações.

Incluindo uma Nova Tarefa

Importante: O horário no qual você cria a tarefa afeta a primeira execução da tarefa.
  • Se o tempo atual estiver dentro dos tempos de início e parada especificados, as tarefas serão executadas no tempo atual. As execuções subsequentes ocorrem em intervalos especificados pelo valor de frequência. Portanto, por exemplo, se o tempo atual estiver dentro do tempo de início e parada e a frequência estiver configurada para 15 minutos, a primeira execução da tarefa iniciará imediatamente após a tarefa ser planejada e a próxima execução ocorrerá 15 minutos mais tarde.
  • Entretanto, se o tempo atual não estiver dentro do tempo de início e de parada, a tarefa aguardará a execução até o próximo tempo de início, no dia atual ou no próximo dia. Por exemplo, se o horário atual fosse 1h da manhã e o próximo horário de início fosse 7h da noite (19h), a tarefa iniciaria às 19h do mesmo dia.

Para incluir uma nova tarefa:

  1. Clique em Incluir uma Nova Tarefa na página Planejamento da Tarefa.
  2. Clique em Tarefa em Qual tipo de tarefa gostaria de incluir? em seguida clique em Próximo.
  3. Insira um nome no campo Nome da Tarefa. Por exemplo: MyTask.
  4. Selecione uma categoria na lista Grupo de Tarefas.
  5. Inclua uma descrição da nova tarefa no campo de texto e clique em Avançar.
  6. Especifique o intervalo de tempo permitido no qual a tarefa pode executar, tal como entre dez horas (10h) da manhã e 6 horas (18h) da noite, toda hora, todo dia.
    1. Especifique o horário de início do intervalo de tempo, usando o formato de relógio de 24 horas: hh:mm:ss Por exemplo: 10:00:00
    2. Especifique o horário de parada do intervalo de tempo, usando o formato de relógio de 24 horas: hh:mm:ss Por exemplo: 18:00:00. É possível também especificar NEVER.
    3. Especifique a frequência com que a tarefa pode ser executada durante o intervalo de tempo permitido, em dias, horas e minutos: d Dias h Horas m Minutos. Por exemplo: 0 Dias 1 Horas 00 Minutos.
  7. Clique em Avançar.
  8. Especifique a instrução de comando que a tarefa deve executar. Por exemplo, se desejar excluir tarefas diversas, você poderá inserir a seguinte instrução: 
    delete from ph_task where tk_group='MISC'
  9. Clique em Avançar. A página de confirmação é exibida.
  10. Verifique as entradas para cada seleção:
    • Se as entradas estiverem corretas, clique em Concluir para confirmar e criar a tarefa.
    • Clique em Voltar para retornar para as páginas anteriores para corrigir os valores incorretos.

Incluindo um Novo Sensor

  1. A partir da página Planejamento da Tarefa, clique em Incluir uma Nova Tarefa.
  2. Clique em Sensor em Qual tipo de tarefa gostaria de incluir? em seguida clique em Próximo.
  3. Insira um nome no campo Nome do Sensor. Por exemplo: MySensor.
  4. Selecione uma categoria na lista Grupo de Sensores.
  5. Inclua uma descrição da nova tarefa no campo de texto e clique em Avançar.
  6. Especifique o intervalo de tempo permitido no qual o sensor pode ser executado, tal como entre 10h e 18h, toda hora, todo dia.
    1. Especifique o horário de início do intervalo de tempo, usando o formato de relógio de 24 horas: hh:mm:ss. Por exemplo: 10:00:00
    2. Especifique o horário de parada do intervalo de tempo, usando o formato de relógio de 24 horas: hh:mm:ss. Por exemplo: 18:00:00
    3. Especifique a frequência para quando a tarefa pode executar durante o intervalo de tempo permitido, em dias, horas e minutos: d Dias h Horas m Minutos. Por exemplo: 0 Dias 1 Horas 00 Minutos.
    4. Especifique o horário de exclusão dos dados, em dias, horas e minutos: d Dias h Horas m Minutos. Por exemplo: 1 Dias 0 Horas 0 Minutos.
  7. Clique em Avançar.
  8. No campo Nome da Tabela de Resultados, especifique o nome da tabela de resultados para conter os dados do sensor. Por exemplo: MyResults
  9. Clique em Avançar.
  10. Conclua a instrução SQL para a tabela de resultados inserindo os nomes de colunas e definições no campo Instrução SQL. Por exemplo, se você desejar coletar o código de retorno de taskA (com task_id 10) na tabela TaskA_ReturnCode, poderá inserir as seguinte instrução SQL: 
    create table TaskA_ReturnCode(task_name varchar(20), errorcode integer)
  11. Especifique a instrução de comando que o sensor deve executar. Por exemplo: 
    insert into TaskA_ReturnCode values('TaskA', (select run_retcode from ph_run
where run_task_id=10))
  12. Verifique as entradas para cada seleção.
    • Se as entradas estiverem corretas, clique em Concluir para confirmar e criar o sensor.
    • Clique em Voltar para retornar às páginas anteriores para atualizar quaisquer valores incorretos.

  3. Selecione Ativar Tarefa.
  4. Clique em Salvar.

Alterando o Intervalo de Tempo Permitido da Tarefa do Planejador

O intervalo de tempo permitido é aquele no qual a tarefa pode ser executada, por exemplo, entre 10h e 18h.

  1. Expanda Planejador de Tarefas > Planejador. Uma lista de tarefas com detalhes de planejamento é exibida na página Detalhes da Tarefa.
  2. Clique em um nome da tarefa para alterar a frequência ou condições específicas dos tempos de execução planejados. É possível atualizar os dias ou horas em que a tarefa é executada.
  3. Especifique o horário de início do intervalo de tempo, usando o formato de relógio de 24 horas: hh:mm:ss. Por exemplo, especificar 10:00:00 representa 10 horas da manhã.
  4. Especifique o horário de parada do intervalo de tempo, usando o formato de relógio de 24 horas: hh:mm:ss. Por exemplo, especificar 18:00:00 representa 6 horas da noite.
  5. Especifique a frequência com que a tarefa pode ser executada durante o intervalo de tempo permitido, em dias, horas e minutos: d Dias h Horas m Minutos. Por exemplo: 0 Dias 1 Horas 00 Minutos.
  6. Selecione Desativar para qualquer dia da semana para o qual você não deseja que a tarefa seja executada.
  7. Clique em Salvar para salvar suas alterações.

Excluindo uma Tarefa do Planejador

Restrição: As tarefas do sistema não podem ser excluídas.

Para excluir uma tarefa definida pelo usuário:

  1. Expanda Planejador de Tarefas > Planejador.
  2. Clique em Excluir uma Tarefa. A coluna Excluir Tarefa é incluída na tabela Lista de Tarefas Cron. Cada tarefa possui um botão Excluir; para tarefas do sistema, o botão Excluir está desativado.
  3. Localize o nome da tarefa e clique em Excluir. Se a tarefa for para um sensor, será solicitado que você exclua a tabela de resultados associada ao sensor.
  4. Determine o que fazer com a tabela.
    • Clique em Sim para excluir a tabela de resultados bem como a tarefa.
    • Clique em Não para manter a tabela quando a tarefa for excluída.
    • Clique em Cancelar para cancelar a exclusão da tarefa e tabela.

Crie uma diretiva externa para otimizar uma instrução SQL usando o IBM OpenAdmin Tool (OAT) for Informix.

Pré-requisitos:
  • No arquivo onconfig do servidor de banco de dados, defina o parâmetro de configuração EXT_DIRECTIVES como 1 ou 2, de modo que as diretivas externas possam ser salvas.   • Assegure-se de que o rastreio de SQL esteja ativado e que a opção Nomes de Tabelas esteja selecionada na página Administrador de Rastreio. Consulte Ativando o Rastreio de SQL e Especificando Opções para o Rastreio de SQL.

É possível criar uma diretiva externa para aplicar a uma instrução SQL. Diretivas são comentários que instruem o otimizador de consulta a executar uma consulta. As diretivas externas fornecem uma maneira de alterar o plano de execução da consulta de uma instrução SQL. Depois que você aplicar diretivas externas a uma instrução SQL, o plano de execução da consulta da instrução SQL é alterado de acordo toda vez que você reexecutar a instrução SQL.

Importante: Quando você executa novamente uma instrução SQL, ela deve corresponder exatamente à instrução SQL aplicada às diretivas. Por exemplo, ela deve ter as mesmas letras maiúsculas e minúsculas e o mesmo espaçamento.

    2. Clique na instrução SQL na lista.
    3. Clique em qualquer linha da lista de execuções da instrução SQL.
  3. Clique na guia Otimizar.
  4. Clique em Incluir.
  5. Na página Incluir Diretivas, selecione a diretiva. Selecione as tabelas e os índices, se necessário. Para incluir mais diretivas à instrução SQL, clique em Incluir Outra ou clique em Incluir para fechar a página.

Limite o acesso às páginas de Administração do IBM OpenAdmin Tool (OAT) for Informix por motivos de segurança. Proteger estas páginas com uma senha é importante porque as páginas de Administração estão acessíveis a partir da página Login e, portanto, abertas para qualquer um.

Um método é usar a autenticação integrada fornecida pelo servidor da Web. Por exemplo, é possível proteger o diretório de instalação, OATINSTALL/admin/.htaccess.

Exemplo: Para proteger o diretório do OAT a partir do Apache:

  1. Crie um arquivo de senha em um diretório que não está acessível a partir da Web usando o utilitário htpasswd fornecido com o Apache.
  2. Inclua diretivas de autenticação em um dos arquivos a seguir:
    • O arquivo OATINSTALL/admin/.htaccess
    • O arquivo httpd.conf
  3. Coloque as diretivas dentro de uma seção <Directory>. Por exemplo:

Configurando HTTPS para OAT (Linux)

Ative o HTTPS para proteger o servidor da web do IBM OpenAdmin Tool (OAT) for Informix contra escutas, violação, falsificação de mensagem e interferência na rede.

Pré-requisito: O OpenSSL deve ser instalado no computador no qual o servidor Apache para OAT está instalado. É possível fazer o download e instalar o OpenSSL a partir do website OpenSSL Project: www.openssl.org.

Quando o HTTPS está ativado, as mensagens de clientes OAT são criptografadas antes de serem enviadas para o servidor da web do OAT. A criptografia evita que usuários não autorizados recebam por meio da linha informações confidenciais e as deturpem. Quando o HTTPS está ativado, os clientes OAT também podem se autenticar no host do OAT para que usuários não autorizados não possam enganar os clientes OAT com falsos servidores da web.

Importante: O HTTPS criptografa a comunicação apenas entre o servidor da web e o cliente do OAT. Ele não criptografa a comunicação entre um servidor de banco de dados IBM Informix e o servidor da web. O IBM Informix versão 9.4, e mais recente, ativa a criptografia de dados entre o servidor Informix e o servidor da web do OAT usando um módulo de suporte de comunicação de criptografia. Para obter informações adicionais sobre criptografia e servidores Informix, leia "Criptografia sobre a Ligação com o IDS 9.40" no website do developerWorks®.

Quando você instala o OAT com o IBM Informix Client Software Development Kit (Client SDK), o programa de instalação fornece o software necessário para executar o OAT, incluindo Apache, PHP e PDO_informix pré-configurados. No Linux, o servidor da web Apache que é instalado com o OAT é ativado para o módulo de criptografia mod_ssl. Esse módulo é necessário para ativar o HTTPS.

Para ativar o HTTPS no OAT no Linux, conclua as etapas a seguir:

  1. Crie uma chave de criptografia e um certificado para o servidor da web para que os clientes OAT possam autenticar o servidor da web com base em seu certificado.
  2. Configure o Apache para ativar o HTTPS.

Para obter informações adicionais sobre a ativação de HTTPS, leia "Assegurando o Servidor da Web do OpenAdmin Tool for Informix com HTTPS" no website do developerWorks.

Criando uma Chave de Criptografia

As chaves são usadas na criptografia e decriptografia. Geralmente, elas vêm em pares, a chave pública e a chave privada. As chaves públicas são usadas para criptografar mensagens e as chaves privadas são usadas para decriptografar mensagens. Um certificado é um documento que autentica um servidor da web. Um servidor da web com HTTPS ativado possui um certificado, assinado por uma autoridade de certificação confiável, para verificar o servidor da web. Depois que um servidor da web é ativado para HTTPS, os clientes escolhem estabelecer uma conexão normal ou uma conexão segura para o servidor da web. Para estabelecer uma conexão normal, o cliente digita http://web server_url em seu navegador da web. Para estabelecer uma conexão segura, o cliente digita https://web server_url.

Para gerar pares de chaves pública e privada e o certificado, use o arquivo executável openssl no diretório bin de sua instalação do OpenSSL.

Ao executar o comando para gerar a chave privada, você é solicitado a inserir uma passphrase para proteger sua chave privada. Você deve inserir novamente essa passphrase toda vez que executar um comando que requeira o uso da chave privada. Por exemplo, você deve inserir a passphrase ao iniciar seu servidor da web Apache ou usar a chave privada para gerar um certificado. No Linux, por motivos de segurança, use uma passphrase para proteger e criptografar sua chave privada.

Para gerar uma chave privada, execute o comando a seguir no computador no qual o OpenSSL está instalado:

openssl genrsa -des3 -out privkey.pem 2048

A chave privada é criada no arquivo privkey.pem. Armazene este arquivo em um local seguro porque ele contém a chave de decriptografia para o servidor da web. Quando você gera o certificado, esse arquivo é usado para gerar a chave pública associada, que está incluída no certificado.

Criando um Certificado

Depois de criar uma chave privada, crie um certificado para autenticar a identidade do servidor da web. Para gerar um certificado, você cria uma solicitação de assinatura de certificado e a envia para uma autoridade de certificação confiável. A autoridade emite um certificado.

Para gerar uma solicitação de assinatura de certificado, execute o comando a seguir:

openssl req -new -key privkey.pem -out cert.csr

Para obter informações adicionais sobre solicitações de assinatura de certificado, entre em contato com sua autoridade de certificação.

Como alternativa, é possível criar um certificado autoassinado sem passar por uma autoridade de certificação, embora os certificados autoassinados não sejam recomendados.

Para gerar um certificado autoassinado, execute o comando a seguir:

openssl req -new -x509 -key privkey.pem -out cacert.pem -days 1095

O OpenSSL solicita a você suas informações. O certificado é armazenado no arquivo cacert.pem. Esse arquivo é exibido para Web clients para verificar sua identidade. Ele também inclui a chave pública para Web clients, para que eles possam criptografar a comunicação.

Para obter informações adicionais sobre chaves de criptografia, consulte a documentação do OpenSSL.

Configurando o Apache para Ativar HTTPS

Para ativar o HTTPS, atualize o arquivo de configuração do Apache e o arquivo de configuração Apache SSL:

  1. Localize o arquivo de configuração do Apache. O caminho inclui a versão do Apache, por exemplo: 
    /oat/installation/directory/Apache_2.4.2/conf/httpd.conf
  2. No arquivo de configuração do Apache, remova o comentário desta linha: #Include conf/extra/httpd-ssl.conf.

    Essa linha é comentada por padrão. Quando o comentário é removido, o arquivo de configuração Apache SSL é incluído no arquivo httpd.conf.

  3. Localize o arquivo de configuração Apache SSL. O caminho inclui a versão do Apache, por exemplo: 
    /oat/installation/directory/Apache_2.4.2/conf/extra/httpd-ssl.conf
  4. No arquivo de configuração Apache SSL, verifique se o número da porta SSL está configurado para uma porta disponível.

    O HTTPS requer uma porta SSL separada. Por padrão, o número da porta SSL está configurado como 443. Se essa porta não estiver disponível, altere a porta SSL na diretiva Listen e na seção Host Virtual.

  5. No arquivo de configuração Apache SSL, verifique se a diretiva SSLCertificateKeyFile indica o local correto do arquivo de chave privado que você criou: privkey.pem
  6. Verifique se a diretiva SSLCertificateFile indica o local correto do arquivo de certificado que você criou: cacert.pem.
  7. Opcional: Para controlar quais cifras o servidor da web aceita, edite a diretiva SSLCipherSuite. Para obter informações sobre a diretiva SSLCipherSuite, consulte a documentação do Apache mod_ssl.

Para obter informações adicionais sobre configurações HTTPS, consulte o website do Apache: www.apache.org.

Criando uma Chave de Criptografia

As chaves são usadas na criptografia e decriptografia. Geralmente, elas vêm em pares, a chave pública e a chave privada. As chaves públicas são usadas para criptografar mensagens e as chaves privadas são usadas para decriptografar mensagens. Um certificado é um documento que autentica um servidor da web. Um servidor da web com HTTPS ativado possui um certificado, assinado por uma autoridade de certificação confiável, para verificar o servidor da web. Depois que um servidor da web é ativado para HTTPS, os clientes escolhem estabelecer uma conexão normal ou uma conexão segura para o servidor da web. Para estabelecer uma conexão normal, o cliente digita http://web server_url em seu navegador da web. Para estabelecer uma conexão segura, o cliente digita https://web server_url.

Para gerar pares de chaves pública e privada e o certificado, use o arquivo executável openssl no diretório bin da nova instalação do servidor da web Apache.

Ao executar o comando para gerar a chave privada, você é solicitado a inserir uma passphrase para proteger sua chave privada. No Windows, o Apache não suporta uma passphrase para uma chave privada, portanto, você deve remover a passphrase conforme explicado nas próximas etapas.

Para criar uma chave privada:

  1. Para gerar uma chave privada, execute o comando a seguir no computador no qual o novo servidor da web Apache está instalado: 
    openssl genrsa -des3 -out privkey.pem 2048
  2. Para remover a passphrase, execute o comando a seguir: 
    openssl rsa -in privkey.pem -out privkey_nopassphrase.pem

Uma versão não criptografada da chave privada é criada no arquivo privkey_nopassphrase.pem. Armazene esse arquivo em um local seguro porque ele contém sua chave privada em um formato não criptografado. Quando você gera o certificado, esse arquivo é usado para gerar a chave pública associada, que está incluída no certificado.

Criando um Certificado

Depois de criar a chave privada, crie um certificado para autenticar a identidade do servidor da web. Para criar um certificado, você gera uma solicitação de assinatura de certificado e a envia para uma autoridade de certificação confiável. A autoridade emite um certificado.

Para gerar uma solicitação de assinatura de certificado, execute o comando a seguir:

openssl req -new -key privkey_nopassphrase.pem -out cert.csr

Para obter informações adicionais sobre solicitações de assinatura de certificado, entre em contato com sua autoridade de certificação.

Como alternativa, é possível criar um certificado autoassinado sem passar por uma autoridade de certificação, embora os certificados autoassinados não sejam recomendados.

Para gerar um certificado autoassinado, execute o comando a seguir:

openssl req -new -x509 -key privkey_nopassphrase.pem -out cacert.pem -days 1095

O OpenSSL solicita a você suas informações. O certificado é armazenado no arquivo cacert.pem. Esse arquivo é exibido para Web clients para verificar sua identidade. Ele também inclui a chave pública para Web clients, para que eles possam criptografar a comunicação.

Para obter informações adicionais sobre chaves de criptografia, consulte a documentação do OpenSSL.

Configurando o Apache para Ativar HTTPS

Para ativar o HTTPS, atualize o arquivo de configuração do Apache e o arquivo de configuração Apache SSL:

  1. Localize o arquivo de configuração do Apache. O caminho inclui a versão do Apache, por exemplo: 
    /oat/installation/directory/Apache_2.2.22/conf/httpd.conf
  2. No arquivo de configuração do Apache, remova o comentário desta linha: #Include conf/extra/httpd-ssl.conf.

    Essa linha é comentada por padrão. Quando o comentário é removido, o arquivo de configuração Apache SSL é incluído no arquivo httpd.conf.

  3. Localize o arquivo de configuração Apache SSL. O caminho inclui a versão do Apache, por exemplo: 
    /oat/installation/directory/Apache_2.2.22/conf/extra/httpd-ssl.conf
  4. No arquivo de configuração Apache SSL, verifique se o número da porta SSL está configurado para uma porta disponível.

    O HTTPS requer uma porta SSL separada. Por padrão, o número da porta SSL está configurado como 443. Se essa porta não estiver disponível, altere a porta SSL na diretiva Listen e na seção Host Virtual.

  5. No arquivo de configuração Apache SSL, verifique se a diretiva SSLCertificateKeyFile indica o local correto do arquivo de chave privado que você criou: privkey_nopassphrase.pem
  6. Verifique se a diretiva SSLCertificateFile indica o local correto do arquivo de certificado que você criou: cacert.pem.
  7. Opcional: Para controlar quais cifras o servidor da web aceita, edite a diretiva SSLCipherSuite. Para obter informações sobre a diretiva SSLCipherSuite, consulte a documentação do Apache mod_ssl.

Para obter informações adicionais sobre configurações HTTPS, consulte o website do Apache: www.apache.org.
