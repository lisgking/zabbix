IBM OpenAdmin Tool (OAT) para Informix

IBM® OpenAdmin Tool (OAT) para Informix® es una aplicación web para administrar y analizar el rendimiento de los servidores de bases de datos IBM Informix. Puede administrar varias instancias de servidor de bases de datos desde una sola instalación de OAT en un servidor web. Puede acceder al servidor web con cualquier navegador para administrar todos los servidores de bases de datos.

Puede utilizar OAT para supervisar y analizar el rendimiento de un servidor de base de datos:

  • Supervise el estado de un servidor de base de datos con informes de estado detallados, información sobre alertas y registros de mensajes.
  • Analice el estado de un servidor de bases de datos y envíe por correo electrónico los resultados y recomendaciones utilizando el plug-in IBM Informix Health Advisor para OpenAdmin Tool (OAT).
  • Supervise las estadísticas de rendimiento durante un tiempo con gráficos detallados del historial de rendimiento.
  • Supervise los datos reunidos por el Planificador, que ejecuta tareas administrativas y reúne información utilizando sentencias SQL.
  • Ejecute comprobaciones de validación del sistema en las bases de datos y las tablas en un servidor de base de datos.
  • Ejecute informes de sistema y mandatos onstat.
  • Expanda las sentencias SQL para revisar los detalles de rendimiento y optimizar las sentencias SQL.

Puede utilizar OAT para realizar estas tareas administrativas:

  • Gestione el servidor de base de datos incluyendo la configuración, los privilegios de usuario y los procesadores virtuales.
  • Cree y supervise las tareas para el Planificador.
  • Gestione el sistema de mantenimiento Actualizar estadísticas automáticamente (AUS), que actualiza de forma automática las estadísticas para optimizar el rendimiento de las consultas.
  • Gestione los objetos de base de datos y tabla utilizando el plug-in IBM Informix Schema Manager para OpenAdmin Tool (OAT).
  • Gestione el espacio de almacenamiento y los registros de recuperación para el servidor de base de datos.
  • Optimice el espacio comprimiendo y desfragmentando tablas.
  • Realice una copia de seguridad de los espacios de almacenamiento del servidor de una base de datos.
  • Ejecute sentencias SQL y realice operaciones SQL estándar en las tablas.
  • Supervise y administre clústeres de alta disponibilidad.
  • Supervise y administre Enterprise Replication utilizando el plug-in IBM Informix Replication para OpenAdmin Tool (OAT).
  • Supervise y administre objetos de base de datos relacionados con una serie de tiempo, incluyendo tablas, índices y tablas virtuales, y calendarios, contenedores y subtipos TimeSeries utilizando el plug-in IBM Informix TimeSeries para OpenAdmin Tool (OAT).

Puede instalar IBM OpenAdmin Tool (OAT) para Informix al instalar IBM Informix Client Software Development Kit (Client SDK) o al instalar el paquete de software de IBM Informix y seleccionar Client SDK o IBM Informix Connect. Este programa de instalación incluye OAT, el plug-in Health Advisor, el plug-in Replication, el plug-in Schema Manager, el plug-in TimeSeries y todo el software necesario para ejecutar OAT, incluyendo Apache, PHP y PDO_informix configurados previamente.

De forma alternativa está disponible OpenAdmin Tool Community Edition en la página de descargas de productos gratuitas de IBM Informix: https://www14.software.ibm.com/webapp/iwm/web/preLogin.do?lang=en_US&source=swg-informixfpd (se abre en una ventana nueva). OpenAdmin Tool Community Edition incluye OAT y los plug-ins opcionales, pero no incluye el software de soporte. Esta edición se debe instalar manualmente.

¿Cómo conectarse a un servidor de base de datos?

Inicio de sesión en un servidor de base de datos

Inicie la sesión en un servidor de bases de datos utilizando IBM OpenAdmin Tool (OAT) para Informix.

Para iniciar sesión en un servidor de base de datos:

  1. Vaya al URL raíz de OAT: http://nombre_servidor/openadmin.
  2. En la página Inicio de sesión para OAT, complete los campos Detalles de servidor.
  3. Pulse Iniciar sesión.

Se conectará al servidor de base de datos mediante OAT.

Sugerencia: De forma alternativa, si el servidor de bases de datos es miembro de un grupo OAT, puede conectarse al servidor de bases de datos iniciando la sesión en el grupo OAT. Un grupo OAT contiene información de conexión para servidores de bases de datos. Cuando inicia la sesión en un grupo OAT, puede conmutar a otros servidores de bases de datos del grupo sin iniciar la sesión otra vez.

Inicio de sesión en un servidor de base de datos en un grupo OAT

Conéctese a un servidor de base de datos iniciando la sesión en un grupo de conexiones de servidor de base de datos en IBM OpenAdmin Tool (OAT) para Informix.

Un grupo OAT contiene información de conexión para servidores de bases de datos. Cuando inicia la sesión en un grupo OAT, puede conmutar a otros servidores de bases de datos del grupo sin iniciar la sesión otra vez.

Para iniciar la sesión en un servidor de bases de datos de un grupo OAT:

  1. Vaya al URL raíz de OAT: http://nombre_servidor/openadmin.
  2. En la página de Inicio de sesión de OAT, en la lista Grupo OAT, seleccione el grupo OAT que contiene el servidor al que se desea conectar.
  3. Entre la contraseña para el grupo OAT.
  4. Pulse Obtener servidores.
  5. Seleccione el servidor de la lista de servidores. Los campos Detalles del servidor se rellenan con la información del servidor.
  6. En el campo Contraseña del área Detalles del servidor, acepte la contraseña si tiene una o escriba una válida, y luego pulse Inicio de sesión.

Conmutación a otro servidor de un grupo OAT

Conéctese a otro servidor de bases de datos de un grupo de conexiones de servidor de base de datos en IBM OpenAdmin Tool (OAT) para Informix.

Un grupo OAT contiene información de conexión para servidores de bases de datos. Cuando inicia la sesión en un grupo OAT, puede conmutar a otros servidores de bases de datos del grupo sin iniciar la sesión otra vez.

Para conmutar a otro servidor de bases de datos de un grupo OAT, seleccione el servidor de bases de datos en la lista Servidor en la parte superior de la página de OAT.

¿Cómo supervisar la actividad del servidor de bases de datos?

Supervisión de estado de servidor de base de datos

Supervise el estado de un servidor de bases de datos específico en tiempo real o vea el estado de todos los servidores de un grupo OAT al mismo tiempo.

Puede supervisar el estado de un servidor de bases de datos específico utilizando un panel de instrumentos. Este panel de instrumentos muestra el uso de memoria y espacio, el número de bloqueos, transacciones y consultas y las tablas con el mayor número de lecturas y grabaciones.

Puede ver el estado de todos los servidores de bases de datos de un grupo OAT en un panel de instrumentos de varios servidores. Desde este panel de instrumentos, puede ir a otras páginas de IBM OpenAdmin Tool (OAT) para Informix para obtener más información.

Para supervisar el estado del servidor de bases de datos:

  1. En el menú de OAT, pulse Centro de salud > Panel de instrumentos.
  2. Para renovar el panel de instrumentos, pulse el icono de renovación o cambie la velocidad de renovación.

Personalización del panel de instrumentos

Cambie la información de estado que el panel de instrumentos muestra para un servidor de bases de datos específico.

Requisito previo: Si el menú de Administración de IBM OpenAdmin Tool (OAT) para está protegido por contraseña, necesita la contraseña para realizar esta tarea.

El panel de instrumentos de OAT muestra información acerca de un servidor de bases de datos; por ejemplo, el uso de memoria y el número de transacciones. Puede especificar la información que se debe visualizar acerca de un servidor de bases de datos personalizando el panel de instrumentos.

Para personalizar el panel de instrumentos:

  1. En el menú Administración de OAT, pulse Gestor del panel de instrumentos.
  2. Para abrir el editor de diseño para un panel de instrumentos, efectúe una doble pulsación en el panel de instrumentos.

Modificar los umbrales de estado para el panel de instrumentos

Modifique los umbrales para los indicadores de estado que se muestran en el panel de instrumentos de varios servidores.

Requisito previo: El plug-in IBM Informix Health Advisor para OpenAdmin Tool (OAT) debe estar instalado.

El panel de instrumentos de varios servidores de IBM OpenAdmin Tool (OAT) para Informix visualiza indicadores de estado para los servidores de bases de datos de un grupo OAT. Por ejemplo, si el espacio libre para un espacio de base de datos en un servidor de bases de datos cae por debajo del umbral especificado, se muestra un indicador de aviso para el servidor de bases de datos.

Puede modificar los umbrales de estado para la memoria, el espacio y el porcentaje de bloques de datos del total de E/S. Estos umbrales son los valores predeterminados del plug-in Health Advisor.

Para modificar los umbrales de estado para el panel de instrumentos:

  1. En el menú de OAT, pulse Centro de salud > Health Advisor.
  2. En la página Perfil, verifique que el perfil actual es el perfil predeterminado.
  3. Pulse la pestaña Alarmas.
  4. Modifique los umbrales de cualquiera de estas alarmas:
    Umbral de alarma del plug-in Health Advisor Umbral de estado de panel de instrumentos
    Memoria libre de SO Memoria
    Espacio libre de espacio de base de datos Espacio
    Operaciones de E/S de bloque de datos E/S: Porcentaje de bloque de datos de total de E/S

Suprimir el registro de mensajes en línea

Suprimir o truncar el archivo de registro de mensajes en línea del servidor de bases de datos.

Requisito previo: Para esta acción, IBM OpenAdmin Tool (OAT) para Informix necesita IBM Informix versión 11.70.xC3.

Para suprimir el registro de mensajes en línea :

  1. En el menú de OAT, pulse Anotaciones > Anotación en línea.
  2. Realice una de las siguientes acciones:
    • Para suprimir el archivo de registro, pulse Suprimir.
    • Para suprimir los mensajes del archivo de registro, pero conservar el archivo de registro, pulse Truncar.

Rotación del registro de mensajes en línea

Habilitar, inhabilitar o modificar la rotación del archivo de registro de mensajes en línea para el servidor de bases de datos.

Requisito previo: Para esta acción, IBM OpenAdmin Tool (OAT) para Informix necesita IBM Informix versión 11.70.xC3.

Puede controlar el tamaño del archivo de anotaciones de mensajes en línea de rotación de registro automático de configuración. Cuando el registro de mensajes en línea rota, el servidor de bases de datos conmuta a un nuevo archivo de registro de mensajes en línea e incrementa en una unidad el número de ID para los archivos de registro anterior. Cuando se alcanza el número máximo de los archivos de anotaciones cronológicas, se suprime el archivo de registro con el ID más alto.

Para rotar el registro de mensajes en línea:

  1. En el menú de OAT, pulse Anotaciones > Anotación en línea.
  2. Pulse el separador Rotación del registro de mensajes en línea. Puede habilitar o inhabilitar la tarea, modificar la planificación o editar el parámetro que determina el número máximo de archivos de registro.

¿Cómo gestionar espacio del sistema?

Supervisión de uso de espacio

Revisar la información sobre el uso del espacio y los espacios de almacenamiento para un servidor de bases de datos. Para un espacio de almacenamiento específico, revisar información sobre el uso y la optimización del espacio, y los bloques, las tablas y los índices en el espacio.

Para supervisar el uso del espacio:

  1. En el menú de IBM OpenAdmin Tool (OAT) para Informix, pulse Administración de espacio > Almacenamiento. La página muestra paneles de información sobre el uso del espacio para el servidor de base de datos.
  2. Para mostrar más información, amplíe los paneles pulsando el botón ampliar del panel.
  3. Para mostrar información sobre un espacio de almacenamiento, seleccione el espacio de la lista. La página muestra paneles de información sobre el uso del espacio para el espacio.
  4. Para mostrar más información sobre la página seleccionada, amplíe los paneles.

Gestión del uso de espacio

Gestionar el espacio de almacenamiento de un servidor de base de datos.

Creación de un espacio

Crear un espacio de base de datos, un espacio de base de datos temporal, un espacio de Blob o un espacio de SB para un servidor de base de datos.

Cuando se crea un espacio utilizando IBM OpenAdmin Tool (OAT) para Informix, también se puede hacer que el espacio sea expandible. El servidor de la base de datos puede ampliar el tamaño del espacio de almacenamiento automáticamente cuando sea necesario, añadiendo o ampliando un bloque. El suministro de almacenamiento automático necesita IBM Informix versión 11.70.

Para crear un espacio:

  1. En el menú de OAT, pulse Administración de espacio > Almacenamiento.
  2. Pulse Acciones > Crear espacio.
  3. Complete los campos de la página Crear un espacio.

Configuración de la conectividad para clústeres de alta disponibilidad

Configure el daemon idsd en un servidor IBM Informix para permitir iniciar un servidor en un clúster de alta disponibilidad o añadir un servidor secundario de disco compartido (SDS).

Requisito previo: Necesita privilegios de root en el servidor Informix para este procedimiento.

Para utilizar IBM OpenAdmin Tool (OAT) para Informix para tareas de administración remotas incluido el inicio de un servidor en un clúster de alta disponibilidad o la adición de un servidor secundario, instale y configure los siguientes daemons en el servidor Informix:

  • Un daemon de servicio de Internet, por ejemplo los daemons de internet xinetd o inetd para UNIX y Linux.
  • idsd, un proceso de daemon que permite a OAT iniciar un servidor Informix remoto. El daemon reside en el mismo host que el servidor Informix y se debe estar ubicado en un directorio accesible al usuario informix. El daemon idsd se invoca mediante los daemons xinetd o inetd.
Importante: No puede utilizar las características de administración remotas en sistemas operativos Windows o MAC.

Para configurar un daemon de internet y el daemon idsd:

  1. Asegúrese de que el daemon idsd se ha instalado con Informix, en el directorio $INFORMIXDIR/bin. Asegúrese de que el propietario y el grupo son informix y que los permisos son 0754.

    Por ejemplo:

    [informix@myhost]/usr/informix/bin% ls -la idsd 
    -rwxr-xr--  1 informix informix 23717 Oct 15 16:34 idsd
  2. En el servidor, configure el cifrado para la conexión. Para Informix 11.70, utilice la herramienta GSKCmd que está disponible como una parte de IBM JRE 1.6 o posterior. Para versiones de Informix anteriores, utilice la herramienta GSKCmd que se incluye en el IBM Global Security Kit.
    1. Cree una base de datos de claves mediante los parámetros de línea de mandatos GSKCmd para su plataforma: 
      mandato -keydb -create -db nombre_bd_clave.kdb -pw contraseña -type cms -stash
      Por ejemplo: 
      gsk8cmd -keydb -create -db mydb.kdb -pw mypassw0rd -type cms -stash
    2. Cree un certificado autofirmado utilizando los siguientes parámetros de línea de mandatos de GSKCmd:
      mandato -cert -create -db nombre_bd_clave.kdb -pw contraseña -label texto_etiqueta
    -dn nombre_distinguido
    -size <1024 | 512 | 2048> -default_cert yes
      Donde: nombre_distinguido es un nombre distinguido de X.500. El nombre distinguido es una serie entrecomillada con el formato siguiente: CN=nombre común (obligatorio), O=organización, OU=unidad de organización, L=ubicación, ST=estado o provincia, C=país.

      Por ejemplo: 

      gsk8cmd -cert -create -db mydb.kdb -pw mypassw0rd -label mycertificatelabel
    -dn "CN=lenexa.ibm.com,O=ibm,OU=IBM HTTP Server,ST=KS,C=US" 
    -size 1024 -default_cert yes
  3. Abra el archivo de configuración xinetd o inetd para editarlo y añadir una entrada de servicio para el daemon idsd.
    Importante: Si el parámetro de configuración GSKIT_VERSION se establece en una versión distinta del valor predeterminado de GSKit, añada el argumento de programa de servidor -g NUMBER a la entrada de servicio xinetd o inetd para especificar la versión de GSKit.
    • Para el daemon de servidor xinetd: /etc/xinetd.conf. 
      service idsd
             {
             disable = no
             socket_type = stream 
             protocol = tcp
             user = root
             wait = no 	    
             server = /usr/informix/bin/idsd
             server_args = -l /usr/informix/idsd.log 
                 -k /usr/informix/idsd -n idsd_label
             }
    • Para el daemon de servidor inetd en UNIX: /etc/inetd.conf: 
      idsd stream tcp nowait root /usr/informix/bin/idsd idsd -l /usr/informix/idsd.log  -k /usr/informix/idsd -n idsd_label
  4. Abra el archivo /etc/services para editarlo y añadir un servicio denominado idsd. Por ejemplo: 
    idsd 10000/tcp
    En este ejemplo, se utiliza el número de puerto 10000, pero puede utilizarse cualquier otro puerto que sea superior a 1024 y que no esté ya utilizado. Sin embargo, el nombre debe ser idsd y el protocolo tcp.
  5. Reinicie los daemon de servicio xinetd o inetd. En la mayoría de las implementaciones, puede reiniciar enviando la señal SIGHUP al proceso inetd o al xinetd. Sin embargo, en algunas plataformas Linux, puede ejecutar el siguiente mandato como root: 
    service xinetd restart

  1. En el menú de IBM OpenAdmin Tool (OAT) para Informix, pulse Administración de servidor > Actualizar estadísticas automáticamente > Config.
  2. Basándose en el rendimiento y los requisitos del sistema, determine si los valores predeterminados son válidos para el sistema.
    1. Para especificar el tiempo máximo entre reconstrucciones de estadísticas, entre el número de días para el parámetro AUS_AGE, por ejemplo 30, 60 o 90.
    2. Para especificar qué porcentaje de cambios en una tabla inicia una nueva generación de estadísticas, introduzca un nombre de parámetro, como por ejemplo, 20 ó 50, en el parámetro AUS_CHANGE.
    3. Para especificar si AUS utiliza las reglas de directrices recomendadas, seleccione On u Off en la lista del parámetro AUS_AUTO_RULES. El valor predeterminado es On.
    4. Para especificar el número de filas por debajo del cual siempre se reconstruyen las estadísticas, entre un número para el parámetro AUS_SMALL_TABLES. El valor predeterminado es 100.
    5. Para especificar la prioridad de PDQ, entre un número de -1 a 100 para el parámetro AUS_PDQ.El valor predeterminado es 10.   3. Pulse Guardar para guardar los cambios.
  4. Para determinar si las tareas AUS están habilitadas, pulse el separador Información y, a continuación, pulse en estas tareas:
    • Evaluación automática de la actualización de las estadísticas
    • Renovación automática de la actualización de las estadísticas
      Nota: Si no hay trabajo que la tarea de renovación de actualización automática de estadísticas pueda realizar, la tarea se inhabilita a sí misma.     Se visualiza la página Detalles de la tarea.
  5. Desplácese hasta el campo Habilitar tarea y asegúrese de que está seleccionado.
  6. Especifique la frecuencia con las que se ejecutan las tareas con los campos que establecen la ventana de ejecución.Puesto que AUS es una operación que utiliza muchos recursos, se puede definir una ventana de ejecución específica. La definición de ventana de ejecución incluye una hora de inicio y una de fin y los días de la semana que van a ejecutarse. De forma predeterminada, el periodo de tiempo de ejecución es de 1:00 a.m. a 5:00 a.m. diariamente.
  7. Pulse Guardar para guardar los cambios.

¿Cómo gestionar tareas del Planificador?

Adición de tareas del Planificador

Puede crear tareas del Planificador que ejecuten tareas o sensores a frecuencias y horas específicas.

  • Las tareas son un medio para ejecutar un trabajo específico a una hora o intervalo específicos.
  • Los sensores recopilan y guardan información.

Adición de una nueva tarea

Importante: La hora a la que crea la tarea afecta la primera ejecución de la tarea.
  • Si la hora actual está entre la hora de inicio y la hora de detención especificadas, la tarea se ejecuta a la hora actual. Las ejecuciones siguientes tienen lugar en los intervalos que especifica el valor de frecuencia. Por ejemplo, si la hora está dentro de la hora de inicio y detención y el valor de frecuencia está establecido en 15 minutos, la primera ejecución de la tarea se inicia inmediatamente después de que se planifique la tarea y la siguiente ejecución tiene lugar 15 minutos más tarde.
  • Sin embargo, si la hora no está dentro de la hora de inicio y detención , la tarea espera a ejecutarse hasta la siguiente hora de inicio, ya sea en el mismo día o al día siguiente. Por ejemplo, si la hora actual es 1:00 AM y la siguiente de inicio es 7:00 PM (19:00), la tarea se iniciaría a las 19:00 del mismo día.

Para añadir una tarea nueva:

  1. Pulse Añadir una tarea nueva en la página Planificador de tareas.
  2. PulseTarea en ¿Qué tipo de tarea desea añadir? y luego, pulse Siguiente.
  3. Introduzca un nombre en el campo Nombre de tarea.Por ejemplo,: MiTarea.
  4. Seleccione una categoría de la lista Grupo de tareas.
  5. Añada una descripción de la tarea nueva en el campo de texto y pulse Siguiente.
  6. Especifique la franja de tiempo permitido en la que la tarea puede ejecutarse, por ejemplo, entre las diez (10.00) de la mañana y las seis (18.00) de la tarde, cada hora, cada día.
    1. Especifique la hora de inicio del intervalo de tiempo, utilizando el formato de reloj de 24 horas: hh:mm:ss. Por ejemplo: 10:00:00
    2. Especifique la hora de detención del intervalo de tiempo, utilizando el formato de reloj de 24 horas: hh:mm:ss. Por ejemplo: 18:00:00. También puede especificar NEVER.
    3. Especifique la frecuencia con la que se puede ejecutar la tarea durante el intervalo de tiempo permitido, en días, horas y minutos: d Días h Horas m Minutos. Por ejemplo: 0 Días 1 Horas 00 Minutos.
  7. Pulse Siguiente.
  8. Especifique la sentencia de mandatos que va a ejecutar la tarea.Por ejemplo, si desea suprimir varias tareas, puede introducir la siguiente sentencia: 
    suprimir de ph_task donde tk_group='MISC'
  9. Pulse Siguiente.Aparece la página de información.
  10. Verifique las entradas de cada selección:
    • Si las entradas son correctas, pulse Finalizar para confirmar y crear la tarea.
    • Pulse Volver para volver a las páginas anteriores si desea corregir algún valor que no sea correcto.

Adición de un sensor nuevo

  1. En la página Planificación de tareas, pulse Añadir una tarea nueva.
  2. Pulse Sensor en ¿Qué tipo de tarea desea añadir? y pulse Siguiente.
  3. Introduzca un nombre en el campo Nombre de sensor. Por ejemplo: MiSensor.
  4. Seleccione una categoría de la lista Grupo de sensores.
  5. Añada una descripción de la tarea nueva en el campo de texto y pulse Siguiente.
  6. Especifique la franja horaria permitida en la que el sensor puede ejecutarse, por ejemplo, entre las 10:00 AM y 6:00 PM, cada día, cada hora.
    1. Especifique la hora de inicio del intervalo de tiempo con formato de 24 horas: hh:mm:ss. Por ejemplo: 10:00:00
    2. Especifique la hora de detención del intervalo de tiempo con formato de 24 horas: hh:mm:ss. Por ejemplo: 18:00:00
    3. Especifique la frecuencia con la que la tarea puede ejecutarse durante el intervalo de tiempo permitido, en días, horas y minutos: d Days h Hours m Minutes.Por ejemplo: 0 Days 1 Hours 00 Minutes.
    4. Especifique la hora de supresión de datos, en días, horas y minutos: d Days h Hours m Minutes.Por ejemplo: 1 Days 0 Hours 0 Minutes.
  7. Pulse Siguiente.
  8. En el campo Nombre de la tabla de resultados, especifique el nombre de la tabla de resultados para mantener los datos del sensor. Por ejemplo: MiResultados
  9. Pulse Siguiente.
  10. Complete la sentencia SQL para la tabla de resultados introduciendo los nombres de columnas y definiciones en el campo Sentencia SQL. Por ejemplo, si desea recopilar los códigos de retorno de taskA (con task_id 10) en la tabla TaskA_ReturnCode, puede introducir la siguiente sentencia SQL: 
    crear tabla TaskA_ReturnCode(task_name varchar(20), errorcode integer)
  11. Especifique la sentencia de mandatos que va a ejecutar el sensor.Por ejemplo: 
    insert into TaskA_ReturnCode values('TaskA', (select run_retcode from ph_run where run_task_id=10))
  12. Verifique las entradas de cada selección.
    • Si las entradas son correctas, pulse Finalizar para confirmar y crear el sensor.
    • Pulse Volver para volver a las páginas anteriores si desea actualizar algún valor que no sea correcto.

  4. Determine qué hacer con la tabla.
    • Pulse Sí para suprimir la tabla de resultados y la tarea.
    • Pulse No para mantener la tabla cuando se suprima la tarea.

Configuración de HTTPS para OAT (Linux)

Habilite HTTPS para proteger el servidor web de IBM OpenAdmin Tool (OAT) para Informix frente a escuchas no autorizadas, manipulación indebida, falsificación de mensajes e interferencias con la red.

Requisito previo: OpenSSL se debe instalar en el sistema donde está instalado el servidor Apache para OAT. Puede descargar e instalar OpenSSL del sitio web de OpenSSL Project: www.openssl.org.

Cuando HTTPS está habilitado, los mensajes de los clientes OAT se cifran antes de enviarse al servidor web de OAT. El cifrado impide que usuarios no autorizados escuchen por la línea y roben información delicada. Cuando HTTPS está habilitado, los clientes OAT también pueden autentificarse en el host OAT para que los usuarios no autorizados no puedan engañar a los clientes OAT con servidores web falsos.

Importante: HTTPS sólo cifra la comunicación entre el servidor web OAT y el cliente. No cifra la comunicación entre un servidor de bases de datos IBM Informix y el servidor web. IBM Informix versión 9.4 y posterior permite el cifrado de datos entre el servidor Informix y el servidor web OAT utilizando un módulo de soporte de comunicación de cifrado. Para obtener más información sobre los servidores de cifrado e Informix, lea "Encryption over the wire with IDS 9.40" en el sitio web de developerWorks®.

Cuando instale OAT con IBM Informix Client Software Development Kit (Client SDK), el programa de instalación proporciona el software necesario para ejecutar OAT, incluyendo Apache, PHP y PDO_informix configurados previamente. En Linux, el servidor web de Apache que se instala con OAT está habilitado para el módulo de criptografía mod_ssl. Este módulo es necesario para habilitar HTTPS.

Para habilitar HTTPS en OAT en Linux, realice los pasos siguientes:

  1. Cree una clave de cifrado y un certificado para el servidor web para que los clientes OAT puedan autenticar el servidor web basándose en el certificado.
  2. Configure Apache para habilitar HTTPS.

Para obtener información adicional acerca de la habilitación de HTTPS, lea "Securing the OpenAdmin Tool for Informix web server with HTTPS" en el sitio web de developerWorks.

Creación una clave de cifrado

Las claves se utilizan en el cifrado y descifrado. Éstas suelen venir en pares, la clave pública y la clave privada. Las claves públicas se utilizan para cifrar mensajes y las claves privadas se utilizan para descifrar mensajes. Un certificado es un documento que autentica un servidor web. Un servidor web con HTTPS habilitado tiene un certificado, firmado por una entidad emisora de certificados de confianza, para verificar el servidor web. Después de habilitar un servidor web para HTTPS, los clientes eligen si se debe establecer una conexión normal o una conexión segura con el servidor web. Para establecer una conexión normal, los tipos de cliente http://url_servidor web en el navegador web. Para establecer una conexión segura, los tipos de cliente https://url_servidor web.

Para generar pares de claves privada y pública y el certificado, utilice el archivo ejecutable openssl en el directorio bin de la instalación OpenSSL.

Cuando ejecute el mandato para generar la clave privada, se le solicitará que entre una frase de contraseña para proteger la clave privada. Se debe volver a escribir esta frase de contraseña cada vez que ejecute un mandato que requiere el uso de la clave privada. Por ejemplo, debe entrar la frase de contraseña cuando inicie el servidor web Apache web o utilizar la clave privada para generar un certificado. En Linux, por razones de seguridad, utilice una frase de contraseña para proteger y cifrar la clave privada.

Para generar una clave privada, ejecute el siguiente mandato en el sistema donde está instalado OpenSSL: 

openssl genrsa -des3 -out privkey.pem 2048

La clave privada se crea en el archivo privkey.pem. Almacene este archivo en una ubicación segura porque contiene la clave de descifrado para el servidor web. Cuando genere el certificado, este archivo se utiliza para generar la clave pública asociada, que se incluye en el certificado.

Creación de un certificado

Después de crear una clave privada, cree un certificado para autenticar la identidad del servidor web. Para generar un certificado, cree una solicitud de firma de certificado y envíe la solicitud de firma de certificado a una entidad emisora de certificados de confianza. La entidad emisora de certificados emite un certificado.

Para generar una solicitud de firma de certificado, ejecute el mandato siguiente:

openssl req -new -key privkey.pem -out cert.csr

Para obtener más información sobre solicitudes de firma de certificado, póngase en contacto con la entidad emisora de certificados.

De forma alternativa, puede crear un certificado autofirmado sin pasar por la entidad emisora de certificados aunque los certificados autofirmados no se recomiendan.

Para generar una certificado autofirmado, ejecute el mandato siguiente:

openssl req -new -x509 -key privkey.pem -out cacert.pem -days 1095

OpenSSL le solicita la información. El certificado se almacena en el archivo cacert.pem. Este archivo se muestra a los clientes web para verificar la identidad. También incluye la clave pública para clientes web, para que puedan cifrar las comunicaciones.

Para obtener más información acerca de las claves de cifrado, consulte la documentación de OpenSSL.

Configuración de Apache para habilitar HTTPS

Para habilitar HTPPS, actualice el archivo de configuración de Apache y el archivo de configuración de SSL de Apache:

  1. Localice el archivo de configuración de Apache. La vía de acceso incluye la versión de Apache, por ejemplo: 
    /oat/installation/directory/Apache_2.4.2/conf/httpd.conf
  2. En el archivo de configuración de Apache, elimine el comentario de esta línea: #Include conf/extra/httpd-ssl.conf .

    Esta línea se comenta de forma predeterminada. Cuando se elimina el comentario, el archivo de configuración SSL de Apache se incluye en el archivo httpd.conf.

  3. Localice el archivo de configuración SSL de Apache. La vía de acceso incluye la versión de Apache, por ejemplo: 
    /oat/installation/directory/Apache_2.4.2/conf/extra/httpd-ssl.conf
  4. En el archivo de configuración SSL de Apache, verifique que el número de puerto SSL esté establecido en un puerto disponible.

    HTTPS necesita un puerto SSL independiente. De forma predeterminada, el número de puerto SSL 443 se establece en 443. Si este puerto no está disponible, cambie el puerto SSL en la directiva Listen y la sección de host virtual.

  5. En el archivo de configuración SSL de Apache, verifique que la directiva SSLCertificateKeyFile indique la ubicación correcta del archivo de clave privada que ha creado: privkey.pem
  6. Verifique que la directiva SSLCertificateFile indique la ubicación correcta del archivo de certificado que ha creado: cacert.pem.
  7. Opcional: Para controlar qué cifrados acepta el servidor web, edite la directiva SSLCipherSuite. Para obtener información sobre la directiva SSLCipherSuite, consulte la documentación de mod_ssl de Apache.

Para obtener más información sobre las configuraciones de HTTPS, consulte el sitio web de Apache: www.apache.org.

Para establecer una conexión normal, los tipos de cliente http://url_servidor web en el navegador web. Para establecer una conexión segura, los tipos de cliente https://url_servidor web.

Para generar pares de claves privadas y públicas y el certificado, utilice el archivo ejecutable openssl en el directorio bin de la nueva instalación de servidor web de Apache.

Cuando ejecute el mandato para generar la clave privada, se le solicitará que entre una frase de contraseña para proteger la clave privada. En Windows, Apache no soporta una frase de contraseña de una clave privada, por lo que debe eliminar la frase de contraseña como se explica en los pasos siguientes.

Para crear una clave privada:

  1. Para generar una clave privada, ejecute el siguiente mandato en el sistema donde está instalado el nuevo servidor web de Apache: 
    openssl genrsa -des3 -out privkey.pem 2048
  2. Para eliminar la frase de contraseña, ejecute el siguiente mandato: 
    openssl rsa -in privkey.pem -out privkey_nopassphrase.pem

Se crea una versión no cifrada de la clave privada en el archivo privkey_nopassphrase.pem. Almacene este archivo en una ubicación segura porque contiene la clave privada en un formato no cifrado. Cuando genere el certificado, este archivo se utiliza para generar la clave pública asociada, que se incluye en el certificado.

Creación de un certificado

Después de crear la clave privada, cree un certificado para autenticar la identidad del servidor web. Para crear un certificado, genere una solicitud de firma de certificado y envíe la solicitud de firma de certificado a una entidad emisora de certificados de confianza. La entidad emisora de certificados emite un certificado.

Para generar una solicitud de firma de certificado, ejecute el mandato siguiente:

openssl req -new -key privkey_nopassphrase.pem -out cert.csr

Para obtener más información sobre solicitudes de firma de certificado, póngase en contacto con la entidad emisora de certificados.

De forma alternativa, puede crear un certificado autofirmado sin pasar por la entidad emisora de certificados aunque los certificados autofirmados no se recomiendan.

Para generar una certificado autofirmado, ejecute el mandato siguiente:

openssl req -new -x509 -key privkey_nopassphrase.pem -out cacert.pem -days 1095

OpenSSL le solicita la información. El certificado se almacena en el archivo cacert.pem. Este archivo se muestra a los clientes web para verificar la identidad. También incluye la clave pública para clientes web, para que puedan cifrar las comunicaciones.

Para obtener más información acerca de las claves de cifrado, consulte la documentación de OpenSSL.

Configuración de Apache para habilitar HTTPS

Para habilitar HTPPS, actualice el archivo de configuración de Apache y el archivo de configuración de SSL de Apache:

  1. Localice el archivo de configuración de Apache. La vía de acceso incluye la versión de Apache, por ejemplo: 
    /oat/installation/directory/Apache_2.2.22/conf/httpd.conf
  2. En el archivo de configuración de Apache, elimine el comentario de esta línea: #Include conf/extra/httpd-ssl.conf .

    Esta línea se comenta de forma predeterminada. Cuando se elimina el comentario, el archivo de configuración SSL de Apache se incluye en el archivo httpd.conf.

  3. Localice el archivo de configuración SSL de Apache. La vía de acceso incluye la versión de Apache, por ejemplo: 
    /oat/installation/directory/Apache_2.2.22/conf/extra/httpd-ssl.conf
  4. En el archivo de configuración SSL de Apache, verifique que el número de puerto SSL esté establecido en un puerto disponible.

    HTTPS necesita un puerto SSL independiente. De forma predeterminada, el número de puerto SSL 443 se establece en 443. Si este puerto no está disponible, cambie el puerto SSL en la directiva Listen y la sección de host virtual.

  5. En el archivo de configuración SSL de Apache, verifique que la directiva SSLCertificateKeyFile indique la ubicación correcta del archivo de clave privada que ha creado: privkey_nopassphrase.pem
  6. Verifique que la directiva SSLCertificateFile indique la ubicación correcta del archivo de certificado que ha creado: cacert.pem.
  7. Opcional: Para controlar qué cifrados acepta el servidor web, edite la directiva SSLCipherSuite. Para obtener información sobre la directiva SSLCipherSuite, consulte la documentación de mod_ssl de Apache.

Para obtener más información sobre las configuraciones de HTTPS, consulte el sitio web de Apache: www.apache.org.

Prueba de la configuración

Debe reiniciar el servidor web para que entren en vigor los cambios de HTTPS. Reinicie el servidor web utilizando apachemonitor.exe o los atajos de menú Inicio.

Después de reiniciar el servidor web, abra OAT en un navegador web utilizando este URL: https://nombrehost:númeropuerto_ssl/openadmin. Se le solicitará que lea y acepte el certificado para el servidor web OAT antes de que se visualice la página de inicio de sesión de OAT.
