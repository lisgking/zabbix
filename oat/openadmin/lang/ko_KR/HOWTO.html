IBM OpenAdmin Tool(OAT) for Informix

IBM® OpenAdmin Tool(OAT) for Informix®는 IBM Informix 데이터베이스 서버의 성능을 관리 및 분석하는 웹 응용프로그램입니다. 웹 서버의 단일 OAT 설치에서 여러 데이터베이스 서버 인스턴스를 관리할 수 있습니다. 브라우저를 통해 웹 서버에 액세스하여 모든 데이터베이스 서버를 관리할 수 있습니다.

OAT를 사용하면 데이터베이스 서버의 성능을 모니터하고 분석할 수 있습니다.

  • 자세한 상태 보고서, 경고 정보 및 메시지 로그를 사용하여 데이터베이스 서버의 상태를 모니터할 수 있습니다.
  • 자세한 성능 히스토리 차트를 사용하여 성능 통계를 계속 모니터할 수 있습니다.
  • 스케줄러가 수집한 데이터를 모니터할 수 있습니다. 스케줄러는 관리 작업을 실행하고 SQL문을 사용하여 정보를 수집합니다.
  • 데이터베이스 서버의 데이터베이스 및 테이블에서 시스템 유효성 검증을 실행할 수 있습니다.
  • 시스템 보고서 및 onstat 명령을 실행할 수 있습니다.
  • SQL문에서 드릴 다운하여 성능 세부사항을 검토하고 SQL문을 최적화할 수 있습니다.

OAT를 사용하면 다음의 관리 작업을 수행할 수 있습니다.

  • 구성, 사용자 권한 및 가상 프로세서를 포함한 데이터베이스 서버를 관리할 수 있습니다.
  • 스케줄러의 작업을 작성하고 모니터할 수 있습니다.
  • 쿼리 성능을 최적화하기 위해 통계를 자동으로 갱신하는 AUS(Auto Update Statistics) 유지보수 시스템을 관리할 수 있습니다.
  • OpenAdmin Tool(OAT)용 IBM Informix 스키마 관리자 플러그인을 사용하여 데이터베이스 및 테이블 오브젝트를 관리할 수 있습니다.
  • 스토리지 영역 및 데이터베이스 서버의 복구 로그를 관리할 수 있습니다.
  • 테이블을 압축 및 일원화하여 영역을 최적화할 수 있습니다.
  • 데이터베이스 서버의 스토리지 영역을 백업할 수 있습니다.
  • SQL문을 실행하고 테이블에서 표준 SQL 조작을 수행할 수 있습니다.
  • 고가용성 클러스터를 모니터하고 관리할 수 있습니다.
  • OpenAdmin Tool(OAT)용 IBM Informix 복제 플러그인을 사용하여 Enterprise Replication을 모니터하고 관리할 수 있습니다.
  • OpenAdmin Tool(OAT)용 IBM Informix 시계열 플러그인을 사용하여 테이블, 인덱스, 가상 테이블, 시계열 하위 형식, 컨테이너, 달력 등 시계열과 관련된 데이터베이스 오브젝트를 모니터하고 관리할 수 있습니다.

OAT는 오픈 소스 소프트웨어 응용프로그램입니다. 복제 플러그인, 스키마 관리자 플러그인 및 시계열 플러그인은 IBM 비보증 소프트웨어 프로그램입니다. 즉, 오픈 소스 소프트웨어 응용프로그램이 아닙니다. OAT의 추가 플러그인을 작성하여 필요한 기능을 추가할 수 있습니다.

OAT는 IBM Informix의 무료 제품 다운로드 페이지(https://www14.software.ibm.com/webapp/iwm/web/preLogin.do?lang=en_US&source=swg-informixfpd)에서 다운로드할 수 있습니다.

데이터베이스 서버 연결 방법

데이터베이스 서버에 로그인

IBM OpenAdmin Tool(OAT) for Informix를 사용하여 데이터베이스 서버에 로그인할 수 있습니다.

데이터베이스 서버에 로그인하려면 다음을 수행하십시오.

  1. 다음 OAT 루트 URL(http://server name/openadmin)로 이동하십시오.
  2. OAT 로그인 페이지에서 서버 세부사항 필드를 채우십시오.
  3. 로그인을 클릭하십시오.

OAT를 통해 데이터베이스 서버에 연결됩니다.

팁: 또는, 데이터베이스 서버가 OAT 그룹의 멤버인 경우 OAT 그룹에 로그인하여 데이터베이스 서버에 연결할 수 있습니다. OAT 그룹에는 데이터베이스 서버에 대한 연결 정보가 있습니다. OAT 그룹에 로그인하면 다시 로그인하지 않아도 그룹의 다른 데이터베이스 서버로 전환할 수 있습니다.

OAT 그룹의 데이터베이스 서버에 로그인

IBM OpenAdmin Tool(OAT) for Informix의 데이터베이스 서버 연결 그룹에 로그인하여 데이터베이스 서버에 연결할 수 있습니다.

OAT 그룹에는 데이터베이스 서버에 대한 연결 정보가 있습니다. OAT 그룹에 로그인하면 다시 로그인하지 않아도 그룹의 다른 데이터베이스 서버로 전환할 수 있습니다.

OAT 그룹의 데이터베이스 서버에 로그인하려면 다음을 수행하십시오.

  1. 다음 OAT 루트 URL(http://server name/openadmin)로 이동하십시오.
  2. OAT 로그인 페이지의 OAT 그룹 목록에서 연결할 서버가 포함된 OAT 그룹을 선택하십시오.
  3. OAT 그룹의 암호를 입력하십시오.
  4. 서버 가져오기를 클릭하십시오.
  5. 서버 목록에서 서버를 선택하십시오. 서버 세부사항 필드가 서버 정보로 채워집니다.
  6. 서버 세부사항 영역의 암호 필드에서 암호가 제공된 경우 이를 수락하거나 유효한 암호를 입력한 후, 로그인을 클릭하십시오.

  • 관련 태스크
  • OAT 그룹 추가
    IBM OpenAdmin Tool(OAT) for Informix에서 데이터베이스 서버 연결 그룹을 정의할 수 있습니다.

OAT 그룹의 다른 서버로 전환

IBM OpenAdmin Tool(OAT) for Informix의 데이터베이스 서버 연결 그룹에 있는 다른 데이터베이스 서버에 연결할 수 있습니다.

OAT 그룹에는 데이터베이스 서버에 대한 연결 정보가 있습니다. OAT 그룹에 로그인하면 다시 로그인하지 않아도 그룹의 다른 데이터베이스 서버로 전환할 수 있습니다.

OAT 그룹의 다른 데이터베이스 서버로 전환하려면 OAT 페이지의 맨 위에 있는 서버 목록에서 데이터베이스 서버를 선택하십시오.

  • 관련 태스크
  • OAT 그룹 추가
    IBM OpenAdmin Tool(OAT) for Informix에서 데이터베이스 서버 연결 그룹을 정의할 수 있습니다.

OpenAdmin Tool(OAT) for Informix 메뉴에서 Health Center > 대시보드를 클릭하십시오.

경고 및 이벤트 경보를 모니터하려면 다음을 수행하십시오.

OpenAdmin Tool(OAT) for Informix 메뉴에서 Health Center > 경고를 클릭하십시오.

  • 관련 참조
  • Event Alarm Parameters (Administrator's Reference)
  • Event Alarm IDs (Administrator's Reference)
  • The ph_alert Table (Administrator's Reference)

전제조건: 영역 최적화 동작의 경우, IBM OpenAdmin Tool(OAT) for Informix에는 IBM Informix 버전 11.50.xC6 이상이 필요합니다. 조각 모음 동작의 경우, OAT에는 Informix 버전 11.70이 필요합니다.

여유 공간을 DBSPACE로 리턴하려면 다음을 수행하십시오.

  1. OAT 메뉴에서 영역 관리 > 스토리지를 클릭하십시오.
  2. 데이터베이스 서버를 선택한 상태에서 테이블 및 인덱스 패널을 펼치십시오.
  3. 테이블 및 테이블 분할을 선택하십시오.
  4. 동작 > 영역 최적화를 클릭하십시오.
  5. 수축을 선택하십시오.
  6. 옵션: 추가적인 동작을 선택하십시오.
    • 테이블 및 테이블 분할을 압축하려면 압축을 선택하십시오.
    • 여유 공간을 통합하려면 리팩을 선택하십시오.
    • 익스텐트 수를 줄이려면 익스텐트 조각 모음을 선택하십시오.

고가용성 클러스터에 대한 연결 구성

IBM Informix 서버에서 idsd 데몬을 설정하여 고가용성 클러스터의 서버를 시작하거나 공유 디스크 보조(SDS) 서버를 추가할 수 있습니다.

전제조건: 이 프로시저를 수행하려면 Informix 서버에 대해 루트 권한이 필요합니다.

고가용성 클러스터의 서버를 시작하거나 보조 서버를 추가하는 등 원격 관리 작업을 위해 IBM OpenAdmin Tool(OAT) for Informix를 사용하려면 Informix 서버에서 다음 데몬을 설치 및 구성하십시오.

  • 인터넷 서비스 데몬(UNIX 및 Linux의 경우 xinetd 또는 inetd 인터넷 데몬)
  • OAT에서 원격 Informix 서버를 시작할 수 있게 하는 데몬 프로세스인 idsd. 이 데몬은 Informix 서버와 동일한 호스트에 존재하며 사용자 informix가 액세스할 수 있는 디렉토리에 있어야 합니다. idsd 데몬은 xinetd 또는 inetd 데몬을 통해 호출됩니다.
중요사항: Windows 또는 MAC OS에서는 원격 관리 기능을 사용할 수 없습니다.

인터넷 데몬과 idsd 데몬을 구성하려면 다음을 수행하십시오.

  1. idsd 데몬이 Informix와 함께 $INFORMIXDIR/bin 디렉토리에 설치되어 있는지 확인하십시오. 소유자 및 그룹이 informix이고 권한이 0754인지 확인하십시오.

    예를 들면 다음과 같습니다. 

    [informix@myhost]/usr/informix/bin% ls -la idsd          
    -rwxr-xr--  1 informix informix 23717 Oct 15 16:34 idsd
  2. 서버에서 연결에 대한 암호화를 설정하십시오. Informix 11.70의 경우, IBM JRE1.6 또는 이후 버전의 파트로 사용 가능한 GSKCmd 도구를 사용하십시오. 이전 Informix 버전의 경우, IBM Global Security Kit에 포함되어 있는 GSKCmd 도구를 사용하십시오.
    1. 사용자 플랫폼에서 다음 GSKCmd 명령행 매개변수를 사용하여 키 데이터베이스를 작성하십시오. 
      command -keydb -create -db keydb_name.kdb -pw password -type cms -stash
      예를 들면 다음과 같습니다. 
      gsk8cmd -keydb -create -db mydb.kdb -pw mypassw0rd -type cms -stash
    2. 다음 GSKCmd 명령행 매개변수를 사용하여 자체 서명 인증서를 작성하십시오. 
      command -cert -create -db keydb_name.kdb -pw password -label label_text
    -dn distinguished name 
    -size <1024 | 512 | 2048> -default_cert yes
      여기서, distinguished name은 X.500 식별 이름입니다. distinguished name은 다음 형식의 인용 문자열입니다. (CN=common name (필수), O=organization, OU=organization unit, L=location, ST=state 또는 province, C=country.)

      예를 들면 다음과 같습니다. 

      gsk8cmd -cert -create -db mydb.kdb -pw mypassw0rd -label mycertificatelabel
    -dn "CN=lenexa.ibm.com,O=ibm,OU=IBM HTTP Server,ST=KS,C=US" 
    -size 1024 -default_cert yes
  3. 편집용 xinetd 또는 inetd 구성 파일을 열고 idsd 데몬의 서비스 엔트리/입력을 추가하십시오.
    중요사항: GSKIT_VERSION 구성 매개변수가 GSKit의 비기본 버전으로 설정되어 있는 경우, -g NUMBER 서버 프로그램 인수를 xinetd 또는 inetd 서비스 엔트리/입력에 추가하여 GSKit 버전을 지정하십시오.
    • xinetd 서버 데몬의 경우: /etc/xinetd.conf. 
      service idsd
             {
             disable = no
             socket_type = stream 
             protocol = tcp
             user = root
             wait = no 	    
             server = /usr/informix/bin/idsd
             server_args = -l /usr/informix/idsd.log -k /usr/informix/idsd -n idsd_label
             }
    • UNIX의 inetd 서버 데몬의 경우: /etc/inetd.conf: 
      idsd stream tcp nowait root /usr/informix/bin/idsd idsd -l /usr/informix/idsd.log  -k /usr/informix/idsd -n idsd_label
  4. /etc/services 파일을 열어 idsd 이름의 서비스를 편집 및 추가하십시오. 예를 들면 다음과 같습니다. 
    idsd 10000/tcp
    이 예에서 포트 번호 10000을 사용했으나 1024보다 큰 포트 번호(사용 중이 아닌)를 사용할 수 있습니다. 그러나 이름은 idsd이어야 하고, 프로토콜은 tcp이어야 합니다.
  5. xinetd 또는 inetd 서비스 데몬을 재시작하십시오. 대부분 구현에서 inetd 또는 xinetd 프로세스 실행에 대한 SIGHUP 신호를 전송하여 재시작할 수 있습니다. 그러나 일부 Linux 플랫폼에서는 루트로 다음 명령을 실행할 수 있습니다. 
    service xinetd restart

고가용성 클러스터를 찾으려면 다음을 수행하십시오.

  1. OAT 메뉴에서 복제 > 클러스터를 클릭하십시오.
  2. 클러스터 찾기를 클릭하십시오.

    클러스터가 발견되면 발견된 클러스터 페이지가 표시되어 클러스터의 서버(기본, 고가용성 데이터 복제(HDR), 공유 디스크 보조(SDS) 및 원격 단독 보조(RSS))를 나열합니다.

  3. 클러스터의 기본 이름을 수락하거나 새로운 고유한 이름을 입력하십시오. 발견된 각 클러스터에 대해 이 단계를 반복하십시오.

클러스터 토폴로지 보기

고가용성 클러스터의 토폴로지를 봅니다.

전제조건: IBM OpenAdmin Tool(OAT) for Informix에서 클러스터의 이름을 찾으십시오. 고가용성 클러스터 찾기의 내용을 참조하십시오.

클러스터의 토볼로지를 보려면 다음을 수행하십시오.

  1. OAT 메뉴에서 복제 > 클러스터를 클릭하십시오.
  2. 클러스터에서 클러스터를 선택하십시오. 클러스터 토폴로지 및 서버 목록이 표시됩니다.

    선택한 클러스터 토폴로지는 고가용성 데이터 복제(HDR), 공유 디스크 보조(SDS) 및 원격 단독 보조(RSS) 서버를 포함한 보조 서버와 기본 서버 간의 관계를 표시합니다.

클러스터의 서버 상태 보기

고가용성 클러스터의 서버 상태를 봅니다.

전제조건: IBM OpenAdmin Tool(OAT) for Informix에서 클러스터의 이름을 찾으십시오. 고가용성 클러스터 찾기의 내용을 참조하십시오.

클러스터의 서버 상태를 보려면 다음을 수행하십시오.

  1. OAT 메뉴에서 복제 > 클러스터를 클릭하십시오.
  2. 클러스터에서 클러스터를 선택하십시오. 클러스터 토폴로지 및 서버 목록이 표시됩니다.

    목록에는 고가용성 데이터 복제(HDR), 공유 디스크 보조(SDS) 및 원격 단독 보조(RSS) 서버를 포함한 보조 서버와 기본 서버의 상태가 표시됩니다.

SDS 서버 추가

공유 디스크 보조(SDS) 서버 연결을 고가용성 클러스터에 추가합니다.

전제조건:
  • IBM OpenAdmin Tool(OAT) for Informix에서 클러스터의 이름을 찾으십시오. 고가용성 클러스터 찾기의 내용을 참조하십시오.
  • idsd 데몬을 Informix 데이터베이스 서버에서 설치 및 구성하십시오. 고가용성 클러스터에 대한 연결 구성의 내용을 참조하십시오.


SDS 서버 연결을 추가하려면 다음을 수행하십시오.

  1. OAT 메뉴에서 복제 > 클러스터를 클릭하십시오.
  2. 클러스터에서 클러스터를 선택하십시오. 클러스터 토폴로지 및 서버 목록이 표시됩니다.
  3. SDS 추가를 클릭하십시오. SDS 서버 추가 마법사를 완료하십시오.

클러스터의 서버 시작 및 중지

고가용성의 서버를 온라인 또는 오프라인 모드로 변경합니다.

전제조건:
  • IBM OpenAdmin Tool(OAT) for Informix에서 클러스터의 이름을 찾으십시오. 고가용성 클러스터 찾기의 내용을 참조하십시오.
  • idsd 데몬을 Informix 데이터베이스 서버에서 설치 및 구성하십시오. 고가용성 클러스터에 대한 연결 구성의 내용을 참조하십시오.

중요사항: 공유 디스크 보조(SDS) 서버를 중지하는 경우 서버는 OAT의 클러스터에 더 이상 표시되지 않습니다. SDS 서버를 OAT 표시로 복원하려면 서버를 수동으로 재시작한 후 클러스터 찾기를 클릭하십시오. 클러스터를 발견하면 SDS 서버가 클러스터에 대한 서버 목록에 표시됩니다.

보조 서버를 시작 또는 중지하려면 다음을 수행하십시오.

  1. OAT 메뉴에서 복제 > 클러스터를 클릭하십시오.
  2. 클러스터에서 클러스터를 선택하십시오. 클러스터 토폴로지 및 서버 목록이 표시됩니다.
  3. 서버 목록에서 중지 또는 시작하려는 서버에 대해 수정을 클릭하십시오. 서버 수정 페이지에 서버의 상태 및 서버의 시작 또는 중지 가능 여부가 표시됩니다.
  4. 시작 또는 중지를 클릭하십시오.

시계열 관리

테이블, 인덱스, 가상 테이블, 시계열 하위 형식, 컨테이너, 달력 등 시계열과 관련된 데이터베이스 오브젝트를 검토하고 관리할 수 있습니다.

시계열은 시간 경과에 따라 변화하는 대로 기록된 데이터 세트입니다. 각 엔트리에는 타임스탬프가 있으며 데이터베이스에 저장됩니다. Informix 시계열 확장은 시계열 데이터 관리를 위해 시계열 자료형, 컨테이너 및 달력을 제공합니다.

시계열을 관리하려면 다음을 수행하십시오.

IBM OpenAdmin Tool(OAT) for Informix 메뉴에서 SQL 도구 상자 > 시계열를 클릭하십시오. 시계열 데이터가 포함된 데이터베이스 서버의 데이터베이스 및 테이블 목록이 표시됩니다.

  • 데이터베이스 또는 테이블에 대한 정보를 보려면 목록에서 클릭하십시오.
  • 특정 데이터베이스 또는 테이블 오브젝트에 대한 자세한 정보를 보려면 오브젝트 패널에서 펼치기 아이콘을 클릭하십시오.

데이터베이스 서버의 구성 매개변수 관리

현재 데이터베이스 서버의 onconfig 파일에서 매개변수 값을 보고 변경할 수 있습니다.

IBM OpenAdmin Tool(OAT) for Informix를 사용하여 현재 IBM Informix 데이터베이스 서버의 구성 매개변수의 값을 상당수 변경할 수 있습니다.

데이터베이스 서버의 구성 매개변수를 관리하려면 다음을 수행하십시오.

  1. OAT 메뉴에서 서버 관리 > 구성을 클릭하십시오.
  2. 매개변수 목록을 필터하려면, 다음과 같이 표시 목록에서 보기를 선택하십시오.
    • 기본: 모든 환경에서 적용 가능한 기본 구성 매개변수를 표시합니다.
    • 동적: 변경할 수 있는 매개변수를 표시합니다.
    • 권장사항: OAT 권장사항을 따르지 않는 설정이 있는 매개변수를 표시합니다. 권장사항을 따르지 않는 설정이 있는 모든 매개변수는 노란색으로 강조 표시됩니다.
  3. 매개변수를 보거나 변경하려면 매개변수 이름을 클릭하십시오.

시스템 유효성 검증 실행

현재 서버에서 하나 이상의 데이터베이스 또는 테이블 유효성 검증

서버 관리 > 시스템 유효성 검증을 펼치십시오.

현재 서버에서 하나 이상의 데이터베이스 또는 테이블을 유효성 검증하려면 다음 옵션 중 하나를 사용하십시오.

  • 모든 데이터베이스의 모든 테이블을 유효성 검증하려면 데이터베이스 이름 필드에서 모두를 선택하고 테이블 이름 필드에서 모두를 선택하십시오.
  • 한 데이터베이스의 모든 테이블을 유효성 검증하려면 데이터베이스 이름 필드에서 데이터베이스 이름을 선택하고 테이블 이름 필드에서 모두를 선택하십시오.
  • 하나의 특정 테이블을 유효성 검증하려면 데이터베이스 이름 필드에서 데이터베이스 이름을 선택하고 테이블 이름 필드에서 테이블 이름을 선택하십시오. 테이블이 속한 데이터베이스를 알 수 없는 경우, 데이터베이스 이름 필드를 기본값인 모두로 유지하십시오. 테이블 이름 목록에서 테이블의 이름을 검색하십시오.

현재 서버에서 하나 이상의 DBSPACE 익스텐트 유효성 검증

  1. 서버 관리 > 시스템 유효성 검증을 펼치십시오.
  2. DBSPACE 이름을 선택하거나 모든 DBSPACE를 선택하려면 "모두"를 선택하십시오.
  3. 익스텐트 검사를 클릭하십시오. 검사한 각 익스텐트에 대한 상태 메시지가 표시되어 검사가 성공적으로 완료되었는지를 확인할 수 있습니다.

가상 프로세서 관리

서버 관리 > 가상 프로세서를 펼치십시오. 프로세서의 전역 보기가 표시됩니다.

이 페이지에서 가상 프로세서를 추가 또는 삭제하거나 가상 프로세서 클래스를 선택하여 VP 클래스 테이블에서 볼 수 있습니다. 클래스의 VP 페이지에서 클래스를 추가 또는 삭제할 수도 있습니다.

사용자 권한 부여 및 제한

사용자에 데이터베이스 레벨 권한 부여

  1. OAT 메뉴에서 서버 관리 > 사용자 권한을 클릭하십시오.
  2. 데이터베이스를 선택하십시오.
  3. 새 권한 부여에서 사용자 이름(예를 들어, myUserName)을 입력하십시오.

    Windows에서 사용자 이름은 20 문자를 초과할 수 없습니다.

  4. 사용자에 대한 데이터베이스 레벨 권한을 선택하십시오.
  5. 권한 부여를 클릭하십시오.

  • 관련 개념
  • Database-level privileges (Database Design and Implementation Guide)

사용자의 데이터베이스 레벨 권한 수정

  1. OAT 메뉴에서 서버 관리 > 사용자 권한을 클릭하십시오.
  2. 데이터베이스를 선택하십시오.
  3. 사용자 이름이 포함된 행에서, 수정을 클릭하십시오.
  4. 사용자에 대한 데이터베이스 레벨 권한을 선택하십시오. 사용자의 권한을 취소하려면 권한 취소를 선택하십시오.
  5. 저장을 클릭하여 권한을 수정하십시오.

  • 관련 개념
  • Database-level privileges (Database Design and Implementation Guide)

사용자의 데이터베이스 레벨 권한 취소

  1. OAT 메뉴에서 서버 관리 > 사용자 권한을 클릭하십시오.

내부 사용자 관리(UNIX, Linux)

내부에서 인증된 사용자 및 호스트 운영 체제에 계정이 없는 사용자를 추가하고 수정합니다.

전제조건:
  • 내부 사용자의 경우, IBM OpenAdmin Tool(OAT) for Informix에는 IBM Informix 11.70이 필요합니다.
  • 데이터베이스 인증을 갖는 사용자를 작성하려면 Informix 11.70.xC2 이상이 필요합니다.
  • Informix 11.70.xC3 이상의 경우, 내부 사용자를 작성하기 위해서는 먼저 맵핑된 사용자 서로게이트 파일을 설정해야 합니다.

  • 관련 개념
  • Internal users (UNIX, Linux) (Security Guide)
  • Connection security (Security Guide)
  • Mapped user surrogates in the allowed.surrogates file (UNIX, Linux) (Security Guide)

서버의 내부 사용자 권한 설정(UNIX, Linux)

AUS(Auto Update Statistics) 관리

테이블 통계에 대해 AUS(Auto Update Statistics) 유지보수 시스템을 검토 및 조정합니다.

AUS 유지보수 시스템은 새로운 옵티마이저 통계가 필요한 테이블과 인덱스를 식별하고 쿼리 성능을 최적화하도록 해당 UPDATE STATISTICS 명령문을 실행합니다. 사용자 시스템에 따라 AUS 만기 정책 또는 스케줄을 조정해야 할 수 있습니다.

AUS 정책 조정 및 모니터링은 서버 관리 > AUS(Auto Update Statistics) > 구성 페이지에서 수행할 수 있습니다.

SQL 도구 상자 > 스키마 관리자 > 통계 페이지에서 OpenAdmin Tool(OAT)용 IBM Informix 스키마 관리자 플러그인을 사용하여 테이블 통계를 검토할 수 있습니다.

AUS(Auto Update Statistics) 구성 및 사용

  1. IBM OpenAdmin Tool(OAT) for Informix 메뉴에서 서버 관리 > AUS(Auto Update Statistics) > 구성을 클릭하십시오.
  2. 시스템 성능 및 요구사항에 기반하여 기본값이 사용자의 시스템에 유효한지 여부를 결정하십시오.
    1. 통계 재빌드 빈도를 지정하려면, AUS_AGE 매개변수에 대해 30, 60 또는 90과 같은 일 수를 입력하십시오.
    2. 테이블의 몇 퍼센트의 변경이 통계 재빌드를 시작하는지를 지정하려면, AUS_CHANGE 매개변수에 대해 20 또는 50과 같은 백분율 수를 입력하십시오.
    3. AUS에서 권장 가이드라인 규칙 사용 여부를 지정하려면 AUS_AUTO_RULES 매개변수에 대한 목록에서 온 또는 오프를 선택하십시오. 기본값은 온입니다.
    4. 테이블에서 통계 재빌드에 필요한 최소 행의 수를 지정하려면 AUS_SMALL_TABLES 매개변수에 대해 100과 같은 수를 입력하십시오.
    5. PDQ 우선순위를 지정하려면 AUS_PDQ 매개변수에 대해 -1부터 100까지의 수를 입력하십시오. 기본값은 10입니다. PDQ(Parallel Database Queries) 우선순위는 데이터베이스 서버에서 병렬 처리를 위해 자원을 사용하는 방법을 결정하는 요소 중 하나입니다.
    6. 통계 갱신 작업을 실행하기 위해 사용할 스레드 수를 지정하려면, 스레드 수를 입력하십시오. OAT는 이 AUS 작업 수를 작업 스케줄러에 실행 요청합니다.이 옵션의 경우, IBM Informix 버전 11.70이 필요합니다.
  3. 저장을 클릭하여 변경을 보존하십시오.
  4. AUS 작업이 사용 가능인지를 판별하려면 정보 탭을 클릭하고 다음과 같은 작업을 클릭하십시오.
    • AUS(Auto Update Statistics) 평가
    • AUS(Auto Update Statistics) 새로 고침
      주: AUS(Auto Update Statistics) 새로 고침 작업에 수행할 사항이 없는 경우 작업이 스스로 비활성됩니다. 다음 번 AUS(Auto Update Statistics) 평가 작업이 실행되고 새로 고침 작업에 대한 작업이 작성되는 경우, 평가 작업이 자동으로 새로 고침 작업을 활성화합니다.
    작업 세부사항 페이지가 표시됩니다.
  5. 작업 사용 가능 필드로 스크롤 다운하여 선택되어 있는지 확인하십시오.
  6. 실행 창을 설정하는 필드를 사용하여 작업 실행 빈도를 지정하십시오. AUS는 자원 집중 조작이므로 특정 실행 창을 정의할 수 있습니다. 실행 창 정의에는 시작 시간 및 종료 시간과 실행하는 매주의 요일이 포함됩니다. 기본적으로 런타임 창은 매일 오전 1:00 – 오전 5:00입니다.
  7. 저장을 클릭하여 변경을 저장하십시오.

현재 AUS(Auto Update Statistics) 설정 보기

AUS(Auto Update Statistics)에 관한 다음 정보를 볼 수 있습니다.

  • 데이터베이스가 갱신된 시기를 판별하려면 마지막 검사 컬럼을 참조하십시오. 이 값은 통계가 갱신되어야 하는 테이블의 마지막 평가 타임스탬프입니다.
  • 현재 그룹의 어떤 데이터베이스에서 AUS가 사용 가능한지를 결정하려면 데이터베이스 컬럼을 참조하십시오.
  • 특정 데이터베이스에 대해 통계가 누락된 테이블 수를 결정하려면 통계 누락 테이블 컬럼을 참조하십시오. 이러한 테이블을 통계 갱신에 권장되는 기본 가이드라인을 준수하지 않습니다.

    통계 및 분산의 최소 집합이 모든 인덱스에서(또는 인덱스가 없는 경우 테이블에서) LOW, 인덱스의 모든 리드 키에서 HIGH 및 리드가 아닌 모든 키에서 MEDIUM 실행으로 구성됩니다.

  • 특정 데이터베이스에서 통계 새로 고침이 필요한 테이블 수를 결정하려면 통계를 새로 고쳐야 하는 소형 테이블 및 통계를 새로 고쳐야 하는 대형 테이블 컬럼을 참조하십시오.
  • 데이터베이스에서 새로 고친 테이블 수를 보려면 새로 고친 테이블 수 컬럼을 참조하십시오.

전제조건: 이 동작의 경우, OpenAdmin Tool(OAT) for Informix에는 IBM Informix 11.70.xC2가 필요합니다.

스케줄러 작업 관리 방법

스케줄러 작업 추가

지정한 시간 및 빈도에서 작업 또는 센서를 실행하는 스케줄러 작업을 작성할 수 있습니다.

  • 작업은 특정 시간 또는 간격에서 특정 작업을 실행할 수 있는 방법을 제공합니다.
  • 센서는 정보를 수집하고 저장합니다.

새 작업 추가

중요사항: 작업을 작성한 시간이 작업의 처음 실행에 영향을 줍니다.
  • 현재 시간이 지정된 시작 및 중지 시간 내에 있을 경우, 작업은 현재 시간에 실행됩니다. 연속 실행은 빈도 값에서 지정한 간격으로 발생합니다. 따라서, 예를 들어 현재 시간이 시작 및 중지 시간 내에 있고 빈도가 15분으로 설정되어 있으면 작업의 첫 번째 실행은 작업이 스케줄된 직후 실행되며 다음 실행은 15분 뒤에 발생합니다.
  • 그러나, 현재 시간이 시작 및 중지 시간 내에 있지 않을 경우 작업은 오늘 또는 내일의 다음 시작 시간까지 실행을 대기합니다. 예를 들어, 현재 시간이 오전 1:00이고 다음 시작 시간이 오후 7:00(19:00)인 경우 작업은 오늘 19:00에 시작합니다.

새 작업을 추가하려면 다음을 수행하십시오.

  1. 작업 스케줄 페이지에서 새 작업 추가를 클릭하십시오.
  2. 추가할 작업 유형에서 작업을 클릭한 후 다음을 클릭하십시오.
  3. 작업 이름 필드에 이름을 입력하십시오. 예를 들어 MyTask와 같습니다.
  4. 작업 그룹 목록에서 범주를 선택하십시오.
  5. 텍스트 필드에서 새 작업에 대한 설명을 추가하고 다음을 클릭하십시오.
  6. 작업이 실행되는 허용 시간 프레임을 지정하십시오(예: 매일, 매시간, 아침 10시(10:00)부터 저녁 6시(18:00) 사이).
    1. 시간 간격의 시작 시간을 24시간 형식(hh:mm:ss)으로 지정하십시오. 예를 들어 10:00:00과 같습니다.
    2. 시간 간격의 중지 시간을 24시간 형식(hh:mm:ss)으로 지정하십시오. 예를 들어 18:00:00과 같습니다. 또한 없음을 지정할 수도 있습니다.
    3. 허용 시간 간격 동안 작업 실행 주기를 일, 시간, 분(d일 h시간 m분)으로 지정하십시오. 예를 들어, 0일 1시간 00분과 같습니다.
  7. 다음을 클릭하십시오.
  8. 작업을 실행하도록 명령문을 지정하십시오. 예를 들어, 기타 작업을 삭제하려면 다음 명령문을 입력하십시오. 
    delete from ph_task where tk_group='MISC'
  9. 다음을 클릭하십시오. 확인 페이지가 표시됩니다.
  10. 각 선택사항에 대해 엔트리를 확인하십시오.
    • 엔트리가 올바르면 완료를 클릭하여 확인하고 작업을 작성하십시오.
    • 잘못된 값을 정정하려면 뒤로를 클릭하여 이전 페이지로 돌아가십시오.

새 센서 추가

  1. 작업 스케줄 페이지에서 새 작업 추가를 클릭하십시오.
  2. 추가할 작업 유형에서 센서를 클릭한 후 다음을 클릭하십시오.
  3. 센서 이름 필드에 이름을 입력하십시오. 예를 들어 MySensor와 같습니다.
  4. 센서 그룹 목록에서 범주를 선택하십시오.
  5. 텍스트 필드에서 새 작업에 대한 설명을 추가하고 다음을 클릭하십시오.
  6. 센서가 실행될 수 있는 허용 시간 프레임을 매일, 매시간, 오전 10:00에서 오후 6:00 사이와 같이 지정하십시오.
    1. 시간 간격의 시작 시간을 24시간 형식(hh:mm:ss)으로 지정하십시오. 예를 들어 10:00:00과 같습니다.
    2. 시간 간격의 중지 시간을 24시간 형식(hh:mm:ss)으로 지정하십시오. 예를 들어 18:00:00과 같습니다.
    3. 허용 시간 간격 동안 작업 실행 주기를 일, 시간, 분(d일 h시간 m분)으로 지정하십시오. 예를 들어, 0일 1시간 00분과 같습니다.
    4. 데이터 삭제 시간을 일, 시간, 분(d일 h시간 m분)으로 지정하십시오. 예를 들어, 1일 0시간 0분과 같습니다.
  7. 다음을 클릭하십시오.
  8. 결과 테이블 이름 필드에서 센서 데이터를 보유할 결과 테이블 이름을 지정하십시오. 예를 들어 MyResults와 같습니다.
  9. 다음을 클릭하십시오.
  10. SQL문 필드에 컬럼 이름 및 정의를 입력하여 결과 테이블에 대한 SQL문을 완료하십시오. 예를 들어 TaskA_ReturnCode 테이블의 TaskA 리턴 코드(task_id가 10인)를 수집하려면 다음 SQL문을 입력하십시오. 
    create table TaskA_ReturnCode(task_name varchar(20), errorcode integer)
  11. 센서를 실행하도록 명령문을 지정하십시오. 예를 들면 다음과 같습니다. 
    insert into TaskA_ReturnCode values('TaskA', (select run_retcode from ph_run where run_task_id=10))
  12. 각 선택사항에 대해 엔트리를 확인하십시오.
    • 엔트리가 올바르면 완료를 클릭하여 확인하고 센서를 작성하십시오.
    • 잘못된 값을 갱신하려면 뒤로를 클릭하여 이전 페이지로 돌아가십시오.

스케줄러 작업 사용 가능

비활성 상태인 스케줄러 작업을 사용 가능하게 합니다.

스케줄러 작업을 사용 가능하게 하려면 다음을 수행하십시오.

  1. 작업 스케줄러 > 스케줄러를 펼치십시오.
  2. 작업을 클릭하십시오. 작업에 대한 작업 세부사항 페이지가 표시됩니다.
  3. 작업 사용 가능을 클릭하십시오.
  4. 저장을 클릭하십시오.

스케줄러 작업 허용 시간 간격 변경

허용 시간 간격은 오전 10:00 - 오후 6:00과 같이 작업을 실행할 수 있는 시간 프레임입니다.

  1. 작업 스케줄러 > 스케줄러를 펼치십시오. 작업 세부사항 페이지에 작업 목록이 스케줄 세부사항과 함께 표시됩니다.
  2. 작업 이름을 클릭하여 스케줄된 런타임 시간의 빈도 또는 특정사항을 변경하십시오. 작업을 실행하는 기간(날짜 또는 시간)을 갱신할 수 있습니다.
  3. 시간 간격의 시작 시간을 24시간 형식(hh:mm:ss)으로 지정하십시오. 예를 들어, 10:00:00으로 지정하면 오전 10시를 나타냅니다.
  4. 시간 간격의 중지 시간을 24시간 형식(hh:mm:ss)으로 지정하십시오. 예를 들어, 18:00:00으로 지정하면 오후 6시를 나타냅니다.
  5. 허용 시간 간격 동안 작업 실행 주기를 일, 시간, 분(d일 h시간 m분)으로 지정하십시오. 예를 들어, 0일 1시간 00분과 같습니다.
  6. 작업을 실행하지 않으려는 매주의 요일에 대해 사용 안함을 선택하십시오.
  7. 저장을 클릭하여 변경을 저장하십시오.

스케줄러 작업 삭제

제한사항: 시스템 작업은 삭제할 수 없습니다.

사용자 정의 작업을 삭제하려면 다음을 수행하십시오.

  1. 작업 스케줄러 > 스케줄러를 펼치십시오.
  2. 작업 삭제를 클릭하십시오. 작업 삭제 컬럼이 크론 작업 목록 테이블에 추가됩니다. 각 작업에는 삭제 버튼이 있습니다. 시스템 작업의 경우, 삭제 버튼이 비활성화되어 있습니다.
  3. 작업 이름을 찾은 후 삭제를 클릭하십시오. 작업이 센서에 대한 것이면, 센서와 관련된 결과 테이블을 삭제하도록 프롬프트됩니다.
  4. 테이블 삭제 여부를 결정하십시오.
    • 작업 뿐만 아니라 결과 테이블도 삭제하려면 예를 클릭하십시오.
    • 작업이 삭제되어도 테이블을 보존하려면 아니오를 클릭하십시오.
    • 작업 및 테이블 삭제를 취소하려면 취소를 클릭하십시오.

성능 정보 탐색 방법

SQL Explorer로 쿼리 성능 분석

SQL Explorer를 사용하려면 SQL 추적을 켜야 합니다.

SQL Explorer를 사용하여 쿼리 드릴 다운을 수행하십시오. SQL Explorer는 SQL 추적을 사용하여 시스템에서 실행된 각 SQL문에 대한 통계 정보를 수집하고 명령문 히스토리를 표시합니다. 다음 절에서는 SQL 추적을 켜는 방법 및 SQL Explorer 사용 방법을 설명합니다.

SQL 추적 켜기

  1. 성능 분석 > SQL Explorer로 탐색하십시오. SQL 추적이 실행 중이지 않으면 다음 메시지가 표시됩니다. SQL TRACING is currently off. Use the Admin option to switch back on.
  2. 관리를 클릭하여 SQL 추적 프로파일 페이지로 탐색하십시오.
  3. 추적 상태 목록에서 온을 선택하십시오.
  4. 페이지 하단으로 스크롤 다운하여 저장을 클릭하여 SQL 추적을 켜십시오.

SQL 추적 끄기

  1. 성능 분석 > SQL Explorer로 탐색하십시오. SQL 추적이 켜저 있으면 SQL Explorer에 대한 탭을 선택할 수 있습니다. SQL 추적이 이미 오프 상태이면 다음 메시지가 표시됩니다. SQL TRACING is currently off. Use the Admin option to switch back on.
  2. 추적 관리 탭을 클릭하여 SQL 추적 프로파일 페이지로 탐색하십시오.
  3. 추적 상태 목록에서 오프를 선택하십시오.
  4. 페이지 하단으로 스크롤 다운하여 저장을 클릭하여 SQL 추적을 끄십시오.

SQL 추적 옵션 지정

SQL Explorer는 현재 서버에서 SQL문 실행에 관한 상세 정보를 표시합니다. 추적 관리 페이지에서 SQL Explorer에서 사용 가능한 추적 대상 정보를 지정할 수 있습니다.

SQL Explorer 세부사항을 설정하려면 다음을 수행하십시오.

  1. IBM OpenAdmin Tool(OAT) for Informix 메뉴에서 성능 분석 > SQL Explorer > 추적 관리를 클릭하십시오.
  2. 다음 SQL 추적 옵션을 지정하십시오.
    • 전역 또는 사용자 추적 모드를 선택하십시오. 이 옵션은 IBM Informix 11.50.xC3 이상의 경우에 표시됩니다. 자세한 정보는 추적 모드의 내용을 참조하십시오.
    • 추적 정보에 테이블 이름을 포함하려면 테이블 이름 선택란을 선택하십시오.
    • 추적 정보에 프로시저 스택을 포함하려면 프로시저 스택 선택란을 선택하십시오.
    • 추적 정보에 호스트 변수에 관한 정보를 포함하려면 호스트 변수 선택란을 선택하십시오.
    • 추적 옵션이 변경 및 저장될 때 추적 버퍼를 비우려면 추적 버퍼 지우기 선택란을 선택하십시오.
    • 추적할 명령문 수를 500 - 2147483647의 숫자 값으로 입력하십시오.
    • 추적 크기를 1 - 100의 숫자 값으로 입력하십시오.
  3. 저장을 클릭하십시오.

추적 모드 옵션은 IBM Informix 11.50.xC3 이상의 경우에 표시됩니다.

    • 사용자 추적 목록을 수정하려면 다음을 수행하십시오.
      • 사용자 추가: 추적하지 않은 데이터베이스 목록에서 데이터베이스 이름을 선택하고 +를 클릭하십시오. 데이터베이스 이름이 추적한 데이터베이스 목록에 표시됩니다.
      • 사용자 삭제: 추적한 데이터베이스 목록에서 데이터베이스 이름을 선택하고 -를 클릭하십시오. 데이터베이스 이름이 추적하지 않은 데이터베이스 목록에 표시됩니다.

SQL문 최적화

SQL문을 최적화하기 위해 IBM OpenAdmin Tool(OAT) for Informix를 사용하여 외부 지시문을 작성할 수 있습니다.

전제조건:
  • 데이터베이서 서버의 onconfig 파일에서 EXT_DIRECTIVES 구성 매개변수를 1 또는 2로 설정하여 외부 지시문을 저장할 수 있습니다. EXT_DIRECTIVES 구성 매개변수를 변경한 후에 변경이 효력을 가지려면 데이터베이스 서버를 재시작해야 합니다.
  • SQL 추적이 켜저 있고 추적 관리 페이지에서 테이블 이름이 선택되어 있는지 확인하십시오. SQL 추적 켜기 및 SQL 추적 옵션 지정을 참조하십시오.

SQL문에 적용하기 위해 외부 지시문을 작성할 수 있습니다. 지시문은 쿼리 옵티마이저가 쿼리를 수행하는 방법을 설명하는 주석입니다. 외부 지시문은 SQL문의 쿼리 실행 계획을 변경할 수 있는 방법을 제공합니다. SQL문에 외부 지시문을 적용한 후에 SQL문의 쿼리 실행 계획은 SQL문을 리턴할 때마다 변경됩니다.

중요사항: SQL문을 리턴할 때 지시문을 적용한 SQL문과 정확히 일치해야 합니다. 예를 들어, 대소문자 및 공백이 동일해야 합니다.

외부 지시문을 작성하려면 다음을 수행하십시오.

  1. OAT 메뉴에서 성능 분석 > SQL Explorer를 클릭하십시오.
  2. SQL문으로 드릴 다운하십시오.
    1. SQL 탭을 클릭하십시오.
    2. 목록에서 SQL문을 클릭하십시오.
    3. SQL문 실행의 목록에서 임의 행을 클릭하십시오.
  3. 최적화 탭을 클릭하십시오.
  4. 추가를 클릭하십시오.
  5. 지시문 추가 페이지에서 지시문을 선택하십시오. 필요하면 테이블 및 인덱스를 선택하십시오. SQL문에 대해 지시문을 더 추가하려면 더 추가를 클릭하십시오. 아니면 추가를 클릭하여 페이지를 닫으십시오.
  6. 지시문을 적용하는 방법을 지정하고 적용을 클릭하십시오. 기본값은 비활성입니다. 기본적으로 SQL문에 지시문을 적용하지 않습니다.

onstat 유틸리티로 서버 모니터링

IBM OpenAdmin Tool(OAT) for Informix에서 onstat 유틸리티를 사용하여 데이터베이스 서버에 대한 성능 및 상태 정보를 표시할 수 있습니다.

onstat 유틸리티를 사용하여 데이터베이스 서버의 활동을 모니터링하고 상태를 검사할 수 있습니다.

onstat 유틸리티를 사용하려면 다음을 수행하십시오.

  1. OAT 메뉴에서 성능 분석을 펼치고 onstat 유틸리티를 클릭하십시오.
  2. onstat 옵션을 입력하십시오. onstat를 입력하지 않아도 됩니다.
  3. 실행을 클릭하십시오.

데이터베이스 및 테이블 관리 방법

서버의 데이터베이스 검토

SQL 도구 상자를 펼치고 데이터베이스를 클릭하십시오. 페이지에 현재 서버의 데이터베이스 목록과 6개의 최대 데이터베이스에서 사용하는 영역의 차트가 표시됩니다.

데이터베이스에 연결

데이터베이스에 연결할 때 데이터베이스 테이블, 스키마, 프로시저 및 루틴을 검토할 수 있습니다.

데이터베이스에 연결하려면 다음을 수행하십시오.

  1. SQL 도구 상자를 펼치고 데이터베이스를 클릭하십시오. 페이지에 현재 서버의 데이터베이스 목록과 6개의 최대 데이터베이스에서 사용하는 영역의 차트가 표시됩니다.
  2. 데이터베이스 목록에서, 데이터베이스 이름을 클릭하십시오. 데이터베이스의 테이블 목록이 테이블 페이지에 표시됩니다. 선택한 데이터베이스 이름이 페이지 오른쪽 맨 위의 데이터베이스 필드에 표시됩니다.
  3. 옵션: 카탈로그 테이블 이름을 포함하려면 데이터베이스 카탈로그 테이블 포함?을 선택하고 실행 요청을 클릭하십시오. 카탈로그 테이블 및 사용자 테이블의 조합 목록이 표시됩니다.
    • 테이블을 찾아보려면, 찾아보기 컬럼에 있는 아이콘을 클릭하십시오.
    • 테이블 컬럼에 대한 정보를 보려면, 이름 컬럼에 있는 테이블을 클릭하십시오.
    • 테이블에 대한 파티션 정보를 보려면 부품번호 컬럼에 있는 값을 클릭하십시오.

스키마 보기

데이터베이스에 연결되어야 합니다. 데이터베이스에 연결의 내용을 참조하십시오.

스키마를 보려면 다음을 수행하십시오.

  1. SQL 도구 상자를 펼치고 스키마 브라우저를 클릭하십시오.

    테이블 목록과 해당 스키마가 표시됩니다. 연결한 데이터베이스 이름이 페이지 오른쪽 맨 위의 데이터베이스 필드에 표시됩니다.

  2. 보려는 테이블의 브라우저 아이콘을 선택하십시오.

    테이블에 대한 SELECT * 명령문의 결과 세트가 표시됩니다.

데이터베이스에 대한 프로시저 및 루틴 검토

데이터베이스에 연결되어야 합니다. 데이터베이스에 연결의 내용을 참조하십시오.

데이터베이스에 대한 프로시저 및 루틴을 검토하려면 다음을 수행하십시오.

  1. SQL 도구 상자를 펼치고 스키마 브라우저를 클릭하십시오.

    테이블 목록과 해당 스키마가 표시됩니다. 연결한 데이터베이스 이름이 페이지 오른쪽 맨 위의 데이터베이스 필드에 표시됩니다.

  2. SPL/UDR 탭을 선택하십시오.

    프로시저/기능 목록이 표시됩니다.

표준 SQL 조작 수행: 쿼리, 삽입, 갱신, 삭제

단순한 쿼리를 작성하여 테이블의 특정 행을 표시한 후 행을 갱신 또는 삭제하거나 예제별 쿼리를 사용하여 테이블에 새로운 행을 삽입합니다.

쿼리 수행

단순한 쿼리를 작성하고 예제별 쿼리를 사용하여 테이블에 특정 행을 표시합니다.

테이블에 대해 쿼리를 수행한 후 결과로 표시되는 행을 검토, 갱신 또는 삭제할 수 있습니다.

쿼리를 수행하려면 다음을 수행하십시오.

  1. IBM OpenAdmin Tool(OAT) for Informix의 메뉴에서 SQL 도구 상자를 펼치고 예제별 쿼리를 클릭하십시오.
  2. 테이블을 찾으려면, 찾기 필드에 테이블 이름의 일부를 입력하고 검색을 클릭하십시오. 또한 데이터베이스를 펼쳐 테이블을 표시할 수 있습니다.
    주: 목록에는 예제별 쿼리(QBE)로 작업할 수 있는 테이블만 표시됩니다. QBE로 작업할 수 없는 테이블은 표시되지 않습니다. 여기에는 시스템 테이블 또는 시스템 데이터베이스(sysadmin 데이터베이스 이외)의 테이블과 QBE에서 지원하지 않는 자료형(BLOB, BYTE, CLOB, DISTINCT TYPES, LIST, MULTISET, NCHAR, NVARCHAR, ROW, SET, TEXT 및 UDT)이 있는 테이블이 포함됩니다.

    선택한 테이블의 컬럼이 쿼리 페이지에 표시됩니다.

  3. 쿼리 기준을 입력하려면 컬럼별로 버튼을 클릭하여 쿼리 기준 입력 페이지를 여십시오.
  4. 연산자 목록에서 연산자를 선택하고 값을 입력한 후 추가를 클릭하십시오.

    쿼리 기준이 테이블에 표시됩니다. 다른 기준을 추가하는 경우 먼저 AND 또는 OR을 선택하십시오.

  5. 확인을 클릭하십시오.

    페이지가 닫히고 컬럼에 대해 쿼리 기준이 표시됩니다.

  6. 쿼리를 수행하려면 실행 요청을 클릭하십시오.

    한 번에 한 행의 쿼리에 대한 결과가 페이지에 행으로 표시됩니다. 쿼리 기준을 입력하지 않고 쿼리를 수행하면 테이블의 모든 행이 표시됩니다.

    행을 스크롤하려면 화살표 단추를 사용하십시오.

테이블 행 갱신

예제별 쿼리를 사용하여 테이터베이스 서버에서 테이블 행을 갱신합니다.

주: 고유 식별자를 가진 행만 IBM OpenAdmin Tool(OAT) for Informix에서 예제별 쿼리를 사용하여 갱신될 수 있습니다. 고유 식별자에는 기본 키, 고유 제한 조건, 고유 인덱스 및 테이블이 분할되지 않은 경우 ROWID가 포함됩니다.

데이터베이스 서버에서 테이블 행을 갱신하려면 다음을 수행하십시오.

  1. 예제별 쿼리를 사용하여 쿼리를 수행하십시오. 쿼리 수행의 내용을 참조하십시오.
  2. 필요한 경우 화살표 단추를 사용하여 행을 스크롤하십시오.
  3. 필드에 행에 대한 변경을 입력하십시오.
  4. 행 갱신을 클릭하십시오.

    데이터베이스 서버에서 행이 갱신됩니다.

테이블 행 삭제

예제별 쿼리를 사용하여 테이터베이스 서버에서 테이블 행을 삭제합니다.

주: 고유 식별자를 가진 행만 IBM OpenAdmin Tool(OAT) for Informix에서 예제별 쿼리를 사용하여 삭제될 수 있습니다. 고유 식별자에는 기본 키, 고유 제한 조건, 고유 인덱스 및 테이블이 분할되지 않은 경우 ROWID가 포함됩니다.

데이터베이스 서버에서 테이블 행을 삭제하려면 다음을 수행하십시오.

  1. 예제별 쿼리를 사용하여 쿼리를 수행하십시오. 쿼리 수행의 내용을 참조하십시오.
  2. 필요한 경우 화살표 단추를 사용하여 행을 스크롤하십시오.
    중요사항: 행을 삭제하는 경우 데이터베이스 서버에서 영구적으로 삭제됩니다. 삭제를 실행 취소할 수 없습니다.
  3. 행 삭제를 클릭하십시오.

    데이터베이스 서버에서 행이 삭제됩니다.

테이블 행 삽입

IBM OpenAdmin Tool(OAT) for Informix에서 예제별 쿼리를 사용하여 데이터베이스 서버에서 테이블에 행을 삽입할 수 있습니다.

데이터베이스 서버에서 테이블에 행을 삽입하려면 다음을 수행하십시오.

  1. OAT 메뉴에서 SQL 도구 상자를 펼치고 예제별 쿼리를 클릭하십시오.
  2. 테이블을 찾으려면, 찾기 필드에 테이블 이름의 일부를 입력하고 검색을 클릭하십시오. 또한 데이터베이스를 펼쳐 테이블을 표시할 수 있습니다.
    주: 목록에는 예제별 쿼리(QBE)로 작업할 수 있는 테이블만 표시됩니다. QBE로 작업할 수 없는 테이블은 표시되지 않습니다. 여기에는 시스템 테이블 또는 시스템 데이터베이스(sysadmin 데이터베이스 이외)의 테이블과 QBE에서 지원하지 않는 자료형(BYTE, TEXT, BLOB, CLOB, NCHAR, NVARCHAR, SET, MULTISET, LIST, ROW, DISTINCT TYPES 및 UDT)이 있는 테이블이 포함됩니다.

    선택한 테이블의 컬럼이 쿼리 페이지에 표시됩니다.

  3. 삽입 탭을 클릭하십시오.
  4. 필드에 값을 입력하십시오.
    팁: 각 필드의 풍선 도움말에서 필요한 데이터 유형(예를 들어, CHAR, DECIMAL 또는 DATETIME)을 식별합니다.
  5. 행 추가를 클릭하십시오.

    데이터베이스 서버에서 행이 테이블에 삽입됩니다.

서버에 액세스 설정 방법

OAT 그룹 추가

IBM OpenAdmin Tool(OAT) for Informix에서 데이터베이스 서버 연결 그룹을 정의할 수 있습니다.

OAT 그룹은 데이터베이스 서버 그룹을 관리하는 방법과 각 개별 서버가 아닌 그룹에 로그온하는 방법을 제공합니다.

OAT 그룹을 추가하려면 다음을 수행하십시오.

  1. OAT 관리 메뉴에서 연결 관리를 펼치고 그룹 추가를 클릭하십시오.
  2. 그룹 이름 필드에 그룹의 문자와 숫자 이름을 입력하십시오.
  3. 암호 필드에 문자와 숫자 암호를 입력하십시오.
  4. 옵션: 그룹에 읽기 전용 권한을 지정하려면 읽기 전용을 선택하십시오.
    중요사항: 읽기 전용 그룹은 사용자가 서버 수정을 할 수 없게 하므로, 서버에 포함된 민감한 데이터에 대한 잠재적 액세스를 방지할 수 있습니다. 읽기 전용 그룹의 멤버십을 서버에서 상세 데이터를 볼 수 있는 권한이 있는 사용자로 제한하십시오.

    읽기 전용 그룹의 제한에 관한 자세한 정보는 읽기 전용 그룹 제한을 참조하십시오.

  5. 추가를 클릭하여 그룹 정보를 저장하고 그룹을 추가하십시오.

그 다음 그룹에 서버 연결을 추가하십시오. 서버 연결 작성을 참조하십시오.

서버 연결 작성

IBM OpenAdmin Tool(OAT) for Informix에 정의되어 있는 데이터베이스 서버 연결 그룹에 서버 연결을 추가할 수 있습니다.

  1. OAT 관리 메뉴에서 연결 관리를 펼치고 연결 추가를 클릭하십시오.
  2. 그룹 이름 목록에서 서버 연결을 추가할 OAT 그룹을 선택하십시오.
  3. 연결 정보를 입력하십시오.
    중요사항: 추가하는 각각의 새 연결에 대해 V3.0 이전의 Informix Connect 또는 IBM Informix Client Software Development Kit(Client SDK) 버전을 사용하는 경우, 웹 서버 컴퓨터에서 해당 연결에 대한 SQLHOSTS 엔트리가 있는지 확인하십시오.
  4. 저장을 클릭하여 연결을 추가하십시오.

연결 정보 내보내기

IBM OpenAdmin Tool(OAT) for Informix의 데이터베이스 서버 연결 그룹에 대한 연결 정보를 XML 파일로 내보낼 수 있습니다.

연결 정보를 내보내려면 다음을 수행하십시오.

  1. OAT 로그인 페이지에서 관리를 클릭하십시오. OAT 관리 메뉴 및 페이지가 표시됩니다.
  2. 연결 관리 > 연결 관리(Admin)를 클릭하십시오.
  3. XML 파일에 사용자 이름 및 암호를 포함하려면 내보낸 XML 파일에 민감한 암호 정보 포함을 선택하십시오. 기본적으로 내보낸 정보에는 OAT 그룹 사용자 이름 및 암호가 포함되지 않습니다.
  4. XML로 내보내기를 클릭하십시오.

연결 정보 가져오기

XML 파일에서 IBM OpenAdmin Tool(OAT) for Informix의 데이터베이스 서버 연결 그룹에 대한 연결 정보를 가져올 수 있습니다.

가져오는 연결 파일은 OAT에서 내보낸 연결 파일과 동일한 형식이어야 합니다. 파일 설명에 대해서는 연결 관리 페이지의 도움말 버튼을 클릭하십시오.

연결 정보를 가져오려면 다음을 수행하십시오.

  1. OAT 로그인 페이지에서 관리를 클릭하십시오. OAT 관리 메뉴 및 페이지가 표시됩니다.
  2. 연결 관리 > 연결 관리(Admin)를 클릭하십시오.
  3. XML 파일에서 연결 정보 가져오기에서 연결 정보를 포함하는 XML 파일을 찾으십시오.
  4. 기존 연결 데이터를 교체하려면 기존 연결 정보 모두 제거를 선택하십시오. 또는 옵션 선택을 지우고 기존 연결 데이터에 정보를 추가하십시오.
  5. XML에서 가져오기를 클릭하십시오.

비기본 로케일이 있는 데이터베이스 액세스

멀티바이트 데이터베이스 로케일을 포함하여 기본 영어(en_us) 이외의 로케일이 있는 데이터베이스를 관리합니다.

  • IBM OpenAdmin Tool(OAT) for Informix에는 ILS(IBM Informix International Language Supplement)가 필요합니다. ILS는 IBM Informix Client Software Development Kit에 포함되어 있습니다. OAT가 설치되어 있는 시스템의 Informix Connect 또는 Client SDK 디렉토리에 ILS를 설치하십시오. Client SDK 매체에서 ILS를 설치할 수 있습니다.
  • OpenAdmin Tool(OAT)용 IBM Informix 복제 플러그인에는 Informix 11.70.xC3가 필요합니다.
  • 데이터베이스 이름은 영어 문자여야 합니다.

OAT를 사용하여 Informix에서 지원하는 모든 데이터베이스 로케일의 데이터를 보고 수정할 수 있습니다. 예를 들어, 이탈리아어 또는 일본어 로케일이 있는 데이터베이스에 액세스할 수 있습니다.

복제 플러그인에서 비기본 로케일이 있는 데이터베이스를 볼 때에는 언어 목록 및 로케일 목록을 사용하여 적절한 데이터베이스 로케일을 선택하십시오.

읽기 전용 권한이 있는 OAT 그룹 작성

그룹의 멤버가 서버의 정보를 볼 수 있지만 수정할 수 없는 데이터베이스 서버 연결 그룹을 IBM OpenAdmin Tool(OAT) for Informix에서 작성하십시오.

중요사항: 읽기 전용 그룹은 사용자가 서버 수정을 할 수 없게 하므로, 서버에 포함된 민감한 데이터에 대한 잠재적 액세스를 방지할 수 있습니다. 읽기 전용 그룹의 멤버십을 서버에서 상세 데이터를 볼 수 있는 권한이 있는 사용자로 제한하십시오.

읽기 전용 그룹의 제한에 관한 자세한 정보는 읽기 전용 그룹 제한을 참조하십시오.

  1. OAT 관리 메뉴에서 연결 관리 > 그룹 추가를 클릭하십시오.
  2. 그룹 이름 필드에 그룹의 문자와 숫자 이름을 입력하십시오.
  3. 암호 필드에 문자와 숫자 암호를 입력하십시오.
  4. 그룹에 읽기 전용 권한을 지정하려면 읽기 전용을 선택하십시오.
  5. 추가를 클릭하여 그룹 정보를 저장하고 그룹을 추가하십시오.

그 다음 그룹에 서버 연결을 추가하십시오. 서버 연결 작성을 참조하십시오.

  • SQL 도구 상자
    • SQL Editor에 액세스할 수 없습니다(마스터 DBA 계정에만 해당).
    • 스키마 브라우저를 사용하여 테이블을 찾아볼 수 없습니다.

이러한 페이지에 대해 허가되지 않는 액세스를 방지하려면 다음 방법 중 하나 또는 둘 모두를 사용하십시오.

  • 필수사항: OAT를 구성하는 관리 페이지에 암호 보호를 제공하십시오. 관리 구성 페이지에 암호 보호 제공: Apache 예제의 내용을 참조하십시오.
  • 권장 사항: 읽기 전용 그룹을 포함하여 서버 그룹에 대한 액세스를 제한하십시오. 읽기 전용 권한이 있는 OAT 그룹 작성의 내용을 참조하십시오.

관리 구성 페이지에 암호 보호 제공: Apache 예제

보안 상의 이유로 IBM OpenAdmin Tool(OAT) for Informix의 관리 페이지에 대한 액세스를 제한하십시오. 관리 페이지는 로그인 페이지에서 액세스할 수 있고 모든 사람에게 개방되므로 암호로 보호하는 것이 중요합니다.

한 방법으로 웹 서버에서 제공하는 내장 인증을 사용할 수 있습니다. 예를 들어, 설치 디렉토리, OATINSTALL/admin/.htaccess를 보호할 수 있습니다.

예: Apache 내부에서 OAT 디렉토리를 보호하려면 다음을 수행하십시오.

  1. Apache와 함께 제공되는 htpasswd 유틸리티를 사용하여 웹에서 액세스할 수 없는 디렉토리에 암호 파일을 작성하십시오.
  2. 다음 파일 중 하나에 인증 지시문을 추가하십시오.
    • OATINSTALL/admin/.htaccess 파일
    • httpd.conf 파일
  3. 지시문을 <Directory> 세션 내부에 넣으십시오. 예를 들면 다음과 같습니다. 
    <Directory OATINSTALL/admin>
AuthType Basic
AuthName "OAT Admin"
AuthUserFile PASSWORDFILE
Require ValidUser
</Directory>
    여기서 PASSWORDFILE은 htpasswd 유틸리티로 암호 파일을 작성한 디렉토리입니다.

자세한 정보 및 예제는 www.apache.org에서 Apache HTTP 서버 버전 2.0 인증, 권한 및 액세스 제어에 관한 기사를 참조하십시오.

OAT에 대한 HTTPS 구성

HTTPS(Hypertext Transfer Protocol over Secure Socket Layer)는 무단 도청, 변조 및 메시지 위조 또는 네트워크를 비밀리에 도청하거나 간섭하려는 사용자로부터 IBM OpenAdmin Tool(OAT) for Informix 웹 서버를 보호합니다.

이 절에서 클라이언트는 OAT 클라이언트(OAT를 보기 위해 사용하는 웹 브라우저)를 의미하며 웹 서버는 OAT 클라이언트에서 사용하도록 구성된 웹 서버를 의미합니다.

웹 서버에 전송하기 전에 OAT 클라이언트에서 발생하는 메시지에 HTTPS 암호화를 사용하면 무단 사용자가 네트워크를 도청하여 민감한 정보를 유출하는 것을 방지할 수 있습니다. HTTPS를 사용하면 클라이언트에 OAT 웹 서버에 대한 인증을 부여하여 무단 사용자가 잘못된 웹 서버로 클라이언트를 속이는 것을 방지할 수 있습니다.

중요사항: HTTPS는 OAT 웹 서버와 클라이언트 사이의 통신만 암호화합니다. IBM Informix 데이터베이스 서버와 웹 서버 사이의 통신은 암호화하지 않습니다. IBM Informix(버전 9.4 이후)는 암호화 통신 지원 모듈을 사용하여 Informix 서버와 OAT 웹 서버 간에 데이터를 암호화할 수 있습니다. 암호화 및 Informix 서버에 대한 자세한 정보는 developerWorks®의 "Encryption over the wire with IDS 9.40" 주제를 참조하십시오.

OAT에서 HTTPS를 사용하려면 다음 단계가 필요합니다.

  1. OAT에 설치된 Apache 웹 서버를 mod_ssl 암호화 모듈에 대해 사용 가능한 다른 Apache 웹 서버로 교체하십시오.
  2. 새 웹 서버에 대한 암호화 키와 인증서를 작성하여, OAT 클라이언트가 사용자 인증서에 기반하여 웹 서버를 인증하도록 하십시오.
  3. HTTPS를 사용하도록 httpd.conf(Apache 구성 파일)를 구성하십시오.

자세한 정보는 developerWorks의 "HTTPS on OAT." 주제를 참조하십시오.

SQL 도구 상자에 대해 암호 보호 사용 또는 사용 안함

IBM OpenAdmin Tool(OAT) for Informix 2.22 이상에는 SQL 도구 상자에 대한 자동 암호 보호 기능이 있습니다. SQL 도구 상자에 연결할 때 유효한 사용자 ID와 암호흫 제공해야 합니다.

전제조건: SQL 도구 상자에 대해 암호 보호를 사용하거나 사용하지 않으려면 OAT 관리 구성 페이지(이 페이지도 암호로 보호할 수 있음)에 액세스할 수 있어야 합니다.
중요사항: 보안을 위해 관리 구성 페이지에 암호 보호를 제공하십시오. 자세한 정보는 관리 구성 페이지에 암호 보호 제공: Apache 예제의 내용을 참조하십시오.

암호 보호를 사용하지 않으려면 다음을 수행하십시오.

  1. OAT 관리 메뉴에서 OAT 구성을 클릭하십시오.
  2. SQL 도구 상자 사용 시 로그인 보증서 필요를 지우십시오.
  3. 저장을 클릭하여 변경을 저장하십시오.
  4. 암호 보호를 다시 사용 가능하게 하려면 SQL 도구 상자 사용 시 로그인 보증서 필요를 선택하고 저장을 클릭하십시오.

중요사항: 플러그인 압축 파일은 플러그인이 설치될 때 삭제됩니다. 플러그인이 설치 제거된 후에는 플러그인 압축 파일이 플러그인 디렉토리에 복원되어야 재설치할 수 있습니다.

  1. 압축 플러그인 파일을 OATDirectory/plugin_install에 복사하십시오. 예를 들면, 다음과 같습니다. C:₩Program Files₩OpenAdmin₩Apache_2.2.16₩htdocs₩openadmin₩plugin_install.
  2. OAT 로그인 페이지에서 또는 OAT 메뉴에서 관리를 클릭하십시오.
  3. 관리 메뉴에서 플러그인 관리자를 클릭하십시오.
  4. 아직 설치되지 않은 플러그인 목록의 해당 플러그인이 포함된 행에서 설치를 클릭하십시오.

isreadonly 함수

isreadonly 함수는 사용자가 읽기 전용 그룹으로 로그인되었는지 여부를 표시합니다.

구문

isreadonly ()

사용

isreadonly 함수를 사용하여 사용자가 읽기 전용 권한이 있는 그룹으로 IBM OpenAdmin Tool(OAT) for Informix에 로그인되었는지 여부를 판별하십시오. 이 정보를 사용하여 사용자가 OAT에서 액세스할 수 있는 사항과 사용자에게 사용 가능한 사항을 판별할 수 있습니다.

리턴 값

리턴 값이 true이면 사용자가 읽기 전용 권한이 있는 그룹으로 로그인되었음을 표시합니다.

리턴 값이 false이면 사용자가 읽기 전용 권한으로 제한되지 않은 그룹으로 로그인되었음을 표시합니다.

예제

이 예제에서는 사용자가 읽기 전용인 경우 오류를 리턴하고 메시지를 표시합니다. 이 예제에서 PermissionRequired는 본질적으로 메시지가 아니라 언어 키워드입니다.

if ( $this->idsadmin->isreadonly() )
{
    $this->idsadmin->fatal_error("{$this->idsadmin->lang("PermissionRequired")}");
    return;         
}

 $db = $this->idsadmin->get_database("sysmaster");

load_lang 함수

load_lang 함수는 모듈의 언어 파일을 idsadmin 언어 배열로 로드합니다.

구문

load_lang($keyword)
변수 용도 제한사항
keyword 모듈에 대해 로드할 언어 파일을 지정하는 키워드입니다. 관례적으로 이 키워드는 모듈의 실행 키워드와 동일합니다. 언어 파일에는 모듈의 텍스트 문자열이 포함되어 있습니다.

사용

load_lang 함수는 lang_keyword.xml(예: lang_admin.xml 또는 lang_health.xml)을 로드합니다. load_lang 함수는 일반적으로 모듈의 클래스 생성자의 파트로 호출됩니다.

예제

이 클래스 생성자의 예제는 헬스 모듈의 언어 파일을 로드합니다.

function __construct(&$idsadmin)
{
        $this->idsadmin = &$idsadmin;

        $this->idsadmin->load_lang("health");
}

lang 함수

lang 함수는 특정 메시지를 현재 언어로 리턴합니다.

구문

lang($item, $parameters=array())
변수 용도 제한사항
item 현재 언어로 리턴해야 할 텍스트 문자열의 이름입니다.
parameters 배열의 매개변수입니다. 매개변수는 '{0}', '{1}' 및 '{2}' 대신 사용됩니다. 선택적입니다.

사용

lang 함수는 로드된 언어 파일에서 지정된 메시지를 찾습니다. 현재 언어가 영어가 아니고 지정된 메시지를 찾을 수 없는 경우에는 메시지가 영어로 리턴됩니다. 메시지를 영어로 사용할 수 없는 경우에는 오류가 리턴됩니다.

예제

이 예제에서는 taskdetailssaved 항목의 텍스트를 현재 언어로 검색하고 해당 텍스트를 상태 메시지로 인쇄합니다.

$this->idsadmin->status($this->idsadmin->lang('taskdetailssaved'));
