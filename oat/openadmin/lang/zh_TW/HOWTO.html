IBM OpenAdmin Tool (OAT) for Informix

IBM® OpenAdmin Tool (OAT) for Informix® 是一種 Web 應用程式，用來管理及分析 IBM Informix 資料庫伺服器的效能。您可以在 Web 伺服器上的單一 OAT 安裝中管理多個資料庫伺服器實例。您可以透過任何瀏覽器存取 Web 伺服器，以管理所有資料庫伺服器。

您可以使用 OAT，來監視及分析資料庫伺服器的效能：

  • 監視具有詳細狀態報告、警示資訊及訊息日誌的資料庫伺服器狀態。
  • 使用 IBM Informix Health Advisor Plug-in for OpenAdmin Tool (OAT)，來分析資料庫伺服器的狀態，並透過電子郵件寄送結果及建議。
  • 隨時間經過監視具有效能歷程詳細圖表的效能統計資料。
  • 監視由「排程器」收集的資料，其使用 SQL 陳述式執行管理作業與收集資訊。
  • 在資料庫及資料庫伺服器上的表格執行系統驗證檢查。
  • 執行系統報告及 onstat 指令。
  • 在 SQL 陳述式往下探查，以檢閱效能明細及最佳化 SQL 陳述式。

您可以使用 OAT，來執行下列管理作業：

  • 管理資料庫伺服器，包括配置、使用者專用權及虛擬處理器。
  • 建立及監視「排程器」作業。
  • 管理「自動更新統計資料 (AUS)」維護系統，可自動化更新統計資料以最佳化查詢效能。
  • 使用 IBM Informix Schema Manager Plug-in for OpenAdmin Tool (OAT) 來管理資料庫及表格物件。
  • 管理儲存體空間及管理資料庫伺服器的回復日誌。
  • 透過壓縮及重組表格以最佳化空間。
  • 備份資料庫伺服器的儲存體空間。
  • 執行 SQL 陳述式及執行表格上的標準 SQL 作業。
  • 監視及管理高可用性叢集。
  • 使用 IBM Informix Replication Plug-in for OpenAdmin Tool (OAT) 來監視及管理 Enterprise Replication。
  • 使用 IBM Informix TimeSeries Plug-in for OpenAdmin Tool (OAT) 來監視及管理與時間序列相關的資料庫物件，包括表格、索引和虛擬表格，以及 TimeSeries 子類型、儲存器和行事曆。

當安裝 IBM Informix Client Software Development Kit (Client SDK) 時，或當安裝 IBM Informix 軟體組並選取 Client SDK 或 IBM Informix Connect 時，您可以安裝 IBM OpenAdmin Tool (OAT) for Informix。 此安裝程式包括 OAT、Health Advisor 外掛程式、「抄寫」外掛程式、「綱目管理程式」外掛程式及 TimeSeries 外掛程式，以及所有執行 OAT 所需的軟體，包括預先配置的 Apache、PHP 及 PDO_informix。

或者，您可以從「IBM Informix 免費產品下載」頁面取得 OpenAdmin Tool Community Edition：https://www14.software.ibm.com/webapp/iwm/web/preLogin.do?lang=en_US&source=swg-informixfpd（在新視窗中開啟）。OpenAdmin Tool Community Edition 包括 OAT 及選用的外掛程式，但不包括支援軟體。 此版本必須手動安裝。

如何連接至資料庫伺服器？

登入資料庫伺服器

使用 IBM OpenAdmin Tool (OAT) for Informix 來登入資料庫伺服器。

若要登入資料庫伺服器，請執行下列動作：

  1. 跳至 OAT 根 URL：http://server name/openadmin。
  2. 在 OAT 的登入頁面中，完成伺服器詳細資料欄位。
  3. 按一下登入。

這時會透過 OAT 將您連接至資料庫伺服器。

提示：或者，如果資料庫伺服器是 OAT 群組的成員，則您可以登入 OAT 群組來連接至資料庫伺服器。OAT 群組包含資料庫伺服器的連線資訊。登入 OAT 群組時，您可以切換到群組中的其他資料庫伺服器，而不需再次登入。

登入 OAT 群組中的資料庫伺服器

登入 IBM OpenAdmin Tool (OAT) for Informix 中的資料庫伺服器連線群組，來連接至資料庫伺服器。

OAT 群組包含資料庫伺服器的連線資訊。登入 OAT 群組時，您可以切換到群組中的其他資料庫伺服器，而不需再次登入。

若要登入 OAT 群組中的資料庫伺服器，請執行下列動作：

  1. 跳至 OAT 根 URL：http://server name/openadmin。
  2. 在 OAT 登入頁面的 OAT 群組清單中，選取包含您要連接之伺服器的 OAT 群組。
  3. 輸入 OAT 群組的密碼。
  4. 按一下取得伺服器。
  5. 從伺服器清單中選取伺服器。伺服器詳細資料欄位中會填入伺服器資訊。
  6. 如果某人已提供或輸入有效的密碼，請在伺服器詳細資料區域的密碼欄位中，接受密碼，然後按一下登入。

  • 相關作業：
  • 新增 OAT 群組
    定義 IBM OpenAdmin Tool (OAT) for Informix 中的資料庫伺服器連線群組。

切換至 OAT 群組中的其他伺服器

連接至 IBM OpenAdmin Tool (OAT) for Informix 中資料庫伺服器連線群組內的其他資料庫伺服器。

OAT 群組包含資料庫伺服器的連線資訊。登入 OAT 群組時，您可以切換到群組中的其他資料庫伺服器，而不需再次登入。

若要切換至 OAT 群組中的其他資料庫伺服器，請選取 OAT 頁面頂端的伺服器清單中的資料庫伺服器。

  • 新增 OAT 群組
    定義 IBM OpenAdmin Tool (OAT) for Informix 中的資料庫伺服器連線群組。

如何監視資料庫伺服器活動？

監視資料庫伺服器狀態

即時監視特定資料庫伺服器的狀態，或同時檢視 OAT 群組中所有伺服器的狀態。

您可以使用儀表板來監視特定資料庫伺服器的狀態。此儀表板會顯示記憶體及空間用量、鎖定、交易及查詢的數目，以及具有最多讀取數及寫入數的表格。

您可以在多台伺服器的儀表板上檢視 OAT 群組中所有資料庫伺服器的狀態。從這個儀表板中，您可以跳至 IBM OpenAdmin Tool (OAT) for Informix 中的其他頁面，以取得相關資訊。

若要監視資料庫伺服器狀態，請執行下列動作：

  1. 在 OAT 功能表上，按一下性能檢測中心 > 儀表板。
  2. 若要重新整理儀表板，請按一下重新整理圖示或變更更新頻率。

自訂儀表板

變更儀表板針對特定資料庫伺服器顯示的狀態資訊。

必要條件：如果 IBM OpenAdmin Tool (OAT) for Informix 中的管理功能表受到密碼保護，則您需要密碼才能執行此作業。

OAT 儀表板會顯示資料庫伺服器的相關資訊；例如，記憶體用量及交易數。您可以自訂儀表板，來指定要顯示的資料庫伺服器相關資訊。

若要自訂儀表板，請執行下列動作：

  1. 在 OAT 中的管理功能表上，按一下儀表板管理程式。
  2. 若要開啟儀表板的佈置編輯器，請按兩下儀表板。

修改儀表板的狀態臨界值

修改多台伺服器儀表板上所顯示之狀態指示器的臨界值。

必要條件：必須安裝 IBM Informix Health Advisor Plug-in for OpenAdmin Tool (OAT)。

IBM OpenAdmin Tool (OAT) for Informix 中的多台伺服器儀表板會顯示 OAT 群組中資料庫伺服器的狀態指示器。 比方說，如果資料庫伺服器上 dbspace 的可用空間低於指定的臨界值，則會針對資料庫伺服器顯示警告指示器。

您可以修改記憶體、空間及 I/O 總計的片段百分比的狀態臨界值。這些臨界值是來自 Health Advisor 外掛程式的預設值。

若要修改儀表板的狀態臨界值，請執行下列動作：

  1. 在 OAT 功能表上，按一下性能檢測中心 > Health Advisor。
  2. 在設定檔頁面上，驗證現行設定檔是否為預設設定檔。
  3. 按一下警示標籤。
  4. 修改下列任一個警示的臨界值：
    Health Advisor 外掛程式警示臨界值 儀表板狀態臨界值
    OS 可用記憶體 記憶體
    Dbspace 可用空間 空間
    片段 I/O 作業數 I/O：I/O 總計的片段百分比

若要監視警示及事件警示，請執行下列動作：

在 OpenAdmin Tool (OAT) for Informix 的功能表上，按一下性能檢測中心 > 警示。

在 OpenAdmin Tool (OAT) for Informix 的功能表上，按一下日誌 > 管理指令。

刪除線上訊息日誌

刪除或截斷資料庫伺服器的線上訊息日誌檔。

必要條件：對於此動作，IBM OpenAdmin Tool (OAT) for Informix 需要 IBM Informix 11.70.xC3 版。

若要刪除線上訊息日誌，請執行下列動作：

  1. 在 OAT 功能表上，按一下日誌 > 線上日誌。
  2. 請執行下列一項動作：
    • 若要刪除日誌檔，請按一下刪除。
    • 若要刪除日誌檔中的訊息，但是保留日誌檔，請按一下截斷。

循環線上訊息日誌

啟用、停用或修改資料庫伺服器的線上訊息日誌檔的自動循環。

必要條件：對於此動作，IBM OpenAdmin Tool (OAT) for Informix 需要 IBM Informix 11.70.xC3 版。

您可以藉由配置自動日誌循環，來控制線上訊息日誌檔的大小。 當線上訊息日誌循環時，資料庫伺服器會切換至新的線上訊息日誌檔，並且將前一個日誌檔的 ID 號碼加一。達到日誌檔數目上限時，會刪除具有最高 ID 的日誌檔。

若要循環線上訊息日誌，請執行下列動作：

  1. 在 OAT 功能表上，按一下日誌 > 線上日誌。
  2. 按一下線上訊息日誌循環標籤。您可以啟用或停用作業、修改排程，或編輯決定日誌檔數目上限的參數。

配置高可用性叢集的連線功能

在 IBM Informix 伺服器上設定 idsd 常駐程式，以便能夠啟動高可用性叢集中的伺服器，或新增共用磁碟次要 (SDS) 伺服器。

必要條件：您在 Informix 伺服器上需要具有 root 專用權，才能執行此程序。

若要使用 IBM OpenAdmin Tool (OAT) for Informix 來執行遠端管理作業，包括啟動高可用性叢集中的伺服器或新增次要伺服器，請在 Informix 伺服器上安裝並配置下列常駐程式：

  • 網際網路服務常駐程式，例如 UNIX 及 Linux 的 xinetd 或 inetd 網際網路常駐程式。
  • idsd，這是一個可讓 OAT 啟動遠端 Informix 伺服器的常駐程序。 此常駐程式位於 Informix 伺服器所在的同一部主機，而且必須位於使用者 informix 可存取的目錄中。 idsd 常駐程式是透過 xinetd 或 inetd 常駐程式來呼叫。
重要事項：在 Windows 或 MAC OS 上，您無法使用遠端管理功能。

若要配置網際網路常駐程式及 idsd 常駐程式，請執行下列動作：

  1. 確保 idsd 常駐程式已隨著 Informix 一起安裝，而且安裝在 $INFORMIXDIR/bin 目錄中。確保擁有者及群組是 informix，且許可權是 0754：

    例如： 

    [informix@myhost]/usr/informix/bin% ls -la idsd 
    -rwxr-xr--  1 informix informix 23717 Oct 15 16:34 idsd
  2. 在伺服器上，設定連線的加密。若為 Informix 11.70，請使用 IBM JRE 1.6 或更新版本中所提供的 GSKCmd 工具。若為舊版的 Informix，請使用 IBM Global Security Kit 中包含的 GSKCmd 工具。
    1. 根據您的平台，使用下列 GSKCmd 指令行參數來建立金鑰資料庫：
      command -keydb -create -db keydb_name.kdb -pw password -type cms -stash
      例如： 
      gsk8cmd -keydb -create -db mydb.kdb -pw mypassw0rd -type cms -stash
    2. 使用下列 GSKCmd 指令行參數來建立自簽憑證：
      command -cert -create -db keydb_name.kdb -pw password -label label_text
    -dn distinguished name 
    -size <1024 | 512 | 2048> -default_cert yes
      其中：distinguished name 是 X.500 識別名稱。distinguished name 是以引號括住的字串，格式如下：CN=common name（必要）, O=organization, OU=organization unit, L=location, ST=state or province, C=country。

      例如： 

      gsk8cmd -cert -create -db mydb.kdb -pw mypassw0rd -label mycertificatelabel
    -dn "CN=lenexa.ibm.com,O=ibm,OU=IBM HTTP Server,ST=KS,C=US" 
    -size 1024 -default_cert yes
  3. 開啟 xinetd 或 inetd 配置檔來編輯，並新增 idsd 常駐程式的服務項目。
    重要事項：如果 GSKIT_VERSION 配置參數設為非預設的 GSKit 版本，請在 xinetd 或 inetd 服務項目中新增 -g NUMBER 伺服器程式引數，以指定 GSKit 版本。
    • 若為 xinetd 伺服器常駐程式：/etc/xinetd.conf。
      service idsd
             {
             disable = no
             socket_type = stream 
             protocol = tcp
             user = root
             wait = no 	    
             server = /usr/informix/bin/idsd
             server_args = -l /usr/informix/idsd.log 
                 -k /usr/informix/idsd -n idsd_label
             }
    • 若為 UNIX 上的 inetd 伺服器常駐程式：/etc/inetd.conf： 
      idsd stream tcp nowait root /usr/informix/bin/idsd idsd -l /usr/informix/idsd.log  -k /usr/informix/idsd -n idsd_label
  4. 開啟檔案 /etc/services 來編輯，並新增名為 idsd 的服務。例如： 
    idsd 10000/tcp
    此範例中使用埠號 10000，但您可以使用未在使用中且大於 1024 的其他任何埠。 不過，名稱必須是 idsd，且通訊協定必須是 tcp。
  5. 重新啟動 xinetd 或 inetd 服務常駐程式。在大部分實作中，您可以將 SIGHUP 信號傳送至執行中的 inetd 或 xinetd 處理程序，以重新啟動。然而，在部分 Linux 平台上，您可以用 root 身分執行下列指令： 
    service xinetd restart

管理自動更新統計資料

檢閱及調整表格統計資料的「自動更新統計資料 (AUS)」維護系統。

AUS 維護系統可識別需要新最佳化統計資料的表格及索引，並執行適當的 UPDATE STATISTICS 陳述式來最佳化查詢效能。根據系統的不同，您可能需要調整 AUS 的到期原則或排程。

您可以在伺服器管理 > 自動更新統計資料 > 配置頁面上，控制及監視 AUS 原則。

您可以在 SQL 工具箱 > 綱目管理程式 > 統計資料頁面上使用 IBM Informix Schema Manager Plug-in for OpenAdmin Tool (OAT)，來檢閱表格統計資料。

配置及啟用自動更新統計資料

  1. 在 IBM OpenAdmin Tool (OAT) for Informix 的功能表上，按一下伺服器管理 > 自動更新統計資料 > 配置。
  2. 根據您的系統效能及需求，判斷預設值是否適用於您的系統。
    1. 若要指定統計資料重建之間的時間上限，請在 AUS_AGE 參數中輸入天數，例如 30、60 或 90。
    2. 若要指定表格的變更必須達到多少百分比，才能開始重建統計資料，請在 AUS_CHANGE 參數中輸入百分比數字，例如 20 或 50。
    3. 若要指定 AUS 是否使用建議的準則規則，請從 AUS_AUTO_RULES 參數的清單中選取開啟或關閉。預設值是開啟。
    4. 若要指定低於多少列時一律重建統計資料，請在 AUS_SMALL_TABLES 參數中輸入數字。預設值為 100。
    5. 若要指定 PDQ 優先順序，請在 AUS_PDQ 參數中輸入 -1 至 100 的數字。預設值是 10。判斷資料庫伺服器如何使用資源來進行平行處理時，「平行資料庫查詢 (PDQ)」優先順序是其中一個因數。
    6. 若要指定用來執行更新統計資料作業的執行緒數目，請輸入執行緒的數目。OAT 會將這個數目的 AUS 作業提交給「作業排程器」。此選項需要 IBM Informix 11.70 版。
  3. 按一下儲存以保存您所做的變更。
  4. 若要判斷 AUS 作業是否已啟用，請按一下資訊標籤，然後按一下這些作業：
    • 自動更新統計資料評估
    • 自動更新統計資料重新整理
      附註：如果「自動更新統計資料重新整理」作業沒有可處理的工作，則作業會自行停用。 下一次執行「自動更新統計資料評估」作業及建立重新整理作業的工作時，評估作業會自動啟用重新整理作業。
    作業詳細資料頁面隨即出現。
  5. 向下捲動至啟用作業欄位，並確定已選取它。
  6. 在欄位中設定執行時段，以指定作業的執行頻率。由於 AUS 是資源密集的作業，您可以定義特定的執行時段。 執行時段定義包括開始時間及結束時間，以及一星期中要在那幾天執行。 依預設，執行時間範圍是每日上午 1:00 – 上午 5:00。
  7. 按一下儲存以儲存任何變更。

  • 相關概念：
  • AUS 到期原則（效能手冊）

在伺服器上檢視重新整理作業及評估作業

如何管理「排程器」作業？

新增排程器作業

您可以建立「排程器」作業，以便依指定的時間及頻率來執行作業或感應器。

  • 作業可讓您依特定的時間或間隔來執行特定的工作。
  • 感應器可收集並儲存資訊。

新增作業

重要事項：您建立作業當時的時間會影響第一次執行作業的時間。
  • 如果現行時間在指定的啟動和停止時間內，則會立即執行作業。 後續就依頻率值所指定的間隔來執行。 因此，舉例來說，假設現行時間在啟動和停止時間內，且頻率設為 15 分鐘， 則排定作業之後會立即開始第一次執行作業，下一次執行則在 15 分鐘之後。
  • 不過，如果現行時間不是在啟動和停止時間內，則作業會等到下一次啟動時間才執行，可能在當日或次日。 例如，假設現行時間是 1:00 a.m.，而下一次啟動時間是 7:00 p.m. (19:00)，則作業會在同一天的 19:00 啟動。

若要新增作業，請執行下列動作：

  1. 從「作業排程」頁面中按一下新增作業。
  2. 在您想要新增何種類型的作業？下按一下作業，然後按下一步。
  3. 在作業名稱欄位中輸入名稱。例如：MyTask。
  4. 從作業群組清單中選取種類。
  5. 在文字欄位中加入新作業的說明，然後按下一步。
  6. 指定容許執行作業的時間範圍，例如，每天從早上 10 點 (10:00) 到晚上 6 點 (18:00)，每小時執行一次。
    1. 使用 24 小時制來指定時間間隔的開始時間：hh:mm:ss。例如：10:00:00。
    2. 使用 24 小時制來指定時間間隔的停止時間：hh:mm:ss。例如：18:00:00。 您也可以指定 NEVER。
    3. 以日數、時數及分鐘數，指定在容許的時間間隔內可執行作業的頻率：d 日 h 小時 m 分鐘。例如：0 日 1 小時 00 分鐘。
  7. 按下一步。
  8. 指定作業要執行的指令陳述式。例如，假設您想要刪除細項作業，您可以輸入下列陳述式： 
    delete from ph_task where tk_group='MISC'
  9. 按下一步。確認頁面隨即出現。
  10. 驗證每一個選擇的輸入項目：
    • 如果輸入項目正確，請按一下完成，以確認並建立作業。
    • 按上一步，以回到前一頁來更正不正確的值。

新增感應器

  1. 從「作業排程」頁面中，按一下新增作業。
  2. 在您想要新增何種類型的作業？下按一下感應器，然後按下一步。
  3. 在感應器名稱欄位中輸入名稱。例如：MySensor。
  4. 從感應器群組清單中選取種類。
  5. 在文字欄位中加入新作業的說明，然後按下一步。
  6. 指定容許執行感應器的時間範圍，例如，每天從 10:00 a.m. 到 6:00 p.m.，每小時執行一次。
    1. 使用 24 小時制來指定時間範圍的開始時間：hh:mm:ss。例如：10:00:00
    2. 使用 24 小時制來指定時間範圍的停止時間：hh:mm:ss。例如：18:00:00
    3. 以日數、時數及分鐘數，指定在容許的時間範圍內可執行作業的頻率：d 日 h 小時 m 分鐘。例如：0 日 1 小時 00 分鐘。
    4. 以日數、時數及分鐘數，指定資料刪除時間： d 日 h 小時 m 分鐘。例如：1 日 0 小時 0 分鐘。
  7. 按下一步。
  8. 在結果表格名稱欄位中，指定用來保留感應器資料的結果表格名稱。例如：MyResults
  9. 按下一步。
  10. 在 SQL 陳述式欄位中輸入直欄名稱及定義，以完成結果表格的 SQL 陳述式。例如，假設您想要在表格 TaskA_ReturnCode 中收集 taskA（task_id 為 10）的回覆碼，您可以輸入下列 SQL 陳述式： 
    create table TaskA_ReturnCode(task_name varchar(20), errorcode integer)
  11. 指定感應器要執行的指令陳述式。例如： 
    insert into TaskA_ReturnCode values('TaskA', (select run_retcode from ph_run where run_task_id=10))
  12. 驗證每一個選擇的輸入項目。
    • 如果輸入項目正確，請按一下完成，以確認並建立感應器。
    • 按上一步，以回到前一頁來更新任何不正確的值。

  3. 選取啟用作業。
  4. 按一下儲存。

變更排程器作業的容許時間間隔

容許的時間間隔是指可以執行作業的時間範圍，例如從 10:00 a.m. 到 6:00 p.m.。

  1. 展開作業排程器 > 排程器。「作業詳細資料」頁面中會顯示作業及其排程詳細資訊的清單。
  2. 按一下作業名稱，以變更排程執行時期的頻率或特性。您可以更新作業執行的日數或次數。
  3. 使用 24 小時制來指定時間範圍的開始時間：hh:mm:ss。例如，指定 10:00:00 代表早上 10 點。
  4. 使用 24 小時制來指定時間範圍的停止時間：hh:mm:ss。例如，指定 18:00:00 代表晚上 6 點。

  7. 按一下儲存以儲存您所做的變更。

  3. 找出作業名稱，然後按一下刪除。如果作業是關於感應器，則會提示您刪除與感應器相關聯的結果表格。
  4. 決定如何處理表格。
    • 按一下是，以刪除結果表格及作業。
    • 按一下否，在刪除作業時保留表格。
    • 按一下取消，以取消刪除作業及表格。

如何探索效能資訊？

使用 SQL 瀏覽器來分析查詢效能

您必須開啟 SQL 追蹤，才能使用「SQL 瀏覽器」。

使用「SQL 瀏覽器」來執行查詢往下探查。「SQL 瀏覽器」會使用 SQL 追蹤，以收集系統上執行的每一個 SQL 陳述式的相關統計資訊，並顯示陳述式歷程。下列章節會說明如何開啟 SQL 追蹤，以及如何使用「SQL 瀏覽器」：

開啟 SQL 追蹤

  1. 導覽至效能分析 > SQL 瀏覽器。如果 SQL 追蹤不在執行中，則會顯示下列訊息：SQL TRACING 目前關閉。請使用「管理」選項來切回開啟。
  2. 按一下管理，以導覽至「SQL 追蹤設定檔」頁面
  3. 從追蹤狀態清單中，選取開啟。
  4. 按一下儲存以開啟 SQL 追蹤。

關閉 SQL 追蹤

  1. 導覽至效能分析 > SQL 瀏覽器。如果 SQL 追蹤已開啟，則您可以選取「SQL 瀏覽器」的標籤。如果 SQL 追蹤已關閉，則會顯示下列訊息：SQL TRACING 目前關閉。請使用「管理」選項來切回開啟。
  2. 按一下追蹤管理標籤，以導覽至「SQL 追蹤設定檔」頁面
  3. 從追蹤狀態清單中，選取關閉。
  4. 按一下儲存以關閉 SQL 追蹤。

指定 SQL 追蹤的選項

「SQL 瀏覽器」顯示現行伺服器上執行的 SQL 陳述式的詳細資訊。 您可以從追蹤管理頁面中，指定「SQL 瀏覽器」中要追蹤及可用的資訊。

若要設定「SQL 瀏覽器」詳細資料，請執行下列動作：

  1. 在 IBM OpenAdmin Tool (OAT) for Informix 的功能表上，按一下效能分析 > SQL 瀏覽器 > 追蹤管理。
  2. 指定下列 SQL 追蹤選項：
    • 選取廣域或使用者追蹤模式。此選項是針對 IBM Informix 11.50.xC3 以及更新版本而顯示。如需相關資訊，請參閱追蹤模式。
    • 選取表格名稱勾選框，以在追蹤資訊中併入表格的名稱。
    • 選取程序堆疊勾選框，以在追蹤資訊中併入程序堆疊。
    • 選取主變數勾選框，以在追蹤資訊中併入主變數的相關資訊。
    • 選取清除追蹤緩衝區勾選框，以在變更及儲存追蹤選項時，清空追蹤緩衝區。
    • 輸入要追蹤的陳述式數目 (500 - 2147483647)。
    • 輸入每個 SQL 陳述式的追蹤大小上限（512 - 102400，以位元組為單位）。
  3. 按一下儲存。

追蹤模式

追蹤模式選項是針對 IBM Informix 11.50.xC3 以及更新版本而顯示。

  • 選取廣域，以追蹤整個伺服器的 SQL 陳述式。 「SQL 瀏覽器」可以顯示所有資料庫及使用者階段作業的資訊，但不包括在資料庫追蹤及使用者追蹤清單中指定的階段作業。
  • 選取使用者，以追蹤一連串資料庫或使用者階段作業的 SQL 陳述式。 「SQL 瀏覽器」只能顯示資料庫追蹤及使用者追蹤清單中指定的資料庫及使用者階段作業的資訊，除非這些清單空白。
    • 若要顯示資料庫追蹤或使用者追蹤清單，請按一下顯示。若要隱藏清單，請按一下隱藏。
    • 若要修改資料庫追蹤清單，請執行下列動作：
      • 新增資料庫：從未追蹤的資料庫清單中，選取資料庫名稱，然後按一下 +。資料庫名稱會顯示於已追蹤的資料庫清單中。
      • 刪除資料庫：從已追蹤的資料庫清單中，選取資料庫名稱，然後按一下 -。資料庫名稱會顯示於未追蹤的資料庫清單中。
    • 若要修改使用者追蹤清單，請執行下列動作：
      • 新增使用者：從未追蹤清單中，選取資料庫名稱，然後按一下 +。資料庫名稱會顯示於已追蹤的資料庫清單中。
      • 刪除使用者：從已追蹤的資料庫清單中，選取資料庫名稱，然後按一下 -。資料庫名稱會顯示於未追蹤的資料庫清單中。

最佳化 SQL 陳述式

使用 IBM OpenAdmin Tool (OAT) for Informix 來建立外部指引，以將 SQL 陳述式最佳化。

必要條件：
  • 在資料庫伺服器 onconfig 檔案中，將 EXT_DIRECTIVES 配置參數設為 1 或 2，以便能夠儲存外部指引。 變更 EXT_DIRECTIVES 配置參數後，請重新啟動資料庫伺服器，變更才會生效。
  • 確保 SQL 追蹤已開啟，且追蹤管理頁面上已選取表格名稱選項。 請參閱開啟 SQL 追蹤及指定 SQL 追蹤的選項。

您可以建立外部指引，以套用至 SQL 陳述式。 指引是一種註解，可告知查詢最佳化工具執行查詢的方式。 外部指引可提供方法，讓您變更 SQL 陳述式的查詢執行計劃。 當您將外部指引套用至 SQL 陳述式之後，每次重新執行此 SQL 陳述式時，將會依此變更 SQL 陳述式的查詢執行計劃。

重要事項：當您重新執行 SQL 陳述式時，此 SQL 陳述式必須完全符合您已套用指引的 SQL 陳述式。例如，必須有相同的字母大小寫及相同的間距。

若要建立外部指引，請執行下列動作：

  1. 在 OAT 功能表上，按一下效能分析 > SQL 瀏覽器。
  2. 往下探查至 SQL 陳述式。
    1. 按一下 SQL 標籤。
    2. 按一下清單中的 SQL 陳述式。
    3. 按一下 SQL 陳述式執行清單中的任一列。
  3. 按一下最佳化標籤。
  4. 按一下新增。
  5. 在新增指引頁面上，選取指引。視需要選取表格及索引。若要新增其他 SQL 陳述式指引，請按一下新增另一個，或按一下新增來關閉頁面。
  6. 指定套用指引的方法，然後按一下套用。預設值為非作用中：不要將指引套用至 SQL 陳述式。

如何設定伺服器的存取權？

新增 OAT 群組

定義 IBM OpenAdmin Tool (OAT) for Informix 中的資料庫伺服器連線群組。

OAT 群組可讓您管理資料庫伺服器的群組，並可讓您登入群組，而無須逐一登入個別伺服器。

若要新增 OAT 群組，請執行下列動作：

  1. 在 OAT 管理功能表上，展開管理連線，然後按一下新增群組。
  2. 在群組名稱欄位中，輸入群組的英數名稱。
  3. 在密碼欄位中，輸入英數密碼。
  4. 選用項目：若要指定群組具有唯讀專用權，請選取唯讀。
    重要事項：唯讀群組會防止使用者修改伺服器；但無法防止使用者存取伺服器內含的可能機密資料。請限制唯讀群組的成員資格，僅限於已授權在伺服器上檢視詳細資料的使用者。

    如需唯讀群組限制的相關資訊，請參閱唯讀群組限制。

  5. 按一下新增，以儲存群組資訊並新增群組。

  3. 輸入連線資訊。
    重要事項：如果針對每一個新增的連線，您是使用 3.0 版以前的 Informix Connect 或 IBM Informix Client Software Development Kit (Client SDK) 版本，請確保該連線在 Web 伺服器電腦上有相對應的 SQLHOSTS 項目。

提供「管理」配置頁面的密碼保護：Apache 範例

基於安全考量，限制對 IBM OpenAdmin Tool (OAT) for Informix 的「管理」頁面的存取。使用密碼來保護這些頁面很重要，因為從登入頁面就可以存取管理頁面，因此每個人都可以存取。

一種作法是使用 Web 伺服器提供的內建鑑別機制。 例如，您可以保護安裝目錄 OATINSTALL/admin/.htaccess。

範例：若要從 Apache 內保護 OAT 目錄，請執行下列動作：

  1. 使用 Apache 隨附的 htpasswd 公用程式，在從 Web 無法存取的目錄中建立密碼檔。
  2. 將鑑別指引新增至下列其中一個檔案：
    • OATINSTALL/admin/.htaccess 檔案
    • httpd.conf 檔案
  3. 將指引放入 <Directory> 區段內。例如： 
    <Directory OATINSTALL/admin>
AuthType Basic
AuthName "OAT Admin"
AuthUserFile PASSWORDFILE
Require ValidUser
</Directory>
    其中，PASSWORDFILE 是您使用 htpasswd 公用程式來建立密碼檔的目錄。

如需相關資訊及範例，請閱讀 Apache Software Foundation 網站上關於 Apache HTTP 伺服器 2.0 版鑑別、授權及存取控制的文章：www.apache.org（在新視窗中開啟）。

配置 OAT 的 HTTPS (Linux)

啟用 HTTPS 以保護 IBM OpenAdmin Tool (OAT) for Informix 的 Web 伺服器，避免遭受竊聽、竄改、訊息偽造，以及與網路相互干擾。

必要條件：在安裝 OAT 的 Apache 伺服器的電腦上，必須安裝 OpenSSL。您可以從「OpenSSL 專案」網站下載並安裝 OpenSSL：www.openssl.org。

當 HTTPS 啟用時，來自 OAT 用戶端的訊息會先加密，然後再傳送至 OAT Web 伺服器。加密可以防止未獲授權的使用者在線路上監聽及竊取機密資訊。當 HTTPS 啟用時，OAT 用戶端也可以向 OAT 主機鑑別，讓未獲授權的使用者無法利用假的 Web 伺服器欺騙 OAT 用戶端。

重要事項：HTTPS 只會加密 OAT Web 伺服器與用戶端之間的通訊。不會加密 IBM Informix 資料庫伺服器與 Web 伺服器之間通訊。IBM Informix 9.4 版以及更新版本可以使用加密通訊支援模組，來啟用 Informix 伺服器與 OAT Web 伺服器之間的資料加密功能。 如需加密及 Informix 伺服器的相關資訊，請閱讀 developerWorks® 網站上的 "Encryption over the wire with IDS 9.40"。

當您安裝 OAT 與 IBM Informix Client Software Development Kit (Client SDK) 時，安裝程式會提供必要軟體來執行 OAT，包括預先配置的 Apache、PHP 及 PDO_informix。 在 Linux 上，會針對 mod_ssl 加密法模組啟用隨著 OAT 一起安裝的 Apache Web 伺服器。若要啟用 HTTPS，則需要此模組。

若要在 Linux 上啟用 OAT 中的 HTTPS，請完成下列步驟：

  1. 為 Web 伺服器建立加密金鑰及憑證，讓 OAT 用戶端可以根據您的憑證來鑑別 Web 伺服器。
  2. 將 Apache 配置為啟用 HTTPS。

如需啟用 HTTPS 的相關資訊，請閱讀 developerWorks 網站上的 "Securing the OpenAdmin Tool for Informix web server with HTTPS"。

建立加密金鑰

金鑰是用於加密與解密。它們通常是成對提供的，即公開金鑰與私密金鑰。公開金鑰是用來加密訊息，而私密金鑰則用來解密訊息。 憑證是鑑別 Web 伺服器的文件。已啟用 HTTPS 的 Web 伺服器具有一個已由信任的憑證管理中心所簽署的憑證，用來驗證 Web 伺服器。 在針對 Web 伺服器啟用 HTTPS 之後，用戶端會選擇要與 Web 伺服器建立一般連線還是安全連線。若要建立一般連線，用戶端會在其 Web 瀏覽器中鍵入 http://web server_url。若要建立安全連線，用戶端會鍵入 https://web server_url。

若要產生私密金鑰與公開金鑰的配對及憑證，您可以使用 OpenSSL 安裝的 bin 目錄中的 openssl 執行檔。

當執行指令來產生私密金鑰時，系統會提示您輸入通行詞組，以保護私密金鑰。每次執行需要使用私密金鑰的指令時，您必須重新輸入這個通行詞組。 例如，在啟動 Apache Web 伺服器或使用私密金鑰來產生憑證時，您必須輸入通行詞組。在 Linux 上，基於安全考量，請使用通行詞組來保護並加密您的私密金鑰。

若要產生私密金鑰，請在安裝 OpenSSL 的電腦上，執行下列指令：

openssl genrsa -des3 -out privkey.pem 2048

私密金鑰會建立在 privkey.pem 檔案中。 因為這個檔案包含 Web 伺服器的解密金鑰，所以請將它儲存在安全的位置。當您產生憑證時，這個檔案會用來產生相關聯的公開金鑰，而該金鑰將併入憑證中。

建立憑證

在建立私密金鑰之後，請建立憑證來鑑別 Web 伺服器的身分。若要產生憑證，您可以建立憑證簽署要求，並將憑證簽署要求傳送至信任的憑證管理中心。此管理中心會發出憑證。

若要產生憑證簽署要求，請執行下列指令：

openssl req -new -key privkey.pem -out cert.csr

如需憑證簽署要求的相關資訊，請聯絡您的憑證管理中心。

或者，您可以建立自簽憑證，而不需要透過憑證管理中心處理，但不建議使用自簽憑證。

若要產生自簽憑證，請執行下列指令：

openssl req -new -x509 -key privkey.pem -out cacert.pem -days 1095

OpenSSL 會提示您輸入資訊。憑證會儲存在 cacert.pem 檔案中。 此檔案會顯示給 Web 用戶端，以驗證您的身分。它也會包括 Web 用戶端的公開金鑰，以便它們可以加密通訊。

如需加密金鑰的相關資訊，請參閱 OpenSSL 文件。

將 Apache 配置為啟用 HTTPS

若要啟用 HTTPS，請更新 Apache 配置檔及 Apache SSL 配置檔：

  1. 找出 Apache 配置檔。路徑包括 Apache 版本，例如： 
    /oat/installation/directory/Apache_2.4.2/conf/httpd.conf
  2. 在 Apache 配置檔中，從下行中移除註解：#Include conf/extra/httpd-ssl.conf。

    依預設，會註銷此行。當移除註解時，Apache SSL 配置檔即會併入 httpd.conf 檔案中。

  3. 找出 Apache SSL 配置檔。路徑包括 Apache 版本，例如： 
    /oat/installation/directory/Apache_2.4.2/conf/extra/httpd-ssl.conf
  4. 在 Apache SSL 配置檔中，驗證 SSL 埠號是否設為可用埠。

    HTTPS 需要個別的 SSL 埠。依預設，SSL 埠號會設為 443。如果此埠無法使用，請變更接聽指引及「虛擬主機」區段中的 SSL 埠。

  5. 在 Apache SSL 配置檔中，驗證 SSLCertificateKeyFile 指引是否指出您所建立之私密金鑰檔的正確位置：privkey.pem。
  6. 驗證 SSLCertificateFile 指引是否指出您所建立之憑證檔的正確位置：cacert.pem。
  7. 選用項目：若要控制 Web 伺服器接受哪些密碼，請編輯 SSLCipherSuite 指引。如需 SSLCipherSuite 指引的相關資訊，請參閱 Apache mod_ssl 文件。

如需 HTTPS 配置的相關資訊，請參閱 Apache 網站：www.apache.org。

    重要事項：將目錄名稱設為 Apache_2.2.22，即使您是安裝更新版本的 Apache，亦然。OAT 安裝及其服務會配置為使用目錄名稱 Apache_2.2.22。 如果新的 Apache 安裝具有不同的目錄名稱，則它們將無法運作。
  5. 編輯 Apache 配置檔：/oat/installation/directory/Apache_2.2.22/conf/httpd.conf。 在這個 httpd.conf 檔案中，新增或解除註解下列幾行： 
    LoadModule php5_module
"c:\oat\installation\dir\PHP_5.2.4\php5apache2_2.dll"
LoadModule ssl_module modules/mod_ssl.so
AddType application/x-httpd-php .php
PhpIniDir 'c:\oat\installation\dir\PHP_5.2.4'
  6. 在 httpd.conf 檔案中，更新下行：Listen 80（或 Listen 8080）。 此行指出 OAT Web 伺服器的埠號。以與原始 OAT Apache Web 伺服器（其隨著 OAT 一起安裝）相同的埠號取代 80 或 8080。
  7. 在 httpd.conf 檔案中，更新下行：ServerName www.example.com:80。此行指出伺服器用來識別本身的名稱及埠。以原始 OAT Apache 配置檔 c:\oat\installation\dir\Apache_2.2.22_noSSL\conf\httpd.conf 中的 ServerName 整行取代此行。
  8. 在 httpd.conf 檔案中，更新下行：DirectoryIndex index.html。此行設定要求目錄時 Apache 提供的檔案。將此行變更為：DirectoryIndex index.html index.php。
  9. 在原始 OAT Apache 配置檔 c:\oat\installation\dir\Apache_2.2.22_noSSL\conf\httpd.conf 中，找出下行：setenv INFORMIXDIR。 此行會在 OAT 的 Apache 環境中設定 INFORMIXDIR 變數。若要設定新 Apache Web 伺服器的變數，請將此行複製到新 Apache Web 伺服器的 httpd.conf 檔案。您可以將此行放置在檔案結尾處。
  10. 將整個目錄 c:\oat\installation\dir\Apache_2.2.22_noSSL\htdocs\openadmin\ 複製到 c:\oat\installation\dir\Apache_2.2.22\htdocs\openadmin\。 所有 OAT 原始碼都位於此目錄中。
  11. 在命令提示字元中執行下列指令，以確定已適當地載入 PHP Apache 處理程式及 mod_ssl 模組： 
    cd c:\oat\installation\dir\Apache_2.2.22\bin\
httpd.exe –M
    此指令會顯示 Apache 模組的清單。 確認 php5_module 及 ssl_module 位於清單上。 現在應該已對 OAT 適當地設定新的 Web 伺服器。
  12. 在開始功能表上，按一下 IBM OpenAdmin Tool for Informix，然後按一下啟動 Apache Service for OpenAdmin Tool。 您可以使用 Web 瀏覽器來存取 OAT。Web 伺服器已啟用 mod_ssl，但 HTTPS 尚未開啟。

建立加密金鑰

金鑰是用於加密與解密。它們通常是成對提供的，即公開金鑰與私密金鑰。公開金鑰是用來加密訊息，而私密金鑰則用來解密訊息。 憑證是鑑別 Web 伺服器的文件。已啟用 HTTPS 的 Web 伺服器具有一個已由信任的憑證管理中心所簽署的憑證，用來驗證 Web 伺服器。 在針對 Web 伺服器啟用 HTTPS 之後，用戶端會選擇要與 Web 伺服器建立一般連線還是安全連線。若要建立一般連線，用戶端會在其 Web 瀏覽器中鍵入 http://web server_url。若要建立安全連線，用戶端會鍵入 https://web server_url。

若要產生私密金鑰與公開金鑰的配對及憑證，您可以使用新 Apache Web 伺服器安裝的 bin 目錄中的 openssl 執行檔。

當執行指令來產生私密金鑰時，系統會提示您輸入通行詞組，以保護私密金鑰。在 Windows 上，Apache 不支援私密金鑰的通行詞組，因此您必須如後續步驟所述來移除通行詞組。

若要建立私密金鑰，請執行下列動作：

  1. 若要產生私密金鑰，請在安裝新 Apache Web 伺服器的電腦上，執行下列指令：
    openssl genrsa -des3 -out privkey.pem 2048
  2. 若要移除通行詞組，請執行下列指令： 
    openssl rsa -in privkey.pem -out privkey_nopassphrase.pem

未加密的私密金鑰版本會建立在 privkey_nopassphrase.pem 檔案中。 因為這個檔案包含未加密格式的私密金鑰，所以請將它儲存在安全的位置。當您產生憑證時，這個檔案會用來產生相關聯的公開金鑰，而該金鑰將併入憑證中。

建立憑證

在建立私密金鑰之後，請建立憑證來鑑別 Web 伺服器的身分。若要建立憑證，您可以產生憑證簽署要求，並將憑證簽署要求傳送至信任的憑證管理中心。此管理中心會發出憑證。

若要產生憑證簽署要求，請執行下列指令：

openssl req -new -key privkey_nopassphrase.pem -out cert.csr

如需憑證簽署要求的相關資訊，請聯絡您的憑證管理中心。

或者，您可以建立自簽憑證，而不需要透過憑證管理中心處理，但不建議使用自簽憑證。

若要產生自簽憑證，請執行下列指令：

openssl req -new -x509 -key privkey_nopassphrase.pem -out cacert.pem -days 1095

OpenSSL 會提示您輸入資訊。憑證會儲存在 cacert.pem 檔案中。 此檔案會顯示給 Web 用戶端，以驗證您的身分。它也會包括 Web 用戶端的公開金鑰，以便它們可以加密通訊。

如需加密金鑰的相關資訊，請參閱 OpenSSL 文件。

將 Apache 配置為啟用 HTTPS

若要啟用 HTTPS，請更新 Apache 配置檔及 Apache SSL 配置檔：

  1. 找出 Apache 配置檔。路徑包括 Apache 版本，例如： 
    /oat/installation/directory/Apache_2.2.22/conf/httpd.conf
  2. 在 Apache 配置檔中，從下行中移除註解：#Include conf/extra/httpd-ssl.conf。

    依預設，會註銷此行。當移除註解時，Apache SSL 配置檔即會併入 httpd.conf 檔案中。

  3. 找出 Apache SSL 配置檔。路徑包括 Apache 版本，例如： 
    /oat/installation/directory/Apache_2.2.22/conf/extra/httpd-ssl.conf
  4. 在 Apache SSL 配置檔中，驗證 SSL 埠號是否設為可用埠。

    HTTPS 需要個別的 SSL 埠。依預設，SSL 埠號會設為 443。如果此埠無法使用，請變更接聽指引及「虛擬主機」區段中的 SSL 埠。

  5. 在 Apache SSL 配置檔中，驗證 SSLCertificateKeyFile 指引是否指出您所建立之私密金鑰檔的正確位置：privkey_nopassphrase.pem。
  6. 驗證 SSLCertificateFile 指引是否指出您所建立之憑證檔的正確位置：cacert.pem。
  7. 選用項目：若要控制 Web 伺服器接受哪些密碼，請編輯 SSLCipherSuite 指引。如需 SSLCipherSuite 指引的相關資訊，請參閱 Apache mod_ssl 文件。

如需 HTTPS 配置的相關資訊，請參閱 Apache 網站：www.apache.org。

測試配置

您必須重新啟動 Web 伺服器，HTTPS 變更才會生效。請使用 apachemonitor.exe 或開始功能表捷徑來重新啟動 Web 伺服器。

在重新啟動 Web 伺服器之後，請在 Web 瀏覽器中使用下列 URL 來開啟 OAT：https://hostname:ssl_portnumber/openadmin。 在顯示 OAT 登入頁面之前，系統會提示您檢視並接受 OAT Web 伺服器的憑證。

idsadmin 類別函數

idsadmin 類別針對 IBM OpenAdmin Tool (OAT) for Informix 的外掛程式提供基本函數庫，包括建立資料庫連線、載入語言檔案，以及以 HTML 呈現輸出。

idsadmin 物件會當作模組建構子的參數，傳遞至每一個模組。

isreadonly 函數

isreadonly 函數指出使用者是否使用唯讀群組登入。 此函數是 idsadmin 類別的一部分，而此類別會針對 IBM OpenAdmin Tool (OAT) for Informix 的外掛程式提供基本函數庫。

語法

isreadonly ()

使用情況

使用 isreadonly 函數來判斷使用者是否使用具有唯讀專用權的群組登入 OAT。您可以使用此資訊來判斷使用者可在 OAT 中存取的項目，以及要提供哪些項目給使用者使用。

回覆值

true 的回覆值指出使用者是使用具有唯讀專用權的群組登入。

false 的回覆值指出使用者是使用未限制只有唯讀專用權的群組登入。

範例

如果使用者唯讀，這個範例會傳回錯誤並顯示訊息。在這個範例中，PermissionRequired 是語言關鍵字，而不是訊息本身。 

if ( $this->idsadmin->isreadonly() )
{
    $this->idsadmin->fatal_error($this->idsadmin->lang("PermissionRequired"));
    return;         
}

get_database 函數

get_database 函數會建立對資料庫伺服器上之資料庫的連線。 此函數是 idsadmin 類別的一部分，而此類別會針對 IBM OpenAdmin Tool (OAT) for Informix 的外掛程式提供基本函數庫。

語法

get_database($dbname="sysmaster",$use_user=false)
變數 用途 限制

use_user 是否使用「SQL 工具箱」使用者來連接至資料庫的指示器。當您在「SQL 工具箱」中時，請將值設為 true。 選用。預設值是 false。

回覆值

回覆值是 PDO 資料庫連線。

範例

這個範例會建立與 sysmaster 資料庫的連線。

 $db = $this->idsadmin->get_database("sysmaster");

load_lang 函數

load_lang 函數會將模組的語言檔案載入到 idsadmin 語言陣列。此函數是 idsadmin 類別的一部分，而此類別會針對 IBM OpenAdmin Tool (OAT) for Informix 的外掛程式提供基本函數庫。

語法

load_lang($keyword)
變數 用途 限制
keyword 指定要針對模組載入哪個語言檔案的關鍵字。 依照慣例，這個關鍵字與模組的 act 關鍵字相同。語言檔案包含了模組的字串。

function __construct(&$idsadmin)

        $this->idsadmin->load_lang("health");
}

lang 函數

lang($item, $parameters=array())
變數 用途 限制
item 要以現行語言傳回的字串名稱。

$this->idsadmin->html->set_pagetitle($this->idsadmin->lang("alerts") );

format_units 函數會以最適合人類讀取的形式格式化整數位元組值，例如 KB、MB 或 GB。此函數是 idsadmin 類別的一部分，而此類別會針對 IBM OpenAdmin Tool (OAT) for Informix 的外掛程式提供基本函數庫。

語法

format_units($value,$prec=2 )
變數 用途 限制
value 整數值（以位元組為單位）。 必須為整數。
prec 已格式化數字的精準度。 選用。必須為整數。預設值為 2，其指定 2 個小數位數。

回覆值

回覆值是已格式化的數字，其字尾指出單位：B、KB、MB、GB、TB、PB 或 EB。

範例

此範例產生 $formatted_page_size 變數，其中保留已格式化的值 2 K。

$page_size = 2048;
$formatted_page_size = $this->idsadmin->format_units($page_size);

語法

setCurrMenuItem($item="")
變數 用途 限制
item 功能表項目的索引鍵名稱。 索引鍵名稱必須存在。傳遞不存在的功能表項目變數，不會引起錯誤，但不會強調顯示功能表中的任何項目。

使用情況

setCurrMenuItem 函數會在網頁呈現於瀏覽器時，藉由強調顯示現行功能表項目來設定它。此函數不會建立功能表項目。

範例

此範例將現行功能表項目設為 Storage。

$this->idsadmin->setCurrMenuItem("storage");
