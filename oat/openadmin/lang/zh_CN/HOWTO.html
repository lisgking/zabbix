IBM OpenAdmin Tool (OAT) for Informix

IBM® OpenAdmin Tool (OAT) for Informix® 是用于管理和分析 IBM Informix 数据库服务器性能的 Web 应用程序。您可以通过 Web 服务器上安装的单个 OAT 管理多个数据库服务器实例。您可以通过任意浏览器访问 Web 服务器来管理所有数据库服务器。

您可以使用 OAT 来监视和分析数据库服务器的性能：

  • 通过详细的状态报告、警报信息和消息日志来监视数据库服务器的状态。
  • 使用 OpenAdmin Tool (OAT) 的 IBM Informix Health Advisor 插件来分析数据库服务器的状态，并通过电子邮件发送结果和建议。
  • 通过详细的性能历史记录图表来监视一段时间内的性能统计信息。
  • 监视调度程序收集的数据，该调度程序会使用 SQL 语句来执行管理任务并收集信息。
  • 对数据库服务器上的数据库和表执行系统验证检查。
  • 执行系统报告和 onstat 命令。
  • 向下钻取 SQL 语句以查看性能详细信息并优化 SQL 语句。

您可以使用 OAT 执行以下管理任务：

  • 管理数据库服务器，包括配置、用户特权和虚拟处理器。
  • 创建和监视调度程序的任务。
  • 管理 Auto Update Statistics (AUS) 维护系统，该系统会自动更新统计信息以优化查询性能。
  • 使用 OpenAdmin Tool (OAT) 的 IBM Informix Schema Manager 插件来管理数据库和表对象。
  • 管理存储空间并管理数据库服务器的恢复日志。
  • 通过对表进行压缩并取消分段来优化空间。
  • 备份数据库服务器的存储空间。
  • 对表运行 SQL 语句并执行标准 SQL 操作。
  • 监视并管理高可用性集群。
  • 使用 OpenAdmin Tool (OAT) 的 IBM Informix Replication 插件来监视和管理 Enterprise Replication。
  • 使用 OpenAdmin Tool (OAT) 的 IBM Informix TimeSeries 插件来监视和管理与时间系列相关的数据库对象，包括表、索引、虚拟表、TimeSeries 子类型、容器和日历。

当您安装 IBM Informix Client Software Development Kit (Client SDK) 或安装 IBM Informix 捆绑软件并选择 Client SDK 或 IBM Informix Connect 时，可以安装 IBM OpenAdmin Tool (OAT) for Informix。此安装程序包含 OAT、Health Advisor 插件、Replication 插件、Schema Manager 插件、TimeSeries 插件，以及运行 OAT 必需的所有软件（包括预配置的 Apache、PHP 和 PDO_informix）。

或者，也可以从以下 IBM Informix Free Product Download 页面下载 OpenAdmin Tool Community Edition：https://www14.software.ibm.com/webapp/iwm/web/preLogin.do?lang=en_US&source=swg-informixfpd（在新窗口中打开）。OpenAdmin Tool Community Edition 包含 OAT 和可选插件，但它不包含支持软件。 此修订版必须手动安装。

如何连接到数据库服务器？

登录到数据库服务器

使用 IBM OpenAdmin Tool (OAT) for Informix 登录到数据库服务器。

要登录到数据库服务器：

  1. 转至 OAT 根 URL：http://服务器名称/openadmin。
  2. 在 OAT 的登录页面上，填写服务器详细信息字段。
  3. 单击登录。

您已通过 OAT 连接到数据库服务器。

提示：如果数据库服务器是 OAT 组的成员，那么您也可以通过登录到该 OAT 组来连接到该数据库服务器。OAT 组包含数据库服务器的连接信息。如果您已登录到某个 OAT 组，那么无需再次登录就可以切换到该组中的其他数据库服务器。

登录到 OAT 组中的数据库服务器

通过登录到 IBM OpenAdmin Tool (OAT) for Informix 中的数据库服务器连接组来连接到数据库服务器。

OAT 组包含数据库服务器的连接信息。如果您已登录到某个 OAT 组，那么无需再次登录就可以切换到该组中的其他数据库服务器。

要登录到 OAT 组中的数据库服务器：

  1. 转至 OAT 根 URL：http://服务器名称/openadmin。
  2. 在 OAT 登录页面上的 OAT 组列表中，选择包含您要连接的服务器的 OAT 组。
  3. 输入该 OAT 组的密码。
  4. 单击获取服务器。
  5. 从服务器列表选择服务器。在服务器详细信息字段植入服务器信息。
  6. 在服务器详细信息区域的密码字段中，接受提供的密码（如果有），或输入有效密码，然后单击登录。

  • 相关任务：
  • 添加 OAT 组
    在 IBM OpenAdmin Tool (OAT) for Informix 中定义数据库服务器连接组。

切换至 OAT 组中的其他服务器

连接到 IBM OpenAdmin Tool (OAT) for Informix 内数据库服务器连接组中的其他数据库服务器。

OAT 组包含数据库服务器的连接信息。如果您已登录到某个 OAT 组，那么无需再次登录就可以切换到该组中的其他数据库服务器。

要切换至 OAT 组中的其他数据库服务器，请在 OAT 页面顶部的服务器列表中选择数据库服务器。

  • 相关任务：
  • 添加 OAT 组
    在 IBM OpenAdmin Tool (OAT) for Informix 中定义数据库服务器连接组。

如何监视数据库服务器活动？

监视数据库服务器状态

实时监视特定数据库服务器的状态，或同时查看 OAT 组中所有服务器的状态。

您可以使用仪表板监视特定数据库服务器的状态。此仪表板显示内存和空间使用率、锁定数、事务数、查询数，以及读取数和写入数最多的表。

您可以在多服务器仪表板上查看 OAT 组中所有数据库服务器的状态。您可以从此仪表板转至 IBM OpenAdmin Tool (OAT) for Informix 中的其他页面，以获取更多信息。

要监视数据库服务器状态：

  1. 在 OAT 菜单上，单击运行状况中心 > 仪表板。
  2. 要刷新仪表板，请单击刷新图标或更改刷新率。

定制仪表板

更改仪表板为特定数据库服务器显示的状态信息。

先决条件：如果 IBM OpenAdmin Tool (OAT) for Informix 中的管理菜单受密码保护，那么您需要提供密码才能执行此任务。

OAT 仪表板显示有关数据库服务器的信息；例如，内存使用率和事务数。您可以通过定制仪表板来指定要显示的有关数据库服务器的信息。

要定制仪表板：

  1. 在 OAT 中的管理菜单上，单击仪表板管理器。
  2. 要打开仪表板的布局编辑器，请双击该仪表板。

修改仪表板的状态阈值

修改多服务器仪表板上显示的状态指示器的阈值。

先决条件：必须安装 OpenAdmin Tool (OAT) 的 IBM Informix Health Advisor 插件。

IBM OpenAdmin Tool (OAT) for Informix 中的多服务器仪表板显示 OAT 组中数据库服务器的状态指示器。 例如，如果数据库服务器上数据库空间的可用空间降至指定的阈值以下，将对该数据库服务器显示警告指示器。

您可以修改内存、空间和块占总计 I/O 的百分比的状态阈值。这些阈值是 Health Advisor 插件中的缺省设置。

要修改仪表板的状态阈值：

  1. 在 OAT 菜单上，单击运行状况中心 > Health Advisor。
  2. 在概要文件页面上，验证当前概要文件是否为缺省概要文件。
  3. 单击警报选项卡。
  4. 修改其中任何警报的阈值：
    Health Advisor 插件警报阈值 仪表板状态阈值
    操作系统的可用内存 内存
    数据库空间的可用空间 空间
    块 I/O 操作数 I/O：块占总计 I/O 的百分比

要删除联机消息日志：

  1. 在 OAT 菜单上，单击日志 > 联机日志。
  2. 执行以下某项操作：
    • 要删除日志文件，请单击删除。
    • 要删除日志文件中的消息，但保留日志文件，请单击截断。

循环交替联机消息日志

启用、禁用或修改数据库服务器的联机消息日志文件的自动循环交替。

先决条件：对于此操作，IBM OpenAdmin Tool (OAT) for Informix 需要 IBM Informix V11.70.xC3。

您可以通过配置自动日志循环交替来控制联机消息日志的大小。联机消息日志循环交替时，数据库服务器会切换至新的联机消息日志文件，并将先前日志文件的标识号加一。达到日志文件数最大值时，会删除具有最高标识的日志文件。

要循环交替联机消息日志：

  1. 在 OAT 菜单上，单击日志 > 联机日志。
  2. 单击联机消息日志循环交替选项卡。您可以启用或禁用任务、修改调度或编辑确定日志文件最大数目的参数。

如何管理系统空间？

监视空间使用情况

复审有关空间使用情况以及数据库服务器的存储空间的信息。对于特定存储空间，复审有关空间使用情况和优化以及空间中块、表和索引的信息。

要监视空间使用情况：

  1. 在 IBM OpenAdmin Tool (OAT) for Informix 的菜单上，单击空间管理 > 存储。此页面显示有关数据库服务器空间使用情况的信息的面板。
  2. 要显示更多信息，请单击该面板中的展开按钮展开面板。
  3. 要显示有关特定存储空间的信息，请在列表中选择该空间。页面将显示有关该空间使用情况的信息的面板。
  4. 要显示有关选定空间的更多信息，请展开面板。

管理空间使用情况

管理数据库服务器的存储空间。

创建空间

为数据库服务器创建数据库空间、临时数据库空间、BLOB 空间或智能大对象空间。

使用 IBM OpenAdmin Tool (OAT) for Informix 创建空间时，您也可以将空间设置为可扩充。数据库服务器会在必要时通过添加或扩展块来自动扩充存储空间的大小。自动存储供应需要 IBM Informix V11.70。

要创建空间：

  1. 在 OAT 菜单上，单击空间管理 > 存储。
  2. 单击操作 > 创建空间。
  3. 填写创建空间页面上的字段。

  • 相关概念：
  • 自动空间管理（管理员指南）
  • 相关参考：

修改空间

指定数据库服务器是否可在必要时通过添加或扩展块来扩充存储空间的大小。设置服务器可用于扩充空间的最小大小。

先决条件：此操作需要 IBM Informix V11.70。

要修改空间：

  1. 在 IBM OpenAdmin Tool (OAT) for Informix 的菜单上，单击空间管理 > 存储。
  2. 展开空间面板。
  3. 在空间页面上，选择空间。
  4. 单击修改。
  5. 填写页面上的字段。

  • 相关概念：
  • 自动空间管理（管理员指南）
  • 相关参考：

删除空间

删除数据库空间、临时数据库空间、BLOB 空间或智能大对象空间。

配置高可用性集群的连接

在 IBM Informix 服务器上设置 idsd 守护程序以支持启动高可用性集群中的服务器或添加共享磁盘辅助 (SDS) 服务器。

先决条件：您需要具有 Informix 服务器上的 root 特权，才能执行此过程。

要使用 IBM OpenAdmin Tool (OAT) for Informix 执行远程管理任务（包括启动高可用性集群中的服务器或添加辅助服务器），请在 Informix 服务器上安装并配置以下守护程序：

  • 因特网服务守护程序，例如，适用于 UNIX 和 Linux 的 xinetd 或 inetd 因特网守护程序。
  • idsd，允许 OAT 启动远程 Informix 服务器的守护进程。 该守护程序位于 Informix 服务器所在的相同主机上，并且必须位于用户 informix 可访问的目录中。idsd 守护程序是通过 xinetd 或 inetd 守护程序调用的。
要点：您不能在 Windows 或 MAC OS 上使用远程管理功能。

要配置因特网守护程序和 idsd 守护程序：

  1. 确保 idsd 守护程序已与 Informix 一起安装在 $INFORMIXDIR/bin 目录中。确保所有者和组是 informix，且许可权是 0754：

    例如：

    [informix@myhost]/usr/informix/bin% ls -la idsd 
    -rwxr-xr--  1 informix informix 23717 Oct 15 16:34 idsd
  2. 在服务器上，设置连接的加密。对于 Informix 11.70，使用作为 IBM JRE 1.6 或更高版本一部分提供的 GSKCmd 工具。对于较早的 Informix 版本，使用 IBM Global Security Kit 中包含的 GSKCmd 工具。
    1. 对您的平台使用以下 GSKCmd 命令行参数来创建密钥数据库：
      command -keydb -create -db keydb_name.kdb -pw password -type cms -stash
      例如：
      gsk8cmd -keydb -create -db mydb.kdb -pw mypassw0rd -type cms -stash
    2. 使用以下 GSKCmd 命令行参数来创建自签名证书：
      command -cert -create -db keydb_name.kdb -pw password -label label_text
    -dn distinguished name 
    -size <1024 | 512 | 2048> -default_cert yes
      其中：专有名称是 X.500 专有名称。专有名称是加引号的字符串，格式如下：CN=公共名称（必需）, O=组织, OU=组织单元, L=位置, ST=省/自治区/直辖市, C=国家或地区。

      例如： 

      gsk8cmd -cert -create -db mydb.kdb -pw mypassw0rd -label mycertificatelabel
    -dn "CN=lenexa.ibm.com,O=ibm,OU=IBM HTTP Server,ST=KS,C=US" 
    -size 1024 -default_cert yes
  3. 打开 xinetd 或 inetd 配置文件进行编辑，并为 idsd 守护程序添加服务条目。
    要点：如果 GSKIT_VERSION 配置参数设置为 GSKit 的非缺省版本，请将 -g NUMBER 服务器程序参数添加到 xinetd 或 inetd 服务条目以指定 GSKit 版本。
    • 对于 xinetd 服务器守护程序：/etc/xinetd.conf。
      service idsd
             {
             disable = no
             socket_type = stream 
             protocol = tcp
             user = root
             wait = no 	    
             server = /usr/informix/bin/idsd
             server_args = -l /usr/informix/idsd.log 
                 -k /usr/informix/idsd -n idsd_label
             }
    • 对于 UNIX 上的 inetd 服务器守护程序：/etc/inetd.conf：
      idsd stream tcp nowait root /usr/informix/bin/idsd idsd -l /usr/informix/idsd.log  -k /usr/informix/idsd -n idsd_label
  4. 打开文件 /etc/services 进行编辑，并添加名为 idsd 的服务。例如：
    idsd 10000/tcp
    在本例中，使用端口号 10000，但您可以使用任何其他还未使用的大于 1024 的端口。但是，名称必须为 idsd，协议必须为 tcp。
  5. 重新启动 xinetd 或 inetd 服务守护程序。在大多数实施中，您可以向正在运行的 inetd 或 xinetd 进程发送 SIGHUP 信号来重新启动。但是，在某些 Linux 平台上，您可以使用 root 身份运行以下命令： 
    service xinetd restart

管理 Auto Update Statistics

复审和调整表统计信息的 Auto Update Statistics (AUS) 维护系统。

AUS 维护系统识别需要新优化器统计信息的表和索引，并运行相应的 UPDATE STATISTICS 语句以优化查询性能。根据您的系统，您可能需要调整 AUS 到期策略或调度。

您可以在服务器管理 > Auto Update Statistics > 配置页面上控制和监视 AUS 策略。

您可以使用 SQL ToolBox > Schema Manager > 统计信息页面上 OpenAdmin Tool (OAT) 的 IBM Informix Schema Manager 插件来复审表的统计信息。

配置并启用 Auto Update Statistics

  1. 在 IBM OpenAdmin Tool (OAT) for Informix 的菜单上，单击服务器管理 > Auto Update Statistics > 配置。
  2. 根据您的系统性能和需求，确定缺省值对于您的系统是否有效。
    1. 要指定两次重建统计信息之间的最长时间，请为 AUS_AGE 参数输入天数，例如 30、60 或 90。
    2. 要指定表的更改达到多少百分比后会启动统计信息重建，请为 AUS_CHANGE 参数输入百分比数字（例如 20 或 50）。
    3. 要指定 AUS 是否使用建议的指导规则，请从列表为 AUS_AUTO_RULES 参数选择启用或禁用。缺省值为启用。
    4. 要指定始终重建统计信息的行数上限，请为 AUS_SMALL_TABLES 参数输入数字。缺省值为 100。
    5. 要指定 PDQ 优先级，请为 AUS_PDQ 参数输入从 -1 到 100 的数字。缺省值为 10。并行数据库查询 (PDQ) 优先级是确定数据库服务器如何将资源用于并行处理的因素之一。
    6. 要指定用于运行更新统计信息任务的线程数，请输入线程数。OAT 会将此数量的 AUS 任务提交到任务调度程序。此选项需要 IBM Informix V11.70。
  3. 单击保存以保存更改。
  4. 要确定是否已启用 AUS 任务，请单击信息选项卡，然后单击以下任务：
    • Auto Update Statistics 评估
    • Auto Update Statistics 刷新
      注：如果“Auto Update Statistics 刷新”任务没有可执行的工作，该任务将自行禁用。 下次“Auto Update Statistics 评估”任务运行并为刷新任务创建工作时，评估任务将自动启用刷新任务。
    此时将显示任务详细信息页面。
  5. 向下滚动至启用任务字段，并确保该字段已选中。
  6. 使用设置运行窗口的字段来指定运行任务的频率。由于 AUS 是资源密集型操作，因此可以定义特定运行期间。运行期间定义包含开始时间和结束时间，以及在每周的哪几天运行。缺省情况下，运行时间窗口为每天凌晨 1:00 到 5:00。
  7. 单击保存以保存任何更改。

如何管理调度程序任务？

添加调度程序任务

您可以创建“调度程序”任务，以在指定时间并以指定频率运行任务或传感器。

  • 任务提供在指定时间或以指定时间间隔运行特定作业的方式。
  • 传感器可收集并保存信息。

添加新任务

要点：创建任务的时间将影响任务的首次执行。
  • 如果当前时间介于指定的启动时间和停止时间之间，那么任务将在当前时间执行。后续执行将以频率值指定的时间间隔进行。因此，例如，当前时间若介于启动时间和停止时间之间，且频率设置为 15 分钟，那么任务的首次执行将在任务调度后立即开始，且下次执行将发生在 15 分钟后。
  • 但是，如果当前时间不在启动时间和停止时间之间，那么任务将等待下次开始时间再执行，可能是当天，也可能是第二天。例如，如果当前时间为凌晨 1:00，而下次开始时间为晚上 7:00 (19:00)，那么任务将在当天的 19:00 开始。

要添加新任务：

  1. 从“任务调度”页面，单击添加新任务。
  2. 单击要添加哪类任务？下方的任务，然后单击下一步。
  3. 在任务名称字段中输入名称。例如：MyTask。
  4. 从任务组列表选择类别。
  5. 在文本字段中添加新任务的描述，然后单击下一步。
  6. 指定可允许任务运行的时间范围，例如在每天上午 10 点 (10:00) 到下午 6 点 (18:00) 之间的每个小时。
    1. 指定时间间隔的开始时间，使用 24 小时制格式：hh:mm:ss 例如：10:00:00
    2. 指定时间间隔的停止时间，使用 24 小时制格式：hh:mm:ss 例如：18:00:00。 还可以指定 NEVER。
    3. 指定在允许的时间间隔内任务可运行的频率（以天、小时和分钟表示）：d 天 h 小时 m 分钟 例如：0 天 1 小时 00 分钟。
  7. 单击下一步。
  8. 指定任务要运行的命令语句。例如，如果您要删除杂项任务，可以输入以下语句： 
    delete from ph_task where tk_group='MISC'
  9. 单击下一步。此时将显示确认页面。
  10. 验证每个选择的条目：
    • 如果条目正确，请单击完成以确认并创建任务。
    • 单击返回以返回到先前的页面，来更正错误的值。

添加新传感器

  1. 从“任务调度”页面，单击添加新任务。
  2. 单击要添加哪类任务？下方的传感器，然后单击下一步。
  3. 在传感器名称字段中输入名称。例如：MySensor。
  4. 从传感器组列表选择类别。
  5. 在文本字段中添加新任务的描述，然后单击下一步。
  6. 指定可允许传感器运行的时间范围，例如每天上午 10:00 到下午 6:00 之间的每个小时。
    1. 指定时间间隔的开始时间，使用 24 小时时钟格式：hh:mm:ss。例如：10:00:00
    2. 指定时间间隔的停止时间，使用 24 小时时钟格式：hh:mm:ss。例如：18:00:00
    3. 指定在允许的时间间隔内任务运行的频率（以天、小时和分钟表示）：d 天 h 小时 m 分钟。例如：0 天 1 小时 00 分钟。
    4. 指定数据删除时间（以天、小时和分钟表示）：d 天 h 小时 m 分钟。例如：1 天 0 小时 0 分钟。
  7. 单击下一步。
  8. 在结果表名字段中，指定用于保留传感器数据的结果表名。例如：MyResults
  9. 单击下一步。
  10. 通过在 SQL 语句字段中输入列名和定义，完成结果表的 SQL 语句。例如，如果您要收集表 TaskA_ReturnCode 中 taskA（其 task_id 为 10）的返回码，可以输入以下 SQL 语句： 
    create table TaskA_ReturnCode(task_name varchar(20), errorcode integer)
  11. 指定传感器要运行的命令语句。例如：
    insert into TaskA_ReturnCode values('TaskA', (select run_retcode from ph_run where run_task_id=10))
  12. 验证每个选择的条目。
    • 如果条目正确，请单击完成以确认并创建传感器。
    • 单击返回以返回到先前的页面，来更新任何错误的值。

优化 SQL 语句

使用 IBM OpenAdmin Tool (OAT) for Informix 创建外部伪指令以优化 SQL 语句。

先决条件：
  • 在数据库服务器 onconfig 文件中，将 EXT_DIRECTIVES 配置参数设置为 1 或 2，这样就可以保存外部伪指令。更改 EXT_DIRECTIVES 配置参数后，请重新启动数据库服务器以使更改生效。
  • 确保已启用 SQL 跟踪，并已在跟踪管理页面上选中表名选项。请参阅启用 SQL 跟踪和指定 SQL 跟踪的选项。

您可以创建外部伪指令以应用于 SQL 语句。伪指令是指示查询优化器如何执行查询的注释。外部伪指令提供了一种用于更改 SQL 语句的查询执行计划的方法。向 SQL 语句应用外部伪指令后，SQL 语句的查询执行计划将在您每次重新运行 SQL 语句时进行相应更改。

要点：当您重新运行 SQL 语句时，该语句必须与您将伪指令应用于的 SQL 语句完全匹配。例如，它必须具有相同的字母大写格式和相同的间隔。

要创建外部伪指令：

  1. 在 OAT 菜单上，单击性能分析 > SQL 资源管理器。
  2. 向下钻取至 SQL 语句。
    1. 单击 SQL 选项卡。
    2. 单击列表中的 SQL 语句。
    3. 单击 SQL 语句执行列表中的任意行。
  3. 单击优化选项卡。
  4. 单击添加。
  5. 在添加伪指令页面上，选择伪指令。需要时可选择表和索引。要为 SQL 语句添加更多伪指令，请单击添加其他，或单击添加以关闭页面。
  6. 指定如何应用伪指令，然后单击应用。缺省值为非活动：不对 SQL 语句应用伪指令。

在 IBM OpenAdmin Tool (OAT) for Informix 中定义数据库服务器连接组。

OAT 组提供了管理数据库服务器组以及登录到组（而不是每个单独的服务器）的方式。

    要点：如果要使用 V3.0 之前的 Informix Connect 或 IBM Informix Client Software Development Kit (Client SDK) 版本，那么对于添加的每个新连接，请确保在 Web 服务器计算机上有该连接的对应 SQLHOSTS 条目。

为管理配置页面提供密码保护：Apache 示例

出于安全性原因，限制对 IBM OpenAdmin Tool (OAT) for Informix 的“管理”页面的访问权。使用密码保护这些页面是很重要的，因为管理页面可从登录页面进行访问，因此对所有人开放。

一种方法是使用 Web 服务器提供的内置认证。例如，您可以保护安装目录 OATINSTALL/admin/.htaccess。

示例：要从 Apache 中保护 OAT 目录：

  1. 使用 Apache 随附的 htpasswd 实用程序，在无法从 Web 访问的目录中创建密码文件。
  2. 将认证伪指令添加至以下某个文件：
    • OATINSTALL/admin/.htaccess 文件
    • httpd.conf 文件
  3. 将伪指令放在 <Directory> 段内。例如：
    <Directory OATINSTALL/admin>

AuthName "OAT Admin"

</Directory>

有关更多信息和示例，请参阅以下 Apache Software Foundation Web 站点上有关 Apache HTTP Server V2.0 认证、授权和存取控制的文章：www.apache.org（在新窗口中打开）。

配置 OAT 的 HTTPS (Linux)

启用 HTTPS 以保护 IBM OpenAdmin Tool (OAT) for Informix 的 Web 服务器免遭窃听、篡改、消息伪造和网络干扰。

先决条件：必须在安装了 OAT 的 Apache 服务器的计算机上安装 OpenSSL。您可以从以下 OpenSSL Project Web 站点下载并安装 OpenSSL：www.openssl.org。

启用 HTTPS 时，来自 OAT 客户机的消息将进行加密，然后再发送至 OAT Web 服务器。加密可防止未经授权的用户侦听该线路并窃取敏感信息。启用 HTTPS 时，OAT 客户机也可以向 OAT 主机认证，这样未经授权的用户就无法使用错误的 Web 服务器欺骗 OAT 客户机。

要点：HTTPS 只加密 OAT Web 服务器与客户机之间的通信。它不会加密 IBM Informix 数据库服务器与 Web 服务器之间的通信。IBM Informix V9.4 和更高版本允许使用加密通信支持模块来加密 Informix 服务器与 OAT Web 服务器之间的数据。有关加密和 Informix 服务器的更多信息，请阅读 developerWorks® Web 站点上的“Encryption over the wire with IDS 9.40”。

当您将 OAT 与 IBM Informix Client Software Development Kit (Client SDK) 一起安装时，安装程序会提供运行 OAT 所需的软件，包括预配置的 Apache、PHP 和 PDO_informix。在 Linux 上，与 OAT 一起安装的 Apache Web 服务器将启用 mod_ssl 密码术模块。需要此模块才能启用 HTTPS。

要在 Linux 上的 OAT 中启用 HTTPS，请完成以下步骤：

  1. 为 Web 服务器创建加密密钥和证书，以便 OAT 客户机可以根据您的证书认证 Web 服务器。
  2. 配置 Apache 以启用 HTTPS。

有关启用 HTTPS 的更多信息，请阅读 developerWorks Web 站点上的“Securing the OpenAdmin Tool for Informix web server with HTTPS”。

创建加密密钥

密钥用于加密和解密。它们通常成对（公用密钥和专用密钥）出现。公用密钥用于加密消息，专用密钥用于解密消息。证书是认证 Web 服务器的文档。启用 HTTPS 的 Web 服务器具有由可信认证中心签署的证书，用于验证 Web 服务器。Web 服务器启用 HTTPS 后，客户机选择与 Web 服务器之间是建立普通连接还是安全连接。要建立普通连接，客户机在其 Web 浏览器中输入 http://web server_url。要建立安全连接，客户机输入 https://web server_url。

要生成专用和公用密钥对以及证书，请使用 OpenSSL 安装的 bin 目录中的 openssl 可执行文件。

当您运行命令以生成专用密钥时，会提示您输入口令来保护专用密钥。每次运行需要使用专用密钥的命令时，您都必须重新输入该口令。例如，启动 Apache Web 服务器或使用专用密钥生成证书时，必须输入口令。在 Linux 上，出于安全性原因，请使用口令保护和加密专用密钥。

要生成专用密钥，请在安装 OpenSSL 的计算机上运行以下命令：

openssl genrsa -des3 -out privkey.pem 2048

专用密钥在 privkey.pem 文件中创建。 此文件包含 Web 服务器的解密密钥，请将其存储在安全位置。生成证书时，此文件用于生成关联的公用密钥（包含在证书中）。

创建证书

创建专用密钥后，请创建证书以认证 Web 服务器的身份。要生成证书，请创建证书签署请求，并将该证书签署请求发送给可信认证中心。认证中心会发放证书。

要生成证书签署请求，请运行以下命令：

openssl req -new -key privkey.pem -out cert.csr

有关证书签署请求的更多信息，请联系您的认证中心。

或者，您也可以不通过认证中心而创建自签名证书（不过，推荐不要使用自签名证书）。

要生成自签名证书，请运行以下命令：

openssl req -new -x509 -key privkey.pem -out cacert.pem -days 1095

OpenSSL 会提示您输入信息。证书存储在 cacert.pem 文件中。 此文件将向 Web 客户机显示，以验证您的身份。它还包含 Web 客户机的公用密钥，以便可以加密通信。

有关加密密钥的更多信息，请参阅 OpenSSL 文档。

配置 Apache 以启用 HTTPS

要启用 HTTPS，请更新 Apache 配置文件和 Apache SSL 配置文件：

  1. 找到 Apache 配置文件。路径包含 Apache 版本，例如： 
    /oat/installation/directory/Apache_2.4.2/conf/httpd.conf
  2. 在 Apache 配置文件中，除去以下行的注释：#Include conf/extra/httpd-ssl.conf。

    缺省情况下，此行已注释掉。除去注释后，Apache SSL 配置文件将包含在 httpd.conf 文件中。

  3. 找到 Apache SSL 配置文件。路径包含 Apache 版本，例如： 
    /oat/installation/directory/Apache_2.4.2/conf/extra/httpd-ssl.conf
  4. 在 Apache SSL 配置文件中，验证 SSL 端口号是否设置为可用的端口。

    HTTPS 需要独立的 SSL 端口。缺省情况下，SSL 端口号设置为 443。如果此端口不可用，请在 Listen 伪指令和“虚拟主机”一节中更改 SSL 端口。

  5. 在 Apache SSL 配置文件中，验证 SSLCertificateKeyFile 伪指令是否指示您所创建的专用密钥文件 privkey.pem 的正确位置
  6. 验证 SSLCertificateFile 伪指令是否指示您所创建的证书文件 cacert.pem 的正确位置。
  7. 可选：要控制 Web 服务器接受的密码，请编辑 SSLCipherSuite 伪指令。有关 SSLCipherSuite 伪指令的信息，请参阅 Apache mod_ssl 文档。

有关 HTTPS 配置的更多信息，请参阅 Apache Web 站点：www.apache.org。

测试配置

创建加密密钥

密钥用于加密和解密。它们通常成对（公用密钥和专用密钥）出现。公用密钥用于加密消息，专用密钥用于解密消息。证书是认证 Web 服务器的文档。启用 HTTPS 的 Web 服务器具有由可信认证中心签署的证书，用于验证 Web 服务器。Web 服务器启用 HTTPS 后，客户机选择与 Web 服务器之间是建立普通连接还是安全连接。要建立普通连接，客户机在其 Web 浏览器中输入 http://web server_url。要建立安全连接，客户机输入 https://web server_url。

要生成专用和公用密钥对以及证书，请使用新 Apache Web 服务器安装的 bin 目录中的 openssl 可执行文件。

当您运行命令以生成专用密钥时，会提示您输入口令来保护专用密钥。在 Windows 上，Apache 不支持对专用密钥使用口令，因此您必须除去口令，如下几步中所述。

要创建专用密钥：

  1. 要生成专用密钥，请在安装新 Apache Web 服务器的计算机上运行以下命令：
    openssl genrsa -des3 -out privkey.pem 2048
  2. 要除去口令，请运行以下命令： 
    openssl rsa -in privkey.pem -out privkey_nopassphrase.pem

专用密钥的未加密版本在 privkey_nopassphrase.pem 文件中创建。 此文件包含 Web 服务器的解密密钥，请将其存储在安全位置。生成证书时，此文件用于生成关联的公用密钥（包含在证书中）。

创建证书

创建专用密钥后，请创建证书以认证 Web 服务器的身份。要创建证书，请生成证书签署请求，并将该证书签署请求发送给可信认证中心。认证中心会发放证书。

要生成证书签署请求，请运行以下命令：

openssl req -new -key privkey_nopassphrase.pem -out cert.csr

有关证书签署请求的更多信息，请联系您的认证中心。

或者，您也可以不通过认证中心而创建自签名证书（不过，推荐不要使用自签名证书）。

要生成自签名证书，请运行以下命令：

openssl req -new -x509 -key privkey_nopassphrase.pem -out cacert.pem -days 1095

OpenSSL 会提示您输入信息。证书存储在 cacert.pem 文件中。 此文件将向 Web 客户机显示，以验证您的身份。它还包含 Web 客户机的公用密钥，以便可以加密通信。

有关加密密钥的更多信息，请参阅 OpenSSL 文档。

配置 Apache 以启用 HTTPS

要启用 HTTPS，请更新 Apache 配置文件和 Apache SSL 配置文件：

  1. 找到 Apache 配置文件。路径包含 Apache 版本，例如： 
    /oat/installation/directory/Apache_2.2.22/conf/httpd.conf
  2. 在 Apache 配置文件中，除去以下行的注释：#Include conf/extra/httpd-ssl.conf。

    缺省情况下，此行已注释掉。除去注释后，Apache SSL 配置文件将包含在 httpd.conf 文件中。

  3. 找到 Apache SSL 配置文件。路径包含 Apache 版本，例如： 
    /oat/installation/directory/Apache_2.2.22/conf/extra/httpd-ssl.conf
  4. 在 Apache SSL 配置文件中，验证 SSL 端口号是否设置为可用的端口。

    HTTPS 需要独立的 SSL 端口。缺省情况下，SSL 端口号设置为 443。如果此端口不可用，请在 Listen 伪指令和“虚拟主机”一节中更改 SSL 端口。

  5. 在 Apache SSL 配置文件中，验证 SSLCertificateKeyFile 伪指令是否指示您所创建的专用密钥文件 privkey_nopassphrase.pem 的正确位置
  6. 验证 SSLCertificateFile 伪指令是否指示您所创建的证书文件 cacert.pem 的正确位置。
  7. 可选：要控制 Web 服务器接受的密码，请编辑 SSLCipherSuite 伪指令。有关 SSLCipherSuite 伪指令的信息，请参阅 Apache mod_ssl 文档。

有关 HTTPS 配置的更多信息，请参阅 Apache Web 站点：www.apache.org。
