IBM OpenAdmin Tool (OAT) for Informix

IBM® OpenAdmin Tool (OAT) for Informix® ist eine Webanwendung für die Verwaltung und Analyse der Leistung von IBM Informix-Datenbankservern. Sie können mehrere Datenbankserverinstanzen über eine einzige OAT-Installation auf einem Web-Server verwalten. Der Zugriff auf den Web-Server kann über einen beliebigen Browser erfolgen, über den Sie alle Ihre Datenbankserver verwalten können.

Sie können OpenAdmin Tool verwenden, um die Leistung eines Datenbankservers zu überwachen und zu analysieren:

  • Überwachen Sie den Status eines Datenbankservers mit ausführlichen Statusberichten, Alertinformationen und Nachrichtenprotokollen.
  • Analysieren Sie den Status eines Datenbankservers und senden Sie die Ergebnisse und Empfehlungen mithilfe des IBM Informix-Plug-ins 'Health Advisor' für OpenAdmin Tool (OAT) als E-Mail.
  • Überwachen Sie die Leistungsstatistik über einen bestimmten Zeitraum mit detaillierten Diagrammen des Leistungsdatenprotokolls.
  • Überwachen Sie vom Scheduler zusammengestellte Daten. Dieser führt Verwaltungstasks aus und erfasst Informationen mithilfe von SQL-Anweisungen.
  • Führen Sie Systemüberprüfungen für Datenbanken und Tabellen auf einem Datenbankserver aus.
  • Führen Sie Systemberichte und onstat-Befehle aus.
  • Führen Sie einen Drilldown für SQL-Anweisungen durch, um die Leistungsdaten zu überprüfen, und optimieren Sie SQL-Anweisungen.

Sie können OAT für die folgenden Verwaltungstasks verwenden:

  • Verwalten Sie den Datenbankserver einschließlich Konfiguration, Benutzerzugriffsrechten und virtueller Prozessoren.
  • Erstellen und überwachen Sie Tasks für den Scheduler.
  • Verwalten Sie das AUS-Verwaltungssystem (Auto Update Statistics), das automatisch Statistikdaten aktualisiert, um die Abfrageleistung zu optimieren.
  • Verwalten Sie Datenbank- und Tabellenobjekte über das IBM Informix-Plug-in 'Schema Manager' für OpenAdmin Tool (OAT).
  • Verwalten Sie Speicherbereiche und verwalten Sie Recovery-Protokolle für den Datenbankserver.
  • Optimieren Sie Speicherbereiche durch Komprimierung und Defragmentierung von Tabellen.
  • Führen Sie ein Backup der Speicherbereiche eines Datenbankservers durch.
  • Führen Sie SQL-Anweisungen aus und führen Sie Standard-SQL-Operationen für Tabellen durch.
  • Überwachen und verwalten Sie Hochverfügbarkeitscluster.
  • Überwachen und verwalten Sie Enterprise Replication über das IBM Informix-Plug-in 'Replication' für OpenAdmin Tool (OAT).
  • Überwachen und verwalten Sie Datenbankobjekte, die zu einer Zeitreihe gehören, einschließlich Tabellen, Indizes und virtuelle Tabellen sowie TimeSeries-Subtypen, -Container und -Kalender über das IBM Informix-Plug-in 'TimeSeries' für OpenAdmin Tool (OAT).

Sie können IBM OpenAdmin Tool (OAT) for Informix installieren, wenn Sie IBM Informix Client Software Development Kit (Client SDK) installieren oder wenn Sie das IBM Informix-Softwarepaket installieren und Client SDK oder IBM Informix Connect auswählen. Dieses Installationsprogramm umfasst OAT, das Plug-in 'Health Advisor', das Plug-in 'Replication', das Plug-in 'Schema Manager' und das Plug-in 'TimeSeries' sowie die gesamte zum Ausführen von OAT erforderliche Software, einschließlich vorkonfiguriertem Apache, PHP und PDO_informix.

Alternativ ist OpenAdmin Tool Community Edition über die IBM Informix-Seite für kostenlose Produktdownloads https://www14.software.ibm.com/webapp/iwm/web/preLogin.do?lang=en_US&source=swg-informixfpd (wird in neuem Fenster geöffnet) verfügbar. OpenAdmin Tool Community Edition umfasst OAT und die optionalen Plug-ins, nicht jedoch die unterstützende Software. Diese Edition muss manuell installiert werden.

Informationen zur Vorgehensweise beim Herstellen einer Verbindung zu einem Datenbankserver

Anmelden an einem Datenbankserver

Melden Sie sich mithilfe von IBM OpenAdmin Tool (OAT) for Informix an einem Datenbankserver an.

Gehen Sie wie folgt vor, um sich an einem Datenbankserver anzumelden:

  1. Wechseln Sie zur OAT-Stamm-URL: http://servername/openadmin.
  2. Füllen Sie auf der Seite Anmeldung für OpenAdmin Tool die Felder Serverdetails aus.
  3. Klicken Sie auf Anmeldung.

Sie sind über OpenAdmin Tool mit dem Datenbankserver verbunden.

Tipp: Wenn der Datenbankserver Mitglied einer OAT-Gruppe ist, können Sie alternativ auch eine Verbindung zu dem Datenbankserver herstellen, indem Sie sich an der OAT-Gruppe anmelden. Eine OAT-Gruppe enthält Verbindungsinformationen für Datenbankserver. Wenn Sie sich an einer OAT-Gruppe anmelden, können Sie zu anderen Datenbankservern in der Gruppe wechseln, ohne sich erneut anzumelden.

Anmelden an einem Datenbankserver in einer OAT-Gruppe

Stellen Sie eine Verbindung zu einem Datenbankserver her, indem Sie sich an einer Gruppe von Datenbankserververbindungen in IBM OpenAdmin Tool (OAT) for Informix anmelden.

Eine OAT-Gruppe enthält Verbindungsinformationen für Datenbankserver. Wenn Sie sich an einer OAT-Gruppe anmelden, können Sie zu anderen Datenbankservern in der Gruppe wechseln, ohne sich erneut anzumelden.

Gehen Sie wie folgt vor, um sich an einem Datenbankserver in einer OAT-Gruppe anzumelden:

  1. Wechseln Sie zur OAT-Stamm-URL: http://servername/openadmin.
  2. Wählen Sie in der Liste OAT-Gruppe anmeldender OAT-Seite Anmeldung die OAT-Gruppe aus, die den Server enthält, zu dem Sie eine Verbindung herstellen wollen.
  3. Geben Sie das Kennwort für die OAT-Gruppe ein.
  4. Klicken Sie auf Server abrufen.
  5. Wählen Sie den Server aus der Serverliste aus. Die Felder unter Serverdetails werden mit den Serverinformationen ausgefüllt.
  6. Akzeptieren Sie im Feld Kennwort im Bereich Serverdetails das bereitgestellte Kennwort oder geben Sie ein gültiges Kennwort ein und klicken Sie auf Anmeldung.

  • Zugehörige Tasks:
  • Hinzufügen einer OAT-Gruppe
    Definieren Sie in IBM OpenAdmin Tool (OAT) for Informix eine Gruppe von Serververbindungen.

Wechseln zu einem anderen Server in einer OAT-Gruppe

Stellen Sie eine Verbindung zu einem weiteren Datenbankserver in einer Gruppe von Datenbankserververbindungen in IBM OpenAdmin Tool (OAT) for Informix her.

Eine OAT-Gruppe enthält Verbindungsinformationen für Datenbankserver. Wenn Sie sich an einer OAT-Gruppe anmelden, können Sie zu anderen Datenbankservern in der Gruppe wechseln, ohne sich erneut anzumelden.

Wenn Sie zu einem anderen Server in einer OAT-Gruppe wechseln wollen, wählen Sie den Datenbankserver aus der Liste Server oben auf der OAT-Seite aus.

  • Zugehörige Tasks:
  • Hinzufügen einer OAT-Gruppe
    Definieren Sie in IBM OpenAdmin Tool (OAT) for Informix eine Gruppe von Serververbindungen.

Informationen zur Vorgehensweise beim Überwachen der Datenbankserveraktivität

Überwachen des Datenbankserverstatus

Überwachen Sie den Status eines bestimmten Datenbankservers in Echtzeit oder zeigen Sie den Status aller Server in einer OAT-Gruppe gleichzeitig an.

Sie können den Status eines bestimmten Datenbankservers mithilfe eines Dashboards überwachen. Dieses Dashboard zeigt die Hauptspeicher- und Speicherbelegung, die Anzahl der Sperren, Transaktionen und Abfragen und die Tabellen mit den meisten Lese- und Schreibvorgängen an.

Sie können den Status aller Datenbankserver innerhalb einer OAT-Gruppe auf einem serverübergreifenden Dashboard anzeigen. Über dieses Dashboard gelangen Sie zu anderen Seiten in IBM OpenAdmin Tool (OAT) for Informix, die weitere Informationen enthalten.

Gehen Sie wie folgt vor, um den Datenbankserverstatus zu überwachen:

  1. Klicken Sie im OAT-Menü auf Diagnosezentrale > Dashboard.
  2. Klicken Sie auf das Aktualisierungssymbol oder ändern Sie die Aktualisierungsrate, um das Dashboard zu aktualisieren.

Anpassen des Dashboards

Ändern Sie die auf dem Dashboard angezeigten Statusinformationen für einen bestimmten Datenbankserver.

Voraussetzung: Wenn das Menü Verwaltung in IBM OpenAdmin Tool (OAT) for Informix kennwortgeschützt ist, benötigen Sie das Kennwort, um diese Task ausführen zu können.

Das OAT-Dashboard zeigt Informationen zu einem Datenbankserver an, z. B. die Hauptspeicherbelegung und die Anzahl der Transaktionen. Sie können die Informationen angeben, die zu einem Datenbankserver angezeigt werden sollen, indem Sie das Dashboard anpassen.

Gehen Sie wie folgt vor, um das Dashboard anzupassen:

  1. Klicken Sie im OAT-Menü Verwaltung auf Dashboard-Manager.
  2. Klicken Sie zum Öffnen des Layouteditors für ein Dashboard doppelt auf das entsprechende Dashboard.

Modifizieren der Statusschwellenwerte für das Dashboard

Modifizieren Sie die Schwellenwerte für die auf dem serverübergreifenden Dashboard angezeigten Statusanzeiger.

Voraussetzung: Das IBM Informix-Plug-in 'Health Advisor' für OpenAdmin Tool (OAT) muss installiert sein.

Das serverübergreifende Dashboard in IBM OpenAdmin Tool (OAT) for Informix zeigt Statusanzeiger für die Datenbankserver in einer OAT-Gruppe an. Wenn z. B. der freie Speicherbereich für einen Datenbankbereich auf einem Datenbankserver unter den angegebenen Schwellenwert fällt, wird ein Warnungsanzeiger für den Datenbankserver angezeigt.

Sie können die Statusschwellenwerte für Hauptspeicher, Speicherbereich und den Prozentsatz an Chunks der gesamten Ein-/Ausgabe modifizieren. Diese Schwellenwerte sind die Standardschwellenwerte des Plug-ins 'Health Advisor'.

Gehen Sie wie folgt vor, um die Statusschwellenwerte für das Dashboard zu modifizieren:

  1. Klicken Sie im OAT-Menü auf Diagnosezentrale > Health Advisor.
  2. Überprüfen Sie auf der Seite Profil, ob das aktuelle Profil das Standardprofil ist.
  3. Klicken Sie auf die Registerkarte Alarme.
  4. Sie können die Schwellenwerte für die folgenden Alarme modifizieren:
    Alarmschwellenwert des Plug-ins 'Health Advisor' Dashboard Statusschwellenwert
    Freier Speicher im BS Speicher
    Freier Speicherbereich im Datenbankbereich Speicherbereich
    Chunk-E/A-Operationen E/A: Prozentsatz an Chunks der gesamten Ein-/Ausgabe

Überwachen von Alerts und Ereignisalarmen

Überwachen Sie Ereignisalarme, die vom Datenbankserver generiert wurden, und Alerts, die vom Task-Scheduler oder von integrierten Tasks und Sensoren generiert wurden.

Gehen Sie wie folgt vor, um Alerts und Ereignisalarme zu überwachen:

Klicken Sie im Menü von OpenAdmin Tool (OAT) for Informix auf Diagnosezentrale > Alerts.

  • Zugehörige Referenz:
  • Event Alarm Parameters (Administrator's Reference)
  • Event alarm IDs (Administrator's Reference)
  • The ph_alert Table (Administrator's Reference)

Überprüfen des Verwaltungsbefehlsprotokolls

Überprüfen Sie die Funktionen der SQL-Verwaltungs-API, die auf dem Datenbankserver ausgeführt wurden, einschließlich der Informationen zum Benutzer, der den Befehl ausgeführt hat, zum Zeitpunkt, zu dem der Befehl ausgeführt wurde, zum Befehl und zur Nachricht, die vom Datenbankserver zurückgegeben wurde.

Gehen Sie wie folgt vor, um Verwaltungsbefehle zu überprüfen:

Klicken Sie im Menü von OpenAdmin Tool (OAT) for Informix auf Protokolle > Verwaltungsbefehl.

Überprüfen Sie den aktuellsten Teil des Onlinenachrichtenprotokolls für den Datenbankserver.

Gehen Sie wie folgt vor, um das Onlinenachrichtenprotokoll zu überwachen:

Klicken Sie im Menü von OpenAdmin Tool (OAT) for Informix auf Protokolle > Onlineprotokoll. Fehler werden in Rot und Warnungen in Gelb hervorgehoben.

  • Zugehörige Konzepte:
  • Error Messages (Administrator's Reference)

Löschen des Onlinenachrichtenprotokolls

Löschen Sie die Onlinenachrichtenprotokolldatei für den Datenbankserver oder schneiden Sie die Datei ab.

Voraussetzung: IBM OpenAdmin Tool (OAT) for Informix erfordert für diese Aktion IBM Informix Version 11.70.xC3.

Gehen Sie wie folgt vor, um das Onlinenachrichtenprotokoll zu löschen:

  1. Klicken Sie im OAT-Menü auf Protokolle > Onlineprotokoll.
  2. Führen Sie eine der folgenden Aktionen aus:
    • Klicken Sie auf Löschen, um die Protokolldatei zu löschen.
    • Zum Löschen der Nachrichten in der Protokolldatei, während die Protokolldatei beibehalten wird, klicken Sie auf Protokoll abschneiden.

Rotieren des Onlinenachrichtenprotokolls

Aktivieren, inaktivieren oder modifizieren Sie die automatische Rotation der Onlinenachrichtenprotokolldatei für den Datenbankserver.

Voraussetzung: IBM OpenAdmin Tool (OAT) for Informix erfordert für diese Aktion IBM Informix Version 11.70.xC3.

Sie können die Größe der Onlinenachrichtenprotokolldatei steuern, indem Sie die automatische Protokollrotation konfigurieren. Wenn das Onlinenachrichtenprotokoll rotiert, wechselt der Datenbankserver zu einer neuen Onlinenachrichtenprotokolldatei und erhöht die ID-Nummern für die vorherigen Protokolldateien um eins. Wenn die maximale Anzahl Protokolldateien erreicht wird, wird die Protokolldatei mit der höchsten ID gelöscht.

Gehen Sie wie folgt vor, um das Onlinenachrichtenprotokoll zu rotieren:

  1. Klicken Sie im OAT-Menü auf Protokolle > Onlineprotokoll.
  2. Klicken Sie auf die Registerkarte Rotation des Onlinenachrichtenprotokolls. Sie können die Task aktivieren oder inaktivieren, den Zeitplan modifizieren oder die Parameter bearbeiten, die die maximale Anzahl Protokolldateien festlegen.

Informationen zur Vorgehensweise beim Verwalten von Systemspeicherbereich

Überwachen der Speicherbereichsbelegung

Überprüfen Sie Informationen zur Speicherbereichsbelegung und zu den Speicherbereichen für einen Datenbankserver. Überprüfen Sie für einen bestimmten Speicherbereich Informationen zur Speicherbereichsbelegung und Optimierung und zu den Chunks, Tabellen und Indizes in dem Speicherbereich.

Gehen Sie wie folgt vor, um die Speicherbereichsbelegung zu überwachen:

  1. Klicken Sie im Menü von IBM OpenAdmin Tool (OAT) for Informix auf Bereichsverwaltung > Speicher. Auf der Seite werden Anzeigen mit Informationen zur Speicherbereichsbelegung für den Datenbankserver angezeigt.
  2. Wenn Sie weitere Informationen anzeigen wollen, erweitern Sie die Anzeigen durch Klicken auf die Schaltfläche zum Erweitern in einer Anzeige.
  3. Wenn Sie Informationen zu einem bestimmten Speicherbereich anzeigen wollen, wählen Sie den Speicherbereich in der Liste aus. Auf der Seite werden Anzeigen mit Informationen zur Speicherbereichsbelegung für den Speicherbereich angezeigt.

Verwalten von Hochverfügbarkeitsclustern

Konfigurieren der Konnektivität für Hochverfügbarkeitscluster

Konfigurieren Sie den Dämon idsd auf einem IBM Informix-Server, um das Starten eines Servers in einem Hochverfügbarkeitscluster oder das Hinzufügen eines SDS-Servers (Shared Disk Secondary) zu ermöglichen.

Voraussetzung: Sie müssen für diese Prozedur über Rootberechtigungen auf dem Informix-Server verfügen.

Installieren und konfigurieren Sie die folgenden Dämonen auf dem Informix-Server, wenn Sie IBM OpenAdmin Tool (OAT) for Informix für Fernverwaltungstasks verwenden wollen, einschließlich des Startens eines Servers in einem Hochverfügbarkeitscluster oder des Hinzufügens eines sekundären Servers:

  • Einen Internet-Service-Dämon wie die Internetdämonen xinetd oder inetd für UNIX und Linux.
  • idsd, einen Dämonprozess, der OpenAdmin Tool das Starten eines fernen Informix-Servers ermöglicht. Der Dämon befindet sich auf demselben Host wie der Informix-Server und muss sich in einem Verzeichnis befinden, auf das der Benutzer informix Zugriff hat. Der Dämon idsd wird vom Dämon xinetd oder inetd aufgerufen.
Wichtig: Sie können die Fernverwaltungsfeatures unter Windows oder MAC OS nicht verwenden.

Gehen Sie wie folgt vor, um einen Internetdämon und den Dämon idsd zu konfigurieren:

  1. Stellen Sie sicher, dass der Dämon idsd mit Informix im Verzeichnis $INFORMIXDIR/bin installiert ist. Stellen Sie sicher, dass der Eigner und die Gruppe informix und die Berechtigungen 0754 lauten.

    Beispiel:

    [informix@myhost]/usr/informix/bin% ls -la idsd 
    -rwxr-xr--  1 informix informix 23717 Oct 15 16:34 idsd
  2. Konfigurieren Sie auf dem Server die Verschlüsselung für die Verbindung. Verwenden Sie für Informix 11.70 das Tool GSKCmd, das als Komponente von IBM JRE 1.6 oder später verfügbar ist. Für ältere Informix-Versionen verwenden Sie das Tool GSKCmd, das in IBM Global Security Kit enthalten ist.
    1. Erstellen Sie mithilfe der folgenden GSKCmd-Befehlszeilenparameter eine Schlüsseldatenbank für Ihre Plattform: 
      befehl -keydb -create -db name_der_schlüsseldatenbank.kdb -pw kennwort -type cms -stash
      Beispiel: 
      gsk8cmd -keydb -create -db mydb.kdb -pw mypassw0rd -type cms -stash
    2. Erstellen ein selbst signiertes Zertifikat mithilfe der folgenden GSKCmd-Befehlszeilenparameter: 
      befehl -cert -create -db name_der_schlüsseldatenbank.kdb -pw kennwort -label kennsatztext -dn definierter_name
    -size <1024 | 512 | 2048> -default_cert yes
      Dabei ist definierter_name ein definierter X.500-Name. definierter_name ist eine Zeichenfolge in Anführungszeichen mit folgendem Format: CN=allgemeiner name (erforderlich), O=organisation, OU=organisationseinheit, P=standort, ST=bundesland oder provinz, C=land.

      Beispiel: 

      gsk8cmd -cert -create -db mydb.kdb -pw mypassw0rd -label mycertificatelabel
    -dn "CN=lenexa.ibm.com,O=ibm,OU=IBM HTTP Server,ST=KS,C=US" 
    -size 1024 -default_cert yes
  3. Öffnen Sie die xinetd- oder inetd-Konfigurationsdatei, um einen Serviceeintrag für den Dämon idsd zu bearbeiten und hinzuzufügen.
    Wichtig: Wenn der Konfigurationsparameter GSKIT_VERSION auf eine Nicht-Standardversion von GSKit gesetzt ist, fügen Sie dem Serviceeintrag xinetd oder inetd das Serverprogrammargument -g zahl hinzu, um die GSKit-Version anzugeben.
    • Serverdämon xinetd: /etc/xinetd.conf. 
      service idsd
             {
             disable = no
             socket_type = stream 
             protocol = tcp
             user = root
             wait = no 	    
             server = /usr/informix/bin/idsd
             server_args = -l /usr/informix/idsd.log 
                 -k /usr/informix/idsd -n idsd_label
             }
    • Serverdämon inetd unter UNIX: /etc/inetd.conf: 
      idsd stream tcp nowait root /usr/informix/bin/idsd idsd -l /usr/informix/idsd.log  -k /usr/informix/idsd -n idsd_label
  4. Öffnen Sie die Datei /etc/services, um einen Service idsd zu bearbeiten und hinzuzufügen. Beispiel: 
    idsd 10000/tcp
    In diesem Beispiel wird die Portnummer 10000 verwendet. Sie können jedoch einen beliebigen anderen Port verwenden, der größer ist als 1024 und noch nicht verwendet wird. Der Name muss jedoch idsd und das Protokoll muss tcp lauten.
  5. Starten Sie den Servicedämon xinetd oder inetd erneut. In den meisten Implementierungen können Sie einen Neustart durchführen, indem Sie das Signal SIGHUP an den aktiven Prozess inetd oder xinetd senden. Auf bestimmten Linux-Plattformen können Sie den folgenden Befehl jedoch als Root ausführen: 
    service xinetd restart

  • IBM Global Security Kit (Sicherheit)

Das AUS-Verwaltungssystem gibt Tabellen und Indizes an, die neue Statistikdaten des Optimierungsprogramms erfordern, und führt die erforderlichen Anweisungen UPDATE STATISTICS aus, um die Abfrageleistung zu optimieren.   1. Klicken Sie im Menü für IBM OpenAdmin Tool (OAT) for Informix auf Serververwaltung > Statistikdaten automatisch aktualisieren > Konfiguration.
  2. Geben Sie auf der Basis der Systemleistung und der Voraussetzungen an, ob für Ihr System die Standardwerte gültig sind.
    1. Geben Sie für den Parameter AUS_AGE eine Anzahl Tage ein, z. B. 30, 60 oder 90, um anzugeben, wie oft Statistikdaten wiederhergestellt werden sollen.
    2. Geben Sie im Parameter AUS_CHANGE einen Prozentsatz an, z. B. 20 oder 50, um festzulegen, welcher für eine Tabelle angefallene Änderungsumfang (in Prozent) eine erneute Erstellung der Statistikdaten initialisiert.

Informationen zur Vorgehensweise beim Verwalten von Scheduler-Tasks

Hinzufügen von Scheduler-Tasks

Sie können Scheduler-Tasks erstellen, die zu angegebenen Zeiten und mit angegebener Häufigkeit Tasks oder Sensoren ausführen.

  • Über Tasks kann ein bestimmter Job zu einer bestimmten Zeit oder in einem bestimmten Intervall ausgeführt werden.
  • Sensoren erfassen und speichern Informationen.

Hinzufügen einer neuen Task

Wichtig: Der Zeitpunkt der Taskerstellung wirkt sich auf die erste Ausführung der Task aus.
  • Wenn die aktuelle Zeit zwischen der angegebenen Start- und Stoppzeit liegt, wird die Task zum aktuellen Zeitpunkt ausgeführt. Nachfolgende Ausführungen finden in Intervallen statt, die über den Häufigkeitswert angegeben sind. Wenn die aktuelle Zeit beispielsweise zwischen der Start- und Stoppzeit liegt und die Häufigkeit auf 15 Minuten gesetzt ist, startet die erste Ausführung der Task unverzüglich nach der Planung der Task und die nächste Ausführung erfolgt 15 Minuten später.
  • Wenn die aktuelle Zeit jedoch nicht zwischen der Start- und der Stoppzeit liegt, wird mit der Ausführung der Tasks bis zur nächsten Startzeit entweder am aktuellen oder am darauffolgenden Tag gewartet. Wenn z. B. 1:00 Uhr die aktuelle Zeit und 19:00 Uhr die nächste Startzeit ist, wird die Task um 19:00 Uhr an demselben Tag gestartet.

Gehen Sie wie folgt vor, um eine neue Task hinzuzufügen:

  1. Klicken Sie auf der Seite Taskzeitplan auf Neue Task hinzufügen.
  2. Klicken Sie unter Welchen Typ von Task wollen Sie hinzufügen? auf Task und anschließend auf Weiter.
  3. Geben Sie einen Namen in das Feld Taskname ein. Beispiel: meinetask.
  4. Wählen Sie eine Kategorie aus der Liste Taskgruppe aus.
  5. Fügen Sie eine Beschreibung der neuen Task in das Textfeld ein und klicken Sie auf Weiter.
  6. Geben Sie den für die Ausführung der Task zulässigen Zeitrahmen an, beispielsweise jeden Tag stündlich zwischen 10:00 und 18:00 Uhr.

Optimieren von SQL-Anweisungen

Erstellen Sie eine externe Steueranweisung zum Optimieren einer SQL-Anweisung mithilfe von IBM OpenAdmin Tool (OAT) for Informix.

Voraussetzungen:
  • Setzen Sie in der ONCONFIG-Datei des Datenbankservers den Konfigurationsparameter EXT_DIRECTIVES auf 1 oder 2, damit die externen Steueranweisungen gespeichert werden können. Wenn Sie den Konfigurationsparameter EXT_DIRECTIVES geändert haben, müssen Sie den Datenbankserver erneut starten, damit die Änderung wirksam wird.
  • Stellen Sie sicher, dass die SQL-Traceverarbeitung aktiviert und die Option Tabellennamen auf der Seite Traceverarbeitungsverwaltung ausgewählt ist. Siehe Aktivieren der SQL-Traceverarbeitung und Angeben von Optionen für die SQL-Traceverarbeitung.

Sie können eine externe Steueranweisung erstellen, die auf eine SQL-Anweisung angewendet werden soll. Steueranweisungen sind Kommentare, die dem Abfrageoptimierungsprogramm vorschreiben, wie eine Abfrage ausgeführt werden muss. Mit externen Steueranweisungen können Sie den Abfrageausführungsplan einer SQL-Anweisung ändern. Nach der Anwendung externer Steueranweisungen auf eine SQL-Anweisung wird der Abfrageausführungsplan der SQL-Anweisung bei jeder erneuten Ausführung der SQL-Anweisung entsprechend geändert.

Wichtig: Wenn Sie eine SQL-Anweisung erneut ausführen, muss diese genau der SQL-Anweisung entsprechen, auf die Sie die Steueranweisungen angewendet haben. Sie muss beispielsweise dieselbe Groß-/Kleinschreibung und dieselben Leerzeichen aufweisen.

Gehen Sie wie folgt vor, um eine externe Steueranweisung zu erstellen:

  1. Klicken Sie im OAT-Menü auf Leistungsanalyse > SQL-Explorer.
  2. Führen Sie einen Drilldown bis zur SQL-Anweisung durch.
    1. Klicken Sie auf die Registerkarte SQL.
    2. Klicken Sie auf die SQL-Anweisung in der Liste.
    3. Klicken Sie auf eine Zeile in der Liste der SQL-Anweisungsausführungen.
  3. Klicken Sie auf die Registerkarte Optimieren.
  4. Klicken Sie auf Hinzufügen.
  5. Wählen Sie auf der Seite Steueranweisungen hinzufügen die Steueranweisung aus. Wählen Sie, falls erforderlich, die Tabellen und Indizes aus. Wenn Sie weitere Steueranweisungen für die SQL-Anweisung hinzufügen wollen, klicken Sie auf Weitere hinzufügen oder klicken Sie auf Hinzufügen, um die Seite zu schließen.
  6. Geben Sie an, wie die Steueranweisungen angewendet werden sollen und klicken Sie auf Anwenden. Der Standardwert ist Inaktiv: Die Steueranweisung wird nicht auf die SQL-Anweisung angewendet.

Bereitstellen des Kennwortschutzes für die Verwaltungskonfigurationsseiten: Apache-Beispiel

Beschränken Sie aus Sicherheitsgründen den Zugriff auf die Verwaltungsseiten von IBM OpenAdmin Tool (OAT) for Informix. Der Schutz dieser Seiten mit einem Kennwort ist wichtig, da über die Seite Anmeldung auf die Seiten Verwaltung zugegriffen werden kann und sie daher jedem Benutzer zugänglich sind.

Eine Methode zum Schutz dieser Seiten ist die Verwendung der vom Web-Server bereitgestellten integrierten Authentifizierung. Sie können beispielsweise das Installationsverzeichnis oat-installation/admin/.htaccess schützen.

Beispiel: Gehen Sie wie folgt vor, um das Installationsverzeichnis von OpenAdmin Tool innerhalb von Apache zu schützen:

  1. Erstellen Sie mithilfe des in Apache bereitgestellten Dienstprogramms htpasswd eine Kennwortdatei in einem Verzeichnis, auf das über das Internet nicht zugegriffen werden kann.
  2. Fügen Sie einer der folgenden Dateien Authentifizierungssteueranweisungen hinzu:
    • oat-installation/admin/.htaccess
    • httpd.conf
  3. Stellen Sie die Steueranweisungen in einen <Directory>-Abschnitt. Beispiel: 
    <Directory OATINSTALL/admin>
AuthType Basic
AuthName "OAT Admin"
AuthUserFile kennwortdatei
Require ValidUser
</Directory>
    Dabei ist kennwortdatei das Verzeichnis, in dem Sie die Kennwortdatei mithilfe des Dienstprogramms htpasswd erstellt haben.

Weitere Informationen und Beispiele finden Sie im Artikel zu Authentifizierung, Berechtigung und Zugriffssteuerung auf der Apache Software Foundation-Website: www.apache.org (wird in neuem Fenster geöffnet).

Konfigurieren von HTTPS für OAT (Linux)

Aktivieren Sie HTTPS, um den Web-Server von IBM OpenAdmin Tool (OAT) for Informix vor dem Ausspionieren, Verfälschung, dem Fälschen von Nachrichten und Netzinterferenzen zu schützen.

Voraussetzung: OpenSSL muss auf dem Computer installiert sein, auf dem der Apache-Server für OAT installiert ist. Sie können OpenSSL über die OpenSSL Project-Website unter www.openssl.org herunterladen.

Gehen Sie wie folgt vor, um ein Upgrade eines Plug-ins für OpenAdmin Tool durchzuführen:

  1. Kopieren Sie die komprimierte Plug-in-Datei in das Verzeichnis oat-verzeichnis/plugin_install. Beispiel: C:\Programme\OpenAdmin\Apache_2.2.16\htdocs\openadmin\plugin_install.
  2. Klicken Sie auf der Seite OAT-Anmeldung oder im OAT-Menü auf Verwaltung.
  3. Klicken Sie im Menü Verwaltung auf Plug-in-Manager.
  4. Klicken Sie in der Liste Noch nicht installierte Plug-ins in der Zeile mit dem Plug-in auf Upgrade. Die vorherige Version des Plug-ins wird von der neueren Version überschrieben. Die aktualisierte Version des Plug-ins wird in der Liste Installierte Plug-ins angezeigt und standardmäßig aktiviert.

Funktionen der Klasse 'idsadmin'

Die Klasse idsadmin stellt eine Bibliothek mit Basisfunktionen für Plug-ins für IBM OpenAdmin Tool (OAT) for Informix bereit, einschließlich der Funktionen zum Erstellen von Datenbankverbindungen, Laden der Datei mit sprachabhängigen Anweisungen und Darstellung der Ausgabe in HTML.

Funktion 'isreadonly'

Die Funktion isreadonly gibt an, ob der Benutzer über eine Gruppe angemeldet ist, die nur über Lesezugriff verfügt. Diese Funktion ist Teil der Klasse idsadmin, die eine Bibliothek mit Basisfunktionen für Plug-ins für IBM OpenAdmin Tool (OAT) for Informix bereitstellt.

Syntax

isreadonly ()

Verwendung

Verwenden Sie die Funktion isreadonly, um festzustellen, ob der Benutzer bei OAT über eine Gruppe angemeldet ist, die nur über Lesezugriff verfügt. Mit diesen Informationen können Sie festlegen, welche Zugriffe der Benutzer in OpenAdmin Tool hat und welche Funktionen dem Benutzer zur Verfügung gestellt werden sollen.

Rückgabewerte

Ein Rückgabewert True gibt an, dass der Benutzer über eine Gruppe angemeldet ist, die nur über Lesezugriff verfügt.

Ein Rückgabewert False gibt an, dass der Benutzer über eine Gruppe angemeldet ist, deren Zugriffsrechte nicht auf Lesezugriff beschränkt sind.

Beispiel

In diesem Beispiel wird ein Fehler zurückgegeben und eine Nachricht angezeigt, wenn der Benutzer nur über Lesezugriff verfügt. In diesem Beispiel ist PermissionRequired das Schlüsselwort einer Programmiersprache und nicht die Nachricht selbst.

if ( $this->idsadmin->isreadonly() )
{
    $this->idsadmin->fatal_error($this->idsadmin->lang("PermissionRequired"));
    return;         
}

Funktion 'get_database'

Über die Funktion get_database wird eine Verbindung zu einer Datenbank auf dem Datenbankserver hergestellt. Diese Funktion ist Teil der Klasse idsadmin, die eine Bibliothek mit Basisfunktionen für Plug-ins für IBM OpenAdmin Tool (OAT) for Informix bereitstellt.

Syntax

get_database($dbname="sysmaster",$use_user=false)
Variable Zweck Einschränkungen
dbname Der Datenbankname. Optional. Der Standardwert ist sysmaster.
use_user Gibt an, ob der SQL ToolBox-Benutzer verwendet wird, um eine Verbindung zu der Datenbank herzustellen. Wenn Sie als Benutzer in SQL ToolBox definiert sind, setzen Sie den Wert auf true. Optional. Der Standardwert ist false.

Rückgabewerte

Der Rückgabewert ist die PDO-Datenbankverbindung.

Beispiel

In diesem Beispiel wird eine Verbindung zu der Datenbank sysmaster hergestellt.

 $db = $this->idsadmin->get_database("sysmaster");

Funktion 'load_lang'

Über die Funktion load_lang wird die Datei mit sprachabhängigen Anweisungen für ein Modul in das idsadmin-Sprachenarray geladen. Diese Funktion ist Teil der Klasse idsadmin, die eine Bibliothek mit Basisfunktionen für Plug-ins für IBM OpenAdmin Tool (OAT) for Informix bereitstellt.

Syntax

load_lang($keyword)
Variable Zweck Einschränkungen
keyword Das Schlüsselwort, mit dem angegeben wird, welche Datei mit sprachabhängigen Anweisungen für das Modul geladen werden soll. Normalerweise ist dieses Schlüsselwort identisch mit dem Schlüsselwort act für das Modul. Die Datei mit sprachabhängigen Anweisungen enthält die Textzeichenfolgen für das Modul.

Verwendung

Die Funktion load_lang lädt die Datei lang_keyword.xml; z. B. lang_admin.xml oder lang_health.xml. Die Funktion load_lang wird normalerweise als Teil des Klassenkonstruktors für das Modul aufgerufen.

Beispiel

In diesem Beispiel eines Klassenkonstruktors wird die Datei mit sprachabhängigen Anweisungen für das Modul Health geladen.

function __construct(&$idsadmin)
{
        $this->idsadmin = &$idsadmin;

        $this->idsadmin->load_lang("health");
}

Funktion 'lang'

Die Funktion lang gibt eine bestimmte Nachricht in der aktuellen Sprache zurück. Diese Funktion ist Teil der Klasse idsadmin, die eine Bibliothek mit Basisfunktionen für Plug-ins für IBM OpenAdmin Tool (OAT) for Informix bereitstellt.

Syntax

lang($item, $parameters=array())
Variable Zweck Einschränkungen
item Der Name der Textzeichenfolge, die in der aktuellen Sprache zurückgegeben werden soll.
parameters Die Parameter in einem Bereich. Die Parameter werden für "{0}", "{1}", "{2}" usw. ersetzt. Optional.

Verwendung

Die Funktion lang sucht die angegebene Nachricht in der geladenen Datei mit sprachabhängigen Anweisungen. Ist die aktuelle Sprache nicht Englisch und die angegebene Nachricht wird nicht gefunden, wird die Nachricht in englisch zurückgegeben. Wenn sie nicht in englisch verfügbar ist, wird ein Fehler zurückgegeben.

Beispiele

In diesem Beispiel wird der Text für den Eintrag taskdetailssaved in der aktuellen Sprache abgerufen und als Statusnachricht ausgegeben.

$this->idsadmin->status($this->idsadmin->lang("taskdetailssaved"));

Der Eintrag ist in der Datei mit sprachabhängigen Anweisungen für das Modul wie in diesem Beispiel dargestellt in deutsch enthalten.

<taskdetailssaved><![CDATA[Taskdetails erfolgreich gespeichert.]]></taskdetailssaved>

In diesem Beispiel wird die Seitenüberschrift des Browsers für die Seite Diagnosezentrale > Alerts in OAT festgelegt.

$this->idsadmin->html->set_pagetitle($this->idsadmin->lang("alerts") );

Dies ist der Eintrag in der Datei mit sprachabhängigen Anweisungen für das Modul in Deutsch und in Spanisch.

<alerts><![CDATA[Alerts]]></alerts>
<alerts><![CDATA[Alertas]]></alerts>

Funktion 'format_units'

Die Funktion format_units formatiert einen ganzzahligen Bytewert in der am besten lesbaren Form, z. B. KB, MB oder GB. Diese Funktion ist Teil der Klasse idsadmin, die eine Bibliothek mit Basisfunktionen für Plug-ins für IBM OpenAdmin Tool (OAT) for Informix bereitstellt.
