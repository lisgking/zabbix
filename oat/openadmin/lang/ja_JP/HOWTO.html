IBM OpenAdmin Tool (OAT) for Informix

IBM® OpenAdmin Tool (OAT) for Informix® は、IBM Informix データベース サーバのパフォーマンスを管理および分析する Web アプリケーションです。Web サーバにインストールされている単一の OAT から複数のデータベース サーバ インスタンスを管理できます。どのブラウザからでも、Web サーバにアクセスしてすべてのデータベース サーバを管理できます。

OAT を使用すると、以下のようにデータベース サーバのパフォーマンスを監視および分析できます。

  • 詳細ステータス レポート、アラート情報、およびメッセージ ログを使用して、データベース サーバの状態を監視できます。
  • IBM Informix Health Advisor Plug-in for OpenAdmin Tool (OAT) を使用して、データベース サーバの状態、電子メールの結果、および推奨事項を分析できます。
  • パフォーマンス履歴の詳細グラフを使用して、長期的にパフォーマンス統計を監視できます。
  • スケジューラが収集するデータを監視できます。スケジューラは管理タスクを実行し、SQL 文を使用して情報を収集します。
  • データベース サーバ上のデータベースおよび表に対してシステム検証チェックを実行できます。
  • システム レポートおよび onstat コマンドを実行できます。
  • SQL 文をドリル ダウンしてパフォーマンス詳細を確認し、SQL 文を最適化できます。

OAT を使用すると、以下の管理タスクを実行できます。

  • データベース サーバ (構成、ユーザ アクセス権、仮想プロセッサなど) を管理できます。
  • スケジューラのタスクを作成および監視できます。
  • 自動統計更新 (AUS: Auto Update Statistics) 保守システムを管理できます。このシステムは、自動的に統計を更新して、問合せパフォーマンスを最適化します。
  • IBM Informix Schema Manager Plug-in for OpenAdmin Tool (OAT) を使用して、データベースおよび表オブジェクトを管理できます。
  • データベース サーバの格納領域を管理すること、およびリカバリ ログを管理することができます。
  • 表を圧縮すること、および表のフラグメント化をなくすことで領域を最適化できます。
  • データベース サーバの格納領域をバックアップします。
  • 表に対して SQL 文を実行すること、および標準 SQL 操作を実行することができます。
  • 高可用性クラスタを監視および管理できます。
  • IBM Informix Replication Plug-in for OpenAdmin Tool (OAT) を使用して、エンタープライズ レプリケーションを監視および管理できます。
  • IBM Informix TimeSeries Plug-in for OpenAdmin Tool (OAT) を使用して、時系列に関連するデータベース オブジェクト (表、インデックス、仮想表、および TimeSeries サブタイプ、コンテナおよびカレンダを含む) を監視および管理できます。

IBM OpenAdmin Tool (OAT) for Informix は、IBM Informix Client Software Development Kit (Client SDK) をインストールするとき、または IBM Informix ソフトウェア バンドルをインストールし Client SDK または IBM Informix Connect を選択すると、インストールできます。このインストール プログラムには、OAT、Health Advisor プラグイン、Replication プラグイン、Schema Manager プラグイン、TimeSeries プラグイン、および OAT を実行するために必要なすべてのソフトウェア (事前構成された Apache、PHP、PDO_informix など) が含まれています。

また、IBM Informix Free Product Download ページ https://www14.software.ibm.com/webapp/iwm/web/preLogin.do?lang=en_US&source=swg-informixfpd (新しいウィンドウで開く) から OpenAdmin Tool Community Edition を入手することもできます。OpenAdmin Tool Community Edition には、OAT とオプションのプラグインは含まれていますが、サポートするソフトウェアは含まれていません。 このエディションは手動でインストールする必要があります。

データベース サーバへの接続方法

データベース サーバへのログイン

IBM OpenAdmin Tool (OAT) for Informix を使用して、データベース サーバにログインします。

データベース サーバにログインするには、次の手順を実行します。

  1. OAT ルート URL: http://server name/openadmin に移動します。
  2. OAT の「ログイン」ページで「サーバ詳細」フィールドに入力します。
  3. 「ログイン」をクリックします。

OAT を介してデータベース サーバに接続されます。

ヒント: データベース サーバが OAT グループのメンバである場合は、代わりに OAT グループにログインして、データベース サーバに接続できます。OAT グループには、データベース サーバの接続情報が含まれています。OAT グループにログインすると、再度ログインすることなくグループ内の別のデータベース サーバに切り替えることができます。

OAT グループ内のデータベース サーバへのログイン

IBM OpenAdmin Tool (OAT) for Informix でデータベース サーバ接続のグループにログインして、データベース サーバに接続します。

OAT グループには、データベース サーバの接続情報が含まれています。OAT グループにログインすると、再度ログインすることなくグループ内の別のデータベース サーバに切り替えることができます。

OAT グループ内のデータベース サーバにログインするには、次のようにします。

  1. OAT ルート URL: http://server name/openadmin に移動します。
  2. OAT の「ログイン」ページの「OAT グループ」リストで、接続先のサーバが含まれている OAT グループを選択します。
  3. OAT グループのパスワードを入力します。
  4. 「サーバの取得」をクリックします。
  5. サーバのリストからサーバを選択します。サーバ情報が「サーバ詳細」のフィールドに取り込まれます。
  6. 「サーバ詳細」領域の「パスワード」フィールドで、入力済みのパスワードを受け入れるか、有効なパスワードを入力し、「 ログイン」をクリックします。

OAT グループ内の別のサーバへの切替え

IBM OpenAdmin Tool (OAT) for Informix でデータベース サーバ接続のグループ内の別のデータベース サーバに接続します。

OAT グループには、データベース サーバの接続情報が含まれています。OAT グループにログインすると、再度ログインすることなくグループ内の別のデータベース サーバに切り替えることができます。

OAT グループ内の別のデータベース サーバに切り替えるには、OAT ページの上部にある「サーバ」リストでデータベース サーバを選択します。

データベース サーバの動作の監視方法

データベース サーバの状態の監視

特定のデータベース サーバの状態をリアルタイムで監視するか、または OAT グループ内のすべてのサーバの状態を同時に表示します。

ダッシュボードを使用すると、特定のデータベース サーバの状態を監視できます。このダッシュボードには、メモリと領域の使用量、ロック数、トランザクション数、問合せ数、および読込みと書込みが最多である表が表示されます。

マルチサーバ ダッシュボードには、OAT グループ内のすべてのデータベース サーバの状態を表示できます。詳細については、このダッシュボードから、IBM OpenAdmin Tool (OAT) for Informix の他のページにアクセスします。

データベース サーバの状態を監視するには、次のようにします。

  1. OAT のメニューで、「ヘルス センタ」>「ダッシュボード」をクリックします。
  2. ダッシュボードをリフレッシュするには、リフレッシュ アイコンをクリックするか、またはリフレッシュ レートを変更します。

ダッシュボードのカスタマイズ

ダッシュボードに表示される特定のデータベース サーバの状態に関する情報を変更します。

前提条件: IBM OpenAdmin Tool (OAT) for Informix の「管理」メニューがパスワードで保護されている場合は、パスワードを入力しないとこのタスクを実行できません。

OAT ダッシュボードには、データベース サーバに関する情報 (例えば、メモリー使用率、トランザクション数) が表示されます。ダッシュボードをカスタマイズすると、データベース サーバに関して表示する情報を指定できます。

ダッシュボードをカスタマイズするには、次のようにします。

  1. OAT の「管理」メニューで、「ダッシュボード マネージャ (Dashboard Manager)」をクリックします。
  2. ダッシュボードのレイアウト エディタを開くには、ダッシュボードをダブルクリックします。

ダッシュボードの状態しきい値の変更

マルチサーバ ダッシュボードに表示される状態標識のしきい値を変更します。

前提条件: IBM Informix Health Advisor Plug-in for OpenAdmin Tool (OAT) をインストールする必要があります。

IBM OpenAdmin Tool (OAT) for Informix のマルチサーバ ダッシュボードには、OAT グループ内のデータベース サーバの状態標識が表示されます。 例えば、データベース サーバ上の DB 領域のフリー スペースが指定のしきい値を下回ると、データベース サーバに対する警告標識が表示されます。

メモリ、領域、および入出力の合計に対するチャンクの割合の状態しきい値を変更できます。これらのしきい値は、Health Advisor プラグインのデフォルト設定です。

ダッシュボードの状態しきい値を変更するには、次のようにします。

  1. OAT のメニューで、「ヘルス センタ」>「Health Advisor」をクリックします。
  2. 「プロファイル」ページで、現行プロファイルがデフォルト プロファイルであることを確認します。
  3. 「アラーム」タブをクリックします。
  4. 以下のアラームのしきい値を変更します。
    Health Advisor プラグインのアラームしきい値 ダッシュボードの状態しきい値
    OS 空きメモリ メモリ
    DB 領域フリー スペース 領域
    チャンク入出力操作 入出力: 入出力合計に対するチャンクの割合

オンライン メッセージ ログの削除

データベース サーバのオンライン メッセージ ログ ファイルを、削除または切捨てます。

前提条件: この操作を実行するには、IBM OpenAdmin Tool (OAT) for Informix に IBM Informix バージョン 11.70.xC3 が必要です。

オンライン メッセージ ログを削除するには、次のようにします。

  1. OAT のメニューで、「ログ」>「オンライン ログ」をクリックします。
  2. 次のいずれかを実行します。
    • ログ ファイルを削除するには、「削除」をクリックします。
    • ログ ファイルを残したままログ ファイルのメッセージを削除するには、「切捨て」をクリックします。

オンライン メッセージ ログのローテーション

データベース サーバのオンライン メッセージ ログ ファイルの有効化、無効化、変更を自動的にローテーションします。

前提条件: この操作を実行するには、IBM OpenAdmin Tool (OAT) for Informix に IBM Informix バージョン 11.70.xC3 が必要です。

自動ログ ローテーションを構成して、オンライン メッセージ ログ ファイルのサイズを制御できます。 オンライン メッセージ ログのローテーション時に、データベース サーバはログ ファイルを新しいログ ファイルに切り替えて、ID 番号を前のログ ファイルから 1 ずつ増加させます。 ログ ファイルがこの最大数に達すると、最も ID 番号の大きいログ ファイルが削除されます。

オンライン メッセージ ログをローテーションするには、次のようにします。

  1. OAT のメニューで、「ログ」>「オンライン ログ」をクリックします。

前提条件: 領域の最適化の操作を実行するには、IBM OpenAdmin Tool (OAT) for Informix に IBM Informix 11.50.xC6 以降が必要です。フラグメント解消の操作を実行するには、OAT に Informix 11.70 以降が必要です。インデックスとシンプル ラージ オブジェクト (テキスト (TEXT) 型およびバイト (BYTE) 型のデータ型) を圧縮する場合は、OAT に Informix 12.10 が必要です。

格納領域の自動最適化

データ格納領域を最適化するために、表とフラグメントの圧縮、リパック、縮小、およびフラグメント解消を自動的に実行するようにデータベース サーバを構成します。

前提条件: 領域最適化ポリシーを適用するには、OpenAdmin Tool (OAT) for Informix に IBM Informix 11.70 以降が必要です。インデックスとシンプル ラージ オブジェクト (テキスト (TEXT) 型およびバイト (BYTE) 型のデータ型) を圧縮する場合は、OAT に Informix 12.10 が必要です。

最適化ポリシーを実装するスケジューラ タスクを有効化または無効化できます。ポリシーによって、表とフラグメントを圧縮、縮小、リパック、およびフラグメント解消する時期を決定するしきい値が設定されます。各操作のしきい値を更新するか、またはこれらのしきい値を無効にすることができます。また、スケジューラ タスクの実行時点を指定することもできます。

表と表フラグメントが圧縮されるときには、シンプル ラージ オブジェクト (TEXT データ型と BYTE データ型) が組み込まれます。

格納領域を自動的に最適化するようにデータベース サーバを構成するには、次のようにします。

  1. OAT のメニューで、「スペース管理」>「格納領域」をクリックします。
  2. 選択済みのデータベース サーバで、「表およびインデックス」パネルを展開します。
  3. 「サーバ最適化ポリシー」タブをクリックします。
  4. ページでフィールドに入力します。

  • 関連する概念:
  • 格納領域最適化 (Storage optimization) (管理者ガイド)

圧縮の見積り

表、インデックス、およびフラグメントに含まれるデータを圧縮することによって節約できる領域の量の見積りを更新します。

前提条件: 表および表フラグメントの圧縮保存を見積もるには、IBM OpenAdmin Tool (OAT) for Informix に IBM Informix バージョン 11.50.xC6 以降が必要です。インデックスの圧縮、およびシンプル ラージ オブジェクト (TEXT データ型と BYTE データ型) が含まれている表の圧縮を見積もるには、OAT に Informix 12.10 が必要です。

圧縮の見積りを使用して、表、インデックス、またはフラグメントに含まれるデータを圧縮するかどうかを決定できます。表と表フラグメントの圧縮見積もりには、シンプル ラージ オブジェクト (TEXT データ型と BYTE データ型) が含まれます。圧縮見積もりが可能な場合は、「スペース管理」>「格納領域」>「表およびインデックス」ページの「領域使用量」列に表示されます。最新見積りの時刻が、吹出しヘルプに示されます。

圧縮見積りを更新するには、次のようにします。

  1. OAT のメニューで、「スペース管理」>「格納領域」をクリックします。
  2. 選択済みのデータベース サーバで、「表およびインデックス」パネルを展開します。
  3. 表、インデックス、およびフラグメントを選択します。
  4. 「アクション」>「データの更新」をクリックします。

タスクの進行を監視するには、「タスク状況」タブをクリックします。

データの圧縮

表、インデックス、およびフラグメントに含まれるデータを圧縮して、必要なディスク領域の量を削減します。

前提条件: 領域の最適化の操作を実行するには、IBM OpenAdmin Tool (OAT) for Informix に IBM Informix 11.50.xC6 以降が必要です。フラグメント解消の操作を実行するには、OAT に Informix 11.70 以降が必要です。インデックスとシンプル ラージ オブジェクト (テキスト (TEXT) 型およびバイト (BYTE) 型のデータ型) を圧縮する場合は、OAT に Informix 12.10 が必要です。
重要: 表または表フラグメントには、2000 行以上が含まれている必要があります。インデックスまたはインデックス フラグメントの基となる表にも、2000 行以上が含まれている必要があります。切り離された B ツリー インデックス (一意なツリー インデックス、重複したツリー インデックス、ツリー インデックスの有向グラフを含む) を圧縮できます。

表、インデックス、またはフラグメントを圧縮すると、圧縮タスクによってコンプレッション ディクショナリが作成され、行が移動されずに行内のデータが圧縮されます。表を圧縮するときには、パーティションに保管されているシンプル ラージ オブジェクト (TEXT データ型と BYTE データ型) を同じ DB 領域に含めることができます。圧縮タスクの一部として、フリー スペースを統合 (リパック) でき、フリー スペースを DB 領域に返却 (縮小) できます。また、エクステントの数を削減 (フラグメント解消) することもできます。

表、インデックス、およびフラグメントを圧縮するには、次のようにします。

  1. OAT のメニューで、「スペース管理」>「格納領域」をクリックします。
  2. 選択済みのデータベース サーバで、「表およびインデックス」パネルを展開します。
  3. 表、インデックス、およびフラグメントを選択します。
  4. 「アクション」 > 「領域の最適化」をクリックします。
  5. 「圧縮」を選択します。
  6. オプション: 以下のとおり、追加の操作を選択します。
    • フリー スペースを統合するには、「リパック」を選択します。
    • フリー スペースを DB 領域に返却するには、「縮小」を選択します。
    • エクステントの数を削減するには、「エクステントのフラグメント解消」を選択します。

タスクの進行を監視するには、「タスク状況」タブをクリックします。

表および表フラグメントの圧縮解除

以前に圧縮した表および表フラグメントを圧縮解除します。

前提条件: 領域最適化操作を実行するには、IBM OpenAdmin Tool (OAT) for Informix に IBM Informix 11.50.xC6 以降が必要です。

圧縮解除すると、挿入操作と更新操作の圧縮の停止、コンプレッション ディクショナリの活動停止、および元のページに収まらない行への新規ページの割り当ても行われます。

表および表フラグメントを圧縮解除するには、次の手順を実行します。

  1. OAT のメニューで、「スペース管理」>「格納領域」をクリックします。
  2. 選択済みのデータベース サーバで、「表およびインデックス」パネルを展開します。
  3. 表およびフラグメントを選択します。
  4. 「アクション」 > 「領域の最適化」をクリックします。
  5. 「圧縮解除」を選択します。
  6. オプション: 圧縮解除操作中のデータへのアクセスを防ぐ目的で、表およびフラグメントに排他的なロックをかけるには、「オフライン」を選択します。

タスクの進行を監視するには、「タスク状況」タブをクリックします。

論理ログの削除

論理ログを削除し、DB 領域内の利用可能なディスク領域の量を増やします。

重要: 現在使用中のログ ファイル、または最新のチェックポイント レコードを含むログ ファイルは削除できません。

  • 書き込みが行われていないログ ファイルを削除すると、ログはデータベース サーバにより即時に削除され、領域が解放されます。

論理ログを削除するには、次のようにします。

  1. OAT のメニューで、「スペース管理」 > 「リカバリ ログ」をクリックします。
  2. 「管理」タブをクリックします。
  3. 「論理ログの削除」セクションで、論理ログ番号のリストからログを選択します。
  4. 「削除」をクリックします。

高可用性クラスタの管理

高可用性クラスタのための接続の構成

IBM Informix サーバで idsd デーモンをセット アップして、高可用性クラスタに含まれるサーバを起動できるようにするか、または共用ディスク副 (SDS) サーバを追加できるようにします。

前提条件: この手順を実行するには、Informix サーバで root 権限を持っている必要があります。

遠隔管理タスク (高可用性クラスタに含まれるサーバの起動、副サーバの追加など) を実行するために IBM OpenAdmin Tool (OAT) for Informix を使用するには、次のデーモンを Informix サーバにインストールして構成します。

  • インターネット サービス デーモン (UNIX および Linux 用のインターネット デーモン xinetd または inetd など)
  • OAT でリモート Informix サーバを開始できるようにするデーモン プロセス idsd。 このデーモンは、Informix サーバと同じホスト上にあり、ユーザ informix がアクセスできるディレクトリに配置されている必要があります。 idsd デーモンは、xinetd デーモンまたは inetd デーモン経由で起動されます。
重要: Windows または MAC OS では遠隔管理機能を使用できません。

インターネット デーモンおよび idsd デーモンを構成するには、次のようにします。

  1. idsd デーモンが Informix とともに $INFORMIXDIR/bin ディレクトリにインストールされていることを確認します。所有者およびグループが informix であり、権限が 0754 であることを確認します。

    次に例を示します。

    [informix@myhost]/usr/informix/bin% ls -la idsd
    -rwxr-xr--  1 informix informix 23717 Oct 15 16:34 idsd
  2. サーバで、接続の暗号化をセット アップします。Informix 11.70 では、IBM JRE 1.6 以降の一部として使用可能な GSKCmd ツールを使用します。以前のバージョンの Informix では、IBM Global Security Kit に含まれる GSKCmd ツールを使用します。
    1. ご使用のプラットフォームに応じた次の GSKCmd コマンド行パラメータを使用して、キー データベースを作成します。 
      command -keydb -create -db keydb_name.kdb -pw password -type cms -stash
      次に例を示します。
      gsk8cmd -keydb -create -db mydb.kdb -pw mypassw0rd -type cms -stash
    2. 次の GSKCmd コマンド行パラメータを使用して、自己署名証明書を作成します。
      command -cert -create -db keydb_name.kdb -pw password -label label_text
    -dn distinguished name 
    -size <1024 | 512 | 2048> -default_cert yes
      distinguished name は、X.500 識別名です。distinguished name は、次の形式の引用符付きストリング です。CN=common name (必須), O=organization, OU=organization unit, L=location, ST=state or province, C=country。

      次に例を示します。

      gsk8cmd -cert -create -db mydb.kdb -pw mypassw0rd -label mycertificatelabel
    -dn "CN=lenexa.ibm.com,O=ibm,OU=IBM HTTP Server,ST=KS,C=US" 
    -size 1024 -default_cert yes
  3. xinetd または inetd 構成ファイルを編集のために開き、idsd デーモンのサービス エントリを追加します。
    重要: GSKIT_VERSION 構成パラメータがデフォルト以外のバージョンの GSKit に設定されている場合は、xinetd または inetd サービス エントリに -g NUMBER サーバ プログラム引数を追加して、GSKit のバージョンを指定します。
    • xinetd サーバ デーモンの場合は /etc/xinetd.conf です。
      service idsd
             {
             disable = no
             socket_type = stream 
             protocol = tcp
             user = root
             wait = no 	    
             server = /usr/informix/bin/idsd
             server_args = -l /usr/informix/idsd.log 
                 -k /usr/informix/idsd -n idsd_label
             }
    • UNIX の inetd サーバ デーモンの場合は /etc/inetd.conf です。
      idsd stream tcp nowait root /usr/informix/bin/idsd idsd 
    -l /usr/informix/idsd.log -k /usr/informix/idsd -n idsd_label
  4. ファイル /etc/services を編集用に開き 、idsd という名前のサービスを追加します。次に例を示します。
    idsd 10000/tcp
    この例では、 ポート番号 10000 を使用していますが、1024 より大きい、既に使用中でない 他の任意のポートを使用できます。ただし、名前は idsd である必要があり、 プロトコルは tcp である必要があります。
  5. xinetd サービス デーモンまたは inetd サービス デーモンを再開します。大部分のインプリメンテーションでは、実行中の inetd プロセス または xinetd プロセスに SIGHUP シグナルを送信することにより再開できます。一方、一部の Linux プラットフォームでは、root として次のコマンドを実行できます。 
    service xinetd restart

高可用性クラスタの検索

IBM OpenAdmin Tool (OAT) for Informix を使用して、高可用性クラスタを特定し、監視および管理ができるようにします。

前提条件: 高可用性クラスタ内のサーバを OAT グループに追加します。 OAT グループ内のサーバ上のクラスタのみが「高可用性クラスタ」ページに表示されます。サーバを OAT グループに追加するには、OAT の「管理」メニューで「接続の管理」>「接続の追加」をクリックします。

高可用性クラスタを検索するには、次のようにします。

  1. OAT のメニューで、「レプリケーション」 > 「クラスタ」をクリックします。
  2. 「クラスタの検索」をクリックします。

    クラスタが検出されたら、クラスタ内のサーバ (主サーバ、高可用性データ レプリケーション (HDR) サーバ、共用ディスク副 (SDS) サーバ、およびリモート スタンドアロン副 (RSS) サーバ) がリストされた、「検索されたクラスタ」ページが表示されます。

  3. デフォルト名を受け入れるか、クラスタの新規の一意の名前を入力します。 見つかったクラスタごとにこのステップを反復します。

時系列の管理

時系列に関連付けられたデータベース オブジェクト　(表、インデックス、仮想表、および TimeSeries のサブタイプ、コンテナ、カレンダを含む) を確認および管理します。

時系列は、時間の経過に伴う変化が記録された一連のデータです。 各エントリは、タイム スタンプが付けられて、データベースに格納されます。Informix TimeSeries 機能拡張は、時系列データを管理するための TimeSeries データ型、コンテナ、およびカレンダを提供します。

時系列を管理するには、次のようにします。

IBM OpenAdmin Tool (OAT) for Informix のメニューで、「SQL ToolBox」>「TimeSeries」をクリックします。データベース サーバ内の時系列を含むデータベースと表のリストが表示されます。

  • データベースまたは表についての情報を表示するには、リストからデータベースまたは表をクリックします。
  • 特定のデータベース オブジェクトまたは表オブジェクトの詳細情報を表示するには、オブジェクト パネルで拡張アイコンをクリックします。

データベース サーバの構成パラメータの管理

現在のデータベース サーバの onconfig ファイルのパラメータの値を表示し、変更します。

現行の IBM Informix データベース サーバの構成パラメータの値の多くは、IBM OpenAdmin Tool (OAT) for Informix を使用して変更できます。

データベース サーバの構成パラメータを管理するには、次のようにします。

  1. OAT のメニューで、「サーバ管理」 > 「構成」をクリックします。
  2. パラメータのリストをフィルタリングするには、「表示」リストから以下のビューを選択します。
    • 基本: すべての環境に適用できる基本的な構成パラメータを表示します。
    • 動的: ユーザが変更できるパラメータを表示します。
    • 推奨事項: OAT の推奨に従っていない値が設定されているパラメータを表示します。 推奨に従っていない設定のパラメータはすべて、黄色で強調表示されています。
  3. パラメータを表示または変更するには、パラメータ名をクリックします。

システムの検証検査の実行

現行サーバにある 1 つ以上のデータベースまたは表を検証するには、次のオプションのうちのいずれかを使用します。

  • いずれかのデータベースのすべての表を検証するには、「データベース名」フィールドでデータベースの名前を選択し、 「表名」フィールドで「すべて」を選択します。
  • 特定の 1 つの表を検証するには、「データベース名」フィールドでデータベースの名前を選択し、 「表名」フィールドで表の名前を選択します。 表がどのデータベースに存在するかが不明な場合は、「データベース名」フィールドをデフォルト値「すべて」のままにします。 「表名」リストで表の名前を検索します。

  3. 「エクステントの検査」をクリックします。検査されるエクステントごとに 状況メッセージが表示されます。このとき、検査が正常に完了されたか どうかが検証されます。

データベース サーバによって格納されたハッシュされたパスワードを使用して証明書を検査することによって、データベース サーバが認証するユーザを追加します。 ユーザがデータベース サーバのホスト コンピュータのオペレーティング システム アカウントを所持しているかどうかに関係なく、各ユーザに適切なユーザ アクセス権およびグループ アクセス権をマップすることによって、それらのユーザにデータベース アクセス権を付与します。 ユーザは、オペレーティング システム ユーザ名またはオペレーティング システム ユーザ ID にマップでき、さらに 1 つ以上のグループ ID にマップできます。/etc/informix/allowed.surrogates で指定されたオペレーティング システム ユーザ名、ユーザ ID、グループ名、およびグループ ID によって、どのオペレーティング システム ユーザおよびグループがマップ ユーザのサロゲートとして機能できるかが決まります。

自動統計更新の管理

表統計用の自動統計更新 (AUS: Auto Update Statistics) 保守システムを確認および調整します。

AUS 保守システムは、新しいオプティマイザ統計が必要な表およびインデックスを識別し、適切な UPDATE STATISTICS 文を実行して問合せパフォーマンスを最適化します。使用しているシステムによっては、AUS 有効期限ポリシーまたはスケジュールを調整する必要がある場合があります。

「サーバ管理」>「自動統計更新」>「構成」ページで、AUS ポリシーを制御およびモニタできます。

「SQL ToolBox」>「スキーマ マネージャ」>「統計」ページで IBM Informix Schema Manager Plug-in for OpenAdmin Tool (OAT) を使用すると、表の統計を確認できます。

自動統計更新の構成および有効化

  1. IBM OpenAdmin Tool (OAT) for Informix のメニューで、「サーバ管理」>「自動統計更新」>「構成」をクリックします。
  2. システムの性能および要件に基づいて、デフォルト値がご使用のシステムにとって有効であるかどうかを判別します。
    1. 統計情報を次に再構築するまでの最大時間を指定するには、AUS_AGE パラメータで日数 (30、60、または 90 など) を入力します。
    2. 表の変更部分が何パーセントになったら統計の再作成を開始するのかを 指定するには、20 または 50 などのパーセント数を AUS_CHANGE パラメータに 入力します。
    3. AUS で推奨されるガイドライン ルールを使用するかどうかを指定するには 、AUS_AUTO_RULES パラメータのリストから「オン」または 「オフ」を選択します。デフォルトは「オン」です。
    4. 統計情報が必ず再構築される行数を指定するには、AUS_SMALL_TABLES パラメーターに数を入力します。 デフォルトは 100 です。
    5. PDQ 優先順位を指定するには、AUS_PDQ パラメータ に -1 から 100 までの数を入力します。デフォルトは 10 です。並列データベース問合せ (PDQ) の優先順位は、データベース サーバが並列処理にリソースを使用する方法を決定する 1 つの要因です。
    6. 統計情報タスクの更新を実行するために使用するスレッドの数を指定するには、スレッドの数を入力します。OAT は、この AUS タスクの数をタスク スケジューラに実行要求します。このオプションを使用するには、IBM Informix バージョン 11.70 が必要です。
  3. 変更内容を維持するには、「保存」をクリックします。
  4. AUS タスクが有効かどうかを判別するには、「情報」タブをクリックし、以下のタスクをクリックします。
    • 自動統計更新の評価
    • 自動統計更新のリフレッシュ
      注: 自動統計更新のリフレッシュ タスクに関して実行する作業がない場合、タスクは自動的に無効になります。 自動統計更新の評価タスクが次回実行されたときに、リフレッシュ タスクの対象となる作業が作成された場合は、評価タスクによってリフレッシュ タスクが自動的に有効になります。
    「タスク詳細」ページが表示されます。
  5. 「タスクの有効化」フィールドまでスクロールダウンし、 このフィールドが選択されていることを確認します。
  6. 実行ウィンドウを設定するフィールドを使用して、タスクの実行頻度を指定します。AUS はリソースを大量に消費する操作であるため、特定の実行ウィンドウを定義できます。実行ウィンドウの定義は、 開始時刻と終了時刻および実行する曜日を含みます。 デフォルトでは、実行時ウィンドウは毎日午前 1:00 から午前 5:00 までです。
  7. 変更をすべて保存するには「保存」をクリックします。

スケジューラ タスクの管理方法

スケジューラ タスクの追加

指定した時間および頻度でタスクまたはセンサーのいずれかを実行するスケジューラ タスクを 作成できます。

  • タスクは、特定のジョブを特定の時刻または間隔で実行する方法を 提供します。
  • センサーは、情報を収集して保存します。

新規タスクの追加

重要: タスクを作成した時刻は、そのタスクの最初の実行に影響します。
  • 現在時刻が、指定した開始時刻と停止時刻の間に入っている場合、 タスクは現在時刻から実行されます。以降の実行は、頻度値に指定した間隔 で行われます。したがって、例えば、 現在時刻が指定した開始時刻と停止時刻の間に入っており、頻度に 15 分を 設定した場合、タスクの最初の実行はタスクをスケジュールした直後に開始 され、次回の実行は 15 分後に行われます。
  • 一方、現在時刻が開始時刻と停止時刻の間に入っていない場合は、 次回の開始時刻までタスクの実行が待機されます。これは、当日であるか翌日であるかの いずれかになります。例えば、現在時刻が午前 1:00 であり、次の開始時刻が午後 7:00 (19:00) の場合、タスクは、同じ日の 19:00 に開始されます。

新規タスクを追加するには、次のようにします。

  1. 「タスクのスケジュール」ページで「新規タスクの追加」をクリックします。
  2. 「追加するタスクのタイプ」の下の「タスク」をクリックしてから「次へ」をクリックします。
  3. 「タスク名」フィールドに名前を入力します。例えば、MyTask です。
  4. 「タスク グループ」リストからカテゴリを選択します。
  5. 新規タスクの説明をテキスト フィールドに追加し、 「次へ」をクリックします。
  6. タスクの実行が許可されるタイム フレームを指定します。例えば、毎日の 午前 10 時 (10:00) から午後 6 時 (18:00) までのすべての時間です。
    1. 時間間隔の開始時刻を 24 時間時計形式 hh:mm:ss を使用して指定します (例: 10:00:00)。
    2. 時間間隔の停止時刻を 24 時間時計形式 hh:mm:ss を使用して指定します (例: 18:00:00)。NEVER を指定することもできます。
    3. 許可される時間間隔の間でタスクを実行できる頻度を、日、時、分 (d Days h Hours m Minutes) で指定します (例: 0 Days 1 Hours 00 Minutes)。
  7. 「次へ」をクリックします。
  8. タスクが実行するコマンド ステートメントを指定します。例えば、その他タスクを削除するには、次のようなステートメントを 入力します。 
    delete from ph_task where tk_group='MISC'
  9. 「次へ」をクリックします。確認ページが表示されます。
  10. 各選択項目の入力内容を確認します。
    • 入力が正しければ「終了」をクリックして確認し、 タスクを作成します。
    • 誤った値を訂正するために前のページに戻るには「戻る」を クリックします。

新規センサーの追加

  1. 「タスクのスケジュール」ページで「新規タスクの追加」をクリックします。
  2. 「追加するタスクのタイプ」の下の「センサー」をクリックしてから「次へ」をクリックします。
  3. 「センサー名」フィールドに名前を入力します。例えば、MySensor です。
  4. 「センサ グループ」リストからカテゴリを選択します。
  5. 新規タスクの説明をテキスト フィールドに追加し、 「次へ」をクリックします。
  6. センサーを実行できる許容可能なタイム フレーム (例えば、毎日午前 10:00 時から午後 6:00 時の間の毎時) を指定します。
    1. 時間間隔の開始時刻を 24 時間時計形式 hh:mm:ss を使用して 指定します。例えば、10:00:00 です。
    2. 時間間隔の停止時刻を 24 時間時計形式 hh:mm:ss を使用して 指定します。例えば、18:00:00 です。
    3. 許可される時間間隔の間でタスクを実行できる頻度を、日、時、分で 指定します。d 日 h 時間 m 分例えば、0 日 1 時間 00 分 です。
    4. データの削除時間を、日、時、分で指定します 。d 日 h 時間 m 分例えば、1 日 0 時間 0 分 です。
  7. 「次へ」をクリックします。
  8. 「結果表の名前」フィールドに センサー データを保持する結果表の名前を指定します。例えば、MyResults です。
  9. 「次へ」をクリックします。
  10. 「SQL 文」フィールドに列名および定義を入力することにより、 結果表の SQL 文を完成します。例えば、taskA (task_id は 10) の戻り値を TaskA_ReturnCode 表に 収集する場合は、次のような SQL 文を入力します。 
    create table TaskA_ReturnCode(task_name varchar(20), errorcode integer)
  11. センサーが実行するコマンド ステートメントを指定します。次に例を示します。
    insert into TaskA_ReturnCode 
    values('TaskA', (select run_retcode from ph_run where run_task_id=10))
  12. 各選択項目の入力内容を確認します。
    • 入力が正しければ「終了」をクリックして確認し、 センサーを作成します。
    • 誤った値をすべて更新するために前のページに戻るには「戻る」を クリックします。

スケジューラ タスクの許容される時間間隔の変更

許容される時間間隔とは、タスクの実行が許可されるタイム フレームであり、 例えば、午前 10:00 から午後 6:00 までの間です。

  1. 「タスク スケジューラ」 > 「スケジューラ」を展開します。「タスク詳細」ページに、 スケジュールの詳細を含むタスクのリストが表示されます。
  2. 頻度またはスケジュールされた実行時間の詳細を変更するには、タスク名を クリックします。タスクを実行する日または時間を 更新できます。
  3. 時間間隔の開始時刻を 24 時間時計形式 hh:mm:ss を使用して 指定します。例えば 、10:00:00 という指定は、午前 10 時を 表します。
  4. 時間間隔の停止時刻を 24 時間時計形式 hh:mm:ss を使用して 指定します。例えば、18:00:00 という指定は、午後 6 時を表します。
  5. 許可される時間間隔の間でタスクを実行できる頻度を、日、時、分 (d Days h Hours m Minutes) で指定します (例: 0 Days 1 Hours 00 Minutes)。
  6. タスクを実行しない すべての曜日について「無効」 を選択します。
  7. 「保存」をクリックして変更を保存します。

パフォーマンス情報の探索方法

SQL Explorer を使用した問合せパフォーマンスの分析

SQL Explorer を使用するには SQL トレースをオンにする必要があります。

SQL Explorer を使用して、問合せのドリル ダウンを実行します。SQL Explorer では 、SQL トレースを使用して、システムで実行された各 SQL 文の統計情報を収集し、 文の履歴を表示します。次のセクションでは、SQL トレースをオンにする方法、および SQL Explorer を使用する方法について説明します。

SQL トレースの有効化

  1. 「パフォーマンス分析」>「SQL Explorer」にナビゲートします。SQL トレースが 実行されていない場合は、次のメッセージが表示されます。「現在 SQL トレースはオフです。 「管理」オプションを使用してオンに戻してください。
  2. 「管理」をクリックして、「SQL トレースのプロフィール」 ページにナビゲートします。
  3. 「トレース状態」リストから「オン」を選択します。
  4. 「保存」をクリックして、SQL トレースをオンにします。

SQL トレースの無効化

  1. 「パフォーマンス分析」>「SQL Explorer」にナビゲートします。SQL トレースが オンであれば、SQL Explorer 用のタブを選択できます。SQL トレースが 既にオフの場合は、次のメッセージが表示されます。「現在 SQL トレースはオフです。 「管理」オプションを使用してオンに戻してください。
  2. 「トレース管理」タブをクリックして、「SQL トレースのプロフィール」 ページにナビゲートします。
  3. 「トレース状態」リストから「オフ」を選択します。
  4. 「保存」をクリックして、SQL トレースをオフにします。

SQL トレースのオプションの指定

SQL Explorer は、現行サーバで実行された SQL 文の詳細情報を 表示します。「トレース管理」ページから、トレース対象であり 、SQL Explorer で使用できる情報を指定できます。

SQL Explorer の詳細を設定するには、次のようにします。

  1. IBM OpenAdmin Tool (OAT) for Informix のメニューで、「パフォーマンス分析」>「SQL Explorer」>「トレース管理」をクリックします。
  2. 次の SQL トレース オプションを指定します。
    • グローバルまたはユーザ トレース モードを選択します。このオプションは、IBM Informix 11.50.xC3 以降で表示されます。詳細については、『トレース モード』を参照してください。
    • 表の名前をトレース情報に含めるには「表名」 チェック ボックスを選択します。
    • プロシジャ スタックをトレース情報に含めるには「プロシジャ スタック」 チェック ボックスを選択します。
    • ホスト変数に関する情報をトレース情報に含めるには 「ホスト変数」チェック ボックスを選択します。
    • トレース オプションが変更されて保存されたときに、トレース バッファを空にするには 「トレース バッファのクリア」チェック ボックスを選択します。
    • トレースする文の数 (500 から 2147483647 まで) を入力します。
    • SQL 文ごとの最大トレース サイズをバイト単位 (512 から 102400 まで) で入力します。
  3. 「保存」をクリックします。

トレース モード

「トレース モード」オプションは、IBM Informix 11.50.xC3 以降で表示されます。

  • サーバ全体の SQL 文をトレースするには「グローバル」を選択 します。SQL Explorer では「データベースのトレース」リストおよび「 ユーザのトレース」リストで 指定されているものを除くすべてのデータベースおよびユーザ セッションの情報を表示できます。
  • データベースまたはユーザ セッションのリストに対する SQL 文をトレース するには「ユーザ」を選択します。SQL Explorer では「データベースのトレース」リストおよび「 ユーザのトレース」リストが空でない場合、これらのリストで 指定されているデータベースおよびユーザ セッションのみの情報を表示できます。
    • 「データベースのトレース」または「ユーザのトレース」のリストを表示するには、「表示」をクリックします。これらのリストを 非表示にするには「非表示」をクリックします。
    • 「データベースのトレース」リストを変更するには、次のようにします。
      • データベースの追加: 「トレース対象外データベース」リストから データベース名を選択し、「+」をクリックします。データベース名が 「トレース対象データベース」リストに表示されます。
      • データベースの削除: 「トレース対象データベース」リストから データベース名を選択し、「-」をクリックします。データベース名が 「トレース対象外データベース」リストに表示されます。
    • 「ユーザのトレース」リストを変更するには、次のようにします。
      • ユーザの追加: 「トレース対象外」リストから データベース名を選択し、「+」をクリックします。データベース名が 「トレース対象データベース」リストに表示されます。
      • ユーザの削除: 「トレース対象データベース」リストから データベース名を選択し、「-」をクリックします。データベース名が 「トレース対象外データベース」リストに表示されます。

トランザクションによるドリル ダウン

前提条件: 問合せドリル ダウンを使用するには、SQL トレースが「オン」に設定されている必要があります。 SQL トレースの有効化を参照してください。

  2. 「トランザクション」タブをクリックして、最新 100 トランザクションのトランザクション時間のリストを表示します。

SQL 文のタイプによるドリル ダウン

前提条件: 問合せドリル ダウンを使用するには、SQL トレースが「オン」に設定されている必要があります。 SQL トレースの有効化を参照してください。

  1. OAT のメニューで、「パフォーマンス分析」 > 「SQL Explorer」をクリックします。SQL Explorer のページが表示されます。システムで実行された文は、SQL 文グループごとのサマリ統計情報とともに、タイプ別にグループ化されます。
  2. 文のタイプに対するドリル ダウン アイコンをクリックします。実行された SQL 文のリストが表示されます。文に対する、次いで文の実行に対するドリル ダウン アイコンをクリックして、ドリル ダウンを継続することができます。文の統計情報が表示されます。

SQL 文の最適化

IBM OpenAdmin Tool (OAT) for Informix を使用して、外部ディレクティブを作成し SQL 文を最適化します。

前提条件:
  • データベース サーバの onconfig ファイルで、EXT_DIRECTIVES 構成パラメータに 1 または 2 を設定して、外部ディレクティブを保存できるようにします。EXT_DIRECTIVES 構成パラメータを変更した後、その変更を有効にするには、データベース サーバを再始動する必要があります。
  • SQL トレースがオンであり、「トレース管理」ページ で「表名」オプションが選択されていることを確認します。『SQL トレースの有効化』および『SQL トレースのオプションの指定』を参照してください。

SQL 文に適用する外部ディレクティブを作成できます。 ディレクティブは、問合せオプティマイザに問合せの実行方法を 指示するコメントです。外部ディレクティブは、SQL 文の問合せ実行計画 を変更する方法を提供します。 外部ディレクティブを SQL 文に適用した後、その SQL 文を 再実行するたびに、その SQL 文の問合せ実行計画が順次変更されます。

重要: SQL 文を再実行するときには、その文が、ディレクティブを適用した SQL 文と完全に一致している必要があります。例えば、大文字化されている文字が同じであり、スペーシングが同じである必要があります。

外部ディレクティブを作成するには、 次のようにします。

  1. OAT のメニューで、「パフォーマンス分析」 > 「SQL Explorer」をクリックします。
  2. SQL 文にドリル ダウンします。
    1. 「SQL」タブをクリックします。
    2. リスト内の SQL 文をクリックします。
    3. SQL 文実行のリスト内の行をクリックします。
  3. 「最適化」タブをクリックします。
  4. 「追加」をクリックします。
  5. 「ディレクティブの追加」ページで、ディレクティブを選択します。必要に応じて、表およびインデックスを選択します。SQL 文に対し複数のディレクティブを追加するには、「さらに追加」をクリックします。「追加」をクリックすると、ページが閉じます。
  6. ディレクティブの適用方法を指定するには、「適用」をクリックします。デフォルトは 「非アクティブ」であり、ディレクティブは SQL 文に 適用されません。

パフォーマンス履歴の表示

プロファイルが作成されたフィールドに関する、自動チェックポイント、ディスクとバッファの動作、メモリ、仮想プロセッサ、および B ツリー スキャナを含む履歴データを表示します。

特定のフィールドにおける長期のデータを表示し、複数のサーバでのパフォーマンスの比較を行うことができます。 例えば、サーバ上の合計ディスク読込みと合計ディスク書込みの比較、または 2 つのサーバ上の合計ディスク書込みの比較を行えます。

パフォーマンス履歴を表示するには、次のようにします。

  1. OpenAdmin Tool (OAT) for Informix のメニューで、「パフォーマンス分析」>「パフォーマンス履歴」をクリックします。
  2. 表示するフィールドを選択します。
  3. オプション: 追加のサーバを 3 つまで選択します。

システム性能レポートの実行

現在のデータと履歴データに基づいて、システム性能レポートを作成します。

システム性能レポートを実行するには、以下のようにします。

  1. IBM OpenAdmin Tool (OAT) for Informix のメニューで、「パフォーマンス分析」 > 「システム レポート」をクリックします。
  2. レポートのデータ タイプのタブ (「現行レポート (Current Reports)」または「履歴レポート (Historical Reports)」) をクリックします。

問合せの実行

例示問合せを使用して、表の特定の行を表示する 簡単な問合せを作成します。

表に対する問合せを実行してから、結果として表示された行の確認、更新、 または削除を行うことができます。

問合せを実行するには、 次のようにします。

  1. IBM OpenAdmin Tool (OAT) for Informix のメニューで、「SQL Toolbox」を展開してから、「例示問合せ」をクリックします。
  2. 表を検索するには、表名の任意の部分を「検索」フィールドに入力し、「検索」をクリックします。 データベースを展開して、表を表示することもできます。
    注: リストには、例示問合せ (QBE) を使用して作業できる表のみが表示されます。QBE で作業できない表は表示されません。例えば、システム表、つまりシステム データベース (sysadmin データベースを除く) 内の表、および QBE が対応していない型である BLOB、バイト (BYTE)、CLOB、ディスティンクト (DISTINCT) 型、リスト (LIST)、マルチセット (MULTISET)、各国語文字 (NCHAR)、各国語可変長文字 (NVARCHAR)、行 (ROW)、セット (SET)、テキスト (TEXT)、およびユーザ定義データ型 (UDT) を含む表などは表示されません。

    選択した表 に含まれる列が「問合せ」ページに表示されます。

  3. 問合せの抽出条件を入力するには、列の横のボタンをクリックして「問合せの抽出条件の入力」ページを開きます。
  4. 演算子のリストから演算子を選択し、値を入力して 「追加」をクリックします。

    問合せの抽出条件 が表に表示されます。別の条件を追加する場合は、 まず「AND」または「OR」を選択します。

  5. 「OK」をクリックします。

    ページが閉じ、問合せの抽出条件が列に表示されます。

  6. 問合せを実行するには「実行」をクリックします。

    ページに、問合せ行を一度に 1 行ずつ表示します。 問合せの抽出条件を一切入力せずに問合せを実行すると、 表のすべての行が表示されます。

    行をスクロールする には、矢印ボタンを使用します。

表の行の更新

例示問合せを使用して、データベース サーバの表の行を更新します。

注: IBM OpenAdmin Tool (OAT) for Informix で例示問合せを使用して更新できるのは、固有 ID を持つ行のみです。固有 ID には、主キー、一意性制約、一意性インデックス、および 表がフラグメント化されていない場合の行 ID が含まれます。

データベース サーバの 表の行を更新するには、次のようにします。

  1. 例示問合せを使用して問合せを実行します。『問合せの実行』を参照してください。
  2. 必要に応じて、矢印ボタンを使用し、行までスクロールします。
  3. 行の変更内容をフィールドに入力します。
  4. 「行の更新」をクリックします。

    データベース サーバで 行が更新されます。

表の行の削除

例示問合せを使用して、データベース サーバの表の行を削除します。

注: IBM OpenAdmin Tool (OAT) for Informix で例示問合せを使用して削除できるのは、固有 ID を持つ行のみです。固有 ID には、主キー、一意性制約、一意性インデックス、および 表がフラグメント化されていない場合の行 ID が含まれます。

データベース サーバの 表の行を削除するには、次のようにします。

  1. 例示問合せを使用して問合せを実行します。『問合せの実行』を参照してください。
  2. 必要に応じて、矢印ボタンを使用し、行までスクロールします。
    重要: 行を削除すると、その行は、データベース サーバで永久的に削除されます。削除を元に戻すことはできません。
  3. 「行の削除」をクリックします。

    データベース サーバで 行が削除されます。

  2. 表を検索するには、表名の任意の部分を「検索」フィールドに入力し、「検索」をクリックします。 データベースを展開して、表を表示することもできます。
    注: リストには、例示問合せ (QBE) を使用して作業できる表のみが表示されます。QBE で作業できない表は表示されません。例えば、システム表、つまりシステム データベース (sysadmin データベースを除く) 内の表、および QBE が対応していない型である バイト (BYTE)、テキスト (TEXT)、BLOB、CLOB、各国語文字 (NCHAR)、各国語可変長文字 (NVARCHAR)、セット (SET)、マルチセット (MULTISET)、リスト (LIST)、行 (ROW)、ディスティンクト (DISTINCT) 型、およびユーザ定義データ型 (UDT) を含む表などは表示されません。

    選択した表 に含まれる列が「問合せ」ページに表示されます。

  3. 「挿入」タブをクリックします。
  4. フィールドに値を入力します。
    ヒント: 各フィールドの吹出しヘルプによって、文字 (CHAR) 型、10 進数 (DECIMAL) 型、または日時 (DATETIME) 型など、必要な型が識別されます。
  5. 「行の追加」をクリックします。

    データベース サーバで の表に行が挿入されます。

非デフォルト ロケールでデータベースへアクセスする

マルチバイトのデータベース ロケールを含む、デフォルトの英語 (en_us) 以外のロケールを持つデータベースを管理します。

前提条件:
  • IBM OpenAdmin Tool (OAT) for Informix には、OAT を介してアクセスするすべてのデータベースに使用する UTF-8 ロケールが必要です。IBM Informix Client Software Development Kit (Client SDK) に組み込まれていないロケールを使用してデータベースにアクセスするには、IBM Informix International Language Supplement (ILS) を使用してロケールをインストールします。OAT をインストールしたら、OAT がインストールされているマシンの Informix Connect または Client SDK ディレクトリに ILS をインストールします。 ILS を使用して、OAT で使用されている Informix Connect または Client SDK ディレクトリに追加のロケール (UTF-8 バージョンなど) をインストールします。
  • IBM Informix Replication Plug-in for OpenAdmin Tool (OAT) には Informix 11.70.xC4 が必要です。
  • データベース名には英文字を使用する必要があります。

Informix でサポートされるデータベース ロケールのデータは、OAT を使用して表示および変更できます。 例えば、イタリア語または日本語のロケールがあるデータベースにアクセスすることができます。

Replication Plug-in でデフォルト以外のロケールを使用してデータベースを表示する場合は、「言語」リストおよび「ロケール」リストを使用して適切なデータベース ロケールを選択します。

読取り専用グループの制約事項

IBM OpenAdmin Tool (OAT) for Informix の読取り専用グループには、以下の制約事項があります。

  • 「ヘルス センタ」:
    • アラートを表示することはできますが、表示するアラートを変更することはできません。
    • Informix Health Advisor Plug-in for OAT を使用して、要求時に現行プロファイルに対して Health Advisor を実行することはできません。 プロファイルをロードすることはできますが、プロファイルを追加したり削除したりすることはできません。プロファイルのアラームを確認することはできますが、アラームを有効化、無効化、または変更することはできません。「スケジュール」ページまたは「通知」ページにアクセスすることはできません。Health Advisor Plug-in には Informix 11.50.C7 以降が必要です。
  • 「ログ」: 制限はありません。
  • 「タスク スケジューラ」: スケジューラ タスクの追加、変更、または削除を行うことは できません。
  • 「スペース管理」
    • 格納領域および領域使用情報を表示できますが、領域およびチャンクを追加、変更、削除することはできません。
    • ストレージ プール エントリの追加、変更、削除はできません。ストレージ プールを使用するには、OAT に IBM Informix バージョン 11.70 が必要です。
    • 「リカバリ ログ」ページから「管理」ページ や「リカバリ ポリシー」ページにアクセスすることはできません。
    • 領域最適化情報を表示することはできますが、表、インデックス、フラグメントを圧縮、リパック、縮小したり、表、フラグメントを圧縮解除したりすることはできません。圧縮見積りを更新できません。 インデックスを圧縮するには、OAT に Informix 12.10 が必要です。
    • 表、フラグメント、またはインデックスのフラグメント解消はできません。フラグメント解消の操作を実行するには、OAT に Informix 11.70 以降が必要です。
  • 「サーバ管理」

「管理」構成のページに対するパスワード保護の提供: Apache の例

セキュリティのために、IBM OpenAdmin Tool (OAT) for Informix の「管理」ページへのアクセスを制限します。「管理」のページは、 「ログイン」ページからアクセスでき、したがってすべてのユーザに 公開されているため、これらのページのパスワード保護が重要です。

これを行う方法の 1 つとして、Web サーバが提供する標準装備の認証機構を使用する 方法があります。例えば、インストール ディレクトリ OATINSTALL/admin/.htaccess を保護できます。

例: Apache 内から OAT ディレクトリを保護するには、次のようにします。

  1. Apache に付属の htpasswd ユーティリティを使用して、Web からアクセスできない ディレクトリにパスワード ファイルを作成します。
  2. 次のいずれかのファイルに認証ディレクティブを追加します。
    • OATINSTALL/admin/.htaccess ファイル
    • httpd.conf ファイル
  3. ディレクティブは、<Directory> セクション内に配置します。次に例を示します。
    <Directory OATINSTALL/admin>
AuthType Basic
AuthName "OAT Admin"
AuthUserFile PASSWORDFILE
Require ValidUser
</Directory>
    ここで、PASSWORDFILE は 、htpasswd ユーティリティを使用してパスワード ファイルを作成したディレクトリです。

詳細情報および例については、Apache Software Foundation の Web サイト www.apache.org (新しいウィンドウで開く) で Apache HTTP サーバ バージョン 2.0 の認証、許可、およびアクセス コントロールに関する記事を参照してください。

OAT 用 HTTPS の構成 (Linux)

HTTPS を使用すると、IBM OpenAdmin Tool (OAT) for Informix の Web サーバを保護でき、Web サーバでの盗聴、改ざん、メッセージ偽造、およびネットワーク妨害を防ぐことができます。

前提条件: OAT 用 Apache サーバがインストールされているコンピュータに、OpenSSL をインストールする必要があります。OpenSSL は、OpenSSL Project Web サイト www.openssl.org からダウンロードしインストールできます。

HTTPS を有効化すると、OAT クライアントからのメッセージは、暗号化されてから OAT Web サーバに送信されます。暗号化することで、無許可ユーザは、回線越しに聴取して機密情報を盗み出すことができなくなります。HTTPS を有効化すると、OAT クライアントは OAT ホストによる認証も行うことができるようになるため、無許可ユーザが OAT クライアントを欺いて不正な Web サーバを使用することができなくなります。

重要: HTTPS では、OAT Web サーバとクライアント間の通信のみが暗号化されます。IBM Informix データベース サーバ と Web サーバ間の通信は暗号化されません。 IBM Informix バージョン 9.4 以降では、暗号通信サポート モジュールを使用して、Informix サーバと OAT Web サーバ間のデータを暗号化できます。暗号化と Informix サーバの詳細については、developerWorks® の Web サイトで『Encryption over the wire with IDS 9.40』を参照してください。

IBM Informix Client Software Development Kit (Client SDK) とともに OAT をインストールした場合は、インストール プログラムによって、OAT を実行するために必要なソフトウェア (事前構成された Apache、PHP、PDO_informix など) が提供されます。Linux では、OAT とともにインストールされた Apache Web サーバは、mod_ssl 暗号化モジュール対応のサーバです。このモジュールは、HTTPS を有効化するために必要です。

Linux 上の OAT で HTTPS を有効化するには、以下の手順を実行します。

  1. OAT クライアントがユーザ独自の証明書に基づいて Web サーバを認証できるようにするため、Web サーバの暗号化キーおよび証明書を作成します。
  2. HTTPS を有効化するように Apache を構成します。

HTTPS の有効化の詳細については、developerWorks Web サイトで『Securing the OpenAdmin Tool for Informix web server with HTTPS』を参照してください。

暗号化キーの作成

キーは暗号化と暗号解読で使用されます。キーは通常、パブリック キーとプライベート キーのペアになっています。パブリック キーはメッセージの暗号化に使用され、プライベート キーはメッセージの暗号解読に使用されます。証明書は、Web サーバを認証する文書です。HTTPS 対応の Web サーバには、信頼できる認証局の署名が付いた証明書があり、これによって Web サーバが検証されます。Web サーバが HTTPS 対応になったら、クライアントは、Web サーバに対して標準接続とセキュア接続のどちらを確立するかを選択します。標準接続を確立する場合は、クライアントは Web ブラウザに http://web server_url と入力します。セキュア接続を確立する場合は、クライアントは https://web server_url と入力します。

プライベート キーとパブリック キーのペアおよび証明書を生成するには、OpenSSL のインストール先の bin ディレクトリに含まれている openssl 実行可能ファイルを使用します。

このコマンドを実行してプライベート キーを生成すると、プライベート キーを保護するためのパスフレーズ (パスワード) を入力するように求められます。プライベート キーの使用が求められるコマンドを実行するときには必ず、このパスフレーズを再入力する必要があります。例えば、Apache Web サーバを起動するとき、またはプライベート キーを使用して証明書を生成するときに、パスフレーズを入力する必要があります。Linux では、セキュリティ上の理由から、パスフレーズを使用して、プライベート キーを保護し暗号化します。

プライベート キーを生成するには、OpenSSL がインストールされているコンピュータで次のコマンドを実行します。

openssl genrsa -des3 -out privkey.pem 2048

プライベート キーが privkey.pem ファイル内に作成されます。 このファイルには Web サーバの暗号解読キーが含まれているため、安全な場所に保管してください。証明書を生成するときには、関連付けられたパブリック キーを生成するために、このファイルが使用されます。パブリック キーは証明書に組み込まれます。

証明書の作成

プライベート キーを作成したら、Web サーバの ID を認証するための証明書を作成します。証明書を生成するには、証明書署名要求を作成し、その要求を信頼できる認証局に送信します。その認証局から証明書が発行されます。

証明書署名要求を生成するには、次のコマンドを実行します。

openssl req -new -key privkey.pem -out cert.csr

証明書署名要求の詳細については、認証局に問い合わせてください。

また、認証局を利用せずに自己署名証明書を作成することもできます。ただし、自己署名証明書は推奨されません。

自己署名証明書を生成するには、次のコマンドを実行します。

openssl req -new -x509 -key privkey.pem -out cacert.pem -days 1095

OpenSSL から、情報を入力するように求められます。証明書は cacert.pem ファイルに保管されます。 このファイルは、ID を検証するために Web クライアントに向けて表示されます。このファイルには、Web クライアントのパブリック キーが含まれているため、通信を暗号化できます。

暗号化キーの詳細については、OpenSSL の資料を参照してください。

HTTPS を有効化するための Apache の構成

HTTPS を有効化するには、Apache 構成ファイルおよび Apache SSL 構成ファイルを更新します。

  1. Apache 構成ファイルを見つけ出します。パスには、以下の例のように、Apache のバージョンが含まれています。 
    /oat/installation/directory/Apache_2.4.2/conf/httpd.conf
  2. Apache 構成ファイルで、#Include conf/extra/httpd-ssl.conf という行からコメントを削除します。

    この行は、デフォルトでコメント化されています。このコメントを削除すると、Apache SSL 構成ファイルが httpd.conf ファイルに組み込まれます。

  3. Apache SSL 構成ファイルを見つけ出します。パスには、以下の例のように、Apache のバージョンが含まれています。 
    /oat/installation/directory/Apache_2.4.2/conf/extra/httpd-ssl.conf
  4. Apache SSL 構成ファイルで、SSL ポート番号が使用可能なポートに設定されていることを確認します。

    HTTPS には個別の SSL ポートが必要です。デフォルトでは、SSL ポート番号は 443 に設定されます。このポートが使用不可である場合は、Listen ディレクティブおよび仮想ホスト セクションで SSL ポートを変更します。

  5. Apache SSL 構成ファイルで、SSLCertificateKeyFile ディレクティブが、作成したプライベート キー ファイル (privkey.pem) の正しい場所を示していることを確認します。
  6. SSLCertificateFile ディレクティブが、作成した証明書ファイル (cacert.pem) の正しい場所を示していることを確認します。
  7. オプション: Web サーバが受け入れる暗号を制御するには、SSLCipherSuite ディレクティブを編集します。SSLCipherSuite ディレクティブについては、Apache mod_ssl の資料を参照してください。

HTTPS 構成の詳細については、Apache Web サイト www.apache.org を参照してください。

構成のテスト

HTTPS の変更を有効にするには、Web サーバを再起動する必要があります。Web サーバを再起動するため、OAT のインストール先のディレクトリに含まれている StopApache スクリプトを実行してから、StartApache スクリプトを実行します。パスフレーズを入力するように求められます。これを入力しないと、Web サーバを起動できません。

Web サーバを再起動したら、Web ブラウザに URL https://hostname:ssl_portnumber/openadmin を入力して OAT を開きます。 OAT ログイン ページが表示される前に、OAT Web サーバの証明書を表示して受け入れるように求められます。

OAT Apache Web サーバの置換

OAT Apache Web サーバを置き換えるには、次のようにします。

  1. Apache Web サイト httpd.apache.org から Apache HTTP Server をダウンロードします。 OpenSSL 0.9.8t (MSI インストーラ) が含まれている Win32 Binary の Apache 2.2.x バージョン (使用可能な最新版) を選択します。
  2. OAT Apache Web サーバを停止します。「スタート」メニューに、OAT のメニュー項目「IBM OpenAdmin Tool for Informix」があります。そこから OAT Apache Web サーバを停止するには、「Apache Service for OpenAdmin Tool の停止 (Stop Apache Service for OpenAdmin Tool)」をクリックします。さらに、システム トレイで Apache Monitor が実行されていないことを確認します。
  3. OAT の Apache_2.2.22 ディレクトリの名前を Apache_2.2.22_noSSL に変更して、Apache バイナリ ファイルのバックアップ コピーを作成します。この Apache ディレクトリ内の構成ファイルの一部を、後述の手順で使用します。
  4. Apache MSI インストーラを実行します。標準インストールを実行し、インストール ディレクトリを /oat/installation/directory/Apache_2.2.22 に設定します。
    重要: 2.2.22 より新しいバージョンの Apache をインストールする場合でも、ディレクトリ名を Apache_2.2.22 にします。OAT のインストールとそのサービスが、ディレクトリ名 Apache_2.2.22 を使用するように構成されています。 これらは、新しい Apache インストールのディレクトリ名が異なる場合は機能しません。
  5. Apache 構成ファイル /oat/installation/directory/Apache_2.2.22/conf/httpd.conf を編集します。 この httpd.conf ファイルに以下の行を追加するか、またはコメントを外します。 
    LoadModule php5_module
"c:\oat\installation\dir\PHP_5.2.4\php5apache2_2.dll"
LoadModule ssl_module modules/mod_ssl.so
AddType application/x-httpd-php .php
PhpIniDir 'c:\oat\installation\dir\PHP_5.2.4'
  6. httpd.conf ファイルで、Listen 80 (または Listen 8080) という行を更新します。 この行は、OAT Web サーバのポート番号を示しています。80 または 8080 を、OAT とともにインストールされた元の OAT Apache Web サーバと同じポート番号に置き換えます。
  7. httpd.conf ファイルで、ServerName www.example.com:80 という行を更新します。この行は、サーバがそれ自体を識別するために使用する名前とポートを示しています。この行を、元の OAT Apache 構成ファイル c:\oat\installation\dir\Apache_2.2.22_noSSL\conf\httpd.conf の ServerName 行全体に置き換えます。
  8. httpd.conf ファイルで、DirectoryIndex index.html という行を更新します。この行は、ディレクトリが要求された場合に Apache から提供されるファイルを設定します。この行を DirectoryIndex index.html index.php に変更します。
  9. 元の OAT Apache 構成ファイル c:\oat\installation\dir\Apache_2.2.22_noSSL\conf\httpd.conf で、setenv INFORMIXDIR という行を見つけ出します。この行は、OAT の Apache 環境の INFORMIXDIR 変数を設定します。新しい Apache Web サーバの変数を設定するには、この行を新しい Apache Web サーバの httpd.conf ファイルにコピーします。このファイルの終わりに行を追加できます。
  10. c:\oat\installation\dir\Apache_2.2.22_noSSL\htdocs\openadmin\ ディレクトリ全体を c:\oat\installation\dir\Apache_2.2.22\htdocs\openadmin\ にコピーします。 すべての OAT ソース コードがこのディレクトリに含まれています。
  11. コマンド プロンプトで以下のコマンドを実行して、PHP Apache ハンドラおよび mod_ssl モジュールが正常にロードされていることを確認します。 
    cd c:\oat\installation\dir\Apache_2.2.22\bin\
httpd.exe –M
    このコマンドにより、Apache モジュールのリストが表示されます。 php5_module および ssl_module がリストに含まれていることを確認します。 これで、新しい Web サーバが OAT 用に正しくセットアップされていることを確認できました。
  12. 「スタート」メニューで「IBM OpenAdmin Tool for Informix」をクリックしてから、「Apache Service for OpenAdmin Tool の停止 (Start Apache Service for OpenAdmin Tool)」をクリックします。Web ブラウザを使用して OAT にアクセスできます。Web サーバは mod_ssl 対応ですが、HTTPS はまだオンに切り替えられていません。

暗号化キーの作成

キーは暗号化と暗号解読で使用されます。キーは通常、パブリック キーとプライベート キーのペアになっています。パブリック キーはメッセージの暗号化に使用され、プライベート キーはメッセージの暗号解読に使用されます。証明書は、Web サーバを認証する文書です。HTTPS 対応の Web サーバには、信頼できる認証局の署名が付いた証明書があり、これによって Web サーバが検証されます。Web サーバが HTTPS 対応になったら、クライアントは、Web サーバに対して標準接続とセキュア接続のどちらを確立するかを選択します。標準接続を確立する場合は、クライアントは Web ブラウザに http://web server_url と入力します。セキュア接続を確立する場合は、クライアントは https://web server_url と入力します。

プライベート キーとパブリック キーのペアおよび証明書を生成するには、新しい Apache Web サーバーのインストール先の bin ディレクトリに含まれている openssl 実行可能ファイルを使用します。

このコマンドを実行してプライベート キーを生成すると、プライベート キーを保護するためのパスフレーズ (パスワード) を入力するように求められます。Windows では、Apache はプライベート キーのパスフレーズをサポートしていないため、以下のステップで説明しているようにパスフレーズを削除する必要があります。

プライベート キーを作成するには、次のようにします。

  1. プライベート キーを生成するには、新しい Apache Web サーバがインストールされているコンピュータで次のコマンドを実行します。
    openssl genrsa -des3 -out privkey.pem 2048
  2. パスフレーズを削除するには、次のコマンドを実行します。 
    openssl rsa -in privkey.pem -out privkey_nopassphrase.pem

privkey_nopassphrase.pem ファイル内に、暗号化されていないプライベート キーが作成されます。 このファイルには暗号化されていない形式のプライベート キーが含まれているため、安全な場所に保管してください。証明書を生成するときには、関連付けられたパブリック キーを生成するために、このファイルが使用されます。パブリック キーは証明書に組み込まれます。

証明書の作成

プライベート キーを作成したら、Web サーバの ID を認証するための証明書を作成します。証明書を作成するには、証明書署名要求を生成し、その要求を信頼できる認証局に送信します。その認証局から証明書が発行されます。

証明書署名要求を生成するには、次のコマンドを実行します。

openssl req -new -key privkey_nopassphrase.pem -out cert.csr

証明書署名要求の詳細については、認証局に問い合わせてください。

また、認証局を利用せずに自己署名証明書を作成することもできます。ただし、自己署名証明書は推奨されません。

自己署名証明書を生成するには、次のコマンドを実行します。

openssl req -new -x509 -key privkey_nopassphrase.pem -out cacert.pem -days 1095

OpenSSL から、情報を入力するように求められます。証明書は cacert.pem ファイルに保管されます。 このファイルは、ID を検証するために Web クライアントに向けて表示されます。このファイルには、Web クライアントのパブリック キーが含まれているため、通信を暗号化できます。

暗号化キーの詳細については、OpenSSL の資料を参照してください。

HTTPS を有効化するための Apache の構成

HTTPS を有効化するには、Apache 構成ファイルおよび Apache SSL 構成ファイルを更新します。

  1. Apache 構成ファイルを見つけ出します。パスには、以下の例のように、Apache のバージョンが含まれています。 
    /oat/installation/directory/Apache_2.2.22/conf/httpd.conf
  2. Apache 構成ファイルで、#Include conf/extra/httpd-ssl.conf という行からコメントを削除します。

    この行は、デフォルトでコメント化されています。このコメントを削除すると、Apache SSL 構成ファイルが httpd.conf ファイルに組み込まれます。

  3. Apache SSL 構成ファイルを見つけ出します。パスには、以下の例のように、Apache のバージョンが含まれています。 
    /oat/installation/directory/Apache_2.2.22/conf/extra/httpd-ssl.conf
  4. Apache SSL 構成ファイルで、SSL ポート番号が使用可能なポートに設定されていることを確認します。

    HTTPS には個別の SSL ポートが必要です。デフォルトでは、SSL ポート番号は 443 に設定されます。このポートが使用不可である場合は、Listen ディレクティブおよび仮想ホスト セクションで SSL ポートを変更します。

  5. Apache SSL 構成ファイルで、SSLCertificateKeyFile ディレクティブが、作成したプライベート キー ファイル (privkey_nopassphrase.pem) の正しい場所を示していることを確認します。
  6. SSLCertificateFile ディレクティブが、作成した証明書ファイル (cacert.pem) の正しい場所を示していることを確認します。
  7. オプション: Web サーバが受け入れる暗号を制御するには、SSLCipherSuite ディレクティブを編集します。SSLCipherSuite ディレクティブについては、Apache mod_ssl の資料を参照してください。

HTTPS 構成の詳細については、Apache Web サイト www.apache.org を参照してください。

構成のテスト

HTTPS の変更を有効にするには、Web サーバを再起動する必要があります。apachemonitor.exe または「スタート」メニューのショートカットを使用して、Web サーバを再起動します。

Web サーバを再起動したら、Web ブラウザに URL https://hostname:ssl_portnumber/openadmin を入力して OAT を開きます。 OAT ログイン ページが表示される前に、OAT Web サーバの証明書を表示して受け入れるように求められます。
