Informix TimeSeries Plug-in for OAT

The IBM® Informix® TimeSeries Plug-in for OpenAdmin Tool (OAT) provides a graphical interface for administering database objects that are related to a time series, including TimeSeries subtypes, containers, and calendars.

Prerequisite:
For Informix 11.50 and earlier, the IBM Informix TimeSeries DataBlade® Module must be installed and registered in the database.

A time series is a set of data that is recorded as it varies over time. Each entry is time stamped and stored in the database. The Informix TimeSeries extension provides the TimeSeries data type, containers, and calendars for managing time series data.

With the TimeSeries plug-in, you can monitor the database objects related to your time series:

  • View the TimeSeries subtypes, containers, and calendars that are used for the time series data in a database.
  • View the tables and indexes that contain TimeSeries subtypes.
  • View the virtual tables that are based on tables that contain TimeSeries subtypes.
  • Monitor the percentage of the space that is used in the containers and in the dbspaces for the containers.

You can also create and drop containers, calendars, and virtual tables.

Reviewing time series

Review database objects that are related to a time series, including tables, indexes, and virtual tables, and TimeSeries subtypes, containers, and calendars.

To review time series:

On the menu of the IBM OpenAdmin Tool (OAT) for Informix, click SQL ToolBox > TimeSeries. A list of the databases and tables in the database server that contain time series data is displayed.

  • To view information about a database or a table, click it in the list.
  • To view more information about a specific database object or table object, click the expand icon on the object panel.

Creating a container

Create a container for a time series. Create multiple containers by specifying more than one container per dbspace or by selecting more than one dbspace for the containers.

To create a container:

  1. On the menu of the IBM OpenAdmin Tool (OAT) for Informix, click SQL ToolBox > TimeSeries.
  2. Click the database that contains the time series.
  3. On the Actions menu, click Create Container.
  4. Complete the fields on the page.

Creating a calendar

Create a calendar to associate with a time series.

Every time series is associated with a calendar. A calendar defines a set of valid times at which the time series can record data; it determines when and how often entries are accepted. A calendar consists of a start date, a pattern, and an interval. Consider this example: the pattern start date is a Sunday; the pattern is 1 off, 5 on, 1 off; and the interval is a day. For this pattern, the information is not gathered the first day, Sunday. Information is gathered everyday for the next five days, Monday through Friday. Information is not gathered the last day, Saturday.

To create a calendar:

  1. On the menu of the IBM OpenAdmin Tool (OAT) for Informix, click SQL ToolBox > TimeSeries.
  2. Click the database that contains the time series.
  3. On the Actions menu, click Create Calendar.
  4. Complete the fields on the page.

Creating a virtual table

Create a virtual table that is based on a table that contains a TimeSeries subtype.

A virtual table is a relational table with the same schema as the base table, except for the TimeSeries column. The TimeSeries column is replaced with the columns of the TimeSeries subtype. The elements in the TimeSeries column are individual columns in the virtual table. The virtual table contains all the data that is in the base table, including the data that is encapsulated in the TimeSeries subtype.

To create a virtual table:

  1. On the menu of the IBM OpenAdmin Tool (OAT) for Informix, click SQL ToolBox > TimeSeries.
  2. In the list, expand the database that contains the base table and click the table.
  3. On the Actions menu, click Create Virtual Table.
  4. Complete the fields on the page.
