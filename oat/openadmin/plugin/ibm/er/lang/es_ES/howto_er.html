Plug-in de Informix Replication para OAT

El plug-in de IBM® Informix® Replication para OpenAdmin Tool (OAT) proporciona una interfaz gráfica para configurar, supervisar y administrar Enterprise Replication (ER). También puede administrar cuadrículas de servidores de réplica interconectados.

Requisitos previos:
Se necesita IBM Informix 11.50.xC4 o posteriores para el plug-in de Replication. Para Connection Manager, IBM OpenAdmin Tool (OAT) para Informix necesita Informix 11.70.xC3 y el Connection Manager que está empaquetado con Informix 11.70.xC3 o con IBM Informix Client Software Development Kit (Client SDK) 3.70.xC3

Con el plug-in de Replication, puede supervisar los dominios y servidores ER:

  • Visualizar una topología del dominio ER y todos los nodos conocidos.
  • Visualizar información de resumen y detallada sobre los nodos del dominio.
  • Visualizar información detallada sobre las réplicas y conjuntos de réplicas del dominio.
  • Establecer umbrales de alerta para el dominio.

Puede completar tareas administrativas de ER:

  • Definir y modificar servidores ER, réplicas y conjuntos de réplicas.
  • Utilizar plantillas para configurar ER.
  • Gestionar el estado de servidores, conjuntos de réplicas y réplicas.
  • Volver a sincronizar datos entre servidores de réplica.
  • Modificar los parámetros de configuración de ER de un servidor.

Puede crear y administrar cuadrículas de servidores de réplica interconectados de un dominio:

  • Visualizar información sobre las cuadrículas de un dominio.
  • Crear una cuadrícula y habilitar servidores de origen y usuarios para ejecutar los mandatos de cuadrícula.
  • Modificar una cuadrícula para añadir servidores, habilitar o inhabilitar servidores de origen o autorizar a usuarios adicionales.
  • Revisar el estado de los mandatos de cuadrícula y volver a ejecutar mandatos de cuadrícula que han fallado.
  • Direccionar conexiones de cliente a servidores de la cuadrícula en función de la calidad de los datos replicados configurando acuerdos de nivel de servicio (SLA) de un Gestor de conexiones.

Puede definir un Gestor de conexiones para gestionar solicitudes de conexión de cliente para una combinación de clústeres, cuadrículas, conjuntos de réplicas y conjuntos de servidores. Puede añadir unidades de conexión a un Gestor de conexiones, iniciar y detener un Gestor de conexiones y crear acuerdos de nivel de servicio (SLA).

Configuración del plug-in de Replication

Configure el plug-in de IBM Informix Replication para OpenAdmin Tool (OAT) para supervisar y administrar Enterprise Replication (ER).

Para configurar el plug-in de Replication:

  1. Instale el plug-in de Replication.
  2. Cree grupos OAT.
  3. Inicie la sesión en OAT con uno de los servidores en un dominio ER existente o cualquier servidor que desee definir para Enterprise Replication.

    Para obtener más información sobre cómo instalar el plug-in de Replication y crear grupos de OAT, consulte la ayuda de OAT. Para acceder a la ayuda, en el menú de OAT, pulse Ayuda > Ayuda de tarea de OAT.

Supervisión de Enterprise Replication

Utilice el plug-in de IBM Informix Replication para OpenAdmin Tool (OAT) para supervisar dominios y servidores de Enterprise Replication.

Visualización de la topología de un dominio ER

Ver la topología o el mapa de direccionamiento de un dominio de Enterprise Replication (ER).

Para ver la topología del dominio actual, en el menú para IBM OpenAdmin Tool (OAT) para Informix, pulse Replication > Domino de ER.

La página Dominio muestra todos los nodos del dominio ER conocidos por el nodo actual; por ejemplo, un nodo hoja se conoce a sí mismo y a su padre. Los nodos se indican con el nombre de grupo definido en el archivo SQLHOSTS.

Importante:
Si está conectado un nodo no raíz o a un nodo hoja, la información que puede ver y las acciones que realizar están limitadas. Para tener acceso completo, cambie a un nodo raíz del dominio.

En la página Dominio, puede:

  • Determinar si algún nodo necesita atención revisando cualquier alerta visual.
  • Ver el nombre del nodo y de cualquier miembro del nodo pasando el puntero del ratón sobre el icono del nodo.
  • Visualizar u ocultar el subárbol de cualquier nodo no hoja pulsando el icono de ampliación.
  • Personalizar su vista:
    • Para aumentar o disminuir el tamaño del dominio, utilice la barra de zoom.
    • Para ver la tasa de renovación, pulse el icono de reloj.
    • Para cambiar la tasa de renovación, pulse el icono de reloj y utilice la barra de desplazamiento para especificar una tasa de renovación entre 0 y 300 segundos.
    • Para ver las líneas de conexión entre nodos, pulse el icono Vista normal.
    • Para ocultar las líneas de conexión, pulse el icono Sin vista de conexión.

Para ver los servidores del dominio en una lista, pulse la pestaña Lista de servidores. Para cada servidor, la lista visualiza el estado, los servidores de miembros, el tipo, el padre (si es aplicable), la versión de IBM Informix y la fecha y hora en que se ha supervisado el servidor por última vez.

Configuración de umbrales de alerta

Configurar los umbrales para las alertas para un dominio de Enterprise Replication (ER).

Cuando se alcanza un umbral de alerta, se envía una notificación al administrador y aparece un icono de alerta en la página Dominio. Los umbrales de alerta son aplicables a todos los nodos de un dominio ER.

Para configurar umbrales:

  1. En el menú para IBM OpenAdmin Tool (OAT) para Informix, pulse Replication > Domino de ER.
  2. En el menú Acciones, pulse Editar umbrales.
  3. Para activar un umbral, seleccione el recuadro de selección Habilitar para el umbral. Para desactivar el umbral, deseleccione el recuadro de selección.
  4. Introduzca o cambie los valores de umbral en el campo.
  5. Pulse Actualizar umbrales para guardar los cambios.

Comprobación de alertas en un dominio de Enterprise Replication

Comprobar si hay alertas en los nodos en un dominio de Enterprise Replication (ER).

Se crea una alerta cuando un nodo alcanza o supera un umbral.

Para comprobar las alertas:

  1. En el menú para IBM OpenAdmin Tool (OAT) para Informix, pulse Replication > Domino de ER.

    Se resaltan los nodos con alertas.

  2. Pulse el nodo y, a continuación, el menú Acciones, pulse Estado del nodo.

    Una página visualiza los datos de perfil y las alertas para el nodo.

  3. Para revisar la actividad de réplica del nodo seleccionado, en el menú Acciones pulse Detalles del nodo.

    La página Resumen muestra estadísticas de alto nivel para el nodo seleccionado. Pulse las fichas para obtener información más detallada sobre el nodo:

    • Capturar: actividad de captura de datos en el nodo seleccionado.
    • Cola de envío: la cola de envío para el nodo seleccionado y sus nodos y duplicados de destino.
    • Red: la actividad de la red entre el nodo seleccionado y sus nodos de ER asociados.
    • Uso de disco: los espacios utilizados por el nodo seleccionado cuando las se rellenan las colas de envió y recepción y las transacciones se colocan en spool en el disco.
    • Aplicar: las transacciones duplicadas aplicadas en el nodo actual.
    • ATS: los archivos de ATS (Spooling de transacciones terminadas anormalmente) para el nodo seleccionado. Puede reparar las transacciones anómalas utilizando los archivos ATS de esta página.
    • RIS: Los archivos RIS (Row Information Spooling - Spooling de información de filas) para el nodo seleccionado. Puede reparar las transacciones anómalas utilizando los archivos RIS de esta página.
    • Errores: los errores relacionados con el nodo seleccionado o el dominio.
    • Configuración: los parámetros de configuración ER definidos para el nodo seleccionado. Puede editar los parámetros desde esta página.

Visualización del estado de un nodo de Enterprise Replication

Ver el estado de un nodo en un dominio de Enterprise Replication (ER).

Puede ver el estado de un nodo en un dominio ER de los siguientes modos:

  • Para ver el nodo al que está conectado IBM OpenAdmin Tool (OAT) para Informix, en el menú de OAT, pulse Replication > Dominio de ER. Se mostrarán las páginas detalladas para el nodo.
  • Para ver un nodo distinto del dominio, pulse el nodo. A continuación, en el menú Acciones, pulse Estado del nodo. OAT se conecta con el nodo seleccionado del domino, y aparecen las páginas detalladas del nodo.

Puede ver información detallada sobre las transacciones y la cola de envío en estas páginas de IBM OpenAdmin Tool (OAT) para Informix: Replication > Detalles de nodo > Resumen, Capturar y Cola de envío.

Visualización del nodo como destino de réplica

Ver un nodo en un dominio de Enterprise Replication (ER) como el servidor de destino.

La cola de recepción es la zona de espera para transacciones que se van a aplicar en el nodo de la base de datos actual. ER aplica transacciones a las bases de datos correctas y lo notifica al nodo de origen cuando la transacción se duplica correctamente.

Puede ver información detallada sobre las transacciones y la cola de envío en estas páginas de IBM OpenAdmin Tool (OAT) para Informix: Replication > Detalles de nodo > Resumen, Cola de envío, Aplicar, ATS y RIS.

Supervisión de información de captura de registro para un nodo

Ver información detallada sobre la actividad de captura de registro en el nodo actual en un dominio de Enterprise Replication (ER).

En el menú de IBM OpenAdmin Tool (OAT) para Informix, pulse Replication > Detalles de nodo > Capturar. La página Capturar visualiza la información siguiente:

  • El estado de la captura de la anotación cronológica ER en el nodo actual: Running (Ejecución), Down (Inactivo) o Uninitialized (Sin inicializar).
  • La posición de la anotación cronológica actual en la que se capturan las transacciones para la réplica. Compare las posiciones de los registros para determinar si ER marcha al mismo ritmo que las transacciones de base de datos que están en marcha.
  • La posición de reproducción de registro, que es el punto desde el que ER empieza a releer la información de registro en los almacenamientos intermedios de actualización de registro después de una anomalía. ER ha aplicado todas las transacciones generadas antes de esta posición en todos de los nodos de destino o dichas transacciones se han guardado en el espacio de la cola estable.
    Importante:
    Para evitar la pérdida de datos, asegúrese de que nunca se sobrescriba la posición de reproducción de la anotación cronológica. Si no avanza la posición de reproducción, es posible que la cola esté llena o que un nodo remoto esté inactivo.
  • Número total de páginas de registro y número de páginas hasta que ER entra en un estado de bloqueo de transacciones, conocido como modalidad DDRBLOCK.
  • Número de páginas hasta que la anotación cronológica lógica se recorta y sobrescribe las transacciones que ER no ha procesado aún.
  • Número de páginas de registro que se han leído durante la captura de registro, desde la memoria caché de captura de registro y desde las entradas de registro en spool en disco, y si hay desbordamientos de memoria caché de registro.
  • Si está habilitado el registro dinámico para evitar la modalidad DDRBLOCK y la información relacionada, incluido el número de registro lógicas dinámicas que se pueden solicitar y el número de solicitudes realizadas para crear registros lógicos dinámicos.

Prevención de DDRBLOCK y de reinicio de anotación cronológica

Evitar que Enterprise Replication (ER) entre en un estado de bloqueo de transacción, conocido como modalidad DDRBLOCK.

Cuando una anotación cronológica lógica se recorta, sobrescribe las transacciones que ER no ha procesado todavía. Las transacciones se pierden antes de que se almacenen en la cola de envío. Si un servidor de base de datos se acerca al recorte de anotaciones cronológicas, ER entra en la modalidad DDRBLOCK. ER se puede configurar para que añada un archivo de anotación cronológica de forma dinámica en lugar de entrar en la modalidad DDRBLOCK.

  • Para supervisar la proximidad a DDRBLOCK, en el menú de IBM OpenAdmin Tool (OAT) para Informix, pulse Replication > Detalles de nodo > Capturar. Revise el campo Proximidad a DDRBLOCK, que muestra el número de páginas del espacio de anotaciones cronológicas lógicas hasta que ER entra en DDRBLOCK.
  • Para evitar la modalidad DDRBLOCK, configure ER para que añada archivos de anotaciones cronológicas lógicas de forma dinámica. En el menú de OAT, pulse Replication > Detalles de nodo > Configurar y establezca los parámetros DYNAMIC_LOGS y CDR_MAX_DYNAMIC_LOGS.
  • Para supervisar la proximidad al recorte de registro, en el menú de OAT, pulse Replication > Detalles de nodo > Capturar. Revise el campo Proximidad al recorte del registro, que muestra el número de páginas del espacio de anotaciones cronológicas lógicas hasta que la anotación cronológica lógica se recorta.

Activación de la asignación de archivos de anotaciones cronológicas dinámicas

Ayudar a prevenir que Enterprise Replication (ER) entre en un estado de bloqueo de transacción, conocido como modalidad DDRBLOCK, configurando ER para que añada archivos de registro lógico de forma dinámica.

Cuando un registro lógico se recorta, sobrescribe las transacciones que ER no ha procesado todavía. Las transacciones se pierden antes de que se almacenen en la cola de envío. Si un servidor de base de datos se acerca al recorte de anotaciones cronológicas, ER entra en la modalidad DDRBLOCK. ER se puede configurar para que añada un archivo de anotación cronológica de forma dinámica en lugar de entrar en la modalidad DDRBLOCK.

Para activar la asignación de archivos de anotaciones cronológicas dinámicos:

  1. En el menú de IBM OpenAdmin Tool (OAT) para Informix, pulse Replication > Detalles de nodo > Configurar.
  2. Pulse Editar configuración.
  3. Para DYNAMIC_LOGS, escriba 2 para activar la asignación dinámica de archivos de anotaciones cronológicas.
  4. Para CDR_MAX_DYNAMIC_LOGS, escriba el número de solicitudes de archivos de anotaciones cronológicas dinámicos que ER pueda realizar en una sesión. O bien escriba -1 para añadir anotaciones cronológicas de forma indefinida.

Supervisión de la cola de envío para un nodo

Supervisar estadísticas sobre la cola de envío en el nodo actual en un dominio de Enterprise Replication (ER) y sobre los nodos de destino y las réplicas para los que el nodo actual es un origen de las transacciones replicadas.

ER recopila la información de los registros de registro y coloca los datos que se deben transferir en la cola de envío. Para supervisar la cola de envío, en el menú de IBM OpenAdmin Tool (OAT) para Informix, pulse Replication > Detalles de nodo > Cola de envío.

la página Cola de envío muestra estadísticas sobre las transacciones de la cola de envío. Proporciona información sobre el espacio de almacenamiento, las transacciones enviadas a los nodos de destino y la memoria necesaria para las transacciones de la cola de envío. También proporciona información sobre los nodos de destino y los duplicados para los que el nodo actual es una fuente de transacciones duplicadas.

  • Para determinar cuántas transacciones están en la cola y si están siendo almacenadas, revise los campos Transacciones en cola, Transacciones en memoria y Transacciones en spool.
  • Para determinar si las transacciones se van a aplicar en el nodo de destino, revise el campo Posición del registro de captura. Compárelo con el campo Últimos datos reconocidos, que indica la posición de registro lógicas de la última transacción aplicada en el nodo.
  • Para determinar si alguno de los nodos de destino no aplican transacciones, revise el campo Nodos suspendidos. Pulse también la ficha Nodo de destino para revisar el estado de la conexión de cada nodo de destino relacionado.
  • Para ver el estado de los nodos de destino en los que el nodo actual duplica datos, pulse la ficha Nodos de destino.
  • Para ver los duplicados para el nodo actual, pulse la pestaña Duplicados.

Visualización de nodos de destino y duplicados relacionados con el nodo actual

Ver los nodos de destino y las réplicas para los que el nodo actual de un dominio de Enterprise Replication (ER) es un origen de las transacciones replicadas.

En el menú de IBM OpenAdmin Tool (OAT) para Informix, pulse Replication > Detalles de nodo > Cola de envío y, a continuación, pulse el separador Nodos de destino o el separador Réplicas.

Supervisión de E/S de red para un nodo de Enterprise Replication

Supervisar un resumen de E/S de red para el nodo actual en un dominio de Enterprise Replication (ER). Revisar información detallada sobre el nodo actual y sus nodos relacionados.

Para comprobar la actividad de red para el nodo actual, en el menú de IBM OpenAdmin Tool (OAT) para Informix, pulse Replication > Detalles de nodo > Resumen. En la página Resumen, la sección Red muestra la información siguiente:

  • El número de nodos conectados
  • El número de mensajes enviados por el nodo actual
  • El número de mensajes recibidos por el nodo actual
  • La tasa de rendimiento en el nodo actual
  • El número de mensajes que no han sido enviados debido a la conexión o a otros problemas

Para comprobar el estado de red del nodo actual y los nodos relacionados, en el menú de OAT, pulse Replication > Detalles de nodo > Red. La página Red muestra información sobre la actividad de la red con nodos asociados que han enviado mensajes desde el nodo actual o que los han recibido desde éste. Los nombres de los nodos asociados aparecen como nombres de grupos tal y como se definen en el archivo sqlhosts.

  • Para determinar la actividad de red reciente en el nodo actual, revise el gráfico Detalles de red, el cual muestra el número de mensajes en el tiempo. Pulse el icono del reloj para ver o modificar la tasa de renovación, que va de 0 a 60 segundos.
  • Para determinar el estado de la conexión de un nodo asociado, revise la columna Estado.
  • Para determinar el estado de la interfaz de red (NIF) de un nodo asociado, revise la columna Conexión.

Supervisión de la utilización del espacio de disco para un nodo

Supervisar cómo se está utilizando el espacio de disco en el nodo actual en un dominio de Enterprise Replication (ER).

En el menú de IBM OpenAdmin Tool (OAT) para Informix, pulse Replication > Detalles de nodo > Uso de disco. La página Uso de disco muestra información sobre las tres áreas de espacio utilizadas por ER:

  • Espacio de datos de filas: Los espacios sb que almacenan los datos que deben duplicarse. Están definidos en el parámetro CDR_QDATA_SBSPACE.
  • Espacio de BD de registros de transacciones: los espacios de base de datos que almacenan los registros de transacciones. Están definidos en el parámetro CDR_QHDR_DBSPACE, si éste está definido. De lo contrario, de forma predeterminada, los registros de transacciones se almacenan en el espacio de base de datos raíz.
  • Espacio de SB de transferencia de páginas del agrupador: Los espacios db utilizados para manejo de transacciones grandes. Se definen primero mediante el parámetro SBSPACETEMP, si éste está definido; a continuación con el parámetro SBSPACENAME, si está definido; de lo contrario, con el parámetro CDR_QDATA_SBSPACE.

Importante:
Si se llenan estos espacios, ER se cuelga hasta que se resuelve el problema que está causando que ponga en spool o asigne el espacio de disco adicional.

Para cambiar los valores de estos parámetros, en el menú de OAT, pulse Replication > Detalles de nodo > Configuración y pulse Editar configuración.

Supervisión de la cola de recepción para un nodo

Supervisar las transacciones en cola y pendientes de la cola de recepción en el nodo actual de un dominio de Enterprise Replication (ER).

En el menú de IBM OpenAdmin Tool (OAT) para Informix, pulse Replication > Detalles de nodo > Resumen. Revise la sección Cola de recepción, que muestra el número de transacciones en cola y pendientes.

  • Las transacciones en cola han sido recibidas en el nodo actual desde un nodo asociado y almacenadas en la cola de recepción.
  • Las transacciones pendientes están esperando ser aplicadas en el nodo de destino.

Supervisión de cómo se aplican los datos duplicados en el nodo de destino

Supervisar cómo se aplican los datos replicados en el nodo actual de un dominio de Enterprise Replication (ER).

Cuando un nodo de destino ha recibido transacciones de otro nodo y las almacena en la cola de recepción, ER aplica las transacciones a las tablas en el nodo de destino.

Para ver un resumen de alto nivel de cómo se aplican los datos replicados en el nodo actual, en el menú de IBM OpenAdmin Tool (OAT) para Informix, pulse Replication > Detalles de nodo > Resumen. Revise la sección Aplicar.

Para ver información detallada sobre cómo se aplican los datos replicados, en el menú de OAT, pulse Replication > Detalles de nodo > Aplicar. Esta página contiene tres tipos de información:

  • Estadísticas cobre cómo se aplican los datos duplicados en el nodo actual.
  • Datos gráficos que muestran las tasa de transacciones aplicadas y las tasas de latencia
  • Estadísticas sobre cada nodo de origen y sobre cómo se aplican los datos de cada nodo en el nodo actual

Supervisión de anomalías de transacciones

Supervisar el número de anomalías de transacciones en el nodo actual de un dominio de Enterprise Replication (ER).

Puede supervisar las anomalías de transacciones utilizando los métodos siguientes:

  • En el menú de IBM OpenAdmin Tool (OAT) para Informix, pulse Replication > Detalles de nodo > Resumen. En la sección Aplicar, revise los campos siguientes:
    • Tasa de anomalías en transacciones: Número medio de anomalías en transacciones por segundo.
    • Anomalías totales: Número total de transacciones anómalas.
    • Recuento de archivos ATS: número total de archivos ATS (Spooling de transacciones terminadas anormalmente) en el nodo. Un número mayor que 0 indica posibles anomalías a nivel de tabla.
    • Recuento de archivos RIS: número total de archivos RIS (Spooling de información de filas) en el nodo. Un número mayor que 0 indica posibles anomalías a nivel de fila.
  • En el menú de OAT, pulse Replication > Detalles de nodo > Aplicar. Revise los campos Tasa de anomalías y Total de errores para el nodo actual, la tabla Aplicar estadísticas por nodo, la cual muestra los nodos de origen que han enviado transacciones. La columna Transacciones totales anómalas muestra cuántas transacciones han terminado anormalmente en el nodo actual, por le nodo que ha enviado las transacciones.
  • En el menú de OAT, pulse Replication > Detalles de nodo > ATS y RIS. Estas páginas muestran los archivos ATS y RIS utilizados para resincronizar transacciones después de que haya habido anomalías.

Configuración de Enterprise Replication

Estos temas describen cómo configurar Enterprise Replication (ER) utilizando el plug-in de IBM Informix Replication para OpenAdmin Tool (OAT).

Definición de un servidor para Enterprise Replication

Convertir un servidor en un nodo nuevo en un dominio de Enterprise Replication (ER) existente o el primer nodo en un dominio de ER nuevo.

Para definir un servidor para Enterprise Replication

  1. En el menú de IBM OpenAdmin Tool (OAT) para Informix, pulse Replication > Domino de ER.
  2. En el menú Acciones, pulse Definir servidor nuevo.
  3. Siga las direcciones en el asistente Definir servidor.

Definición de un conjunto de duplicados

Crear un conjunto de duplicados nuevo.

Un conjunto de duplicados combina varios duplicados para formar un conjunto que se puede gestionar conjuntamente como una unidad. Puede utilizar un solo mandato para iniciar, detener, suspender o reanudar todos los duplicados del conjunto.

Para definir un conjunto de duplicados:

  1. En el menú de IBM OpenAdmin Tool (OAT) para Informix, expanda Replication y, a continuación, pulse Réplicas.
  2. Pulse la pestaña Conjuntos de duplicados.
  3. En el menú Acciones, pulse Definir conjunto de duplicados.
  4. Escriba un nombre para el conjunto de duplicados. El nombre debe ser exclusivo y no puede ser igual que un nombre de duplicado.
  5. Seleccione los duplicados para el conjunto.
    • Para añadir duplicados, selecciónelos en la lista Duplicados disponibles y pulse Añadir o Añadir todos.
    • Para suprimir duplicados, selecciónelos en la lista Este conjunto de duplicados y pulse Suprimir o Suprimir todos.
  6. Opcional: Para hacer que este conjunto sea un conjunto de réplicas exclusivo, seleccione Los duplicados son exclusivos de este conjunto.
  7. Para establecer la opción de frecuencia para el conjunto de duplicados, pulse Siguiente, o para crear el conjunto de duplicados con el valor por omisión, pulse Finalizar.
  8. Opcional: Especifique la Frecuencia de la réplica. El valor por omisión es inmediatamente cuando llegan los datos.
  9. Para crear el conjunto de réplicas, pulse Finalizar.

  • Tareas relacionadas:
  • Conjunto de réplicas exclusivo (Guía de Enterprise Replication)

Definición de un duplicado

Definir los participantes y los atributos de la duplicación incluyendo la frecuencia con la que duplicar los datos y cómo gestionar conflictos.

Con la acción Definir duplicado, se define un duplicado maestro estricto y sus participantes. Un duplicado maestro comprueba que las tablas duplicadas de distintos servidores tienen tipos de datos de columnas coherentes. Un duplicado maestro estricto también comprueba que las tablas duplicadas tengan nombres de columnas idénticos. Todos los servidores de bases de datos que contienen duplicados maestros debe poder establecer una conexión directa con el servidor de bases de datos de duplicados maestros.

Requisito previo:
Una tabla debe tener una clave primaria para ser un participante de duplicación. Además, la tabla debe ser una tabla real. No puede ser un sinónimo, una vista ni una secuencia. Además, no puede ser una tabla externa. Sólo aparecen en la lista de tablas seleccionables las tablas que pueden ser participantes de duplicación.

Para definir un duplicado y los participantes de duplicación:

  1. En el menú de IBM OpenAdmin Tool (OAT) para Informix, expanda Replication y, a continuación, pulse Réplicas.
  2. Pulse la pestaña Duplicados.
  3. En el menú Acciones, pulse Definir duplicado.
  4. Escriba un nombre exclusivo para el duplicado.
  5. Seleccione el grupo de servidores para utilizarlo como base de la réplica maestra en la lista Servidor de réplica maestro.
  6. Para crear automáticamente tablas en el servidor de destino que existe en el servidor de duplicados maestros pero no en el servidor de destino, seleccione Crear tablas que faltan en servidores de destino.
  7. Seleccione las tablas y servidores de duplicación que desee definir como participantes en el duplicado utilizando los filtros.
    1. Para Filtro de base de datos, escriba todo o parte del nombre de la base de datos.
    2. Para Filtro de tabla, escriba todo o parte del nombre de la tabla.Siga filtrando la lista hasta que aparezca un solo participante para cada servidor.
    3. Para seleccionar todos los participantes especificados por los filtros, pulse Seleccionar.

      También puede seleccionar o eliminar una tabla pulsándola en la lista.

    Nota:
    El servidor de duplicado maestro debe ser un participante del duplicado nuevo. Cuando seleccione las tablas para participar en el duplicado, debe seleccionar una tabla en el servidor duplicado maestro.
  8. Especifique el tipo de participante: primario o de sólo recepción.El valor por omisión es primario: el participante primario envía y recibe datos. Para cambiar esta opción, pulse el icono.
  9. Especifique qué privilegios se utilizan para aplicar los datos duplicados.El valor por omisión es propietario de tabla. Para cambiar esta opción, pulse el icono.
  10. Para establecer opciones avanzadas para el duplicado, pulse Siguiente, o para crear el duplicado con los valores por omisión para loas opciones avanzadas, pulse Finalizar.

    ER comprueba que las tablas de distintos servidores tienen tipos de datos de columnas coherentes y nombres de columnas idénticos. Si no están disponibles Siguiente y Finalizar, las tablas no serán coherentes.

  11. Opcional: Especifique las opciones avanzadas:
    • Ámbito de resolución de conflictos. El valor por omisión es transacción.
    • Regla de resolución de conflictos. El valor por omisión es indicación de la hora.
    • Formato de coma flotante. El valor por omisión es IEEE.
    • Activar spooling de transacciones terminadas anormalmente (ATS). Por omisión, ATS no está habilitado.
    • Activar spooling de información de filas (RIS). Por omisión, RIS no está habilitado.
    • Duplicar fila completa. Por omisión, se duplican filas completas. Esta opción está habilitada.
    • Activar los activadores en el destino. Por omisión, los activadores no se activan en la tabla de destino.
    • Conservar filas suprimidas. Por omisión, las filas suprimidas no se conservan.
    • Frecuencia de duplicación. El valor por omisión es inmediatamente cuando llegan los datos.
  12. Para crear la réplica, pulse Finalizar.

Utilización de plantillas para configurar Enterprise Replication

Utilizar plantillas para configurar y desplegar Enterprise Replication.

Una plantilla utiliza información de esquema acerca de una base de datos, un grupo de tablas, columnas y claves primarias para definir un grupo de duplicados maestros y un conjunto de duplicados. Primero, defina una plantilla, y a continuación cree una instancia de ella en los servidores donde desee duplicar datos.

Definición de una tarea

Crear una plantilla. Especificar las tablas que deben utilizarse para el servidor de la plantilla, de la base de datos y de Enterprise Replication (ER) en el que están ubicadas las tablas, y las opciones de la plantilla.

Una plantilla define un grupo de duplicados maestros y un conjunto de duplicados. Una plantilla consta de información de esquema acerca de una base de datos, un grupo de tablas, atributos de columna y las tablas primarias que identifican las filas.

Para definir una plantilla:

  1. En el menú de IBM OpenAdmin Tool (OAT) para Informix, expanda Replication y, a continuación, pulse Réplicas.
  2. Pulse la pestaña Plantillas.
  3. En el menú Acciones, pulse Definir plantilla nueva.
  4. Escriba un nombre para la plantilla. El nombre debe ser exclusivo y no puede ser igual que un nombre de duplicado o de conjunto de duplicados.
  5. En la lista Servidores, amplíe los nombres y las bases de datos de servidores para visualizar en las tablas.
    Sugerencia:
    Para filtrar la lista, comience escribiendo el nombre en el recuadro de texto Filtro de servidores y el nombre de la base de datos en el recuadro de texto Filtro de bases de datos.
  6. Seleccione las tablas para la plantilla.
    • Para añadir a la lista todas las tablas de una base de datos, seleccione la base de datos la lista Tablas y pulse Añadir.
    • Para añadir una tabla específica, seleccione la tabla y pulse Añadir.
    • Para suprimir una tabla de la plantilla, selecciónela y pulse Suprimir o pulse Suprimir todas.
  7. Para crear la plantilla con las opciones por omisión, pulse Finalizar, o pulse Siguiente para especificar cualquiera de las siguientes opciones y pulse Finalizar.
    • Ámbito de resolución de conflictos. El valor por omisión es transacción.
    • Regla de resolución de conflictos. El valor por omisión es ignorar.
    • Formato de coma flotante. El valor por omisión es IEEE.
    • Activar spooling de transacciones terminadas anormalmente (ATS). Por omisión, ATS no está habilitado.
    • Activar spooling de información de filas (RIS). Por omisión, RIS no está habilitado.
    • Activar los activadores en el destino. Por omisión, los activadores no se activan en la tabla de destino.
    • Conservar filas suprimidas. Por omisión, las filas suprimidas no se conservan.

Ahora puede iniciar la plantilla de los servidores de las bases de datos del dominio ER utilizando la acción Crear plantilla.

Importante:
Para duplicar datos en un dominio ER basado en los duplicados de la plantilla, debe crear una instancia de la plantilla en al menos dos servidores.

Creación de una plantilla

Crear una instancia de una plantilla en servidores Enterprise Replication (ER).

Requisito previo:
Antes de poder crear una instancia de una plantilla, debe definir la plantilla. Consulte Definición de una plantilla

Para crear una instancia de plantilla:

  1. En el menú de IBM OpenAdmin Tool (OAT) para Informix, expanda Replication y, a continuación, pulse Réplicas.
  2. Pulse la pestaña Plantillas.
  3. En la lista de plantillas, seleccione la plantilla para la que desee crear una instancia.
  4. En el menú Acciones, pulse Crear plantilla.
  5. En la lista Servidores disponibles, seleccione los servidores en los que crear la instancia de plantilla.
    Sugerencia:
    Comience escribiendo el nombre del servidor en el recuadro de texto Filtro de servidores para filtrar la lista.
  6. Para crear la instancia de la plantilla en los servidores con las opciones por omisión, pulse Finalizar, o pulse Siguiente para especificar cualquiera de las siguientes opciones y pulse Finalizar.
    • Crear tablas que faltan. Cuando se selecciona esta opción, las tablas que existen en el servidor del duplicado maestro pero no en los servidores de destino se crean automáticamente en los servidores de destino. Esta opción se selecciona por omisión. El espacio db para las tablas creadas se establece por omisión en el mismo espacio db que la base de datos.
    • Aplicar datos duplicados con privilegios de propietario de tabla. Especifique que el propietario de la tabla de origen es el propietario de las tablas creadas cuando se crea la plantilla. Por omisión, las tablas son propiedad del usuario informix.
    • Establecer el servidor para recibir solamente. Especifique que los servidores de destino sean servidores de sólo recepción. Por omisión, los servidores envían y reciben datos duplicados.
    • Sincronizar datos. Especifique un servidor para que actúe como origen de sincronización de datos en los servidores en los que se crea la instancia de la plantilla. Especifique cómo manejar las filas en los servidores de destino que no estén en el servidor de origen de datos. Por omisión, las filas adicionales se suprimen de los servidores de destino.

Supresión de una plantilla

Suprimir una plantilla de Enterprise Replication (ER) y cualquier conjunto de duplicados creados cuando se crea la instancia de la plantilla.

Importante:
Al suprimir una plantilla no se suprimen los duplicados creados cuando se crea una instancia de la plantilla.

Para suprimir una plantilla:

  1. En el menú de IBM OpenAdmin Tool (OAT) para Informix, expanda Replication y, a continuación, pulse Réplicas.
  2. Pulse la pestaña Plantillas.
  3. Seleccione la plantilla.
  4. En el menú Acciones, pulse Suprimir plantilla.

Volver a ejecutar un mandato de cuadrícula

Vuelva a ejecutar un mandato que ha fallado en uno o más servidores de una cuadrícula de servidores de réplica interconectados de un dominio de Enterprise Replication (ER).

Requisito previo:
Para esta acción, IBM OpenAdmin Tool (OAT) para Informix necesita IBM informix versión 11.70.

Los mandatos de cuadrícula que se ejecutan en un servidor de origen se propagan a todos los demás servidores de la cuadrícula. Puede volver a ejecutar un mandato en los servidores en los que este ha fallado.

Al añadir un servidor a la cuadrícula, los mandatos ejecutados anteriormente en la cuadrícula tienen un estado de pendiente para ese servidor. Si desea ejecutar un mandato de cuadrícula anterior en un nuevo servidor de cuadrícula, vuelva a ejecutar el mandato. Si no desea ejecutar un mandato de cuadrícula anterior en un nuevo servidor, elimine el mandato.

Un acuerdo de nivel de servicio (SLA) es un contrato entre las aplicaciones cliente especificadas y sus proveedores de servicio, en este caso, servidores de bases de datos de Informix. Cuando no se supervisa la calidad de los datos, Connection Manager puede direccionar las conexiones de cliente basándose sólo en la carga de trabajo. Para especificar las anomalías de aplicación y latencia en un SLA, habilite la supervisión de la calidad de los datos modificando un servidor que sea miembro de la cuadrícula en la página Replication > Dominio de ER.

Al reiniciar la réplica y sincronizar los datos, puede beneficiarse de una reparación de indicación de la hora.

Enterprise Replication no sincroniza las transacciones si un duplicado de un conjunto de duplicados está suspendido. Por ejemplo, una transacción que actualiza las tablas X e Y se dividirá si la duplicación para la tabla X está suspendida.

Resincronización de datos entre servidores de duplicación

Corregir incoherencias de datos entre tablas duplicadas. Buscar incoherencias de datos, reparar incoherencias durante una comprobación, sincronizar datos o reparar transacciones anómalas con archivos ATS (Spooling de transacciones terminadas anormalmente) y RIS (Spooling de información de filas).

Con el plug-in de IBM Informix Replication para OpenAdmin Tool (OAT), puede reparar las incoherencias de datos entre tablas replicadas de la maneras siguientes:

  • Comprobar la coherencia y reparar filas incoherentes: Una comprobación de coherencia compara los datos entre un servidor de referencia y uno o más servidores de destino y luego genera un informe en el que se describen las incoherencias. Puede optar por reparar las filas incoherentes durante la comprobación de coherencia. Puede realizar una comprobación de coherencia y una sincronización opcional en un único duplicado o en un conjunto de duplicados.
  • Sincronizar datos: La sincronización duplica cada una de las filas del duplicado o conjunto de duplicados especificado del servidor de referencia a todos los servidores de destino especificados. Puede utilizar la sincronización directa para rellenar un servidor de destino nuevo, o un servidor de destino existente que haya pasado a ser muy incoherente.
  • Reparar transacciones utilizando archivos ATS o RIS: Una reparación mediante un archivo ATS o RIS repara las filas asociadas con la única transacción que hay registrada en el archivo ATS o RIS especificado.
Importante:
La ejecución de una comprobación de coherencia o de una sincronización puede ocupar una gran cantidad de espacio en los archivos de anotaciones cronológicas. Asegúrese de que tiene suficiente espacio antes de ejecutar este mandato.

Reparación de transacciones anómalas con archivos ATS

Reparar transacciones anómalas utilizando archivos ATS (Spooling de transacciones terminadas anormalmente) La tarea reparación de archivos ATS reconcilia las filas que no se han podido aplicar en el servidor de destino.

Requisito previo:
Cree un directorio para almacenar los archivos ATS, habilite la generación de archivos ATS en el servidor Enterprise Replication (ER) y active la generación de archivos ATS para el duplicado.

Un archivo ATS se genera cuando una transacción duplicada no puede ser aplicada en la base de datos de destino debido a un conflicto y toda la transacción termina de forma anómala. Cuando se repara la transacción utilizando el archivo ATS, se repara las filas asociadas ala transacción grabadas en el archivo ATS.

Para reparar transacciones utilizando archivos ATS:

  1. En el menú de IBM OpenAdmin Tool (OAT) para Informix, expanda Replication y, a continuación, pulse Réplicas.
  2. Pulse la pestaña ATS.
  3. En la tabla Spooling de transacciones terminadas anormalmente (ATS), seleccione los recuadros de selección para los archivos ATS.

    Si no desea reparar todos los archivos ATS disponibles al mismo tiempo, es posible que algunas tareas terminen de forma anómala debido a problemas para mantener las relaciones entre las tablas (limitaciones de integridad referenciales).

  4. Pulse Reparar.
  5. Revise los archivos ATS seleccionados para ser reparados.

    Para ver los detalles de un archivo ATS, pulse el nombre del archivo.

  6. Pulse Reparar.
  7. Supervise el progreso de la tarea y revise los resultados en la lista Estado de la tarea de reparación de ATS.

    Para ver los detalles de una tarea, pulse el nombre en la lista.

Un archivo RIS se genera cuando una transacción duplicada que tiene el ámbito de filas definido para ser aplicado en la base de datos de destino debido a un conflicto y la transacción termina de forma anómala. Cuando se repara la transacción utilizando el archivo RIS, se repara la fila que está grabada en el archivo RIS.

  2. Pulse la pestaña RIS.
  3. En la tabla Spooling de información de filas (RIS), pulse los recuadros de selección para los archivos RIS.

    Si no desea reparar todos los archivos RIS disponibles al mismo tiempo, es posible que algunas tareas terminen de forma anómala debido a problemas para mantener las relaciones entre las tablas (limitaciones de integridad referenciales).

  4. Pulse Reparar.
  5. Revise los archivos RIS seleccionados para ser reparados.

    Para ver los detalles de un archivo RIS, pulse el nombre del archivo.

  6. Pulse Reparar.
  7. Supervise el progreso de la tarea y revise los resultados en la lista Estado de la tarea de reparación de RIS.

    Para ver los detalles de una tarea, pulse el nombre en la lista.

Creación de un acuerdo de nivel de servicio

Cree un acuerdo de nivel de servicio para una cuadrícula, un clúster o un conjunto de duplicados mediante Connection Manager.

Requisitos previos:
Para Connection Manager, IBM OpenAdmin Tool (OAT) para Informix necesita Informix 11.70.xC3 y el Connection Manager que está empaquetado con Informix 11.70.xC3 o con IBM Informix Client Software Development Kit (Client SDK) 3.70.xC3. La cuadrícula, el clúster o el conjunto de réplicas deben estar definidos como una unidad de conexión para Connection Manager.

Un acuerdo de nivel de servicio (SLA) es un contrato entre las aplicaciones cliente especificadas y sus proveedores de servicio, en este caso, servidores de bases de datos de Informix. En función de los acuerdos de nivel de servicio, el Gestor de conexiones conecta las aplicaciones cliente a los servidores correspondientes. El Gestor de conexiones acepta las solicitudes de conexión de cliente y redirecciona las solicitudes al servidor correspondiente o funciona como un servidor proxy entre las aplicaciones cliente y los servidores.

Un Gestor de conexiones proporciona equilibrio de carga y direcciona las solicitudes de conexión de cliente al servidor correspondiente en una unidad de conexión en función de los acuerdos de nivel de servicio (SLA). Cuando se supervisa la calidad de datos, Connection Manager puede direccionar las conexiones de cliente a los servidores de réplica basándose en el número de anomalías de aplicación, la latencia de transacción y la carga de trabajo. Cuando no se supervisa la calidad de los datos, Connection Manager puede direccionar las conexiones de cliente basándose sólo en la carga de trabajo. Para especificar las anomalías de aplicación y latencia en un SLA, habilite la supervisión de la calidad de los datos modificando un servidor que sea miembro de la cuadrícula en la página Replication > Dominio de ER.

Para crear un acuerdo de nivel de servicio:

  1. En el menú de OAT, pulse Replication > Connection Manager.
  2. Pulse Connection Manager.
  3. Pulse la unidad de conexión en la lista.
  4. En el menú Acciones, pulse Crear un SLA.
  5. Complete el asistente.
