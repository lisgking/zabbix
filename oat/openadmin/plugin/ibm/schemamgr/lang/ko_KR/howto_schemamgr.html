OAT용 Informix 스키마 관리자 플러그인

OpenAdmin Tool(OAT)용 IBM® Informix® 스키마 관리자 플러그인은 데이터베이스 및 테이블을 보고 관리할 수 있는 그래픽 인터페이스를 제공합니다.

스키마 관리자 플러그인을 사용하여 다음 작업을 수행할 수 있습니다.

  • 데이터베이스, 테이블 및 인덱스를 작성, 및 삭제할 수 있습니다.
  • 외부 테이블을 사용하여 데이터베이스 서버 외부에서 관리 및 저장되는 데이터를 로드/언로드합니다.
  • 테이블 및 분할 통계 대한 정보를 모니터할 수 있습니다.
  • 데이터 분산 통계가 분할된 테이블에서 변경되는 분할에서만 계산되는지 또는 전체 테이블에서 계산되는지 여부를 지정할 수 있습니다.
  • 스토어드 프로시저 및 함수, 순서, UDT(사용자 정의 자료형), 권한, DataBlade® 모듈, 집계, 캐스트 및 연산자 클래스를 포함한 데이터베이스 오브젝트에 대한 정보를 표시합니다.
  • 컬럼, 인덱스, 참조, 권한, UDT, 제한 조건, 분할 및 트리거를 포함한 테이블 오브젝트에 대한 정보를 표시합니다.

데이터베이스 오브젝트 보기

스토어드 프로시저 및 함수, 순서, UDT(사용자 정의 자료형), 권한, DataBlade 모듈, 집계, 캐스트 및 연산자 클래스에 대한 정보를 표시합니다.

데이터베이스 오브젝트를 보려면 다음을 수행하십시오.

  1. IBM OpenAdmin Tool(OAT) for Informix 메뉴에서 SQL 도구 상자 > 스키마 관리자를 클릭하십시오.
  2. 목록에서 데이터베이스를 선택하십시오.

    각 데이터베이스 오브젝트에 대한 정보 패널이 페이지에 표시됩니다.

  3. 데이터베이스 오브젝트에 대한 더 자세한 정보를 표시하려면 오브젝트 패널의 펼치기 버튼을 클릭하십시오.

테이블 오브젝트 보기

컬럼, 인덱스, 참조, 권한, UDT, 제한 조건, 분할 및 트리거에 대한 정보를 표시합니다.

테이블 오브젝트를 표시하려면 다음을 수행하십시오.

  1. IBM OpenAdmin Tool(OAT) for Informix 메뉴에서 SQL 도구 상자 > 스키마 관리자를 클릭하십시오.
  2. 테이블을 찾으려면, 찾기 필드에 테이블 이름의 일부를 입력하고 검색을 클릭하십시오. 또한 데이터베이스를 펼쳐 테이블을 표시할 수 있습니다.
  3. 옵션: 목록에 더 많은 테이블을 포함하는 필터를 선택하십시오.
    • 시스템 카탈로그를 포함하려면 카탈로그 포함을 선택하십시오.
    • 뷰를 포함하려면 뷰 포함을 선택하십시오. 뷰는 지정된 SELECT문을 기반으로 한 가상 테이블입니다.
    • 동의어를 포함하려면 동의어 포함을 선택하십시오. 동의어는 테이블, 뷰 또는 순서 오브젝트의 대체 이름입니다. 일반적으로 동의어는 현재 데이터베이스에 없는 테이블을 참조합니다.
  4. 목록에서 테이블을 선택하십시오.

    각 테이블 오브젝트에 대한 정보 패널이 페이지에 표시됩니다.

  5. 테이블 오브젝트에 대한 더 자세한 정보를 표시하려면 오브젝트 패널의 펼치기 버튼을 클릭하십시오.

테이블 통계 검토

통계가 마지막으로 기록된 시기, 데이터 분산 및 변경된 행 퍼센트를 포함하여 테이블 및 분할 통계에 대한 정보를 검토하십시오. 테이블 통계 자동 갱신 옵션을 지정하십시오.

전제조건:
테이블 통계 자동 갱신 옵션을 지정하고 일부 정보(예: 변경된 행의 비율)를 표시하려면 IBM OpenAdmin Tool(OAT) for Informix에 IBM Informix 버전 11.70이 필요합니다.

테이블 통계를 검토하려면 다음을 수행하십시오.

  1. OAT 메뉴에서 SQL 도구 상자 > 스키마 관리자를 클릭하십시오.
  2. 목록에서 데이터베이스를 펼치십시오.
  3. 목록에서 테이블을 선택하십시오.
  4. 통계 패널을 펼치십시오.
  5. 조각 수준의 통계를 표시하려면 분할 통계를 클릭하십시오. 이 정보는 IBM Informix 버전 11.70의 경우에 표시됩니다.

데이터베이스 작성

데이터베이스를 작성하고 데이터베이스 로케일을 지정할 수 있습니다.

전제조건:
데이터베이스를 작성하려면 OpenAdmin Tool(OAT) for Informix에 IBM Informix 11.70.xC2가 필요합니다.


데이터베이스를 작성하려면 다음을 수행하십시오.

  1. IBM OpenAdmin Tool(OAT) for Informix 메뉴에서 SQL 도구 상자 > 스키마 관리자를 클릭하십시오.
  2. 동작 메뉴에서 데이터베이스 작성을 클릭하십시오.
  3. 데이터베이스 작성 마법사의 지시사항을 따르십시오.

  • 관련 개념
  • Use CREATE DATABASE (Database Design and Implementation Guide)
  • The database locale (GLS User's Guide)
  • 관련 참조
  • CREATE DATABASE statement (SQL Syntax)
  • create database argument: Create a database (SQL administration API) (Administrator's Reference)

데이터베이스 삭제

데이터베이스를 제거하고 데이터베이스에 저장된 테이블 및 데이터를 삭제할 수 있습니다.

전제조건:
데이터베이스를 삭제하려면 OpenAdmin Tool(OAT) for Informix에 IBM Informix 11.70.xC2가 필요합니다.


데이터베이스를 삭제하려면 다음을 수행하십시오.

  1. IBM OpenAdmin Tool(OAT) for Informix 메뉴에서 SQL 도구 상자 > 스키마 관리자를 클릭하십시오.
  2. 데이터베이스를 선택하십시오.
  3. 동작 메뉴에서 데이터베이스 삭제를 클릭하십시오.

  • 관련 참조
  • DROP DATABASE statement (SQL Syntax)

테이블 및 인덱스 관리

테이블을 작성, 삭제 및 절단합니다. 인덱스를 작성, 비활성 및 삭제합니다.

테이블 작성

테이블을 작성하고 테이블이 작동하는 데 필요한 속성을 정의합니다.

테이블을 작성하려면 다음을 수행하십시오.

  1. IBM OpenAdmin Tool(OAT) for Informix 메뉴에서 SQL 도구 상자 > 스키마 관리자를 클릭하십시오.
  2. 테이블을 작성할 데이터베이스를 선택하십시오.
  3. 작업 메뉴에서 테이블 작성을 클릭하십시오.
  4. 테이블 작성 마법사의 지침을 수행하십시오.

테이블 삭제

테이블과 연관된 인덱스 및 데이터와 함께 테이블을 제거합니다.

테이블을 제거할 경우 테이블에 저장된 데이터와 인덱스, 제한 조건, 트리거, 액세스 권한 및 로컬 동의어도 삭제하십시오. 테이블을 기반으로 한 모든 뷰와 테이블과 연관된 모든 위반 및 진단 테이블도 제거하십시오.

테이블을 삭제하려면 다음을 수행하십시오.

  1. IBM OpenAdmin Tool(OAT) for Informix 메뉴에서 SQL 도구 상자 > 스키마 관리자를 클릭하십시오.
  2. 목록에서 테이블이 포함된 데이터베이스를 펼치십시오.
  3. 테이블을 선택하십시오.
  4. 작업 메뉴에서 테이블 삭제를 클릭하십시오.

테이블 절단

테이블에서 모든 행을 삭제하고 연관된 스토리지 영역을 확보하거나, 테이블 및 해당 인덱스에서 다시 사용하기 위한 스토리지 공간을 예약합니다.

기존의 스토리지 공간을 다시 사용하면 테이블을 주기적으로 비우고 새로운 행으로 다시 로드되기 때문에 응용프로그램에서 스토리지를 더 효율적으로 관리할 수 있습니다.

테이블을 절단하려면 다음을 수행하십시오.

  1. IBM OpenAdmin Tool(OAT) for Informix 메뉴에서 SQL 도구 상자 > 스키마 관리자를 클릭하십시오.
  2. 목록에서 테이블이 포함된 데이터베이스를 펼치십시오.
  3. 테이블을 선택하십시오.
  4. 작업 메뉴에서 테이블 절단을 클릭하십시오.
  5. 옵션: 테이블에 할당된 스토리지 공간을 유지하려면 스토리지 공간 예약을 선택하십시오.

인덱스 작성

테이블에서 하나 이상의 컬럼에 대한 인덱스를 작성합니다.

인덱스를 작성하려면 다음을 수행하십시오.

  1. IBM OpenAdmin Tool(OAT) for Informix 메뉴에서 SQL 도구 상자 > 스키마 관리자를 클릭하십시오.
  2. 목록에서 데이터베이스를 펼치고 테이블을 선택하십시오.
  3. 작업 메뉴에서 인덱스 작성을 클릭하십시오.
  4. 인덱스 작성 마법사의 지침을 수행하십시오.

  • 관련 개념
  • Indexes and index performance considerations (Performance Guide)
  • 관련 참조
  • CREATE INDEX statement (SQL Syntax)

인덱스 사용 또는 사용 안함

인덱스를 사용하지 않도록 설정하거나, 사용 안함으로 설정된 인덱스를 활성화합니다.

인덱스는 기본적으로 사용하는 것으로 설정되어 있습니다. 데이터베이스 서버에서는 인덱스를 인식하고, 삽입, 삭제 및 갱신 작업 중에 인덱스를 갱신합니다. 인덱스가 사용 안함으로 설정된 경우 이 작업 동안 인덱스가 더 이상 갱신되지 않습니다.

인덱스의 사용 여부를 지정하려면 다음 작업을 수행하십시오.

  1. IBM OpenAdmin Tool(OAT) for Informix 메뉴에서 SQL 도구 상자 > 스키마 관리자를 클릭하십시오.
  2. 목록에서 인덱스가 적용되는 테이블이 포함된 데이터베이스를 펼치고 테이블을 선택하십시오.
  3. 인덱스 패널을 펼치십시오.
  4. 인덱스를 선택하십시오.
  5. 패널의 작업 메뉴에서 인덱스 사용 또는 인덱스 사용 안함을 클릭하십시오.

  • 관련 개념
  • How the Database Server Treats Disabled Indexes (SQL Syntax)
  • Definitions of Database Object Modes (SQL Syntax)

인덱스 삭제

인덱스(예: 더 이상 유효하지 않은 인덱스)를 삭제합니다.

인덱스를 삭제하려면 다음을 수행하십시오.

  1. IBM OpenAdmin Tool(OAT) for Informix 메뉴에서 SQL 도구 상자 > 스키마 관리자를 클릭하십시오.
  2. 목록에서 인덱스가 적용되는 테이블이 포함된 데이터베이스를 펼치고 테이블을 선택하십시오.
  3. 인덱스 패널을 펼치십시오.
  4. 인덱스를 선택하십시오.
  5. 패널의 작업 메뉴에서 인덱스 삭제를 클릭하십시오.

  • 관련 태스크
  • Dropping indexes for table-update efficiency (Performance Guide)
  • 관련 참조
  • DROP INDEX statement (SQL Syntax)

외부 테이블을 사용하여 데이터 로드/언로드

외부 테이블을 사용하여 데이터베이스 서버 외부에서 관리 및 저장되는 데이터를 로드/언로드합니다.

전제조건:
외부 테이블을 사용하여 데이터를 로드 및 언로드하려면 IBM OpenAdmin Tool(OAT) for Informix에 IBM Informix 버전 11.50.xC6 이상이 필요합니다.

외부 테이블을 사용하여 데이터 로드

외부 테이블을 사용하여 데이터베이스 서버 외부에 있는 소스로부터 데이터베이스 테이블로 데이터를 로드합니다.

전제조건:
외부 테이블을 사용하여 데이터를 로드 및 언로드하려면 IBM OpenAdmin Tool(OAT) for Informix에 IBM Informix 버전 11.50.xC6 이상이 필요합니다.

외부 테이블을 사용하여 데이터를 로드하려면 다음을 수행하십시오.

  1. OAT 메뉴에서 SQL 도구 상자 > 스키마 관리자를 클릭하십시오.
  2. 데이터를 로드할 데이터베이스 테이블을 선택하십시오.

    테이블을 찾으려면, 찾기 필드에 테이블 이름의 일부를 입력하고 검색을 클릭하십시오. 또한 데이터베이스를 펼쳐 테이블을 표시할 수 있습니다.

  3. 작업 메뉴에서 테이블 로드를 클릭하십시오.
  4. 테이블 로드 마법사의 지침을 수행하십시오.

외부 테이블에 데이터 언로드
