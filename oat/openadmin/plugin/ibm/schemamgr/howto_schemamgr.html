Informix Schema Manager Plug-in for OAT

The IBM® Informix® Schema Manager Plug-in for OpenAdmin Tool (OAT) provides a graphical interface for viewing and administering databases and tables.

With the Schema Manager plug-in, you can perform these tasks:

  • Create and drop databases, tables, and indexes.
  • Load and unload data that is managed and stored outside of the database server by using external tables.
  • Monitor information about the statistics for tables and fragments.
  • Specify whether data distribution statistics are calculated on only the fragments that change in a fragmented table or on the entire table.
  • View information about database objects including stored procedures and functions, sequences, user-defined data types (UDTs), privileges, DataBlade® modules, aggregates, casts, and operator classes.
  • View information about table objects including columns, indexes, references, privileges, UDTs, constraints, fragments, and triggers.

Viewing database objects

View information about stored procedures and functions, sequences, user-defined data types (UDTs), privileges, DataBlade modules, aggregates, casts, and operator classes.

To view database objects:

  1. On menu for the IBM OpenAdmin Tool (OAT) for Informix, click SQL Toolbox > Schema Manager.
  2. Select the database in the list.

    The page displays a panel of information for each of the database objects.

  3. To display more details about a database object, click the expand button on the object panel.

Viewing table objects

View information about columns, indexes, references, privileges, user-defined data types (UDTs), constraints, fragments, and triggers.

To view table objects:

  1. On menu for the IBM OpenAdmin Tool (OAT) for Informix, click SQL Toolbox > Schema Manager.
  2. To find a table, type any part of the table name in the Find field, and click Search. You can also expand a database to display the tables.
  3. Optional: Select the filters to include more tables in the list:
    • To include system catalogs, select Include Catalogs.
    • To include views, select Include Views. A view is a virtual table based on a specified SELECT statement.
    • To include synonyms, select Include Synonyms. A synonym is an alternative name for a table, a view, or a sequence object. Typically, synonyms refer to tables that are not in the current database.
  4. Select the table in the list.

    The page displays a panel of information for each of the table objects.

  5. To display more details about a table object, click the expand button on the object panel.

Reviewing table statistics

Review information about table and fragment statistics including when the statistics were last recorded, the data distribution, and the percent of changed rows. Specify the options for automatically updating statistics for the table.

Prerequisite:
The IBM OpenAdmin Tool (OAT) for Informix requires IBM Informix version 11.70 to specify the options for automatically updating statistics for the table and to display some of the information, for example, the percent of changed rows.

To review table statistics:

  1. On the OAT menu, click SQL Toolbox > Schema Manager.
  2. Expand the database in the list.
  3. Select the table in the list.
  4. Expand the Statistics panel.
  5. To display the fragment-level statistics, click Fragment Statistics. This information is displayed for IBM Informix version 11.70.

Creating a database

Create a database and specify the database locale.

Prerequisite:
To create a database, the OpenAdmin Tool (OAT) for Informix requires IBM Informix 11.70.xC2.


To create a database:

  1. On menu for the IBM OpenAdmin Tool (OAT) for Informix, click SQL Toolbox > Schema Manager.
  2. On the Actions menu, click Create Database.
  3. Follow the directions in the Create Database wizard.

  • Related concepts:
  • Use CREATE DATABASE (Database Design Guide)
  • The database locale (GLS User's Guide)
  • Related reference:
  • CREATE DATABASE statement (SQL Syntax)
  • create database argument: Create a database (SQL administration API) (Administrator's Reference)

Dropping a database

Remove a database and delete the tables and data stored in it.

Prerequisite:
To drop a database, the OpenAdmin Tool (OAT) for Informix requires IBM Informix 11.70.xC2.


To drop a database:

  1. On menu for the IBM OpenAdmin Tool (OAT) for Informix, click SQL Toolbox > Schema Manager.
  2. Select the database.
  3. On the Actions menu, click Drop Database.

  • Related reference:
  • DROP DATABASE statement (SQL Syntax)

Managing tables and indexes

Create, drop, and truncate tables. Create, disable, and drop indexes.

Creating a table

Create a table and define the attributes necessary to make a table fully functional.

To create a table:

  1. On menu for the IBM OpenAdmin Tool (OAT) for Informix, click SQL Toolbox > Schema Manager.
  2. Select the database in which to create the table.
  3. On the Actions menu, click Create Table.
  4. Follow the directions in the Create Table wizard.

Dropping a table

Remove a table with its associated indexes and data.

When you remove a table, you also delete the data stored in it and any indexes, constraints, triggers, access privileges, and local synonyms. You also remove all views based on the table and any violations and diagnostics tables associated with it.

To drop a table:

  1. On menu for the IBM OpenAdmin Tool (OAT) for Informix, click SQL Toolbox > Schema Manager.
  2. In the list, expand the database that contains the table.
  3. Select the table.
  4. On the Actions menu, click Drop Table.

Truncating a table

Delete all rows from a table and free the associated storage space, or reserve the storage space for reuse by the table and its indexes.

Reusing the storage space can make storage management more efficient in applications in which a table is periodically emptied and reloaded with new rows.

To truncate a table:

  1. On menu for the IBM OpenAdmin Tool (OAT) for Informix, click SQL Toolbox > Schema Manager.
  2. In the list, expand the database that contains the table.
  3. Select the table.
  4. On the Actions menu, click Truncate Table.
  5. Optional: Select Reserve the storage space to keep the storage space allocated to the table.

Creating an index

Create an index for one or more columns in a table.

To create an index:

  1. On menu for the IBM OpenAdmin Tool (OAT) for Informix, click SQL Toolbox > Schema Manager.
  2. In the list, expand the database and select the table.
  3. On the Actions menu, click Create Index.
  4. Follow the directions in the Create Index wizard.

  • Related concepts:
  • Indexes and index performance considerations (Performance Guide)
  • Related reference:
  • CREATE INDEX statement (SQL Syntax)

Disabling or enabling an index

Disable an index, or enable an index that was disabled.

An index is enabled by default. The database server recognizes the index and updates it during insert, delete, and update operations. When an index is disabled, it is no longer updated during these operations.

To enable or disable an index:

  1. On menu for the IBM OpenAdmin Tool (OAT) for Informix, click SQL Toolbox > Schema Manager.
  2. In the list, expand the database that contains the table that the index applies to and select the table.
  3. Expand the Indexes panel.
  4. Select the index.
  5. On the Actions menu for the panel, click Enable Index or Disable Index.

  • Related concepts:
  • How the Database Server Treats Disabled Indexes (SQL Syntax)
  • Definitions of Database Object Modes (SQL Syntax)

Dropping an index

Drop an index; for example, to remove an index that is no longer efficient.

To drop an index:

  1. On menu for the IBM OpenAdmin Tool (OAT) for Informix, click SQL Toolbox > Schema Manager.
  2. In the list, expand the database that contains the table that the index applies to and select the table.
  3. Expand the Indexes panel.
  4. Select the index.
  5. On the Actions menu for the panel, click Drop Index.

  • Related tasks:
  • Dropping indexes for table-update efficiency (Performance Guide)
  • Related reference:
  • DROP INDEX statement (SQL Syntax)

Loading and unloading data with external tables

Load and unload data that is managed and stored outside of the database server by using external tables.

Prerequisite:
To load and unload data with external tables, the IBM OpenAdmin Tool (OAT) for Informix requires IBM Informix version 11.50.xC6 or later.

Loading data with an external table

Load data into a database table from a source that is external to the database server by using an external table.

Prerequisite:
To load and unload data with external tables, the IBM OpenAdmin Tool (OAT) for Informix requires IBM Informix version 11.50.xC6 or later.

To load data with an external table:

  1. On the OAT menu, click SQL Toolbox > Schema Manager.
  2. Select the database table to load the data into.

    To find a table, type any part of the table name in the Find field, and click Search. You can also expand a database to display the tables.

  3. On the Actions menu, click Load Table.
  4. Follow the directions in the Load Table wizard.

Unloading data to an external table

Unload data from a database table to an external table.

Prerequisite:
To load and unload data with external tables, the IBM OpenAdmin Tool (OAT) for Informix requires IBM Informix version 11.50.xC6 or later.

To unload data to an external table:

  1. On the OAT menu, click SQL Toolbox > Schema Manager.
  2. Select the database table from which to unload the data.

    To find a table, type any part of the table name in the Find field, and click Search. You can also expand a database to display the tables.

  3. On the Actions menu, click Unload Table.
  4. Follow the directions in the Unload Table wizard.

Creating an external table

Create a table that is managed and stored outside of the database server to load and unload data.

Prerequisite:
To create external tables, the IBM OpenAdmin Tool (OAT) for Informix requires IBM Informix version 11.50.xC6.

To create an external table:

  1. On the OAT menu, click SQL Toolbox > Schema Manager.
  2. Select the database for which to create the table.
  3. On the Actions menu, click Create Table.
  4. Follow the directions in the Create Table wizard.

    For Table type, select an external table type:

    • External Fixed: A table in fixed-width text format whose data is managed and stored outside of the Informix database server.
    • External Delimited: A table in delimited text format whose data is managed and stored outside of the Informix database server.
    • External Informix: A table in internal Informix format whose data is managed and stored outside of the Informix database server.

      External tables are used to load and unload data for your database.

Viewing external table information

View information about an external table.

This information is displayed for Informix 11.50.xC6 or later.

To view information about an external table:

  1. On menu for the IBM OpenAdmin Tool (OAT) for Informix, click SQL Toolbox > Schema Manager.
  2. To find a table, type any part of the table name in the Find field, and click Search. You can also expand a database to display the tables.
  3. Select the table in the list.

    The page displays a panel of information for each of the table objects.

  4. To display more details about a table object, click the expand button on the object panel.

    The Table Information panel and Columns panel display information specific to external tables.

Viewing the status of load and unload tasks

View the status of the tasks submitted through the IBM OpenAdmin Tool (OAT) for Informix to load and unload data by using external tables.

Prerequisite:
To load and unload data with external tables, the IBM OpenAdmin Tool (OAT) for Informix requires IBM Informix version 11.50.xC6 or later.

To view the status of load and unload tasks:

  1. On the OAT menu, click SQL Toolbox > Schema Manager.
  2. Click the database or the table that the task runs on.
  3. Expand the Information panel.
  4. Click the Task Status tab.

    To display more information about a task, including the error codes and return messages, click the details icon for the task.

  • Related concepts:
  • Error Messages (Administrator's Reference)

Running saved load and unload tasks

Run tasks that load and unload data by using external tables.

Prerequisite:
To load and unload data with external tables, the IBM OpenAdmin Tool (OAT) for Informix requires IBM Informix version 11.50.xC6 or later.

When you load or unload data by using external tables, the task is saved. You can run the task again or delete it.

To run a saved load or unload task:

  1. On the OAT menu, click SQL Toolbox > Schema Manager.
  2. Click the database or the table that the task runs on.
  3. Expand the Information panel.
  4. Click the Tasks tab.
  5. Select the tasks to run.
  6. On the Actions menu for the panel, click Submit.

Deleting load and unload tasks

Delete a load or unload task or the status of a load or unload task.

Prerequisite:
To load and unload data with external tables, the IBM OpenAdmin Tool (OAT) for Informix requires IBM Informix version 11.50.xC6 or later.

When you load or unload data by using OAT, the task is saved. You can submit the task again or delete it. You can also delete the status of the tasks that are submitted.

Important:
When you delete a task on the Task page, the results of the task are also deleted from the Task Status page. When you delete a task status, only the task status is deleted.

To delete load and unload tasks:

  1. On the OAT menu, click SQL Toolbox > Schema Manager.
  2. Click the database or the table that the task runs on.
  3. Expand the Information panel.
  4. Select the appropriate tab:
    • To delete the task and the task status, click the Tasks tab.
    • To delete only the task status, click the Task Status tab.
  5. Select the task.
  6. Click Actions > Delete.

Creating a group of saved load or unload tasks

Group saved tasks so that they can be submitted together.

Prerequisite:
To load and unload data with external tables, the IBM OpenAdmin Tool (OAT) for Informix requires IBM Informix version 11.50.xC6 or later.

When you load or unload data by using OAT, the task is saved. You can run the task again, delete it, or group it with other saved tasks.

To create a group of saved load or unload tasks:

  1. On the OAT menu, click SQL Toolbox > Schema Manager.
  2. Click the database or the table that the task runs on.
  3. Expand the Information panel.
  4. Click the Tasks tab.
  5. Select the tasks to include in the group. Load and unload tasks can be included in the same group.
  6. On the Actions menu for the panel, click Create Group and enter a name for the group.
